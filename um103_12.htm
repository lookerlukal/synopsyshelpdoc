10.3.12 Surface slope control (ASC, ACS)                                        Next Previous TOC

 

During optimization, lenses occasionally become bent to such an extent that the slope at the margin approaches the hemisphere point.  This is usually undesirable for two reasons: first, the lens is hard to make and coatings are hard to apply uniformly, and second, in that situation rays are often very close to failing – even though the merit function seems to be under control – and very small changes to the lens can cause failures.  Then the optimization procedure gets stalled.

 

This is one of several kinds of monitors that operate continuously during optimization.

 

One can control the slope of the surface at a single point with the DSLOPE aberration.  Another option lets you automatically control the slope of all lenses in the system.  The AANT file entry

 

 ASC [ ANGLE WT WINDOW ] ]        (Automatic Slope Control)

 

will add an aberration that depends on the amount by which the absolute value of the surface slope angle at each surface exceeds the target value.  The aberration is calculated so that:

 

1.  Any angle less than the target gets zero penalty.

2.  Any angle within WINDOW of the target gets a penalty that increases quadratically with the difference and is equal to WT when the error equals WINDOW.

3.  Any angle more than this amount from the target gets a penalty that increases linearly from where the quadratic penalty left off.

 

With this option, lenses are monitored continuously along the upper and lower rays at the full-field point, and those that tend to become too steep are automatically controlled.  The default ANGLE is 60 degrees, the default weight is 1.0, and the default WINDOW is one degree.  This aberration applies to all surfaces whose curvature is varying.  If you want to allow angles steeper than the default 60 degrees, just enter your own target and weight.

 

The value of the ASC aberration is calculated after other aberrations in the merit function are computed, regardless of where the declaration occurs in the AANT file.  For most lenses this makes no difference – but if the lens is a ZFILE zoom lens, the calculation is therefore performed at the last zoom that was specified in the merit function.  Thus, for example, if the last aberration was specified at zoom 7, then the ASC penalty will also be computed in zoom 7.  If this behavior is not what you really want, you are advised to use the DSLOPE aberration instead of ASC.

 

The form

 

 ACS [ ANGLE WT WINDOW ] ]        (Automatic CAO Slope Control)

 

operates in much the same way, but examines the slope at the current CAO aperture instead of monitoring at the rim rays.  For most lenses they are equivalent -- but if you are optimizing a ZFILE zoom lens, the monitors are evaluated after all other aberrations, which means that the system is in the last zoom position at that time.  The ASC version would therefore monitor the slopes at that zoom along the rim rays, while the ACS version would check at the current CAO, which is set to the maximum value needed for all zooms (at surfaces with a soft CAO).  The latter is therefore usually preferred when working with zoom lenses.

 

One can evaluate the current slopes of all surfaces with the command SLOPE. the SLOPE operand is available for controlling the slope of a given surface at a given ray in the AANT file, and CSLOPE will return the slope at the current CAO..
