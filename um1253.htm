12.5.3 Fabrication Adjustments in MC                                        Next Previous TOC

 

The above sections describe the operation of the Monte-Carlo feature (MC), which simulates the manufacturing errors of a lens and calculates performance statistics.  That feature is intended for the case where you make an entire lens in accordance with the tolerance budget, assembling the elements into a cell within the accuracy specified by the budget.  Then it runs the optimization program, with the goal of improving performance when adjustments are made in accordance with the results.

 

In some cases it is not practical to attempt to produce a lens in the manner just described.  Lenses such as the objectives used to expose the wafers for microlithography, as well as some astronomical systems, require tolerances of only a few microns on some parameters, and trying to hold such tolerances is extremely difficult.  The assembly adjustments available to MC may not work well enough in those cases.

 

A lens maker has recourse to another procedure in this situation.  As each element is manufactured, the measured radii and thickness are input to the computer and the system is reoptimized with the remaining design parameters as variables.  Then another element is made, according to the new specs, and the process repeated, and so on, until all elements are finished.  The performance of the system is then close to the nominal performance even though all parameters were in error.  At assembly, the desired airspaces are those produced by the last iteration, not the starting design, and only the effects of tolerances on those airspaces, as well as element wedge, tilt, and decenter then influence image quality.  Many shops also adjust the centration of one or more elements at assembly to reduce those effects as well.  The resulting lens performance is usually well within the diffraction limit, and at no time did anyone have to hold micron-level tolerances on radii or thicknesses.  Note that the concept of testplate matching has little relevance in this scenario; as each lens is made and the system reoptimized, the radii invariably depart from any testplate radii that have been previously matched.  For systems this difficult, each element is given as much attention as required, and the required radius is invariably achieved very closely without recourse to testplates.

 

This procedure of course is much more expensive to carry out and requires much more time than the drop-in-in-the-cell procedure that MC emulates.  It is only practical when but a few systems are to be made and resources are available to underwrite the effort.

 

The SYNOPSYS feature to help you calculate a useful budget and manufacturing sequence in this situation is the Fab. Adjust MC feature (FAMC).  Proceed as follows:

 

1.  Optimize the lens.

 

2.  Run BTOL, giving your actual image requirements.  Do not declare testplates matched with TPR … as usual.  If the resulting budget is technically feasible and affordable, you do not need to use FAMC (but you might need to run it again with testplates matched and declared).  If the budget has tolerances too tight to hold in the shop, read on.

 

3.  Identify the most critical elements, and list them in order of difficulty.

 

4.  Rerun BTOL, adjusting the input so that the tolerances are looser.  You will have to iterate a few times, so to start with you might double the degradation that you asked for earlier.  The point here is to wind up with a budget that is affordable, and then to test it with FAMC.  One way that is sometimes convenient is to specify global fixed tolerances with the BTOL input, as described in Section 12.1.

 

5.  Now prepare and run a MACro with the following lines:

 

FAMC NSAMPLES LIBLOC [ QUIET [ QTOL { QNUM / ALL } QLIB [ TSTAT ]

PASSES NPASS

FAORDER SN SN SN …

FAORDER SN SN SN …

 

PHASE 1

PANT

(variables to adjust axially symmetric errors)

END

AANT

(merit function to correct those errors)

END

SNAP

EVAL

 

PHASE 2

PANT

(variables to correct non-symmetric errors)

END

AANT

(merit function to correct those errors)

END

SNAP

SYNO NPASS

 

PHASE 3

 

In this input, the arguments on the FAMC line are the same as those on the MC input, which you should review if necessary.  (FAMC currently does not have a multiconfiguration option.) NPASS is the number of iterations of optimization you want for the first phase, and FAORDER gives the fabrication order of the elements.  You generally want to make the fussiest elements first, so this should be the order that you listed in step 3.  Each SN in this list is the surface number of the first side of the element.  Every element in the lens should be included in the list somewhere, or it will not be perturbed.  Cemented lenses only require one SN, as all contiguous elements will automatically be included.

 

The input for PHASE 1 specifies those radii, thickness, and airspaces that you intend to recalculate as each element is manufactured.  For greatest benefit, you should list all radii and airspaces, but probably not glass thicknesses, which can become impractical too quickly, unless you set rather close limits to those variables.

 

Note that in the MACro, PHASE 1 ended with EVAL instead of the usual SYNO… input.  What we are doing at this point is just defining the optimization parameters, not running the optimization.  When the program later perturbs each element, it will run the actual optimization at that time, calling for the number of passes requested in the PASSES input.  As each element is perturbed, the PANT file parameters that refer to that element are deleted from the variable list, so the optimization uses only the remaining variables.  Then the next element is perturbed, starting at the latest design values, and the process repeated.  In this way, tolerances are already applied to the lens elements themselves when this phase is finished, and the ill effects are optimized away.

 

When phase 1 is completed, the program inserts all airspace, wedge, tilt, and decenter tolerances that are present in the budget, to simulate assembly errors, and sends the lens for optimization once again – this time using the PHASE 2 input.  Here, you only want to vary those parameters that will actually be used to compensate for assembly errors.  These would not, of course, include any lens parameters that were recalculated in phase 1.  The merit function should contain targets for both positive and negative field points, and the "F" option on ray sets, since the lens will not be axially symmetric at this stage.

 

When it encounters the PHASE 3 input, the program cycles the entire procedure, simulating the next case.

 

When all of this is finished, you can examine the statistics of the results with

 

 MC STAT

 

and

 

 MC PLOT

 

as you would for a normal MC evaluation.  Look at the yield that resulted, and if it is not at the statistical level you need, repeat the procedure from step 4, asking for higher or lower quality to offset the departure you found, or altering the fixed tolerances if those were input.  This will produce a new budget, and after running FAMC again, the statistics may be more to your liking.  When the results are satisfactory, the BTOL budget from that iteration is the budget you should send to the shop.

 

It goes without saying that you must be sure that they understand what you have done, which elements are to be made in which order, and the importance of sending back to the designer the measured data as each element is made so the lens can then be reoptimized with SYNOPSYS.  Close cooperation between the designer and the shop is vital to the success of this approach.  If anyone neglects this interaction, your lens will not work.

 

FAMC displays the statistics of only the last set of adjustment parameters, because the other variables do not have the same nominal value on successive iterations, and listing the departure from those changing values has little meaning.

 

You can ask for statistics on other quantities with the MC IZn input, described in the section on MC.  This input should occur just before the PHASE 3 input.

 

FAMC Example:

 

 

This lens is found in X25.RLE.  First we optimize the lens with the merit function that we are going to use for the fab adjust.

 

PANT

VLIST RAD 1 2 3 7 6 7 8 9 10

VLIST TH 2 4

END

AANT

M 1 10 A FNUM

ETN 1 10 2

GSR .75 1 4 1 0 0

GSR .75 1 4 2 0 0

GSR .75 1 4 3 0 0

GNR .75 1 3 1 .75 0

GNR .75 1 3 1 1.0 0

GNR .75 1 3 2 .75 0

GNR .75 1 3 2 1.0 0

GNR .75 1 3 3 .75 0

GNR .75 1 3 3 1.0 0

END

SNAP

SYNO 10

 

(The edge clearance at surface 3 is likely to become negative, so we control it with the ETN aberration.)  When the optimization is finished, we store the lens in library location 3.

 

Now we want to see how tight the tolerances must be in order to maintain a spot variance of 0.0016.  Let us assume that melt data are already applied to the glasses, so the values are exact and need not be perturbed.

 

CCW

GET 3

SPE

BTOL 2

TOL SPOT .0016

EXACT INDEX 1 3 6 8 9

EXACT VNO 1 3 6 8 9

PREP MC

GO

 

The budget that comes out requires some thickness tolerances of 18 microns, and a centration of 6 microns.  Let us see if we can achieve this kind of quality with looser tolerances by using fabrication adjustments.  First, we rerun the BTOL analysis, this time requesting a spot variance of 0.002.  Tolerances are about three times greater than the previous.  We save the budget with BTOL SAVE, so we do not have to create it over again later if we start again another time.  The file name is just the current log number.  Let us see what FAMC will do:

 

CCW

GET 3

BTOL FET 335                ! assuming the file name was 335.BTO

FAMC 100 3 QUIET -1 ALL 5

FAORDER 8 6 3 1

PASSES 10

 

PHASE 1

PANT

VLIST RAD 1 2 3 7 6 7 8 9 10

VLIST TH 2 4

END

AANT

M 1 10 A FNUM

ETN 1 10 2

GSR .75 1 4 1 0 0

GSR .75 1 4 2 0 0

GSR .75 1 4 3 0 0

GNR .75 1 3 1 .75 0

GNR .75 1 3 1 1.0 0

GNR .75 1 3 2 .75 0

GNR .75 1 3 2 1.0 0

GNR .75 1 3 3 .75 0

GNR .75 1 3 3 1.0 0

END

SNAP

EVAL

 

PHASE 2

PANT

VY 3 YDC 2

VY 3 XDC 2

END

AANT

GNR .75 1 4 1 0 0 0 F

GNR .75 1 4 2 0 0 0 F

GNR .75 1 4 3 0 0 0 F

GNR .75 1 3 1 .75 0 0 F

GNR .75 1 3 1 1.0 0 0 F

GNR .75 1 3 2 .75 0 0 F

GNR .75 1 3 2 1.0 0 0  F

GNR .75 1 3 3 .75 00  F

GNR .75 1 3 3 1.0 0 0 F

GNR .75 1 3 1 -.75 0 0 F

GNR .75 1 3 1 -1.0 0 0  F

GNR .75 1 3 2 -.75 0 0  F

GNR .75 1 3 2 -1.0 0 0  F

GNR .75 1 3 3 -.75 0 0  F

GNR .75 1 3 3 -1.0 0 0 F

END

SNAP

SYNO 10

PHASE 3

 

 

The histogram on the left shows the ordinary MC statistics for this budget with only a focus solve on the final airspace.  The requested performance of 0.002 is met quite well at the two-sigma level in this case, as one would expect, but this budget will not achieve the desired level of 0.0016.  The plot on the right is the performance of FAMC, and it is clear that the performance is now about 0.0016 at the same level.

 

This example shows how one may sometimes achieve tolerances that are two or three times looser than would otherwise be required, by employing the process of fabrication adjustments.  Let us comment on some of the above input.

 

The elements in this case will be manufactured starting from those at the back of the lens, since the budget was somewhat tighter for the rearmost elements.  If this is not convenient, one can easily examine what happens with a different construction order, and advise the shop whether it matters.  The AANT file for phase 2 included raysets for both positive and negative values of HBAR, since at that phase the lens will no longer be axially symmetric and we don’t want the program to adjust the lens so the image at field point +1 is excellent and wind up with a lens with a poor image at –1.

 

A worst-case example will be found in library location 5, as called for by the FAMC input.

 

In case it is not obvious, the FAMC feature does not utilize the adjustment MACro MCFILE.MAC that MC does.

 

A run on FAMC takes many times longer to run than does an MC analysis.  Because of the complexity of the calculations, the process only takes place in core 0, and there is no advantage in requesting more cores.
