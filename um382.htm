3.8.2 Global Coordinates                                        Previous Next TOC

 

As mentioned in section 2.4.1, there are five possible rules for entering tilts and and decenters.  This section discusses the GLOBAL rules, which apply when you want to enter three Euler angles to define the orientation of a surface and give the location in the coordinate system of surface 1.

 

GLOBAL data may be edited in the SPS dialog, as well as by the WorkSheet.  In an RLE or CHG file, a surface may be defined GLOBAL with input of the form

 

 

SN GLOBAL

or

SN GLOBAL POSITION XG YG ZG

and

SN GLOBAL ANGLE AG BG GG.

 

The first form is used only to change a surface that has previously been entered RELATIVE or LOCAL; it merely translates the present surface location into GLOBAL coordinates.  (This should only be used in a CHG file, since the global positions are not calculated until after the original RLE file has been ended.)

 

The second form gives the position and angles of the surface, in the coordinate system of surface number 1.  (It is also possible to enter or change any of the global coordinates in AI, with specific mnemonics -- see Chapter 15.)  If the surface is not yet global, the current position will be changed to global coordinates before the new data are processed.

 

In the above input, the mnemonic ANGLE specifies the Euler-angle order (α,β,γ), where AG is alpha, and so on.  You may specify a different order by substituting one of the following mnemonics:

 

 ABG

 AGB

 BAG

 BGA

 GAB

 GBA.

 

For example, “5 GLOBAL BAG 5 10 3” says that the Euler angles are to be taken in the order (β,α,γ), where alpha is 5, beta is 10, and gamma is 3, and assigned to surface 5.

 

You can vary the global positions with variables in the PANT file using the mnemonics

 

AG        global alpha tilt angle

BG        global beta tilt angle

GG        global gamma tilt angle

 

XG        global X-position

YG        global Y-position

ZG        global Z-position ;

 

for example: VY 33 YG, to vary the global Y-coordinate of surface 33.

 

You can also target them in the AANT file with the same mnemonics, for example

 

M 22.7756 1 A ZG 21

 

giving a target value of 22.7756 to the global Z-coordinate of surface 21.

 

If a surface has been defined GLOBAL (or LOCAL), the thickness of the previous surface no longer controls its position, since the GLOBAL (or LOCAL) data override any TH that may have been entered.  However, a thickness must still be defined for purposes of paraxial raytracing.  Even though the paraxial programs ignore tilts and decenters, in the case of GLOBAL surfaces the global data are the only position specification available.  From these data an effective "thickness" must be derived, and this is done in accordance with one of two options, selected with the system parameters GTS and GTZ (see 3.2).  Since these options can, in some cases, produce unexpected results, the use of paraxial solves following GLOBAL (or LOCAL) surfaces is not recommended unless you are very careful.

 

1.

The vertex separation of the global surface and the preceding one is used for TH (default).  This is given the sign of the Z-component. (GTS)

 

2.

Only the Z-component of the vertex separation is taken as the thickness.  This is done in the coordinate system of the global surface. (GTZ)

 

Global input is incompatible with thickness pickups or solves on the previous surface.  Surface number 1 cannot be made GLOBAL or LOCAL, and if that surface is tilted or decentered, other surfaces that are GLOBAL will follow it.  (But we do not recommend putting a tilt or decenter on surface 1.)  Global data may be converted to relative with SN RELATIVE, to group with SN GROUP, to local with SN LOCAL, and removed entirely with SN NAS.

 

Referring to the tilted lens system shown above, suppose that surfaces 3, 4, and 5 were to be entered as global surfaces.  The input might be:

 

CHG

3 GLOBAL POSITION 0 0 1.85

3 GLOBAL ANGLE 45 0 0

4 GLOBAL POSITION 0 -1.12 1.85

4 GLOBAL ANGLE 90 0 0

5 GLOBAL POSITION 0 -1.47 1.85

5 GLOBAL ANGLE -90 0 0

END

 

Surface 6 has not been redefined (it could also have been GLOBAL), to illustrate the fact that it is still in the same place relative to 5, regardless of how surface 5 was placed where it is, because it is a RELATIVE surface – by default).

 

In the above example, the vertex-to-vertex spacing was the logical selection for obtaining a paraxial thickness (the default).  Now we will use global coordinates to decenter the center element of a triplet (and show how AI can change the lens).  In this case, the Z-component of the separation should be taken as the thickness.  The input for changing the lens is:

 

CHG

GTZ

5 GLOBAL

END

AI

Change YG of surface 3 to 1.5
