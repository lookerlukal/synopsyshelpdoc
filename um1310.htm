13.10 Capturing printout (PON, POFF)                        Previous Next TOC

 

Output that can be listed in a Command Window may be captured for later printing or stored in a named data file for offline processing.  The commands

 

 PON

 …

 POFF

 

will capture all data generated between the PON and POFF commands.  If you later follow this with another PON, then data capture resumes, adding to what was captured before.

 

To close the capture file and send the data to the printer, use POF C or POF CLOSE.  To close the capture file and discard it without printing anything, use POF D.

 

You can also request that the program save the captured data to a named file, rather than to the printer.  To do so, use the form

 

 PON [ filename [ NEW ] ].

 

The named file will have a file extension of “.TXT” if no extension is given in the name.  Capture stops when the POFF command is entered, as before, and may be resumed with another PON filename.  In this way, selected data may be extracted from your output.  (You must reenter the same filename for this to work properly.)

 

When the POFF C or POFF CLOSE command is entered, the capture file is closed and is ready to be opened at any time in Notepad or another text editor program.  The output data in the file are thus available for processing with any program that is capable of reading a .TXT file.  This is a convenient way to create a file that you want to send by email to someone.  You can of course copy and paste the text itself directly from the Command Window into an email client, but in that case the font will most likely be a default proportional-spaced font, and columns of data will not line up correctly.  The .TXT file preserves the font information and looks correct when opened later.

 

If no filename was given in the PON command, then when the file is closed the data are sent to a temporary new Command Window, which is then processed for printing.  This will show all of the captured data along with a Print dialog box where the user can select from the available printers and then click OK.  If the user clicks CANCEL, no printing occurs.

 

You can reopen a named capture file and replace the previous contents with new data, even if it has not yet been closed.  For this, add the optional NEW in word three of the PON command.

 

The command

 

 POFF D

 

will close the current capture file and discard all contents.

 

Printout may be captured with PON even if the system is currently in QUIET mode.  This may save some time if the printout is lengthy and you don't want to see it on the screen first.

 

The font size of hardcopy printouts may be adjusted via the menu item Options|Set Printer Offset, Margins, Font.
