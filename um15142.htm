15.1.4.2 The Zn parameters                                                Previous Next TOC

 

The second form of AI equation evaluation involves Al sentences with less restrictive format rules than the COMPOSITE form, and a set of 20 parameters called Z1 to Z20.  In this case the Zn parameters are defined in an ordinary AI sentence, and the evaluation takes place when the equation line is input (the COMPOSITE command is not required).  The first example of the previous section could be input

 

Z1 = thickness 2

Z2 equals radius 3

= Z1 / Z2

 

and the second might be

 

Z1 is YA on surface 4 for YEN = 1

Z2 = YA on 2 at full aperture

= Z2 - Z1

 

The results of the equation evaluation are displayed on the screen and automatically placed into FILE location 1.

 

Note that the values of the Zn may be set by AI in only two ways:

 

 Zn = any quantity that may be evaluated by AI

or

 Zn = a fixed number

 

It would be incorrect to attempt the form “Z1 = Z2 - Z3”, for example, since an equation may be evaluated only with a line that begins with the equals sign.

 

The value of the parameters Zn may also be changed in optimization, with the variable mnemonic "VZN."  This is most often used in connection with the CLINK aberration, which can execute a command or re-run the alternate MACro – where the Zn parameter may appear in place of a number.  See the above link for more information.

 

If you evaluate your equation as described in this section, the program does not utilize the COMPOSITE aberration evaluation and therefore does not overwrite any AANT file data you might have input earlier.  In contrast, the COMPOSITE formula creates its own very small AANT file, and you lose any previous definition.
