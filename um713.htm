7.13 Adding Tolerances to your Drawings (USE BTOL)                         Next Previous TOC

 

The tolerancing program BTOL will calculate a sophisticated tolerance budget for your lens and print a table showing the results.  Often you will want to prepare element manufacturing and assembly drawings with those data included, and it would be convenient if the program added them to the drawings for you.  There a problem with this idea, however: Whereas BTOL gives you a budget according to a flexible set of user requirements, there is much about the lens mounting design and the capabilities of your optics shop that it does not know about.  For these reasons, you will always want to study the BTOL output carefully to ensure that it is reasonable under your particular circumstances, and you might wish to change the budget in some cases from what BTOL has calculated.

 

Examples where you will modify the tolerance budget yourself are easy to imagine.  A modern optics manufacturer will usually have a numerically-controlled lathe, which can make the lens cell with mechanical precision that is much greater than what you can require from the lens makers.  In this case, you would assign all of an airspace tolerance to the sag tolerance on an element, if that sag affects the airspace.  If your shop is not so modern, on the other hand, the tolerances on the lens cell might have to be larger – because that is all your shop can do – and then you would reduce the sag tolerance on the lens that fits into that cell.  Perhaps two elements should share a sag tolerance, and so on.

 

So it is not wise for the program to simply add the raw BTOL tolerance values to your element drawing: once the drawing is annotated, it is not easy to change the numbers.  It is easy, however to edit the lines of text that you add yourself with the Annotation Editor (Click on the button when the graphics window is open.), so the program adds the required tolerance data in that form instead.

 

The USE BTOL command, which may be entered before an ELD or DWG command, will read the data file that is saved by BTOL and transfer all of the tolerances found in that file to the element drawn by ELD or to the assembly tolerances on DWG drawings.  Since those lines of annotation are formatted in the same way as text lines created by the annotation editor, you can select, edit, repostion, or delete them if you wish.

 

We will illustrate this feature with the lens in library location 1, the infrared MIT lens.

 

 

First, we create a tolerance budget.

 

GET 1

CHG

NCOP

END

 

SPEC

BTOL 2

RANGE 3 RL1 .1

RANGE 5 RL2 .1

DEGRADE SPOT .2

TPR ALL

GO

 

This input causes BTOL to create a data file named BTOL_ELD.DAT, in the current lens directory.

 

Now let’s make an element drawing of the second element;

 

USE BTOL

ELD 3

ID EXAMPLE LENS NAME

ID ELEMENT 2

ID YOUR COMPANY NAME

PLOT

 

 

Now you see the tolerances that apply to this element, taken directly from the BTOL file, in their proper place on the drawing.  Here is an assembly drawing:

 

USE BTOL

DWG -2

 

 

Here, the tilt, decenter, and airspace tolerances are annotated for you.

 

The program will position these annotation lines across the top and bottom portions of the plot, using space that is adequate for 15 lens elements.  If your lens contains more than 15 element, you may have to move things around a bit to avoid overlap -- or else draw your lens in sections.

 

It goes without saying that you must ensure that the lens that you are drawing must be the same one for which the tolerance budget was calculated.  The program does not check this for you, so be careful.
