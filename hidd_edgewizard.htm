Edge Wizard Dialog

 

The Edge Wizard dialog can help you create and edit the parameters that define the geometry of the edge of a lens element. These data can also be entered in the RLE or CHG file, but most users will prefer the Wizard because it lets you watch the edges change as you click on spin buttons in the dialog. You can access the Wizard by clicking on the PAD button , with the menu Wizard | Edge-definition wizard, or the command MEW (menu Edge Wizard). When the dialog opens, you select a lens by clicking on the first side of an element in the PAD display, or by entering the desired surface number and clicking “Update”. Switch 39 must be ON for these edges to be shown.  You can navigate to successive elements with the buttons “Prev. El.”  and “Next El.”.

 

If the lens element does not have any EFILE data when you start the Wizard, you can create a default starting edge by selecting “Explicit” and clicking “Update”. Or you can click “Create all” to generate starting edges for the entire lens. Initially, the newly created edges will be declared EXPLICIT (see below). Edge data cannot be assigned to lenses where the second side is tilted or decentered with respect to the first, or for other special conditions that will be explained in the dialog.

 

Edges can be assigned to one of seven categories:

 

OFF

No EFILE data are in effect.

 

UP

Edges can change as the lens changes, in accordance with rule set 1, below.

 

DOWN

Edges follow rule set 2.

 

MIXED

Edges follow rule set 3

 

EXPLICIT

Edge data are fixed at the entered values. This mode is best for cemented surfaces.

 

EXPLICIT MIRROR

Same as EXPLICIT, except the back surface of the mirror is drawn as well.

 

COPY MIRROR

Used for mirrors where the back surface is a copy of the front surface.

 

Categories 6 and 7 apply to mirrors in air. You may change the current category of an element by selecting the appropriate radio button and clicking “Update”.

 

Edges are drawn in accordance with the data assigned to six points, called A through E in the figure below, and optional angles ANG1 and ANG2.  These angles specify the slope of the normally flat portions from A to B and D to E, in degrees, and are usually set to zero.  (The bevels are always at 45 degrees.)  If you want to specify a conical portion, for when you grind the back of a mirror at an angle to the axis that is not 90 degrees, for example, then use a non-zero ANG1 or ANG2.  A positive angle on either side slopes that face into the lens.  ( An example is given in the installation directory SYNOPSYS\USER in the file 4.RLE.)

 

 

For convex surfaces, points A and B, (or E and D) should normally be coincident unless the lens is molded in plastic with a flange that extends beyond the clear aperture. The lens system as a whole has two other parameters, the MARGIN and BEVEL, which are used by some but not all of the EFILE options. The rules applying to these options are as follows:

 

1.

For the UP option, point A is found by starting at the clear aperture of the surface and adding the current MARGIN. Initially, point B is at the same place, and point C is found by adding the local BEVEL to point B. The local BEVEL is set equal to the system BEVEL on a create operation, but afterward can be changed independently. The same procedure is done for the second side of the element, and the values of C are compared. The larger value is then assigned to both surfaces, and the side that had the smaller C is adjusted so that the new point B (or D) is below the new C by the BEVEL, and if convex, point A (or E) goes to that point as well. Concave surfaces acquire a face dimension AB (or ED) when adjusted. This option redefines the edges whenever the clear apertures of either side, or the MARGIN, changes. You can enter explicit values for the bevels and face dimensions in the Wizard.

 

2.

The DOWN option works from the top down; a value of C is entered by the user, and ponts B and D are found by going down by the current BEVEL dimension. A face value may be entered as well, and points A and E are then placed that distance below the start of the bevel. This option does not employ the current MARGIN

 

3.

The MIXED option combines the above two: point C is entered explicitly, and then points B and D are found by subtracting the local BEVEL dimensions. Points A and E are found by adding the system MARGIN to the current clear aperture of the surface, except for convex surfaces, where the face dimension is kept at zero.

 

4.

The EXPLICIT option differs in several respects from those listed above. Points A through E are entered explicitly (although the Wizard will alter them if necessary to keep the bevel and face dimensions positive). These dimensions are therefore independent of the clear apertures on either side of the element, and care must be exercised to ensure that the edges clear these apertures. The most important difference, however, is that with this option, points D and E and angle ANG2 are independent of the geometry of the second surface itself. This is important when dealing with cemented elements.

 

Let me explain. Once assigned to a surface, the edge geometry becomes part of the lens file and is saved with the lens. But the RLE file data refer to a single surface at a time, not to a whole element. You may enter data in the RLE file that define the dimensions AB and BC for that surface, and then for the next, and so on. But the drawing programs require points A through E on the first side of each element before they can draw the lens properly. So options UP through MIXED have to transfer the missing data from the second to the first side to complete the description. All this is transparent to the user, but it has the consequence that the edges of a cemented surface are coupled in a way that is often undesirable. So for cemented elements the EXPLICIT option is strongly recommended.

 

5.

The EXPLICIT MIRROR option is used for mirrors in air, where you want to emulate the thickness and geometry of the rear surface (which is not present in the lens file).  The usual EFILE entries for points D and E then apply to the rear surface, and you can enter and change the mirror thickness as well as the slope angle on the flat portion between points D and E.  Rectangular outside aperture data (RAO) will be shown if present, and the current CAO if not.  You cannot show a combination of circular and rectangular shapes for mirrors, as you can on lenses.  The EFILE format for this type is SN EFILE MIRROR THICKNESS.  The normal EFILE coordinates A,B,C,D,E apply to the front and back of the mirror and are entered in the usual manner, or with the Wizard.

 

6.

The rear surface of a mirror defined with the above option is normal to the axis, as is the second side of all EFILE elements.  If a mirror is strongly curved or decentered -- or has a UAP aperture -- the COPY MIRROR option is recommended instead.  The rear surface then is assigned a copy of the geometry of the front side, separated from it by the entered thickness.  It is thus not necessarily axially symmetric or centered.  The EFILE format for this type is given by SN EFILE CMIRROR THICKNESS PARENTCAO.  If the parent CAO is not large enough to cover the entire area of the active surface (as given by CAO and RAO data) the surface will only extend to the size of the parent.  Thus you can define a combination of circular and rectangular edges for this type.

 

If EFILE data are in effect, then points A and E are tested along with any hard CAO that may be assigned to a surface, when vignetting is evaluated.  The other EFILE points are ignored.

 

For convenience, we have provided four buttons labeled "-B" and "-F" on the Wizard.
These buttons will make the value of the current bevel or flat portion of the corresponding side go to zero.

 

The button "This element" redefines the drawing at the top of the PAD screen, showing only the currently selected element.  The button "This range" draws the selected range of surface numbers.  Use these buttons only if you want to redefine the display.  If you just want to redraw the picture, use the "Update" button instead.

 

The button "No CAOs" has been provided to remove any user-entered CAO data on the selected element.  Normally one will specify either hard CAOs or EFILE edges – but not both, to control the edge dimension explicitly.  The reason for this is that once the EFILE data are entered, they fill the function that the hard CAO data fill when EFILE data are not present.  They govern the edge that appears on drawings, and are honored when vignetting is tested.  And if the CAO data are allowed to float (i.e., if hard CAOs are removed), some of the EFILE options (UP, MIXED) float along with them – which is a nice feature.  And last, if you alter the EFILE data in the Wizard such that points A or E fall inside the clear aperture, you will see a warning message on the Wizard.  It is therefore helpful if the aperture data represent that needed by the rays, which is the case with default apertures, but not necessarily true with user-entered hard apertures.  This button saves you the trouble of removing the CAOs using a different window.

 

The button “Set CAOs” does the opposite:  Clicking here will transfer the dimensions of points A and E to hard clear apertures on both sides of the element.  The main use of this feature is when you have created exactly the amount of vignetting you want, using EFILE edges, and now you want to create a VFIELD description of the resulting vignetting.  The command FVF will do that for you – but that feature looks at the CAO data, not the EFILE data.  Once the data are transferred, FVF will do the job.  If the active surface is a mirror in AIR, only one surface acquires CAO data in response to this button.
