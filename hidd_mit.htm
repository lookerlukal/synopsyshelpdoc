Image Tools Dialog

 

Command shortcut: MIT

 

The Image Tools dialog can show the effect of lens aberrations and diffraction on the image of a variety of targets.  The target field is constructed with a 400x400 array of bins, filled with the selected target intensities, and then blurred in accordance with the effect to be modeled.  To use this feature,

 

1. Enter the desired image scale in the data box “Reference dimension”.  Select the units.  If you select lp/mm, the reference dimension is the inverse of the spatial frequency; thus, 20 lp/mm scales to 0.05 mm if the lens is in units of MM, or 0.001969 if it is in INCHes.

 

2. Select the effect you want to model from the following list:

 

Geometric

This option expands each picture element as a spot diagram

 

Diffraction

The diffraction point-spread function is applied to each bin.  If the diffraction pattern spills out of the sample area (i.e. if the aberrations are too large), then select the W option.  The area will be doubled, and you are advised to double the number of rays requested as well.  (This affects the image model, not the Reference dimension.)

 

Coherent

The image is analyzed by the PARTC program, and rendered as a map of intensities.  If the phase entry is zero, the target is taken as an intensity-modulated pattern of unity contrast.  If any non-zero number is entered for the phase, then the target has zero contrast but a phase difference given by the entered number.

 

The coherent effect is recommended for analyzing a point target.  See below.

 

This analysis was originally intended for microlithography lenses, which are always telecentric and without pupil aberrations.  It has been enhanced in order to model the effect of the incidence angle at the image, when that is not 90 degrees -- but only in X or Y, not both.  And it does not consider mapping of the raygrid geometry at the exit pupil.  In other words, the OPD map is taken in entrance pupil coordinates.  So do not use this analysis for beam shaping optics, where the geometry differs between the two pupils.

 

Partially coherent

Same as above, except the coherence may be entered into a data box.  A value of 0 is fully coherent, while a value of 2.0 is fully incoherent.  In practice, a value of 1.0 is almost identical to 2.0 -- and runs four times faster.

 

Gaussian

If the illumination has a Gaussian intensity profile instead of uniform, check this box and enter the relative beam size in the box to the right.  If the edge of the illuminated patch occurs at the 1/e**2 point, this number is 1.0

 

3. Select the target to image, from the following list:

 

Sinewave

The number of cycles drawn over the display can be selected by entering a multiplier in the X--> box, where 1.0 yields four cycles over the displayed area, and one cycle over the reference dimension.

 

Squarewave

You may select a desired duty cycle between 0 and 1, where 1 is always up and 0 is always down.  The number of cycles may be selected by the X--> box.

.

3-bar target

Three bars, at a width of ½ of the reference dimension.

 

1-bar target

One bar, at a width of ½ of the reference dimension.

 

Edge

A knife edge, from bright to dark

 

Text

A sample of text, in Times New Roman font, approximately 2.3 characters per reference dimension

 

Slit

A slit, one bin in width and as long as the display

 

Point

A single point.  This gives the spot diagram or the point-spread function, if those effects are selected.  For this target, the coherent effect is recommended, however, rather than the other two, since a point source is always coherent with itself, the calculation is faster than is the diffraction integral, and the results are more accurate.  See below.

 

Circle

A circle whose diameter is the reference dimension

 

Double star

Two point sources, separated by the requested number of bins.  If the points are too close together and the cross section shows bin overlap, use the Smooth option to reduce this artifact – or try the PARTC program, which can draw a smoother curve.

 

Square

A square whose side length is the reference dimension

 

Photograph

With this feature you can process an image of your choice.  See below.

 

AF Target

A pattern of 3-bar targets of varying frequencies, derived from the Air Force resolution targets.  See below.

 

Array

This option opens a box where you select the dimensions of each element of a target array, which may be composed of circles or rectangles, and the array spacing.

 

4. Select the orientation, X or Y.  (The text, point, and photograph options ignore this selection.)

 

5. Enter the desired number of rays, field point, and color.  You may enter “P” for the primary color.

 

6. Select from “Perfect” or “With aberrations”.  The Perfect option will show the original target without any effects at all, if the geometric model is selected.  If the diffraction option is selected, the perfect option will cause the OPD errors to be set to zero – but will still show the effects of apodization and vignetting on the diffraction pattern.

 

7. Click “Process”.  The target is convolved with the image model, and displayed in a large box.  The image model itself is shown in a separate, smaller box (except for the coherent options, which do not involve an image model).  Here is an example of text as blurred by a lens:

 

 

8. Enter a higher number in the “Gain” box if you want to see the fainter portions of the image more clearly.  This will, however, saturate the peaks, which then do not appear in their true colors.  (When converting the display to a graphics display, the gain setting is honored, but the cross-section and other plots always show the raw data, unaffected by the gain setting.)  A gain of 1.0 covers the entire image linearly (except photographs).  You can also select the "Nonlinear" checkbox to display intensities on a logarithmic scale, to better allow for the small contrast range available on the monitor.  Low intensities will appear brighter, but high will be about the same, avoiding saturation.  This option is recommended when you want to see the point-spread model more clearly – but is not appropriate for analyzing the blur of a photograph, for which the image should be a linear model.  The cross-section and other plots are affected by the nonlinear setting.

 

For your convenience, a slider is provided at the left of the dialog to adjust the gain by small increments.  This is especially useful when you model a photograph and want to get the best brightness level.

 

9. You can convert the display to a normal graphics picture with the button “Convert to graphics image”.  This picture can be printed or saved.

 

10. Clicking on the Section button will create a picture showing a cross section of the displayed image in X or Y, selected by radio buttons.  You can also select a slit trace, a knife-edge trace, or pass a square detector over the image in either direction.  The slit and square may be assigned a dimension, in bins.  The reference dimension is 100 bins wide, and the default slit or square trace is but a single bin in size.  You may enter a number between 1 and 400 for the No. Bins.  If you want the center of the slit or square to be centered at zero at the zero point on the trace, be sure to ask for an odd number of bins.  Otherwise there is no center bin.  The plot will be normalized so the peak is at 1.0 if normalilzation is selected.  Otherwise it will show the fraction of the total energy that is on the detector (except for the cross-section plot, which models a slit 1 bin wide and is always normalized.)

 

11. If the lens is FOCAL, the MIT dialog will display a focus slider below the image pane.  Clicking on the slider or sliding the thumb will alter the back focus distance by an amount that depends on the value in the focus range data box, with a default of 0.01, which you may edit, and will then recalculate the display.  No paraxial raytrace is performed, so this function works even if the lens has a YMT solve before the image.  When you leave MIT, the back focus remains where MIT left it – but if there is a solve in effect, it will disappear the next time a PXT is done.  So if you want it to be permanent, please delete the solve first.

 

It goes without saying that, to use the focus slider, you should select the target and effect so the procedure executes very quickly.  Otherwise the time lag may be excessive.

 

Since the MIT display is divided into 400 bins, the requested trace will be discontinuous.  To smooth the trace, select the Smooth box before you request the trace.  Here is an example of the cross-section trace.

 

 

11. You can also get a printed list of the 400 data points in the trace if you select the List checkbox before requesting the trace.

 

12.  The processed display shows the selected target convolved with the point-spread function of the lens.  It is possible to sample that image with a detector array, to evaluate the effect of binning on the processed signal.  This is the function of the Post Process button.  The program creates a row of detectors aligned in a selected direction, and each bin contains all of the energy within that area.

 

The alignment of the array is always perpendicular to the (X,Y) target selection, except for the double star, where is it parallel.  Thus, if you select a sinewave target in Y, the stripes are up and down (along the Y axis) but the modulation information, and the detector array, is in X.

 

Before you can use this feature, you must fill in the data boxes below the button, giving the X and Y dimensions of each bin, and optionally a decenter value as well.  These data are lens units or um, if those units are selected, or in lens units if the cy/mm option is selected.  The largest allowable bin covers the entire sample area, 400 x 400 pixels.

 

To clarify the several steps involved in this calculation, let us summarize the operations.

 

1.  When you click Process, the program creates a model of the target geometry.  This is displayed briefly for some targets but not for others.  (You can see it by selecting Perfect and Geometric, if you want.)

 

2.  It then calculates the point-spread function of the lens, which is displayed in the small box on the upper right.

 

3.  Then it convolves the PSF with the target geometry and displays the blurred image.

 

4.  If you click the Post Process button (after entering appropriate data), the program masks the displayed image with a detector array of the entered dimensions and superimposes this array on the display.

 

5.  If you click any of the buttons Section, Knife, Slit, or Square, the program creates a plot of the requested analysis of the displayed image.  If the Post Processor was invoked, the plot will show two curves, the second being a trace through the detector array.  In this case, only the Section option is available.  Note that the Bins dimension on the Square trace is separate from the array dimensions for the post processor – and only one at a time can be used.

 

6.  The Solid option is available for most target types and converts the display into a form that can be zoomed, rotated, and panned with the rotating solid model window.  This is intended primarily for examining a point image, and for this target the coherent effect is recommended.

Several things should be noted:

 

1. The geometric image is actually an X-Y-plane projection of the image, oriented with the Y-axis of the image plane pointing up.  If the image surface is curved, this projection shows a foreshortened image. (This behavior is the same as all of the spot-diagram options in SYNOPSYS.)

 

2.  The spot diagram is fit into a bin of 60x60 elements prior to processing.  This allows us to indicate the intensity of the image at each bin by the RGB values displayed at that point, and also facilitates showing the image with approximately the same values that a viewer would see.  The normal spot diagram options, on the other hand, show each ray as a spot (or symbol) with a fixed intensity and with color either taken from a fixed table (if switch 85 is off) or calculated to correspond roughly to what the eye would see (if that switch is on).  The result with the dialog is therefore more like the visual appearance of the image than is a spot diagram, but it suffers from loss of resolution in the event that several rays fall into the same bin.  The effect is most pronounced when the spot consists of a small intense core surrounded by a large faint flair.  Since the program scales the bin array to fit the entire spot, in this case the small core might be sampled with but a few bins – giving a rather coarse resolution in the processed target.  A spot that is more spread out, on the other hand, gives a smoother display.  For these reasons, a point target is best analyzed with the coherent effect, which avoids most of the issued associated with binning.  On the other hand, since the spot diagram is calculated much more quickly than the other image models, the geometric effect is recommended when you want to examine the image as it would appear as you shift the focus with the Focus shift slider.

 

3. The diffraction image is shown as it would appear looking normal to the image surface, with the upward direction oriented in the direction of the marginal ray as seen from the chief ray (and this direction may not be toward the local Y axis).  The ray coordinates are rotated in the order (alpha, beta) so that the ray angles are measured from the surface normal, if the image surface is not flat.  The geometry and dimensions of the targets are fixed; that is, there is no foreshortening of the target shape due to the incidence angle of the light, even if the image surface is tilted or curved, since the view is looking normal to the surface.  If the image is poorly corrected, the wide-area option should be selected in order to avoid loss of information from parts of the image beyond the default sample area.  If the wide option does not encompass the entire diffraction pattern, or if the target is a point source, then the coherent effect is recommended instead of the diffraction effect.

 

4.  The text target requires significantly more processing than most of the other targets, especially if the image model blur is large compared to the individual letter dimensions.  For a large blur, the whole process can take many minutes, and you are advised to first examine the image with the point option to verify that its size is reasonable.  In any event, you can click the Stop button on the progress bar to abort a lengthy process.

 

5.  The colors displayed approximate the visual effect of the wavelengths analyzed.  Infrared and ultraviolet wavelengths are shown in red and violet, however, even though the wavelength itself is invisible to the eye.

 

Keep in mind that if you are trying to examine the way an image would appear to the eye, you may not get what you expect.  It is not easy to get a true visual appearance, because the brightness range of each RGB color that is displayed is only 256 shades – while a real-world image will have range of perhaps a million or so.  But if you assign 10 wavelengths spread across the visible spectrum with the sensitivity of the eye giving the spectral weights (the Spectrum Wizard MSW makes this easy to do), then you should get a more-or-less accurate image.  If, on the other hand, you try to make do with, say, three or five wavelengths, then the processed image of a white photograph will show some coloration.

 

A further note applies to multicolor analysis:  The RGB values on the screen are adjusted by the program so the display shows approximately what the eye would see when looking at the image.  If you request a trace or other evaluation, you will analyze that set of RGB values, not the original input energy distribution.  So the relative intensity of the colors of a multicolor image will usually not model that distribution very accurately.  This is especially important when some wavelengths are outside the visual range; in that case, the program arbitrarily assigns them a red or violet color on the monitor.  Obviously, a trace of that pattern will not faithfully reproduce the raw energy distribution.  If your goal is to get an accurate trace, be sure to turn off switch 35 first.  Then the program displays the pattern as a greyscale, with no attempt at color mapping to the eye sensitivity curves.

 

6.  The coherent and partial coherent effects utilizes the PARTC program, and you are advised to read the section in that link to get a feeling for the counter-intuitive results that can be produced by this kind of analysis.  For example, if the system is obscured, the contrast will fall to zero for both higher and lower spatial frequencies.  Some of the targets that can be selected are calculated in only a single dimension (the sinewave, for example) and then simply duplicated across the picture.  Those targets are analyzed by PARTC.  Other targets require every point in the display to be evaluated (the photograph, array, etc.) and in that case the program uses a special 2-dimensional Fast Fourier Transform version of PARTC.  This analysis takes somewhat longer, especially if the aperture filling is not zero.

 

There is an important difference between the diffraction and coherent effects that you should be aware of.  The former evaluates a diffraction integral, modeling the location of each point in the image surface and in the exit pupil.  All geometric effects are therefore taken into account.  The coherent effect, on the other hand models the wavefront with a Zernike expansion over the entrance pupil, and the image analysis considers only the wavefront shape, not the geometry that shows up at the image.  If the chief-ray incidence angle is non zero, of course the image is stretched as a result, but that is not considered in the the FFT analysis.  SYNOPSYS will stretch the image in either X or Y, if required, which is a simple added step.  But that does not work if the incidence angle is compound and requires stretching in both directions at once.  The program will detect such cases and issue a warning message.  You are then advised to rely instead on the diffraction rather than the coherent effect.

 

If your target is a photograph (see the next paragraph), then analyzing a coherent phase mask requires that the photo consist of only pure black and pure white.  Black areas are given the desired phase shift.  If there are any grayscale values in the picture, the results will have little meaning.

 

When you analyze a coherent image, keep in mind that the edge of the sampled picture or array is part of the picture.  That is, bars that go to infinity are analyzed as if they ended right at the edges, which of course they do in this model.  Those (sharp) edges are then subject to the same blurring and coherence effects as everything else in the picture, resulting in “edge effects”, where artifacts can appear that would not be present if the bars actually went to infinity.  An example of this effect is shown below.

The coherent effect is recommended for all point source targets, instead of the geometric or diffraction effects.  The former ignores diffraction and is therefore not accurate unless the image is much larger than the Airy diffraction disk – while the latter ignores energy beyond a sample area that is about 6 times larger than that disk, or twice that if the wide area option is selected.  And both are subject to binning effects, since the image is partitioned into a 60x60 array of bins covering the entire image, and any structure finer than that is lost.  Although the latter effect is small for a diffraction image, it can be major for a poorly-corrected geometric image.  If the system has a large obscuration, the coherent image analysis is more accurate if you request a partial coherence analysis with a fill factor of 2.0.  (It is not necessary to use a large fill if there is no obscuration, since a point target is fully coherent with itself.)  This is to avoid artifacts arising from describing a circular optics problem with a square array of pupil samples, many of which are obscured.

 

While the largest image area that can be sampled by the wide-area diffraction-effect option is about 12 times the size of the Airy disk, as mentioned above, the coherent analysis can go considerably further, out to about 86 times, which means that you can analyze a rather poorly-corrected image with this effect.  Beyond that limit, the algorithm does not treat spatial frequencies high enough to employ the entire aperture of the pupil, and a warning message will be displayed.  If this happens, you should reduce the size of the reference dimension, which reduces the area that is to be analyzed.

 

Owing to the complexity of the sampling and FFT algorithm, the coherent analysis does not support a case where both HBAR and GBAR are nonzero.

 

7.  The option to process your own photograph requires some preparation.  The image to be processed must be on the Windows clipboard before MIT can access it.  The photo must be in bmp format with a color depth of 24 bits.  One way to put it on the clipboard is to open the file in your favorite graphics-editing program (we use IrfanView, which is powerful and free) and then use that program’s copy feature (usually just Ctrl+C).  If the picture you want to process is displayed on a SYNOPSYS graphics window, just click on the Bitmap copy button, , and it will go to the clipboard.  Another simple way to load the picture is to go first to the dialog MFB, which lets you open a disk file of type .bmp.  Then click on the Bitmap button, and the clipboard will be loaded for you.

 

SYNOPSYS will read the upper-left 400 x 400 pixels of the picture on the Windows clipboard.  The area on the MIT dialog that the picture will occupy is about four times the reference dimension.  So, for example, if your image will show up at a size of 0.12 inches horizontally at the film or CCD, and you want this area to fill the MIT display, then you would specify a reference dimension of 0.03, which is ¼ of the total.  You may have to adjust the gain slider to get the most pleasing picture.  If you want to process a color photo, be sure to request the multicolor setting in MIT, so that all colors of the image model are included.

 

If your monitor is set at a high resolution, such as 120 dpi instead of the normal 96, then there will be some stretching in the processed image.  The industry has cleverly decided that individual pixels will not be square at that resolution, and the bits-per-inch densities in X and Y are different.  MIT remaps the pixels to a square array before processing, which is pretty accurate, but if you really want a faithful copy of the original, we advise you to use the default resolution, usually 96 dpi

 

There are subtleties to all of this, owing to display technology.  Suppose the lens has a single wavelength of pure yellow.  There is no yellow phosphor in the monitor, so to display this image the program computes the equivalent RGB color reference and displays a combination of red and green, which the eye interprets as yellow.  If you were to pass that beam through a prism, you would of course get a single yellow spot on the image, displaced by the prism – but to show that on the screen, MIT must render it in red and green.  Now replace that source with a mixture of red and green light – again appearing to be a pure yellow spot to the eye.  Pass that through the prism and you get two images, red and green, separated by the prism.  You can see that just processing the image as the eye sees it, with the RGB model used by MIT, does not necessarily lead to a physically correct answer.  In the second case one would have to include the colors red and green in the WAVL description, so that MIT would know about the color separation.  One must ensure that the wavelengths in the lens include all of the wavelengths in the picture to be processed.  And if your lens is monochromatic, the target should be a black&white picture.  Then the effects of the RGB model do not appear.

 

8.  The image created by the coherent option does not suffer from the sampling limitations of the diffraction image model, and is somewhat more precise for this reason.  If the precision of your slit or knife-edge trace is critical, you are advised to use this function of MIT.  The image model options with DMODEL and WMODEL plus the image dissection routines in FOR are more flexible but slightly less precise.

The Image Tools dialog runs in what is termed “non-modal” mode.  This means you can minimize the window, do other things, and when you restore the window the display comes back as you left it.  So you don’t have to recreate the display each time you activate it – which can take several minutes if you select some of the more complicated kinds of analysis, such as partial coherence.  The only drawback to this procedure that we know of is that you have to minimize the window if you want to produce and show other kinds of plots.  This seems pretty obvious, but we thought we’d mention it, for the record.

 

When you are finished with the MIT analysis of your current lens and want to begin work on a different one, it is wise first to click the CLOSE button on the MIT dialog.  Then, when you open it again, the program will re-initialize everything.  This is particularly important if the units of the two lenses are not the same.
