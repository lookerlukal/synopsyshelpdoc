2.1.3 Character Set                                                                Previous Next TOC

 

Command-level input must be composed of members of the following character set.

 

letters A-Z and a-z

numerals 0-9

special characters + - . , ; >: ! [ ] / $ ‘ _ # HT and blank

 

Lens identification lines beginning with “ID” may contain any character except the quote mark (").

The AI program can also accept other characters, for symbol definition.

 

Lines containing the comment characters (exclamation point (!) and semi-colon (;)) are not processed beyond that character.  Thus you may add annotation to a command line by prefixing it with those characters, or bypass a line entirely by putting that character in front.  The only exceptions are lines with glass type names, which must be spelled exactly, with no extra characters.  The dollar sign "$" in column 1 is also supported as a comment character, for compatibility with older versions, but its use is discouraged since other comment characters are now available.

 

The horizontal tab character (HT) is permitted and can help to line up comments nicely.  It is treated as a space for parsing purposes.

 

Words beginning with a numeral may not contain any letter except E, which denotes exponent data.  (Example: 0.340000E-6)

 

The slash mark "/" may be entered within an input line to indicate that the characters that follow are to be treated as though they were entered on a separate line.  (Example: “3 GTB S / BK7”.)  The slash is, however, treated as an ordinary graphic character in lines beginning with the characters "ID", "PROJECT", or "=".

 

The following are examples of valid input data to SYNOPSYS:

 

PRT

1 RD 62.2 TH 4.6 AIR ; this is the first surface in the lens

OBA 1.0E4 -22.629 4.6

ID F/2.5 LENS (SPECIAL VERSION)

FETCH A:EXAMPLE.RLE

! this line is ignored because of the “!” character

CHG / 2 TH 3.4356 / 4 CAO 33.6 / END

Find the largest clear aperture.

@: increase ympl by 0.1 / em mtf / what is the back focus?

 

If numeric data are required by a program but not entered with the input, the values are assumed zero.  If numeric data are optional and not entered, default values are assumed. The difference is shown in the manual by square brackets “[ ]” around optional data.
