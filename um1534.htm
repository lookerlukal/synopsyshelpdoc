15.3.4 Entering lens requirements                                        Previous Next TOC

 

After the START or ALTER input is entered, XSYS gives the following menu:

 

 

 

SELECT REQUIREMENTS AS FOLLOWS:

 

 

MAXSN

number (maximum number of surfaces)

MINSN

number (minimum number of surfaces)

FNUM

number weight (desired F/number)

AFMAG

number weight (AFOCAL magnification)

BACK

number weight (desired back focal distance)

TOTL

number weight (desired total length of lens)

WFOV

weight (FOV match weight, default 10.0)

CWT

weight (chromatic weight, default 1.0)

FWT

4 weights (field weights, default 1.0, 1.0, .9, .7)

WYB

weight (YB match weight, default 1.0)

WUB

weight (UB match weight, default 1.0)

WPUP

weight (pupil aberration match weight, default 1.0)

ROOM

weight (neg. TH weight, default 10.0)

SNAP

number (for graphical display)

PLOT

[AUTO] for DWG of potential starting points

PASS

number (iterations for optimization, default 10)

GO

(when all requirements are input).

--NOTE--

 

see User's Manual for other options. (M for Menu.)

 

 

The program will utilize library location 10 for temporary storage during the match process.  If you do not normally have switch 42 turned on (automatic storing of changed lenses), be sure to save any lens you have placed in this location.

 

Note that XSYS finds potential starting points without doing any optimization to speak of, so the requested targets will usually be met only approximately.

 

MAXSN and MINSN

 

give the range of surface numbers that will be allowed when finding starting points.  If you want lenses that have from 7 to 12 surfaces, for example, you would enter

 

MAX 12

MIN 7

 

Normally, MIN is not required -- but if your file contains a number of doublets that you know are not suitable, they can be avoided.  When you START a lens, you must specify a MAX.

 

FNUM, AFMAG, BACK, and TOTL

 

 

refer to first-order quantities.  If you are starting from scratch, it is best to specify only one of these, since XSYS can only scale lenses and adjust the relative aperture if no starting lens is available.  In ALTER mode, the possibilities are expanded, and it is possible to specify, for instance, both the FOCL and BACK, with appropriate weights. AFMAG is unique in that the magnification of an AFOCAL telescope is independent of scaling and aperture: in START mode, XSYS can only find the lens that comes closest, while in ALTER mode the result can be controlled.

 

CWT

gives the relative weight of chromatic aberration.  The default is 1.0, and is applied to the ray errors in colors 1 and 3.  If the system is monochromatic (WAV(l) = WAV(3)), CWT equals 0.0.

 

FWT

gives the relative weights of four field points. XSYS always analyzes and rates potential matches at the field points HBAR = 0., .5, .75, and 1.0, with default field weights of 1.0, 1.0, .9, .7.  To alter these, you could enter, for example,

 

FWT 1.0 .8 .6 .4

 

WFOV, WYB, WUB, WPUP, and ROOM

 

 

give a weighting to certain kinds of mismatch between the initial lens (if any) and a potential match in the example file.  These are explained in the next section.

 

SNAP

results in a graphical snapshot of each potential starting point, using the PAD graphics program.  You must define the display prior to entering XSYS; otherwise the default is a D-display at the top of the screen.

 

PLOT [ AUTO ]

 

 

specifies that a normal plot is to be created showing all potential matches as selected by the user, or all matches [ AUTO ], during the XSYS session.  An example is shown below. PLOT may be requested in addition to a snapshot display.

 

PASS

gives the number of iterations of the optional automatic optimization portion of XSYS (see below).

 

GO

initiates the XSYS matching program.

 

After the targets and weights have been entered, as explained above, XSYS tries to find alternate starting points that have the potential to yield excellent lenses when later optimized.
