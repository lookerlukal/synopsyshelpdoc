10.1.1 Metric Input, Variable Metric                                        Next Previous TOC

 

When the program calculates the amount by which each variable should be altered at each iteration, the result is driven in part by a feature inherent in all DLS algorithms: the requirement that the variable change not be too large.  The details of this process differ from one program to another, and from one mode of operation of SYNOPSYS to another; but in all cases there must be a common measure of what constitutes a large or a small change.

 

If you change a curvature, a thickness, or an index of refraction by one unit you get, usually, very different results.  The rule by which such differences are reconciled is called the metric, and in SYNOPSYS the metric is implemented by assigning to each class of variables a value that scales the dimensions of that class and provides the common measure.

 

The initial default metric used by SYNOPSYS assigns to all parameters a metric value of unity except for dispersions, for which the value is 0.1.  Because the PSD algorithm senses differences in linearity; it is relatively insensitive to the initial metric, and this simple rule is sufficient.  An optional mode of SYNOPSYS employs another set of rules to continually update the metric.  You can activate this mode by placing switch 30 on (for best results, also place switches 2 and 8 off, to defeat the PSD method).  This combination is not always as effective as the default algorithm described above, but is occasionally better for some problems.

 

You may also employ your own metric while using the default PSD algorithm, by entering

 

METRIC RDMET THMET INDMET VMET TDCMET GMET OBJMET

 

along with the optimization input files, before the SYNOPSYS command.  (Be sure that switches 2 and 8 are on and 30 is off.)  In this input, each parameter is the metric value to be applied to a class of variables. These are, in order: radii, thicknesses, indices, dispersions, tilts and decenters, aspheric deformations, and object parameters.

 

Most users will have no need for this option, but, for example, should you particularly want, say, thicknesses to be varied more than radii, you might enter your own metric with a smaller value for radii and a larger one for thicknesses.  Switch 30 must be off, or the new metric will affect only the first pass.
