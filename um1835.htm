18.3.5 Cavity                                                                        Previous Next TOC

 

This input file creates a cavity with two mirrors, each at the focal point of the other.  Light enters through a small hole in surface 4 (via surface 2), and undergoes successive reflections until it happens to exit via the same hole.  In the PER drawing below, the path of a single ray is shown.

The two mirrors are both LOOSE and CAPTURE; surface 2 is LOOSE so that the exiting beam can get to surface 5, which is CAPTURE for the same reason.  The program will track a ray that oscillates as this one does for up to 200 cycles, and then abandon it (as in this illustration).  The limit may be increased to 1000 intercepts by placing switch 12 on.

 

When making drawings of nonsequential systems such as this one, using PER or SOLID, remember that there are two ways to request rays on the drawing: with individual RAY commands and with the TRACE command after a suitable PUP definition has specified the ray configuration (see 7.2 and 8.4).  The distinction is as follows: the RAY command shows the requested ray whether or not it gets to the image plane, while the TRACE options show only rays that arrive there (and are not vignetted, if VIG mode is on).  Since many of the possible nonsequential paths result in the eventual loss of the ray, these rays will not be drawn with TRACE, and must be requested individually, with RAY.
