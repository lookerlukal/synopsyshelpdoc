5.5.3 Plotted Ghost Image Features (GHPLOT)                Next Previous TOC

 

SYNOPSYS can analyze and plot the ghost-image properties of a lens with real rays in several formats.  Input is:

 

GHPLOT NRYS { ICOL/P } TSCF HBAR GBAR MODE {R/L/C}

R REFL {ALL / SN SN SN …}

[ SINGLE JREFH JREFL ]

[ BURIED JREFH JREF ]

[ DRAD DRAD ]

[ SIMAGE [ XS / BEST ] ]

[ NARCISSUS SURF ]

[ SLOG NRYS ]

[ PER EL AZ [ TSCF [JSSS JSPS [ V/XOS YOS ] ] ]

PLOT

 

NRYS

The number of rays in the entrance pupil.  The program will construct an (XY) grid of rays to approximate this number.  Each ghost will be represented by the actual number that is traced and gets through to the image surface.  In VIG mode, rays that violate clear apertures within the lens will be deleted.

 

ICOL / P

Color number.  Note that GHPLOT does not have a multicolor option.  P is the principal color.

 

TSCF

Times Scale Factor.  This affects only the MODES 1 and 2 plot.

 

Mode 1:  this controls the size of the area plotted.

 

Mode 2:  the peak height of the plot  is 3.5 inches times TSCF.

 

HBAR

Fractional field in the Y-direction

 

GBAR

Fractional field in the X-direction.  A skew field must have been defined before you can use this parameter.

 

MODE

The form of output is controlled by this selection, which must be an integer from 1 to 4.  The selections are:

 

1.  Plotted spots

2.  Oblique-perspective drawing

3.  Colored boxes indicating intensity

4.  PER drawing showing the paths of rays in a single ghost

 

{ R / L / C }

This input only applies to MODE 2, and specifies a view direction for the 3-D surface.  The default is C (centered).

 

SINGLE …

Use this input to request analysis of only a single ghost, involving surfaces JREFH and JREFL.  If not entered, the program analyzes all two-bounce ghosts not involving dummy surfaces or reflecting surfaces.

 

BURIED

This option is the only one that involves a single reflection rather than two.  If your system has a buried ghost image, this is a good way to look at the ray paths.  Mode 4 (perspective drawing) is the only mode that makes sense in this case.  The ghost that is analyzes starts with a reflection at JREFH, and the ray paths are drawn going backward as far as JREF.

 

DRAD …

This gives the radius of a circular detector whose area is used to normalize the ghost intensity.  A ghost that falls entirely within this area is assigned an intensity equal to the product of the two reflectances, while larger ghosts are less intense according to the area ratio.

 

SIMAGE …

This gives the diameter of the image area that is to be sampled.  The default is a square that encloses the CAO of the image surface.  See below.

 

NARCISSUS ...

GHPLOT can show the pattern at the image that arises from light reflected from that surface and then reflected again from surfaces within the lens.  This is an issue in IR systems with a cooled detector: the retroreflected flux lowers the total output of the detector when a small ghost image is formed on the detector, and one observes a dark smudge near the center of the display.  The paraxial sources of this effect can be found with the NAR command.  The NAR feature of GHPLOT analyzes much the same thing, but in this case with real rays, and the output is graphical.  The surface number of the first reflection is given by the SURF input.  This is usually just the detector itself, in FOCAL SYSTEMS.  If the lens is AFOCAL, then that surface is not actually present, so one specifies instead the first of the two dummy surfaces, which are required in that mode, and which have zero curvature and thickness.  The light that is traced actually starts at the object, and then reflects from the specified surface and propagates to each prior surface.  Since the image is usually very small, this procedure is essentially identical to starting at a point on the detector.  The reflectance of the detector is set at 1.0 for this feature.

 

SLOG …

Another use of GHPLOT is to find the source of stray light that originates out of the field.  If the SLOG option is requested, the program will list the first NRYS rays that get completely through the system.  From this list you can see which surfaces were responsible, and then examine those ray paths in mode 4.  The SLOG option only works in mode 1.

 

PER ..

This input is identical to the usual PER input, and only applies when MODE 4 is selected.  None of the other input lines used with PER may be entered following this line, however; the ray definitions are taken from the other GHPLOT input.  You must also specify BURIED or SINGLE  when selecting this mode.

 

In the event that you wish to analyze a ghost where the image surface is JREFH, the PER option will not work unless you add a dummy surface at the image.

 

PLOT

Ends the GHPLOT input file.

Let us now run GHPLOT and look at the energy distribution from all ghosts.  First we select MODE 1.  Input is

 

CHG

CFIX        ; we do not want to see vignetted rays, so we make all apertures hard

END

 

SSS .003                  ; there will be lots of rays, so make each spot small

GHPLOT 4000 P 10 0.5 0 1  ; the picture will be drawn at 10X scale, at half field

R .01 ALL                 ; reflectivity of all surfaces is 0.01

DRAD .0004               ; detector radius is very small

PLOT

 

In this input, we elected to analyze the case where light enters from HBAR = 0.5, or half field.  GHPLOT is the only ghost-analysis feature that lets you specify a field point; all of the others analyze the on-axis object only.  A nonzero field angle will spread the several ghosts out so they can more easily be seen on the plot.  Here is the result:

 

 

There seems to be a rather dense concentration of rays located above the center point.  Let’s zoom in on that point:

 

 

There seems to be a caustic there, which could make the ghost appear brighter than predicted by the GHOST listing above.  (Remember, that feature is a paraxial analysis only, and it therefore ignores aberrations in the ghost beam.)  Let’s look at this energy distribution with MODE = 2:

 

GHPLOT 20000 P 1 .5 0 2 L

…

 

Here we increase the number of rays to 20000 so that the result will look smooth.  Since the distribution is a function of local ray density, too few rays will make a bumpy picture.  (GHPLOT is not restricted to the 20000 maximum ray limit that applies to most of the image analysis features; the number can be as large as necessary to produce a smooth plot.)

 

 

Indeed, there is a strong spike.

If you want to print this output on a black-and-white printer, it is best to run the analysis again, with switch 35 OFF.  Then the distribution will then be shown as a grayscale, from black to white.

 

The maximum intensity shown here is 1.266E-10, which is somewhat larger than the paraxially-predicted value of 7.8E-11.  Here we must digress a bit to discuss the issues that affect the accuracy of this prediction.

 

First, this analysis gave a field point HBAR of 0.5, so the ghosts were separated on the picture.  The paraxial calculation applies only at the center of the field, where all the ghosts are superimposed.  As you go off axis, the sum will usually decrease, due to the separations.

 

A more important matter has to do with the size of the sample area.  The program divides this area into a 60 x 60 array of bins, and counts the rays that arrive in each one.  Consider two cases: where the bins are larger than the detector, and where they are smaller.  In the first case, if there were a concentration of energy within a small portion of the bin, the program would not see it and the reported energy would be lower than it should be.  In the second cases, the program finds the intensity at the most intense bin, which may not apply to the entire detector area, and calculates the intensity on this area from that value.  So the reported energy will usually be higher than it should be.  The most accurate energy analysis often results when the bin area is equal to that of the detector, and this is the result when you specify SIMAGE BEST.  But there is a catch:  if the bins are sparsely populated, which happens with a large ghost blur and a small detector, then the stochastic noise in that calculation overwhelms the illumination calculation, and the results have little meaning.  There seems to be no general solution, and you are advised to examine the MODE 3 plot to see if it looks smooth.  If not, then use a larger SIMAGE and more rays.  If the ghosts you see look smooth and extend over an area of several bins, then the energy calculation should not suffer from these issues.

 

The calculated ghost intensity will also be strongly affected if any of the ghosts exhibit a caustic.  This is not an artifact, since the intensity in fact can become very large in such cases. In the example above, the spot analysis (MODE 1) looked suspiciously like a caustic pattern.  Let us examine this pattern.  Here we will request only a single ghost, and ask for a PER drawing, with MODE = 4.

 

GHPLOT 4000 P 10 .5 0 4 L

DRAD .0004

SINGLE 6 1

PER 0 0 0 1 99

R .01 ALL

PLOT

 

 

In order to keep the picture simple enough to interpret, the program allows only a single ghost to be analyzed in this mode.  Here we select the ghost that we suspect has a caustic, involving surfaces 6 and 1.  Also, we reduced the ray count to 4000 because fewer rays are easier to see in this mode.  (The program takes the (X,Y) grid of rays that comes close to the requested ray number, and for MODE 4, takes only a single slice of that grid, giving a TFAN of 71 rays in this case.)  Here, one can clearly see the caustic in the beam following the two reflections.

 

Let’s show this caustic in more detail, using the RGHOST feature.  Run a MACro that contains

 

RGH 6 1 0 AIP      ; the fractional aperture is in AIP

ORD = FILE 1       ; and the Y-height at the image is in FILE location 1

 

and then type DO MACRO FOR AIP = 0 TO 1

 

The resulting plot clearly shows how the ray height from this ghost is very nonlinear.

 

 

The paraxial analysis, which assumes that this plot is a straight line, did not in fact give a precise estimate of the energy.  Here is an example where GHPLOT may be superior.

Buried Ghost Analysis

 

Another use of GHPLOT is to look at the ghost rays within a lens, rather than at the image.  This is especially important in high-power laser work, where an interior image can burn coatings if the concentration is great enough.  Here is an example:  After running BGI, we see that there is a buried ghost image caused by reflection at surface 13, with a focus point between surfaces 9 and 10.  Let us look at some ray paths.  Here is the MACro:

 

GHPLOT 200 P 1 0 0 4

R .01 ALL                ; not really relevant here, since only the paths are of interest

BURIED 13 9

PER 0 0 0 5 15

PLOT

 

Here we requested analysis of a buried ghost from surface 13, with rays traced backward as far as surface 9.  The results show the ghost in question:

 

 

GHPLOT is not compatible with nonsequential raytracing.  It employs its own logic – which is itself nonsequential – and if you run it on a nonsequential system you will get a warning message saying that the analysis treats it as sequential.  This can lead to rays being vignetted at a roof prism, because the UAP apertures on the prism faces are too small to pass the entire beam.  To resolve this issue, make a CHG file, and include SN NAP for each of the roof faces, to remove these apertures.  Or you can put the system in NOVIG mode, but this will then show ghost rays even if they are vignetted.

 

If you ever need to evaluate a buried ghost in nonsequential mode, a single case can be run; look at the example in Section 18.3.6.

                                                 

All of the ghost analysis routines may be run from the dialog MGH.
