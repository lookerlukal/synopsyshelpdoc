8.5.12 Illumination Pattern (IPAT)                                         Next Previous TOC

 

When designing illumination systems, one often wants to examine the uniformity of the beam at the final surface; this is the purpose of the IPAT command.  The formats are as follows:

 

IPATTERN [ NCELLS  [ RANDOM ] ]

[ OBI INDEX JX JY ]

TRACE {ICOL/P} HBAR GBAR NRYS

[ OBI INDEX JX JY ]

TRACE …

…

PLOT

 

These mnemonics have their usual meaning: the color selection {ICOL/P} gives the color number, or the principal color if “P” is entered.  The fractional field of view in Y and X are given by HBAR and GBAR, and the number of rays to be traced is given by NRYS.

 

The ray number is not limited to the usual 20,000 rays, and you are encouraged to use a larger number in order to obtain a smooth display, which is calculated by filling an array of 300x300 cells covering the final surface, at the larger of the CAO or RAO dimensions, with rays, and then assigning a color to each box depending on how many rays arrived there.  Additional TRACE commands may be entered as needed, and the resulting rays will be added to those already accumulated.  You can thus model the output from your light source as fully as required.  You can override the number of cells with the NCELLS entry, where the data must be from 2 to 300.  A smaller number smooths out the distribution but loses resolution.  The ray pattern is by default a rectangular grid in the entrance pupil, but you can request instead a random set of rays with the RANDOM request.  This is useful for systems with an array of lenslets, such as for homogenizers for microlithographic illuminators, where a regular grid will generate moire patterns with the regular lens array.

 

The optional OBI… input can be used only if the system has been assigned an object of type OBI.  See Section 3.1.6.
