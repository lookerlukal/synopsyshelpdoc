5.60_RAS_Recast Aspheric to G-series

                                 Previous TOC

 

The command RAS will recast a surface shape into a basic power-series surface.  The format is as follows:

 

RAS sn [ SYMM  / RSYMM [ nterms]  ]

 

If the optional SYMM is included, only terms with bilateral symmetry will be used. RSYMM requests only terms with rotational symmetry.

 

If nterms is entered, the RSYMM conversion will only vary the requested number of terms, in the order 3, 6, 10, 16, 18, 19, 20, 21, and 22.  Thus, RSYMM 3 will produce a surface with terms 3, 6, and 10 adjusted for best fit, and the other terms zero.

 

The program calculates the sag of the original surface sn at an array of points over the surface.  Then it submits an optimization run that varies all the enabled coefficients plus the base curve and conic constant and targets the surface sag at those locations.  When this completes, the surface has been redefined and has a shape one hopes will be very close to that of the previous.  Then one can optimize the new system, varying the G terms as usual.

 

To illustrate this feature, we start with the relay telescope and cast the primary mirror to a power-series surface.

 

FET 4

RAS 1 SYMM

 

The program fits the currently spherical shape and returns a set of coefficients, all of which are very small.

 

Be aware that it is not possible to match most of the exotic aspheric shapes with a normal power series -- which is why those shapes are available in the first place.  If the attempt does not converge to a very small merit function, it is likely you cannot substitute that shape in this way.
