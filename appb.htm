The following table summarizes the function of the mode control switches.  Unless changed by the user, switches

When the MACro SYSTART.MAC is run, an additional set of switches is turned on:

(See section 2.9 for a description of switch usage.)  The dialog MSS gives a description of the function of each switch, and indicates which switches are on and which are off.

1

Drops bounded variables for remaining iterations.  Otherwise, restores variables each pass.

 

Optimization is somewhat faster if this switch is on, but there is a drawback: the optimization program honors the edge-control monitors AEC, etc., to prevent feathered edges.  But if a lens thickness reaches the upper or lower end of its range, that variable will be dropped if this switch is on, and then those monitors will ignore the edge on that element if you run the annealing program.

19

This term is used for designing spectrographs, where the image is usually long and narrow, intended to fall on a slit.  The usual diffraction analysis routines cannot model this kind of image if it is larger than the limited sample region of those features.  If this switch is turned on, all of the image-analysis features that create a grid of rays and calculate the OPDs will take out the Y-defocus term from the Zernike wavefront coefficients before performing the analysis.  The ZCOE routine must have been run immediately before the analysis to be performed, at the same field point and in the same color as the analysis that follows.

 

The features that honor this switch include PSPRD, WPSPRD, PUPIL, and DMODEL.  BTOL and the image routine MIT will also honor the switch and do not require first running ZCOE, since they calculate their own coefficients.  The diffraction and coherent effects will be adjusted by MIT.

 

This feature will turn off the multicore option, which does not support it.

 

The switch also zeros out coefficient 5 in DIFF, which does much the same thing.

Force sequential raytrace, even for nonsequential systems.  In some nonsequential systems, such as those involving a roof prism, the image quality as determined by sequential raytracing is identical to that found by nonsequential tracing, and during optimization one would like the greater speed of the former.  If this switch is turned on, therefore, one will get the greater speed with no loss of accuracy.  But for showing the system with the perspective drawing programs PAD/PP or PERSPECTIVE, one should turn off this switch so the nonsequential ray paths are shown correctly.

55

This switch causes the limits on TH variables to be controlled via the monitors ACC and ACM instead of via hard limits on the variables themselves.

 

You can override individual variable limits in the PANT file, and add your own monitors, if you want to change the defaults, which in this case are ACC 25.4 1 1 and ACM 1 1 1 if this switch is turned on (dimensions in MM).  The hard limits on TH are set very wide by this switch, so the monitors will control them instead.

 

This option often returns results different from those produced with the switch turned off, and they are sometimes better.  So this is an option that is worth exploring.

61

If switch 42 is turned on, the program automatically saves the lens in library location 10 whenever it is changed by most features.  If switch 61 is also on, the program compares the current value of the merit function with that of the lens most recently stored in location 10, and if the newer lens has a larger merit function, the store does not take place.

 

If you alter the merit function definition or want the program to forget the previous value, simply delete the lens in location 10 with DLL 10.  Then the next auto store will work and the test value will be set to the merit function of that lens.

66

Diffraction integral is performed over next-to-last surface, rather than projected to infinity. (Removes Debye approximation.)

 

67

Optimization uses two-sided derivatives.  This makes the optimization run more slowly, but the derivatives are more accurate.  The difference shows up mainly when a lens is near optimum and you want to get as close as possible to that point.

 

If this switch is off, the optimization will initially run more quickly.  Later on, when the convergence becomes slower -- and more accurate derivatives are called for -- the program will automatically switch to the two-sided derivatives..  Thus you get the speed of the simpler algorithm at the start of the job but the performance of the more accurate one near the end.  Users are encouraged to experiment with this switch setting to see which works best on your lens.

 

Besides the difference in speed, the two settings will sometimes lead to very different solutions, especially for runs on DSEARCH, where the configuration is initially undefined. This is a reflection of the essentially chaotic nature of the design space: a tiny change initially produces large changes later on. Again, experiments will show which is better for a given problem.

85

Geometric image analysis approximates the visual color of the wavelength analyzed.  In order for this to work, the program has to make a black background, or a black box, behind any picture that shows a spot diagram.  Otherwise a well-corrected image, which can look like a white blob, would be invisible if printed on white paper.  This switch therefore overrides the selection of White Background in the graphics window, for that portion of the picture.  Plots such as FANS, on the other hand, are drawn over the current background, which can be white, but the intensity of the line color is reduced by about 60% if drawn on white so that a yellow line (which would be very hard to see) is more visible.

 

The composite image picture is made by doing an OR operation on the RGB color reference applied to each wavelength.  So blue, green, and red show up white, as they must.  Unfortunately, not all printer drivers are smart enough to know how to do that operation.  We have tested two different pdf drivers, and neither is able to.  In those cases, the last color drawn overwrites the others, which is not what you want.  If your goal is to attach the picture to an email or .doc file, do not use the pdf format.  Instead, in the Graphics Window, change to a white background (which affects some parts of the picture) and then click the Bitmap copy button.  This puts the picture on the Windows clipboard.  Then, in your document or email, just do a paste at the point where you want the picture to appear.  The colors will then be correct.  Make the bitmap copy after you open Word, if you use it, since that program erases the clipboard.

 

86

This switch is intended for rare cases where the optimization feature fails to complete the first pass.  The reasons for this are obscure, but it does happen.  In such cases, if this switch is on, the program will then run the simulated annealing program at a temperature of 100 and cooling rate of 5 – which may kick the program into converging better.

92

This switch defeats the default graphics feature that automatically switches to a white background when you print a graphics window that is displayed with a black background.  The display on the monitor looks very nice with a dark background, and many users have that option selected all the time -- but when the picture is printed, one usually wants to see black lines on white paper, and not the reverse.  The lines would be hard to see on a black page, especially on inkjet printers, where the ink tends to bleed somewhat and covers up the thin lines.  And your expensive black toner would be quickly used up.  That is the reason for the default behavior.

 

But for some displays you need to show the picture on a black background, especially for the MAP program, which can show many spot diagrams and where the colors and the different field points can overlap on the page.  If the background is black, then the program can combine the colors with OR logic, where red + blue + green = white.  That logic shows fairly realistic spots.  On a white background, on the other hand, it does not work very well: a white spot on a white page would be hard to see.  So the program then does not combine colors in that manner, but makes each color overwrite the previous color instead.

 

So if your display really needs to be printed with a black background, then select that option in the graphics window and turn on switch 92 before printing the page.  And be sure to turn the switch off afterwards!  Otherwise you will soon run out of black toner.

 

93

This switch causes the PAD display to clip the drawing in the top pane so it does not run over onto the lower pane.  If your lens has fold mirrors, this is likely to happen, and you can elect either to see the portion that falls on the lower pane or not with this switch.

 

94

Causes GRAY (global raytrace) to output data at maximum precision.  This is intended for users who wish to capture those data and import them into other programs without round-off errors.

 

95

If this switch is on, KICK and ANNEAL will not increment conic constants, G variables, or glass model data.  (To avoid introducing ray failures in systems that are close to failing already.)

 

96

When this switch is on, AI plots make a small symbol at each data point on the curve.

 

97

Aspheric surfaces can be difficult to raytrace, especially if a ray has more than a single intersection point with the surface.  This switch enables a slower but more reliable binary search algorithm.  If your aspheres are very shallow, you can turn this switch off and get faster performance.  If it is very steep, on the other hand, and flies off with a high slope, the search can also fail.  The program iterates by first finding the intersection of the ray with the vertex plane, and then to the surface at that (X,Y) coordiniate.  But if the actual intersection is far from the vertex intersection, it might not converge.  In that case, turn this switch off.

 

This switch is usually a good place to start, but if you experience ray failures that you believe should not occur, turn it off and see if the problem goes away.

 

If there is more than a single intersection, this switch should find the correct one.

 

98

This switch causes all references to random numbers (except for MC) to use the same set of numbers.  The goal is so someone reading the lessons in my book will get the same results as in that book.  Since many lessons involve simulating annealing, the results would be different every time otherwise, and the reader would be confused and suspect that they made an input error.  For normal users, this switch should always be off.

 

99

This switch makes the optimization program honor the CORE setting.  All of the search program utilize multicore processing automatically, but the SYNOPSYS command, which runs a single optimization, normally executes in a single core.

 

If the process is lengthy, you might experience a speed improvement by authorizing as many cores as your PC has, and then turning this switch on.  If the process runs in only a few seconds in a single core anyway, then this switch will likely make it run more slowly, rather than faster, owing to the overhead involved in starting and managing the several processes; so it should be used with discretion.

 

The command C99 number STATIC sets some arguments that apply to this switch.
