13.3.4 SnapShot                                                                Previous Next TOC

 

The SnapShot feature can is invoked with the command

 

SNAP [ NPASSES ]

 

which must precede the SYNOPSYS optimization command.

 

This command causes the optimization program to update the PAD display every NPASSES iterations so you can monitor the progress of the design.

 

The form of the display that is shown must be selected before the optimization input is run.  The default PAD display is a YZ profile on the top and a set of rayfans on the bottom.  You will sometimes want to display the OPD fans on the bottom, or select the PP (PAD Perspective) display on top if the lens has tilts or decenters that are not in the meridional plane.

 

If you have defined more than a single PAD display format with the buttons , you can switch between them at any time, even during an optimization run.

 

If the optimization process begins to go in a direction that you do not want, just click on the Stop Process button to halt it at the next iteration.

 

To prevent the SnapShot display, if it was enabled earlier, type NOSNAPSHOT.
