10.3.1.1 Automatic generation of ray aberrations                        Next Previous TOC

 

The automatic ray-generating feature constructs a ray pattern of a selected type, and adds selected properties of the rays to the merit function.  This feature (except for IPVAR) does not require the “M TAR WT …” line described in the above section, since the target and weight of each ray or blur size is assigned by the program according to the rules implied in the pattern mnemonic.  Input consists of one or more of the following lines:

 

 

GNR RT WT DEL ICOL HBAR GBAR [ SN [ F [ XWT ] ] ]

transverse coordinates

 

GXR RT WT DEL ICOL HBAR GBAR [ SN [ F ] ]

only correct XC coordinates

 

GYR RT WT DEL ICOL HBAR GBAR [ SN [ F ] ]

only correct YC coordinates

 

GSR RT WT DEL ICOL HBAR GBAR [ SN [ F ] ]

sagittal fan only, correct XC

 

GTR RT WT DEL ICOL HBAR GBAR [ SN [ F ] ]

tangential fan only, correct YC

 

GPR RT WT DEL ICOL HBAR GBAR [ SN [ F [ XWT ] ] ]

errors from principal ray

 

 

 

 

GNO RT WT DEL ICOL HBAR GBAR [ 0 F ]

OPD targets

 

GSO RT WT DEL ICOL HBAR GBAR [ 0 F ]

sagittal fan only

 

GTO RT WT DEL ICOL HBAR GBAR [ 0 F ]

tangential fan only

 

GPO RT WT DEL ICOL HBAR GBAR [ 0 F ]

reference at principal ray

 

 

 

 

GO2 RT WT DEL ICOL HBAR GBAR [ 0 F ]

OPD targets squared

 

 

 

 

GNN 0 WT DEL ICOL HBAR GBAR [ SN ]

correction to centroid

 

 

 

 

GNV 0 WT DEL ICOL HBAR GBAR [ 0 F ]

wavefront variance

 

GPV 0 WT DEL ICOL HBAR GBAR [ 0 F ]

RT is an aperture-dependent weighting factor, from which the weight of each ray is determined as follows:

 

weight = WT / (Xen**2 + Yen**2) **RT

 

Xen and Yen are the fractional entrance pupil coordinates of the ray.  If RT is zero, the ray weight is just equal to WT for all rays generated.  If RT is nonzero, the weighting of each ray will depend upon its fractional aperture (FA); a value of 1 will cause the rays to be weighted as 1 / FA**2.  As a rule, a zero value tends to produce an image of high contrast, while a value of 1 produces lower contrast but higher resolution.  Intermediate values will produce intermediate results, and you should select a value of RT depending on the requirements of the individual lens.

 

A good rule to remember is that, to optimize the RMS spot size, the value of RT should be zero.  This weights all rays equally, as they are when calculating the RMS spot.  To get the highest MTF, on the other hand, a value up to 1.0 may work best; and when that has gone as far as it can, you can switch to MTF aberrations or the GSH method of optimizing a given OPD shear.  For example, if you want to peak the MTF at 0.33 of cutoff, the SHEAR value would be 033.  For maximizing the Strehl ratio with OPDs, the value should usually be 1.0, since that weight applies to the wavefront variance, from which the Strehl ratio is computed.

 

If you enter a non-zero value for RT, the program alters the WT parameter in such a way that the overall impact of the set of rays is roughly the same as with a zero RT.  This is done to ensure that the balance between ray aberrations and any mechanical targets you may have requested is not greatly altered when you change the value of RT.  Otherwise, rays near the center of the pupil might get a very large weight and force the program to upset that balance, ignoring other requirements in order to control only those rays.  Thus, when WT is adjusted, rays near the center will have a greater weight, while rays near the edge will have a lower weight, causing the image structure to change but also maintaining control over your mechanical or other targets.  The value of RT should in most cases be between 0 and 1.  Larger values will defeat the adjustment of WT and will probably upset the balance.

 

If mode switch 83 is on, the weight given above is multiplied by the color weight assigned to that wavelength in the lens file.

 

You may substitute “P” for the color number in all of the above definitions.  This requests just the primary color. You may also use the letter "M", which causes a set of ray aberrations to be generated at all defined colors.  If you want different weights on each color, you have to enter separate requests for each one.

 

The "F" option on these lines causes the program to generate rays on both sides of the pupil.  Normally, rays are traced on one side only, since most lenses exhibit bilateral symmetry and correction on one side automatically corrects the other.  But if your lens is not symmetric, you can use this option to control the entire pupil.  Doing so increases the time required to evaluate the MF, so it should not be requested unless needed.

 

A GNR request will generate a YC and an XC aberration for each ray in a grid, the resolution of which is given by the entry DEL.  This entry represents the number of partitions to be made to the semi-aperture, as shown below.

 

 

XWT if entered, is the weight that applies to the X-component of the ray aberration, which therefore may differ from that given by WT to the Y-component.  The default is XWT = WT.  For monochromators that disperse in the X-direction it is sometimes useful to set XWT to a small value and WT to a larger value.

 

For GNR requests, each ray is traced only once even though two aberrations are generated. If GNO is entered, a single aberration, namely the OPD of the ray, is generated for each ray. GSR and GTR generate rays in the sagittal or tangential fan only. GPR and GPO define the ray error with reference to the principal ray location, rather than the chief ray.  (The chief ray is always taken in the primary color, while the principal ray is in the color of the rayset.)  This is useful for designing spectrometers, where the images in several colors are widely separated.  It is also useful when an image is sampled by several detectors, and the colors do not have to coincide because the differences can be compensated in software; in that case, you might find it useful to create a polynomial describing the mapping of the image in the various colors.  See CDIST.

 

A message is printed giving the number of rays generated by these options.  If switch 29 is turned off prior to the AANT command, the optimizing program will look for the optional [ P ] on the AANT command.  If it is there, the program lists the number of rays that were generated.  If not, then the full details of each ray are listed.

 

If a negative number is input for DEL, the rays will be generated on the left side of the pupil.  If the optional F is included in word 9 of the request, both sides of the pupil will be traced, with twice the number of rays.  This should be input if the lens does not exhibit bilateral symmetry.

 

The value of DEL should normally be between 2 and 6.  A value of 4 generates 26 rays, and the number goes as the square of this value.  For most systems, a value of 4 is adequate.  For GSR and GTR ray sets, which do not fill a grid, a larger number is sometimes appropriate.

 

You may specify individual rays at will in the same merit function as automatically generated rays.  For example, if lateral color is a problem that is not easily controlled by the usual GNR aberrations, one might use

 

AANT

 

GSR .5 2 5 2 0

Corrects 5 rays in color 2, on axis

GNR .5 1 4 1 1

Ray grid, color 1, full field

GNR .5 1 4 2 1

same, color 2

GNR .5 1 4 3 1

and color 3

M 0 10 A 1 YA 1

Corrects chromatic aberration, with higher weight

S 3 YA 1

(See the next section for definitions.)

END

 

 

This input will generate 26 rays each in three colors at the full-field point.  The on-axis image will have a relative weight of 2, and the off-axis images a weight of 1.  The zonal rays at each field point will be weighted more heavily than the marginal rays, by a factor of 1 / 2*FA**2, since we entered a value of .5 for RT.  To improve the lateral color, we specified that the rays in colors 1 and 3 at full field should have the same Y-intercept, with a weight of 10.  (This example assumes that the lens has been set up with colors 1 and 3 for the long and short wavelengths, and color 2 for the primary color.)

 

GNN corrects the rays relative to the centroid of that set of rays rather than to the chief ray intercept, and always traces over the full pupil (since the centroid of a pencil over only half the pupil is itself decentered and would be inappropriate).  While GNR generates two aberrations for each ray, GNN generates only one aberration for the whole set: the mean squared spot size measured from the centroid.  For best results, a lens should be optimized as far as possible with individual ray aberrations or GNR requests, and the GNN option used only to peak up the final image.  The GNN option also permits the centroid coordinates to be controlled explicitly. (See section 10.3.3.)  Since the GNN option ignores the location of the chief ray, it will not automatically control later color, and specific targets should be added for that purpose.

 

GNV causes the variance in the wavefront to be computed for all of the generated rays.  It is useful in the final stages of a design to peak up the performance.  Note that neither GNN nor GNV honors a nonzero RT entry; all rays are weighted equally.  The variance is taken relative to a sphere centered at the primary-color chief ray point.  The effects of lateral color are therefore corrected automatically if the requested color is not the primary color.  (For peaking the MTF at a given frequency, the GSHEAR option is superior.)

 

The GPV option is similar to GNV, except that the OPD reference sphere is centered at the image point in the requested color.  This is the point that minimizes the variance in that color, and it is usually not exactly at the chief-ray point.  GPV is useful when you want each color to form a sharp image, but don't care about lateral color, which is not controlled by this option.  If all colors are to focus together, see IPVAR below.

 

GO2 is similar to GNV but is usually more powerful.  Although minimizing the variance (with GNV) in principle should maximize the Strehl ratio, it suffers from two defects.  First, the variance is not sensitive to the average OPD, since it is defined as the average of the squares minus the square of the average.  So if both of these are large the program only controls the difference, and the OPDs themselves are not strongly driven toward a value of zero.  Also, the process discards the sign information of each OPD.  In contrast, GO2 calculates the square of each OPD and then assigns the sign of the OPD itself to the result.  The net effect is to reduce the sum of the squares of the OPDs, which reduces the variance as well if the average is zero, while at the same time trying to reduce each OPD to zero to make this the case.

 

GSHEAR is an alternative to MTF aberrations, which work but can only be used when the design is very close to perfect already.  GSHEAR also works best if it is already close to a good solution, but it is more forgiving and can be used earlier in the process. This form creates traces two rays for each point in the pattern, sheared in X or Y in the pupil with respect to that point.  The purpose is to improve the convolution MTF at the entered shear value.  It also accepts colors "M" and "P".  The shear value is a fraction of the semi-aperture.  Thus, a shear of 1.0 corresponds to the cutoff frequency, and values of 0.5 or less are usually appropriate.  The RT value does not apply to this form.  Larger shear values produce fewer rays, since rays sheared out of the pupil are ignored.

 

The IPVAR option is another alternative to GNV.  The latter calculates the wavefront variance over a grid of rays, where the OPD of each ray is referenced to the path length of the chief ray.  If the image centroid is located close to the chief-ray point, this is a valid operand that should help to refine a lens that has already been optimized as well as possible with individual rays or OPDs.  But if the peak of the diffraction pattern is shifted away from the centroid, it is better to take the wavefront variance relative to that location, which is what IPVAR does.  Note the format: this request requires that the line contain the entries M tar wt A ... as for ray aberrations, as noted in the above table.  See Section 8.6.8 for a discussion of the calculation of the image point, which applies to this operand as well.  The target should be zero, and you should adjust the weight according to the values of other elements of the MF.  This operand involves more calculating than the other forms, so it runs more slowly. The number of rays therefore should not be excessively large; from 50 to 100 is a good range.  The RT parameter is ignored for this operand; all rays are equally weighted.  Do not specify this or any other variance operand for a lens that is initially not well corrected.  It is intended only to peak up a design that is close to optimum already.  This operand is similar to GPV, described above, except that in IPVAR, all colors utilize the same image point reference, while the latter uses a different reference in each color.

 

It makes sense only to consider rays that are in the entrance pupil and which therefore contribute to the final image, and the above options have several ways of implementing this need.  Most of the sophisticated pupil-search options are rather time-consuming, and one would like to use them only when needed.  For this reason there are several levels of approximation that are much faster and which may be requested where appropriate.  The slower and more accurate methods use the WAP 2 and 3 designations, while the faster ones modify the paraxial pupil according to simple rules.  One therefore gets a fairly accurate way to quickly optimize the lens, while expecting to change to the more accurate pupil description later, for final optimization.  The VFIELD option is intermediate between the two; the data are found by a search, and used thereafter to modify the paraxial pupil.

 

To use the simplest vignetting option, assign a paraxial pupil to the lens and then declare that the pupil is vignetted with the input

 

 VSET FRAC POS                (Vignetting SET).

 

The two parameters specify by how much the beam is reduced in size and decentered in the paraxial pupil.  This option is appropriate when you can estimate the nature of the vignetting that you expect will show up in the final lens.  So you design to that pupil, and then later specify the clear apertures and pupil definition that actually give that vignetting.  This approach is very fast and often sufficiently accurate.

 

The VSET command must appear in the AANT file before the command that generates the rays.  The quantity FRAC is the fraction to which the relative Y-coordinates of all the rays are reduced at full field, and POS is the fractional offset in the Y-direction, at full field.  It is positive if the shift is toward the upper rim ray.

 

Thus, to compress the full-field ray set to 70% of its normal height and shift the center of the resulting elliptical pupil in the direction of the upper rim ray by 25% of the paraxial  pupil radius, you would enter

 

VSET .7 .25

 

and follow this with the GNR request.  Other field points are affected in proportion, and the on-axis point is not affected.  Only the Y-coordinate is reduced.  Rays are distributed uniformly in the compressed pupil; the central ray of the rayset is thus shifted by POS, but the chief ray from which the intercept errors are measured is still centered in the nominal pupil.  If you would prefer that the chief ray be centered in the reduced and shifted pupil, then the VSET option is not recommended, and the VFIELD option should be used instead.

The main distinction between the VSET and VFIELD features is that the latter becomes a permanent part of the lens, and the reduced pupil it defines applies to all features of SYNOPSYS.  VSET, on the other hand is a temporary setting that only applies to the ray grid geometry used for optimization.  The intent is to give you a quick way to evaluate various vignetting factors (with VSET) during optimization – and then, when your design looks promising and you know how much vignetting will be required, you can assign the appropriate VFIELD to the lens.  Then you can analyze the system, find the clear apertures that actually give that vignetting (with CFIX), perhaps create an updated set of VFIELD parameters reflecting those apertures (with FVF), and, if necessary, reoptimize the lens again with that pupil definition (and no VSET).  One simple way to find the apertures giving the desired vignetting is with the WorkSheet, where you can assign and then vary the CAO of a desired surface while watching the ray fans plots to see at which CAO value the fan is properly truncated.

 

A current VSET command remains in effect until altered within the same AANT file, and for alternate configuration optimization, additional VSET commands may be entered as required prior to each rayset request.  The VSET entry also reduces the aperture used for evaluating edge feathering with the ECP and ECN requests (see 10.3.7).

 OBS P1 R1                (or OBSET R1)        (OBScuration Set)

 

will cause an obscuration at P1 of fractional radius R1 to be deleted from the automatically generated sets of rays.

 

 

GTP generates a TFAN of rays all passing through the center of the entrance pupil.  The fan in this case is a collection of HBAR points.  This feature is used to correct the spherical aberration of the pupil on a given surface.  Be warned that if your lens uses any of the wide-angle (WAP) pupil options or the VFIELD, then the chief rays will in general not go through the center of the stop, and using the GTP feature may not make sense.
