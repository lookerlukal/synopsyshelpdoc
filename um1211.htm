TPR

gives the surface numbers of all radii for which testplates already exist.  These surfaces will be given tighter tolerances of but a few fringes, and other tolerances will usually be looser as a result.  Flat surfaces need not be listed since by default they are assumed matched.  If the word ALL is entered instead of a list of surface numbers, then all radii that would otherwise be toleranced in this run will be assumed matched to testplates.

 

Never calculate a tolerance to be assigned to a testplate itself.  Have the testplate made and measured, and then fit the lens to that value.  If you give the testplate itself a tolerance, then the statistics of the surface are skewed by whatever error finally turns up, and the predictions of BTOL are less accurate.  See Section 12.6 for a discussion of this issue.  The radius tolerance applies to the lens surface, not the testplate.

 

If the shop does not plan on using testplates, then the radius tolerance applies to the radius, this declaration should not be used, and the statistics of that tolerance should be accurate.

 

PFTEST

This declaration is appropriate only for aspheric surfaces.  It is intended for surfaces that will be measured with a profilometer, rather than with a testplate or a null setup.  The latter are more accurate, but difficult and expensive to set up.

 

The program models the figure error of power-series aspheres with coefficient G 15, and for Zernike surfaces with term 5, both of which describe an astigmatism error.  For other shapes, the program implements the general-purpose cylinder option GPCYLINDER, which will add a cylinder whose shape in Y is opposite to that in X.  In every case, the modeled error cannot be improved with a focus adjustment at the on-axis point.

 

The idea is that, if the errors are small, the effect on the MTF is a function of the wavefront variance and is not very sensitive to the particular aberrations that may be present.  So we use these terms as a proxy for figure errors in general.

 

To test such a surface, one runs the profilometer over the surface in a raster fashion, measuring the sag at each location and subtracting the sag calculated for the ideal surface.  Then one finds the standard deviation of the errors (the average of the squares minus the square of the average), and takes the square root of the result.  If this is less than the standard deviation tolerance printed by BTOL, the surface is in tolerance.

 

This procedure should work for most gentle surface errors, but may not be appropriate for errors that change rapidly over the surface and generate large values of the slope.  If the surface resembles a Fresnel lens or a phonograph record, you have serious problems.

 

Surfaces so declared are not given a curvature, irregularity or rolled-edge tolerance, since one does not want to include the same error twice, and those are also figure errors.

 

If aspherics of the form ZERNIKE. BICONIC, or TORIC are not so declared, they will be treated the same as regular surfaces, with irregularity modeled with a cylinder term and (except for Zernikes) rolled edges modeled with a high-order aspheric.  This would be appropriate for surfaces tested via a null setup, where one can observe fringes and measure exactly those errors.  Other aspheric shapes are not assigned a tolerance if not declared with PFTEST.

 

It is an error to specify this attribute for a surface that is not declared aspheric in some manner.

 

If ALL is specified instead of surface numbers, the program automatically assigns this attribute to all surfaces within the active range that already are aspheric.

 

The default RANGE for the modeled cylinder error is about 300 waves of sag at the aperture.  This is then reduced as necessary to create the final budget.  To enter a different range, calculate the actual maximum sag error you want at the full aperture, in lens units, and divide by the square of that aperture radius, giving the sag at unit aperture.  Then enter that value with

 

RANGE sn PFTEST sag.

 

EXACT…

gives the variables for which exact measurements are available.  These will be bypassed by BTOL and will contribute nothing to the tolerance budget.  For example:

 

EXACT ALL            ; all variables are exact. (No numbers entered.)

 

EXACT TH 2 4 6     ; these three thicknesses are exact

 

EXACT ALL INDEX ; all indices of refraction are exact

 

If you enter FIX or RANGE input with a zero argument, the program converts that input to EXACT.

 

RANGE…

defines the maximum permissible tolerance on a particular variable.  Any tolerance that tends to be greater than this amount will be set at that value.

 

Although this parameter sets a maximum, it is not strictly used for that purpose.  (Nobody ever complained about a tolerance that was too loose!)  Instead, by increasing the range assigned to a variable that is coming out too tight, that tolerance will be made looser by the budget process when you run BTOL again. That is its true purpose.

This is the only BTOL variable for which the expected mean value is nonzero.  Rolled edges are almost always down, not up, and it is unlikely that an error on any surface will be compensated by an opposite error elsewhere – which is the case for other variables.  This affects the statistics of the calculation, and to account for this special case the program returns a tolerance that is reduced by square root of 1/3.  MC also models this reduced tolerance, always applying it to a turned-down edge, and the image quality statistics then come out as expected.

 

This variable is not available for some of the more exotic shapes, such as Zernike surfaces.

Element TILT

005 in. TIR at CAO radius.  Enter tilt values in degrees.  Tilt and decenter tolerances will involve a single element unless a relative tilt or decenter is already in effect.  In that case, the tolerance applies to the same group size as in the nominal lens. The global variety, described in Section 12.1, as well as the TTR input of this section, is always input in TIR units.

 

If an element or a group of elements has been declared a GROUP, the default tolerance on tilt and decenter is disabled and the group will instead be given tolerances on all six degrees of freedom: X, Y, Z, and alpha, beta, and gamma.

 

Element decent.

.005 in.  Element tilts and decenters refer to a single element unless the element is cemented; then it refers to the entire group.  This parameter is related to the roll options, RL1, and RL2, which are discussed below.

 

Note that the default option is to tolerance the Y-decenter only, assuming that the X-decenter will have a similar tolerance.  But for some off-axis mirror systems this is not the case.  If your MC analysis differs substantially from the predictions of BTOL, try adding the XDC tolerances to the variable list.  For example, if surface 2 is a tilted and decentered mirror, you might add

 

RANGE 2 XDC 0.1

 

to the declarations section of your BTOL input, and check whether the MC analysis is more on target.

 

You can also declare the surface or element a GROUP, and BTOL will generate tolerances in all six degrees of freedom, which is even better.  See the comment under TILT, above.

 

RL1, RL2

These refer to the roll of an element, which is a decenter combined with a tilt such that the element remains in contact with a mechanical structure to the left or the right.  See below for a discussion.  Since this option must be explicitly requested, there is no default range.  See also Section 12.1.1.1 for a discussion of the use of these parameters with cemented surfaces.

 

PEP

This is the piston error of the faces of a prism.  See the discussion of prisms, below.

 

PAX

The angular error in X (the beta direction), used only for roof prisms.

 

PAY

The angular error in Y, normally used for prisms, for faces other than roofs.

 

This parameter can also be used when you want the wedge tolerance to be evaluated on side 1 of an element instead of side 2, which is the default.  To use it in this way, first disable the wedge tolerance with

 

EXACT WEDGE SIDE2

 

where SIDE2 is the surface number of the second side, whose wedge you wish to disable.  Then add

 

RANGE SIDE1 PAY TANGENT

 

where SIDE1 is the first side and TANGENT is the tangent of the range angle.  For a range of 6 arcminutes, for example, range would be 0.001745.

 

The budget comes back with a tolerance on term G 2, which is the aspheric term linear in Y.  This is effectively the tangent of the wedge angle tolerance on side 1.

 

ZYDC

This is valid only for ZFILE zoom lenses, and applies to the centration error of zoom groups.  The default range is the same as for element decenters, described above.  For this variable, SN refers to the group number, not the surface number.

 

PFTEST

See the details for the global assignment of this attribute, above.

 

GPANG

This parameter applies only to surfaces that are already declared GROUP.  They will be given a tolerance of three angles, alpha, beta, and gamma (in that order).  Data are input in degrees, and the default is 0.143 degrees, or 8.59 arcminutes.

 

GPPOS

This also applies only to surfaces declared the start of a GROUP.  They will be assigned tolerances in X, Y, and Z.  Data are in lens units, and the default is 0.005 inches or 0.127 mm.  With these group tolerances, you know the alignment requirements in all six dimensions.

 

 

CEMENT

Use this option when you have modeled a cement layer as an explicit surface with very small thickness.  There are several consequences of doing so, which are explained in Section 12.1.1.1.

 

 

The derivative increment is taken at the current RANGE.  Since the accuracy of the derivative is questionable if the resulting change of quality is excessive, the program will reduce the increment in stages until the resulting changes are reasonable if this should occur.  (For SPOT and WAVE descriptors, the degradation is allowed to approach 5 times the limit; for MTF and STREHL descriptors, the limit is given by the IMAX parameter.)  When this happens the program will display an informative message saying that the increment has been reduced.  You may disable this message with the option NODMESSAGE.

To enter non-default values for any of these quantities, enter a positive number in the actual lens units.  The surface number used for element tilt and decenter should be the first side of an element, while element wedge is assigned to the second side.

 

FIX...

assigns fixed tolerances to particular variables.  Any variable entered with this option will be assigned the fixed tolerance when the budget is prepared, regardless of its sensitivity.  Too large a fixed tolerance can prevent a successful budget from being found.  (Note: BTOL assumes that the fixed tolerance on a variable represents a range with uniform probability distribution – which is also the case for tolerances that it calculates.  The standard deviation of such a variable is the tolerance divided by the square root of 3.  If a variable is expected to be at its tolerance instead of at a random place within it, the fixed tolerance input should then be increased by 1.732, since the program calculates the effect of what it believes is the standard deviation, which is smaller.  See sections 12.3 and 12.6.

 

The special operands ATH and GTH can be declared FIXed; those apply to all airspace thicknesses and glass thicknesses, and override any FIX assigned to TH variables.  Those operands can also be assigned a RANGE.

 

GROUP LEFT, GROUP RIGHT

 

 

are used when a thickness variation is offset by a neighboring thickness or airspace.  Depending on the details of how the lens elements are mounted, a given thickness error may simply displace all other elements to the right or left, or may only affect those neighboring surfaces which depend on the same reference surface in the lens cell.  In the sketch below are shown two groups of lenses, where the leftmost group is positioned against a reference surface to the right of the group, and the rightmost group against a surface on the left.  This is a common occurrence in lens cells, and it is apparent that if an element in the left group were made thicker than nominal, the airspace to the left of the group would be reduced a corresponding amount.  Similarly, an increased thickness in the right group would cause a reduction in the airspace to the right.  These thickness and airspaces are therefore not independent variables, and to properly evaluate their effect on tolerances the correlations should be stated with the BTOL input.

 

 

 

In this example we would input, for the two groups:

 

GROUP LEFT 5 9

GROUP RIGHT 11 15

 

indicating that surfaces 5 through 9 are mounted so that any thickness errors are offset by a corresponding error of the opposite sign to the left of surface 5, and the errors of surfaces 11 through 15 are offset to the right of surface 15.

 

THC...

may also be used to specify that a given thickness is to be correlated with a neighboring thickness or airspace. If surface 6 were to be correlated with surface 9, for example, we would input

 

         THC 6 9

 

which means that an increased thickness of surface 6 causes a reduced thickness of surface 9.  Note that the reverse is not automatically true; thickness 9 may have an error of its own that is independent of any other surface.

 

This optional input should not be overlooked, since many thickness tolerances vary by more than a factor of two when the correlation is considered.  The default considers each thickness independent of all others.

 

The surface numbers specified may not correspond to those assigned to the start and end of a ZFILE zoom group.  They may apply to surfaces within a group, however.

 

INCL

may be input if the program is to consider only a portion of a lens, as for example, to tolerance the objective of a telescope without considering the eyepiece.  Only the surfaces numbered from JSSS through JSPS will be treated.  The default is to tolerance the entire lens.  Only one INCL entry may be input.

 

EXCL

This input will exclude those surfaces from SN to SN from the tolerance analysis and budget preparation.  Any number of EXCL entries may be input.  Automatically excluded are reference surfaces (with no index change) and the object and image surfaces (except for image tilt, which is included).

 

 

 

Calculating a tolerance for element centration requires knowing something about the mounting scheme for that element.  The default parameter DECENTER (or YDC) will treat the perturbation as a pure decenter in the Y-direction.  This might happen if the lens is rather thick and it is held by gravity against the ID of the lens cell, or when there is a flat area that is pressed against a shoulder.  In other cases, however, as when a curved surface is constrained by a shoulder or flange on one side and a retaining ring on the other, the lens is more likely to roll against the shoulder, acquiring a small tilt in the process, as well as a decenter.  Thus, the element decenter and tilt are coupled.

Since the program has no way of knowing which mounting scheme you intend to use, there is no default for the roll options: you must give an explicit RANGE yourself in the BTOL input.  Also, you can only use one of the options DECENTER, RL1, and RL2 on a given element.

 

There is no conflict between the above parameters and the element TILT parameter, however, which you can see are both treated in the above samples.  Thus, you may consider the TILT tolerance to govern the machining of the cell, and the RL1 or RL2 tolerance to apply to the location of the lens within that cell.

 

The RL2 option will roll about the next surface that is followed by AIR.  This means that if your lens has a cemented doublet or triplet, the actual surface that defines the roll geometry is the last surface of the cemented group, not necessarily the second side of the element you assign the RL2 to.  Further, note that it makes no sense to specify an RL2 range to other than the first element in such groups; each element can only roll about the cemented interface to its left, which requires the RL1 option for all but the first, which can use either option.  See Section 12.1.1.1 for more details.

 

The Monte-Carlo program MC perturbs the lens in accordance with the selection you made with the BTOL input.  If you selected one of the roll options, then that program assigns both a decenter and tilt to the element, in addition to any element tilt that also was given a tolerance.  The only restriction is that the tilt and the remote tilt due to the decenter are always in the same sense, either alpha or beta – since relative tilts can only be in one sense at a time.  We do not believe that this has any ill effect on the statistics of the result, since the two can either add or subtract depending on the random sign chosen.

 

The roll options RL1 and RL2 are distinct from the rolled edge tolerance.  The former describe position errors in a given element, while the latter is an error in the shape of a given surface.

Prism tolerances

 

Prisms, taken from the prism library, require special treatment for tolerancing.  The optical faces themselves are given a figure tolerance, as are all other flat optical surfaces.  But in some prisms a given face is encountered more than once.  To model this, the program automatically assigns a curvature pickup where needed, as well as an index pickup to all interior spaces.

 

The thickness and slope errors require three additional variable types, not available for ordinary surfaces.  These are:

 

PEP        the piston error in each face except the first

PAX        the angular error in X

PAY        the angular error in Y.

 

BTOL automatically adds the variables listed above when tolerancing a prism, modeled via G-coefficients.  Thus, term G11, which is linear in X, provides a way to model an error in the beta-tilt assigned to roof surfaces, and is varied by the parameter PAX, while term G2, which is linear in Y, models the alpha-tilt error on all faces that are not part of a roof, according to parameter PAY.  Term G17, which is a constant term in the aspheric description, can be viewed as a piston error: the surface is located either above or below its ideal location.  This is the function of parameter PEP.

 

To understand how all this works, consider the Schmidt prism below.

 

 

Here, light enters at surface 10, reflects from 11, goes to the two roof surfaces at 12 and 13, reflects from 14, and exits through 15.  Surfaces 10 and 14 are really the same, so the program assigns a curvature pickup on 10.  Similarly, surface15 is assigned a PCV of surface 11.  (Note that the surface with the larger CAO is the pickup target; this is so that when BTOL calculates the rolled-edge tolerance REDGE it puts that rolled edge at the larger CAO.)  When surface 14 is assigned a PAY variable by BTOL, modeling an error in alpha tilt, surface 10 gets the same tilt via the PCV, and similarly for surface 11 and 15.

 

The roof surfaces at 12 and 13, on the other hand, are described by a local beta tilt, and to show the effect of an error in this tilt the program assigns the variable PAX.  Thus, both surfaces of the roof then get tolerances on this variable.

 

One more point should be mentioned.  In a system with a roof prism such as this, the program automatically puts the system into nonsequential mode.  In this mode, the order in which rays encounter the roof surfaces depends on which side of the pupil they are on.  Although this mode gives meaningful raytrace paths and the lens drawing programs show the correct paths, raytracing is very slow.  Especially for optimization and BTOL, where one wants speedy results, the loss of speed is significant.

 

To cope with this situation, we have provided a mode switch, number 37, which will cause the program to always trace rays sequentially, even if the system is in nonsequential mode.  It turns out that the image quality one calculates is not dependent on the sequential order of the rays through a roof prism, so one can get the faster speed of sequential raytracing even if the system contains such as prism.  For producing an accurate drawing, on the other hand, one must turn that switch off.

 

BTOL will run properly with either setting of switch 37, but much faster if it is on.  In that case, one gets the faster speed when it calculates the tolerances on all surfaces except roofs.  For those surfaces (only) the program temporarily turns the switch off, calculates the tolerances (which depend on the surface order now) and then turns it on again, if it was on, when it has finished with the roof surfaces.

ZFILE zoom lens tolerances

 

When zoom lenses are assembled, elements within a given zoom group are mounted in a sleeve, and then that sleeve is mounted inside larger tubes in which the cam curves have been machined.  Thus there are two centration tolerances of concern: the elements within the sleeve and the group inside the tube.  BTOL assigns two separate tolerances for those two assembly errors.

 

The element centration tolerance is given in the budget as a decenter, as usual, and a separate list gives the centration tolerance of each zoom group.  The Monte-Carlo analysis (MC) honors both of these, yielding statistics for random combinations of both errors.
