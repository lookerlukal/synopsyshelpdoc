UNITS { INCH / MM / CM / M }

 

Defines the lens units.  Default is INCH unless switch 24 is ON, in which case it is mm.  If units other than inches are selected, the text in this manual may be interpreted by reading the entered units wherever "inches” is written in reference to lens or image dimensions.  (But not in reference to the page margins, spot symbol size, etc.)

 

APS NB

is the surface number of the stop.  Positive values yield a paraxial pupil solve, negative values a real-ray pupil solve . (See section 2.6.)  If NB = 1, or if this entry is omitted, no pupil solve is performed, and the values of YP1 and XP1 (if any) entered with the subsequent object specification are used.  APS input also removes the wide-angle pupil option.

 

 

WAP [1-3 ]

sets the wide-angle pupil options on.  See section 2.6.3.

 

VFIELD NHBAR …

 

initiates the pupil vignetted with field option.  See section 2.6.3.1.

 

NOP/

NSOLVE

NOP removes all pickups and solves in the lens, and NSOLVE removes all thickness and curvature solves, leaving pickups alone.

 

SYMM

removes all asymmetries caused by tilts and decenters, and all deformations.  Torics and biconics will be changed to spherical surfaces.

 

BTH NB

adds the value of NB to the back focal distance if the latter was computed by a thickness solve.  To remove a current BTH, enter BTH 0 in a CHG file, or remove the solve.  To vary this parameter in the PANT file, use VY 0 BTH.

 

This parameter may also be given if the lens is in APERFECT mode.   In this case, the image will be evaluated at a position that is shifted by NB from the focal plane of the simulated perfect lens.

 

VIG

turns on the automatic vignetting option, which deletes rays that violate clear apertures or encounter feathering or trace failures, in most of the analysis programs.  (This is the default option.) See section 2.6.2.

 

NOVIG

turns off the vignetting option.  Only ray failures will cause ray deletion.

 

CFIX {C/E/R)

fixes the CAO's that have been previously calculated with CAP as though they had been entered by the user.  Normally, CAO's computed with CAP are ignored when vignetting is calculated, but when CFIX is entered they are changed to hard apertures as though entered explicitly.  You should call CAP before including this command in a CHG file.  Used alone, the CFIX directive fixes all CAOs in the system.

 

The optional arguments are for systems declared XCAP, which makes the default apertures rectangular.

The form CFIX {C/E/R} (with no surface number) will convert the aperture of all surfaces into a circular (C), elliptical (E), or rectangular (R) hard aperture.  .

 

You can also use the form SN CFIX {C/E/R} to fix the CAO of only a single surface. with the same options..

 

CFREE

frees all CAO's from vignetting consideration.  User-entered apertures are discarded

 

AFOCAL

causes the final ray angles in radians to be output instead of ray intercepts at the image surface.  OPD output represents the departure of the wavefront from a plane wave.  When using this option, the last two surfaces in the lens must be flat and coincident, and the last must contain no tilts or decenters (or undos).  This option should be used if the distance to the image is extremely large (i.e., if UMC 0 is used on the last lens element) to avoid errors inherent in using very large numbers.

 

The ray angles output by SYNOPSYS are defined in the following illustration, where the values Φ and Ψ give the angle to the ray as measured from a point on the ray projected to the XZ or YZ plane.

 

 

This geometry has some subtle implications, which are discussed in Section 7.5.1.

 

For best accuracy, the last surface in the lens should be in the vicinity of the exit pupil.  If it is not, and the rays are not exactly parallel to the chief ray, then the OPD calculation varies with the distance along the beam where the wavefront is calculated, and results are imprecise.

 

The AFOCAL option and its variants all model the image at a specified distance with a curvature centered at the last surface, or infinity for pure AFOCAL.  Occasionally you may encounter a system operating at a very high F/number (which normally calls for AFOCAL mode) but where the image surface is flat and not a sphere centered about the last surface.  Thus it does not fit the AFOCAL model very well.  Another mode is available for such cases.  Declare the system SLOW, and some of the precision issues are avoided.  See that link for details.

 

APERFECT EFL

This option is intended for use with a lens whose output is collimated and which ordinarily would require the AFOCAL option.  It will evaluate the image quality and location that one would see if a perfect lens of the specified focal length were placed behind the system.  Output is then in linear lens units instead of angular units.

 

Like the AFOCAL option, this mode requires that the last two surfaces be flat, but in this case the two are not coincident; the program will assign a separation equal to the requested EFL, plus any BTH value that may be in effect.

 

A subtle difference arises because the image in AFOCAL mode is evaluated as it would appear if you looked directly along the chief ray, while the image from APERFECT mode is evaluated looking perpendicular to the image surface, as in FOCAL mode with a flat image surface.  As a result, the image size and shape will be different in the two cases.

 

See Section 7.5.1 for more information.

 

FOCAL

restores the default FOCAL mode.

 

ACCOM DIOPTERS

 

is used for a system that is nominally AFOCAL but may be slightly defocused.  Intended primarily for eyepiece design, this option requires two final surfaces, as does AFOCAL mode, but automatically places the last at a distance of 1M/DIOPTERS from the previous one, and gives the image a curvature centered at the previous surface.  An accommodation of minus one diopter would therefore yield a surface at a distance of minus one meter with a radius of curvature of one meter.  If the eye of the user were focused at this accommodation, the image surface would be everywhere in focus.  RAY output is in radians, as with AFOCAL mode, and all image analysis that is referenced to the chief ray is in units of angular arc length along this curved image surface rather than X and Y coordinates.  OPD errors are measured from a reference wave converging to the image surface.

 

Be warned that this feature skirts the very situation for which the AFOCAL option was intended: an image formed by a very slow beam.  This is an issue for the following reason.  If all of the rays converge to the chief-ray point, then the OPDs will of course be zero.  But if a small aberration causes a ray to become parallel to the chief ray, as it might in an extremely slow beam, then the OPDs are again zero.  Simple geometry shows that there are always two solutions to the minimum-OPD curve, and a distant, accommodated, image puts them very close together.

 

This problem is largely avoided by disabling the Debye approximation and referencing the OPD values to the actual distance from the pupil to the ray intercept.  For this reason, the Debye approximation is automatically bypassed in ACCOM mode.  This precaution moves the two regions further apart, and OPDs are more likely to be correct.  In any event, be sure to examine the TAP and OPD curves to verify that the second is the integral of the first, and make sure that the next-to-last surface is indeed at the exit pupil.  You can bypass the Debye calculation yourself by turning switch 66 ON.

 

It is possible to change the accommodation setting during optimization, for cases where the field is allowed to curve but the image must be sharp.  See Section 10.3.1.

 

FILLSTOP

causes the paraxial quantity YMP1 to be replaced by a new value calculated so that the marginal paraxial ray intercepts the stop surface at the user-entered clear aperture.  This calculation is performed each time a paraxial raytrace is executed. See 2.6.5.

 

NOFILL

removes the FILLSTOP and CSTOP options, retaining the most recent values of YMP1 and CAO at the stop.

 

CSTOP [ REAL X Y [ MARGIN ] ]

 

 

sets the CAO of the stop surface equal to the axial marginal ray height on that surface.  Default is the paraxial ray, but a real ray at pupil coordinates (X Y) may be requested. See 2.6.4.

 

PCAO

causes default clear apertures to be based on a paraxial raytrace, rather than on a real raytrace.  This is useful when roughing out a system where there are real ray failures, and the lens drawings have no useful aperture shown.  The default apertures are not used for vignetting analysis, as mentioned in section 6.2.1.

 

If you turn on switch 28, this option is defeated.  In that case, the program will automatically enter PCAO mode temporarily if it finds ray failures in the clear aperture calculation resulting in zero apertures (and surfaces missing from the drawing!)

 

RCAO

removes the PCAO option.  Default clear apertures will be based on real rays, as described in section 6.2.1.

 

CFOV

SFOV

specifies that the field of view in X is the same as in Y.  CFOV declares it to be circular, while SFOV makes it square.  The program automatically sets the X-field size XP0 equal to YP0.

 

If CFOV is selected, the program causes CAP to treat YP0 as the maximum field for determining clear apertures, and programs that analyze the field in both X and Y clip the field at a circle whose radius is YP0.

 

If SFOV is selected, the program uses the semi-diagonal dimension of the field, tracing it in the Y-Z plane, to find clear apertures, and programs that analyze the field in both X and Y go out to the corners of the square field.

 

You must still input both HBAR and GBAR parameters to utilize both the X and Y extent of the field when tracing rays or analyzing a given point in the field.

 

RFOV

specifies that the field is rectangular.  This command does not change the current value of XP0, but causes CAP to trace the semi-diagonal dimension of the field to find clear apertures (traced in the Y-Z plane).  RFOV is the default.

 

CPUPIL

specifies that the paraxial pupil is circular.  This is the default, and if you change YMP1 manually, the value of XMP1 will automatically follow.  If you attempt to change XMP1, the new value is ignored.

 

EPUPIL

specifies that the paraxial pupil may be non-circular. If you change YMP1 manually, the value of XMP1 will not change, and the pupil will in general then be elliptical in shape.  A CSTOP in this instance will yield an elliptical aperture on the stop.  Also turns on XPXT mode.

 

RPUPIL

specifies that the paraxial pupil is rectangular. If you change YMP1 manually, the value of XMP1 will not change.  A CSTOP in this instance will yield a rectangular aperture on the stop.  Also turns on XPXT mode.

 

RHAND / LHAND

These entries control the definition of positive X and ZZ.  The default is a left-handed coordinate system. See Section 2.4.

ADD NB

adds the entered NB to all following surface number references.  This can be useful for renumbering a section of a lens file.  Type LE, and then insert the ADD … line before the section you wish to renumber. Another form of the ADD directive takes a surface number, and only renumbers surfaces beyond a given point.

 

GLOBAL

turns on the global option for the system.  Global output may be requested giving surface positions (see 4.6) and ray paths (see 6.3.2). This mode is automatically set when you request any analysis that requires it, or if any surfaces are global, local, or have tilts or decenters, or the external option is on.  See section 2.4.2.

 

RELATIVE

may only be entered to change a lens that currently is in GLOBAL mode.  All surfaces will be made RELATIVE, using normal tilts and decenters to define the position – unless doing so would require multiple tilts on any given surface.  In that case an informative message will be printed.  System GLOBAL mode remains on.  (To turn off system global mode, use the LE lens editor, and remove all global references in the lens, or enter "SYMM" in a CHG file.)  See 3.8.1

 

ICR XEN YEN

Image Chief Ray: defines the pupil coordinates of the ray to be used as a reference for image quality features.  This includes all chief-ray-related features such as FANs, SPTs, etc. -- but does not affect the ray used by the real pupil feature, which is always at the center of the pupil.

 

GTS

Global thickness = Separation (see 3.8.2).  This is the default.

 

GTZ

Global Thickness = Z-component.  See 3.8.2.

 

XPXT

specifies paraxial raytracing in both X-Z and Y-Z planes.

 

YPXT

specifies only Y-Z paraxial trace (default).  Also turns off the EPUPIL and RPUPIL options, if on.

 

NPXT

disables the paraxial raytrace performed automatically when a lens is input or changed in any way.  Only the raytracing is defeated; options such as pickups, reverse tilts, etc. are still honored – but paraxial solves cannot be processed and should not be present.  This is intended for systems in which the real ray and paraxial-ray behavior are not coupled, such as in nonsequential systems.  Certain features of SYNOPSYS, such as NAR, BEAM, BGI, etc. are also defeated in this mode.

 

PXT

restores the paraxial raytrace option, in YPXT mode.  This parameter is distinct from the PXT command.  Also turns off the EPUPIL and RPUPIL options, if on.

 

NCAP

turns off the automatic clear aperture calculation performed before DWG, PER, and certain other features of SYNOPSYS.  This is intended for systems in which the default rays selected for CAP are not appropriate or do not even trace, such as an off-axis strip-field telescope, which has no "on-axis" field point.  If this option is used, every surface should have a user-entered CAO or other aperture.

 

CAP

turns on the automatic CAP option (default).  This parameter is distinct from the CAP command.

 

YCAP

This causes default clear apertures to be calculated only in the Y-Z plane.  If the system has a skew field defined, the calculation is otherwise taken at an angle corresponding to the maximum corner of the field.  If the field is non-circular, that default may not be appropriate, especially if you plan to cut the lenses to a rectangular shape.

 

XCAP

This option calculates default clear apertures in both the X and Y directions, producing rectangular apertures on all surfaces not assigned a hard aperture.  It is intended mainly for strip-field designs, where the skew field is different from the meridional.  It also honors the DCCR directive, which assigns the center of the aperture to the midpoint between the upper and lower rim rays.

 

NOSEQUENTIAL

specifies a system in which rays may encounter surfaces in other than numerical sequence.  See chapter 18 for a description of this feature.

 

SEQUENTIAL

restores sequential raytracing.

 

FFIELD

alters the object height so that the absolute value of GIHT equals the user-entered CAO on the last surface.  In AFOCAL mode this CAO is entered in angular units.  NFFIELD removes this option.  This is a paraxial feature.  The OCALC command is a real-ray equivalent.

 

Note that FFIELD is probably not compatible with a user-entered value of YP1.  The latter sets the chief-ray height on surface 1, while FFIELD changes the definition of full field.  That implies a value of YP1 that very likely is not what the user entered.

 

VACUUM

sets the index of airspaces to 0.9997076.  This entry changes all airspaces in the lens; you may also change individual surfaces.  See section 3.3.4.

 

IMMERSE { WATER / SEAWATER / AIR / CUSTOM Nd Vd }

This entry declares the entire lens system immersed in water, seawater, or a custom material.  All airspaces will be raytraced with the index of that material, as defined by GLM parameters.  Those parameters closely match the measured index values for water and seawater, and you should evaluate any custom material you wish to use to see if the glass model is adequately described by the index and Abbe number.  If so, it should be safe to use those data as well.

 

You cannot assign the VACUUM property to individual surfaces when this is in effect.  Enter IMMERSE AIR to restore normal AIR in airspaces.

 

The system PRESSURE is reset to 100% if this is entered.

 

AIR

restores index of all airspaces to 1.0.

 

PRESSURE P

sets the index of airspaces to a value that depends on the value of P, which is the atmospheric pressure expressed as a percent of normal.  Thus, if P = 100, the result is ordinary AIR, while 0 is vacuum.  Indices so affected will be listed with the material (AIR) instead of AIR, alerting you to the distinction.  The system pressure will be annotated in SPEC and other listings only if it is not equal to the default 100%.

 

AI can also show and change the system pressure, with the AI input

 

PRESSURE?

PRESSURE = value

 

TEMPERATURE TEMP

sets the system temperature at TEMP degrees C.  This directive alters the index of refraction of all glass-table glasses that have been assigned thermal-index coefficients, or any indices that have user-entered coefficients.  It has no effect on other dimensions in the lens.  See Section 3.3.4.  The system temperature will be included in SPEC and other listings only if it is not equal to the default 20 degrees C, or to inform you if any glass indices were recalculated to that temperature.

 

Note that the Schott and Ohara catalog index values are measured at temperatures of 22 and 25 degrees, not the expected 20 degrees.  The program will recalculate the indices at the system temperature (which defaults to 20 degrees) if any glasses are from those catalogs.  You can therefore expect to see a slight difference between the index values reported by SPEC and PRT and those in the vendor’s catalog.

 

AI can also show and change the system temperature, with the AI input

 

TEMP?

TEMP = value

 

FNAME 'name'

This entry is filled in by the program whenever you fetch or save a file.  The name entry is just the filename, which is then automatically restored after a GET or by any feature that uses the library for temporary storage.  You will usually not have any reason to change this entry yourself.

 

If you do want to specify a filename here, you have 31 characters available, and should avoid special characters such as !@#$%^&*()|\?/><~ and blanks.

 

NCOP/NTOP/NIOP

These entries will remove all curvature pickups and solves, all thickness pickups or solves, or all index pickups in the entire lens.  Another version, following a surface number, affects only that surface.

 

LRAYS/RRAYS

Use LRAYS if light enters the lens going to the left.  The default is RRAYS, which propagates to the right.

 

NAS

Removes all relative tilts and decenters in the lens.

 

NDEF

Removes all general aspheric deformations in the lens.

 

ALPHA data

This entry gives the default coefficient of linear thermal expansion (CTE) or alpha, of the material used for the lens housing.  The THERM feature will by default assign this coefficient to airspace dimensions, in accordance with whatever mounting scheme you have declared. See that link for details.

 

The second entry may either be an expansion coefficient or one of the names on the following list.

 

A5086 aluminum alloy 5086, coefficient 23.7E-6

A6061 aluminum alloy 6061, coefficient 23.4E-6

BECU beryllium copper alloy, coefficient 18.0E-6

BER beryllium I-70H, coefficient 11.4E-6

IRON gray cast, coefficient 10.8E-6

FCBRASS free-cutting brass, coefficient 20.5E-6

BRASS yellow, coefficient 20.3E-6

COPPER UNS C14310, coefficient 17.0E-6

FUSILICA, coefficient 0.55E-6

GRANITE, coefficient 6.3E-6

INVAR, coefficient 1.2E-6

SAPPHIRE, coefficient 4.0E-6

S304 stainless steel 304, coefficient 17.3E-6

S316 stainless steel 316, coefficient 16.0E-6

S440 stainless steel 440, coefficient 10.2E-6

ZERODUR, coefficient 0.5E-6

PYREX, coefficient 4.2E-6

 

PRRULES

This optional system entry enables principal-ray rules, which state that the reference point for OPDs and images that are not in the major color will be the principal ray in the requested color instead of the chief ray in the major color.  This is intended primarily for designing spectrometers, where one wants a sharp image in each color but does not want the images to fall at the same place.  If this option is selected, the effects of lateral color will be ignored.  In this mode, it makes sense to ask for single-color analyses, in order to examine each image in turn.  You can also ask for a multicolor analysis, but the images will then be superimposed as though there were no image separation, which may be confusing.

 

The optimization request OPD will function the same as OPP, and GNO will be the same as GPO, so you may use either for optimization with OPD targets.  For transverse-ray targets, the GPR form should be used, since GNR will still reference the chief-ray intercept, which would no longer be appropriate.

 

To remove this option, use CRRULES (chief-ray rules).

 

SLOW

If your lens works at a very slow speed (F/100 or more) but cannot be analyzed in AFOCAL mode because of special geometry, turn this option on to put into effect a special F/number calculation used by the diffraction image analysis routines. Normally, the F/number at the image is calculated by analyzing the convergence angle of selected rays.  But for slow systems, that angle will be very small, and geometric aberrations may alter it by an amount not much different, thus making the F/number calculation, and the diffraction analysis which depends on it, unreliable.  If this option is on, the calculation is instead done by analyzing the size of the beam at the next-to-last surface, which, together with the back focal distance, gives a more reliable figure for the F/number. This only works if the image plane is flat and without tilts and decenters or undos.  The thickness before the image should be large compared with the geometric aberrations.

 

To remove this option and restore the default, enter FAST.

 

PSPACE

This declaration can be assigned to a surface that follows a grazing-incident reflection. If rays headed to the right are still going in that direction after a reflection, they become strange rays.  Those rays follow different rules for intersections, and elements in negative space are rendered differently.  But if all rays in the beam are strange rays, it may be more convenient to declare the surface following the mirror PSPACE.  Then the normal rules for intersections and rendering will apply.

 

Note that strange rays following a mirror will intercept the next surface according to the strange rules, even if that surface is declared PSPACE, since the rays themselves are still in negative space until they reach that surface.  Thus it may be necessary to declare that surface INTERCEPT NEAR as well.

 

To remove the PSPACE declaration, use ASPACE to put the automatic space declaration in effect.

 

PCV CV

This option causes curvature pickups to transfer the base radius only, not aspheric terms or the conic constant.  It is used for tolerancing symmetric systems, where a given lens is supposed to be identical to its mirror image but has to be assigned separate tolerances.

 

EVD {X/Y/U/OFF}

Diffraction image analysis by default uses scalar diffraction theory, which means that if two wavefronts are in phase they add, and if out of phase they cancel.

 

But this theory is imprecise in the case of a very fast lens, such as a microscope objective, where the cone angle can be steep.  If rays from opposite sides of the pupil arrive at the image at 90 degrees relative to each other, they cannot interfere and cannot contribute to the interference pattern that makes up the image.  So resolution is lower than scalar theory predicts in such cases.  A perfect lens working at F/0.628, for example, has a scalar MTF at 1/2 of cutoff of about 0.4, while a vector-diffraction analysis yields an MTF of 0.37 in X and 0.26 in Y, if the object is linearly polarized in Y.  One would like to improve the accuracy of the scalar calculation in such cases.

 

If you declare the object to be polarized, this effect is taken into account -- but not all features can analyze the effects of polarization.  For example, the Fourier-transform DMTF takes it into account and does a vector-diffraction analysis, while the other MTF features, which use a convolution method, do not.

 

The EVD (Emulate Vector Diffraction) options provide a way to compensate for this shortcoming.  If it is activated, the programs that would otherwise do a scalar analysis will emulate the results from the more accurate vector analysis, yielding, in the case of the MTF mentioned above, results that agree within about 0.5%.  This emulation ignores reflection losses and coatings, since it is not a true polarization analysis, but is nonetheless useful when you want a realistic estimate of the resolution of your lens.  If polarization mode is turned on, this option is ignored for those features that can utilize that mode.

 

The polarization mode to be emulated is given by the second word; X or Y for linear polarization in that direction, U for unpolarized (a superposition of X and Y), and OFF turns off the emulation.

 

 


REPUPIL B


Most users will not have to use this option.  When the program requires a real chief ray, declared by a minus sign on the APS entry, it performs an iterative search for the ray that passes through the center of the stop surface.  The wide-angle WAP ... options can modify the path, but the first step is to find that ray.

 

Normally, the first iteration tries the paraxial ray path, and if that ray traces successfully, the program begins a search for the central ray starting from there.  But this calculation is more complicated than you might expect, and we have seen many wide-angle lenses for which there are two solutions, owing to the presence of a caustic related to the pupil aberrations.  In the event that the first iteration happens to fall on the wrong side of the caustic, the search usually fails.

 

The REPUPL B option can in some cases succeed in finding the correct chief ray even then.  It ignores the paraxial path and begins a systematic search, performed on a reversed copy of the lens, tracing from the stop surface out to the original object, which is then at the end.  When a ray is found that goes to the desired object point, the original lens is restored and that path becomes the first iteration. (This search is automatically performed if the paraxial path will not trace.)

 

The REPUPIL B option is intended only for rare cases where the paraxial path will trace but finds the wrong solution. The default is REPUPIL A, which starts with the paraxial path if it traces successfully.

 

Be aware that occasionally one encounters a lens for which there is no solution to the pupil search.  A tactic we have sometimes used in such cases is to revert to a previous lens that traced correctly, and then add a target in the merit function so the real chief-ray intercept is close to the paraxial value.  That reduces pupil aberrations, and the real pupil is less likely to fail thereafter.  For example, one might use the following in the AANT file:

 

M 0 1 A P YA 1 0 0 0 1

S YP1.
