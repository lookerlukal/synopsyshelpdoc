5.24 Creating a VFIELD (FVF)                Next Previous TOC

 

The command

 

FVF [ HBAR HBAR …]

 

creates a VFIELD vignetted-pupil array based on user entered hard apertures.  See section 2.6.2 for a discussion of vignetting models.  The HBAR list may contain up to nine non-negative field points listed in order of increasing value.  Rays are traced to determine the maximum extent of the unvignetted pupil at each of the field points.  These data are then converted into a VFIELD array that gives the same pupil geometry at those fields.  If no HBAR values are given on the command and a VFIELD set of HBARs has previously been entered, the current HBARs are used.  There must currently be CAO or EFILE data assigned to the stop surface.  Apertures assigned to other surfaces are evaluated in the same way as for WAP 3 pupils.

 

The lens must have been assigned a real stop before you can use FVF.  This means that if you have modeled an implied stop by varying the paraxial quantity YP1, you must assign a real stop location before running FVF.

 

This feature is a very efficient substitute for the more general WAP 3 option (See 2.6.3).  That option reduces the pupil so as to clear all entered apertures for every analysis that requires raytracing, which is a lengthy process.  The VFIELD, on the other hand, models the results with a stored table and finds the pupil when needed by interpolation, rather than by performing the search all over again.  It is appropriate only when the lens is stable, however, since if the apertures change the data are no longer meaningful.
