13.15 Displaying Error Messages                                         Next Previous TOC

 

Some error conditions can cause the lens file to be incomplete, rays to fail, or the analysis to be inaccurate, and these must be called to the attention of the user.   There are three categories:  Highest are urgent messages about conditions that must be remedied immediately, or else nothing makes sense.  These messages are displayed in a small box whenever they are encountered, and are listed in the Command Window as well, if switch 54 is on.

 

The second category includes errors that may or may not affect the validity of results.  These are listed in the Command Window – but not displayed immediately – so the processing of any MACro then in progress is not interrupted.  Errors in both of these categories also light up the toolbar button , which is normally shown in grey.  Clicking on this button will display all of the error messages that have been flagged since the last time the button was clicked or the flags reset.

 

The last category is for conditions that may not be important.  These are shown in the Command Window but do not trigger display of an error box.

 

We recommend that new users keep switch 54 on (this is the default), because the listed error message is then associated with the command that produced it.  It also helps in documentation, since a condition that may affect the validity of the results is then noted in any hardcopy that you may print.  The disadvantage of keeping the switch on is that some operations, such as optimization, may encounter the condition repeatedly, and produce many copies of the same message in the Command Window.

 

In cases where a message of the first or second category is irrelevant to your system and becomes a nuisance, you can defeat the display of that message by clicking the checkbox at the bottom of the dialog when the message is displayed.  Be warned that if you do this and have switch 54 off as well, you will then get no message of any kind for that error condition, which is somewhat dangerous.  You can reset all error messages, so that they will be shown, by typing the command

 

 

SAE        Show All Errors

 

or by clicking on the menu item Options | Show Error Light for All Errors.

 

In the unlikely event that the display of urgent messages becomes a nuisance – and the messages are not urgent for your project – you can defeat the display of all messages in this category by turning switch 79 on.

 

The error button is automatically turned off by any GET, FETCH, RLE, CHG, or WorkSheet changes.

 

You can optionally display any waiting error messages with the command

 

ERRORS

 

and clear all messages with the command

 

ERRORS CLEAR
