7.8 Edge description file (EFILE, ELIST)        Next Previous TOC

 

In order to show the edges of lens elements as they are to be manufactured, it is often necessary to define certain dimensions that are not normally part of a lens description, such as the bevel width, etc.  This additional information is most easily created with the Edge Wizard, but may also be entered with the RLE data,  or directly (and temporarily) within the data for ELD.

 

An older format for entering edge data uses the EFILE, which is largely superseded by the Wizard.  It has the useful options of creating a default set of edges or deleting or listing the entire set.  For other purposes we recommend the Wizard (MEW) instead.

 

Once the edge data are defined, they become part of the lens description and are saved with the lens (except data entered directly into ELD, which are temporary).  Thus they may be later edited with LE or the Edge Wizard (Command MEW).  The EFILE edges will be honored by the lens drawing programs PERSPECTIVE, RPER , SOLID, RSOLID, ELD, DWG, CDWG, WGT, and ZDWG, provided that switch 39 is ON.  Also, the PAD D and PP displays show these edges and give immediate graphical feedback when the Wizard is used.

 

EFILE edges are defined by five aperture values, A, B, C, D, and E, as explained in the section on the Edge Wizard.

 

The command EFILE CREATE creates a whole new set of edge data, based on rules similar to those described in the ELD section. EFILE ERASE deletes the entire set, and the command ELIST gives a listing of the current edge description data.  You can also use the form

 

EFILE

{ CREATE / LIST / ERASE }

END

or

 

EFILE CREATE SNA SNB

 

The last form generates default EFILE data only for the lens at the entered surface numbers.

 

EFILE data can only be assigned to real surfaces (not dummies, with no index change), and cannot be assigned to surfaces whose shape is not rotationally symmetric or elements involving a HOE, DOE, or grating, or to surfaces declared HYPER or found to require a hyperspherical shape by the CAP program.

 

There are many possible combinations of aperture specifications and EFILE edge descriptions, and many, but not all, are permitted; these are described in detail below.  Note that the rules for lenses and for mirrors are slightly different.

You may assign both EFILE edges and CAO or RAO data to a lens element or to a mirror.  For a lens, the RAO dimensions should be the same on both sides, and the optional angles must all be zero.  The dimensions so provided interact with the EFILE data in special ways, depending on whether they are assigned to a lens or to a mirror.  In brief:

 

1. Soft apertures (default CAOs) show up on drawings if the lens does not have EFILE data.

 

2.  If EFILE data are assigned, then the EFILE data show up, and soft CAOs have no effect on the drawings.

 

3.  If both EFILE and hard CAO data are assigned but the CAO is centered, the latter again has no effect.  But if the CAO is decentered, it is treated as a cored element, which is an element made by coring out a section from a larger parent.  In this case, the parent element is assumed to be edged as usual in accordance with the EFILE parameters, and then a circular section is cored out, with the radius and location given by the hard CAO.  This rule is the same for both lenses and mirrors.  This feature is intended for off-axis paraboloids, among other things.

 

4.  If both EFILE and RAO data are assigned, then lenses and mirrors are treated differently.  Lenses will again first be edged with the circular EFILE data, and then cut off as required to fit inside the RAO rectangle.  This applies whether the RAO is centered or not.  For mirrors, on the other hand, the EFILE data will no longer be considered circular.  This lets you define a rectangular mirror whose bevels on front and back extend down the length of the rectangle rather than around a circular rim, in accordance with the bevel dimensions in the EFILE data.

 

5.  The CMIRROR designation is the only one where the back of the mirror is not necessarily perpendicular to the axis.  In this form, the back is drawn as an exact copy of the front of the mirror, but shifted in the local Z direction according to the entered mirror thickness.  The aperture of the surface is then the common area between the parent circle and any UAP, RAO or CAO in effect, which may be decentered.  Note that, if the apertures are decentered, this is the only valid assignment for a mirror.

Mirrors may be assigned EFILE data with an optional thickness parameter.  The simplest way to create a rectangular edge is to assign the desired RAO first, before adding EFILE data.  Then go to the Edge Wizard, select the surface, select the “Explicit” option (which is required for this feature), and click “Update”.  For best results, the EFILE parameter A should equal the semi-aperture of the RAO in Y.  If this is not the case, just click on the “-F” button, which will make points A and B equal.  The Edge Wizard will then let you directly edit the dimensions of the resulting rectangular edge.  To change the Y-dimension of a rectangular edge, change the EFILE parameter “C” on the dialog; to change the X-dimension, alter the data labeled “X/Y”.

 

 

The picture above shows several mirrors with RAO apertures that have been assigned EFILE edges and thicknesses.

 

You can optionally enter all of the edge parameters yourself in the RLE or CHG file, in the format given in Section 3.10.  The most useful of these options are

 

SN EFILE EX1 A B C ANG1

SN EFILE EX2 E D ANG2

 

With this input you define points A through E and the slope angle of any flat portion.  This may be done easily via the WorkSheet, where you can change the values with the Selection slider – but parameters changed in this way are not automatically coupled to others that have to change together.  For that reason, the Wizard is really better.

It is possible for a lens to pick up the edge parameters from another element.  This uses the RLE option

 

 SN PEFILE { +/- } JSN

 

For example, if the lens is symmetric and the element starting at surface 8 is identical to that on surface 1, but flipped over, you would use

 

{ RLE / CHG }

…

 

8 PEFILE –1

 

END

 

Here, the negative surface number means that the first and second sides of the second lens are interchanged.  If the surfaces are both mirrors, where the second side does not really exist but can be shown on drawings as though it did, a negative number causes the apparent mirror thickness to be reversed in sign.
