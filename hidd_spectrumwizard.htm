Spectrum Wizard Dialog

 

This Wizard is accessed via the menu Wizard | Spectrum Wizard, or via the command MSW.

 

This dialog shows the spectral curves of a source and a target, as well as their product.  You select the wavelength interval of interest by typing into the appropriate data boxes – or by clicking on the “spin buttons” just to the right of the data boxes.  When you have the desired range displayed, select a source and a target, and then click Get Spectrum.  The curves are shown on the dialog.

 

The next step is to utilize these data for multicolor image analysis.  You can transfer the wavelengths and weights to lens file by selecting the number of wavelengths you want and then clicking Apply to lens.  You can then use any of the image analysis features that permit a multicolor “M” entry to analyze the image in those colors with those weights.

 

If the lens contains any indices that do not permit recalculation, you cannot use the above feature since it requires that rays can be traced at the new wavelengths, and this requires new indices at those wavelengths.

 

If you select a discrete source instead of one of the blackbody (continuous) sources, the above options will select only wavelengths that are actually present in the spectrum of that source and within the current wavelength range.  (There may be none, producing an error message).  With this feature it is likely that you will not get the requested number of colors or wavelengths.  But as long as at least one spectral line is in the range, you will get at least that data, and you can then perform the image analysis at that (or those) wavelengths.

 

The button To Clipboard sends a bitmap copy of the Spectrum Wizard dialog to the Windows clipboard.  Then you can paste it into any third-party program that can accept graphics, like Word or any graphics editor.

 

You can also print a copy of the dialog by clicking the Print button.  SYNOPSYS will send a bitmap-copy of the dialog window to the printer.

 

When you click the Apply to lens button, the program normally assigns new wavelengths and weights, as noted above.  But if you select the option to Keep current wavelengths, the program finds the weights at the wavelengths currently in the lens and only alters the weights at those wavelengths, according to the selected source and detector, keeping the wavelengths themselves constant.

 

Caution:

 

When you apply these data to the lens file, you usually change the primary wavelength, and this will result in different paraxial solves, if any are present.  In particular, if the lens contains curvature solves, the radii that result will be different.  This can have profound effects on the entire system if it goes unnoticed, and a warning message will be displayed if this is the case.

 

Of course the transmission of the optics strongly influences spectral weights, as well as do the source and detector.  The Spectrum Wizard does not model this transmission at the present time, however (a future update will augment the entire glass catalog with transmission data for all glasses, and then the Wizard will be able to model the optics as well).  So in order to make a realistic selection of wavelengths and weights, always select a wavelength region which you know is transmitted by the optics, when you use the Spectrum Wizard.  If you want to modify the weights generated by this Wizard, go to the MMC dialog, which shows the weights (and wavelengths) that were generated, and then type your modified weights into the data boxes.

 

If you intend to view the image with Image Tools (command MIT), or the PSVISUAL PSPRD option, do not use the “Visual” weighting curves.  Although this seems counter-intuitive, keep in mind that your eye will add the eye curve automatically, and you don’t want it added twice.  For all other image analysis, however, you should incorporate the eye response curve into the weights before performing your image analysis, if the system is intended for visual use.

 

The detector data presented in this dialog are derived from a variety of sources, including vendor data sheets.  Since OSD does not control these products or data, it makes no warranty of any kind regarding the accuracy or completeness of the data presented.  Always verify these data with the vendor prior to making a commitment regarding a choice of detector.  The curves for CCDs and photographic films are all normalized prior to display, and you should therefore make no assumptions about the relative sensitivity or efficiency of one as compared to another based on the screen display.
