5.18 Aligning components along a ray (AIM)         Next Previous TOC

 

The AIM command will align components so that a given real ray intercepts selected surfaces at the origin.  This is primarily useful when a system has compound tilts of fold mirrors and it is difficult to find the exact angles required.  To use this feature, the system must be set up initially with approximate positions, preferably so that the desired ray will trace.  Then the input file

 

AIM [ HBAR GBAR { ICOL / P }]

TILT SN SN...

DECENTER SN SN...

GO

 

is entered.  The AIM command specifies the ray along which the components are to be aligned with the arguments HBAR. GBAR and ICOL (the axis ray in the primary color by default), and the TILT and DECENTER arguments specify those components that are to be adjusted.

 

The program works in stages; the first surface given in the tilt or decenter list is given global coordinates and sent to the optimization program, where the variables are the global angles and/or position, and the targets are the (X,Y) coordinates of the selected ray at the next surface in the list.  When that run finishes, the ray intercepts the next surface at the origin.  That surface is then varied in turn to target the ray at the third in the list, and so on.  The final run adjusts the last variable in the list to center the ray at the image surface.  The decenter variable is useful primarily when it is the first (or only) variable, since otherwise the centration of the ray at the varying surface (from the previous variable) is lost.

 

This feature is only appropriate when all of the relevant surfaces are in GLOBAL mode.  It does not vary relative tilts, if there are any.  For this reason, if your lens has fold mirrors that are set up as relative tilts with an equal tilt on the next surface to follow the new axis, as is usually the case, this is not the feature to use.

 

You may run your AIM command file via the MACro editor, but do not run it as the alternate MACro.  AIM creates its own alternate MACro to do the optimization, so this must be vacant.

 

The AIM command runs the optimization program in the background, and if you later ask for the final aberrations with FINAL, expecting to see the results of your own optimization done earlier, you will instead see the results from AIM.
