3.8 Asymmetrical Data: Tilts and Decenters   Previous Next TOC

 

Most of the lens data input described above applies to systems with rotational symmetry. The input described below applies to non-symmetrical systems.

 

For best results, surface 1 should not have any tilts or decenters; although this works properly, some other features become confusing: since lens drawings are oriented according to the starting surface, if that is tilted it will skew the whole drawing (which is otherwise correct); also, global coordinates will then be with respect to the tilted coordinate system of 1.

 

There are five possible ways to input tilt and decenter data on a surface: RELATIVE, REMOTE, GROUP, GLOBAL, and LOCAL.  All of them may be edited in the SPS dialog.  In addition, the system as a whole may be given a position and orientation in some EXTERNAL coordinate system, and then analyzed in that system.

 

These options are summarized as follows:

 

 

Family

Features

Restrictions

RELATIVE

Simple input, automatic undo; can also decenter.

Only a single tilt direction at a time.  Tilt axis is can be anywhere on optical axis.

 

REMOTE

Simple input, automatic undo, tilt axis can be anywhere; can also decenter.

Only a single tilt direction at a time.

 

GROUP

Allows three tilt directions, automatic undo; can also decenter.

Euler order always alpha, beta, gamma.

 

LOCAL

Allows three tilt and decenter directions, any Euler order.

 

No automatic undo.

 

GLOBAL

Allows three tilt and decenter directions, any Euler order.

 

No automatic undo.

 

EXTERNAL

Origin can be anywhere.

For analysis only, not for input.
