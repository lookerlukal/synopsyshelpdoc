7.9 Three-Parameter Plots (PA3)                Next Previous TOC

 

AI can prepare a plot showing how a given parameter affects some other quantity, giving you a two-parameter plot.  The feature described in this section prepares a plot where two parameters change independently, and a third is evaluated and plotted as a function of those two.  This feature requires AI mode to be on and uses the Zn parameters to contain the data to be changed and evaluated.  Input must be entered into a MACro for this feature.  The format is

 

[ CONTOUR ]

PA3 { TEST / LOOP / AGAIN / QUIT } [ HT [ R / L ] ]/[COLOR]

RZ1 START END

RZ2 START END

RZ3 MIN MAX

[NO]SMOOTH

XLABEL "label string"                (up to 32 characters, refers to Z1)

YLABEL "label string"                (refers to Z2)

ZLABEL "label string"                (refers to Z3)

LOOP

 

(Any SYNOPSYS commands, where parameters Z1 and Z2 are used as input somewhere)

[Any evaluation that writes to the AI output buffer]

 

Z3 = (something that AI can refer to)

PA3

 

The first PA3 input initializes everything and specifies the height and orientation of the 3-D plot to be made. HT is the height of the highest point on the plot measured from the zero point for that parameter pair.  The R / L choice refers to a right or left rotation of the display, which is rendered as a solid surface.  Entering neither of these gives a straight view, without rotation.  If the PA3 command follows a CONTOUR directive, then the height and orientation data are ignored, and the data are contour plotted, with exactly 10 levels drawn encompassing the entire data range.

 

If the word “COLOR” appears instead of the HT [ R / L ] entry, the display will be formatted and shown with colored pixels instead of the oblique-perspective view.  The color spectrum is applied to the entire range of data from Z3, but any data value that is exactly equal to zero is shown in black.

 

The parameters to be altered are always Z1 and Z2.  The ranges of Z1 and Z2 are given by the RZ1 and RZ2 input.  The range of Z3 for which data will be plotted on the Z-axis may be given by the RZ3 input.  Any data outside this range will be clipped at the end that is violated, and shown by the color option as black if clipped at the low end and white if at the high end.  The default range is +/- 1.0E10.

 

Labels may be entered with the XLABEL to ZLABEL input, and will be shown on the plot if entered.  It is wise to repeat the range points in the labels, since they are otherwise not documented.  The LOOP command ends the PA3 input and sends control back to the command interpreter, which then executes the remaining commands in the usual fashion.  When the second PA3 command is reached, the program loops back to the beginning, alters the variables, executes the command sequence again, and continues in this fashion until the entire parameter space has been evaluated.

 

The influence of parameters Z1 and Z2 results from those symbols showing up somewhere in the command sequence, which may contain ordinary commands or AI strings, but which must not contain any command that utilizes the same 3-D display by which PA3 renders its output, which would corrupt those data, or which themselves loop back to the MACro.  Thus you may not request any of the following commands in this sequence:

 

[W]DMTF, [W]PSPRD, DMODEL, M2F, SURP, SPROFILE, ANNEAL.

 

The quantity to be evaluated as a function of the two variables must be put into the Z3 parameter before the second PA3 command is reached.  This is done in the usual fashion, where Z3 is set equal to some lens parameter, such as

 

Z3 = 4 TH

Z3 = BACK.

 

Or you may execute any command not in the above list that places a copy of its output in the AI buffer, and then load Z3 from there, for example:

 

 Z3 = FILE 4

 

Most commands fill the buffer, and you check by simply executing that command and then entering the AI sentence.

 

BUFFER?

 

If your answers are displayed, note which file location contains the number you want, and then use that number as the FILE location.

 

This feature normally executes the loop a total of 3481 times, giving data for a 59x59 array of points.  It is often a very lengthy feature to run, and it is wise to test your input before committing to the entire loop.  To test it, just enter "TEST" in word 2 of the PA3 command, and run your MACro.  The program will do the loop only once and list the values of Z1, Z2, and Z3 at the end, showing all of the output from the command sequence.  If you time this loop (use the TIME command) and multiply by 3481, you get an idea of how long the process will take.  If word 2 contains "LOOP", the entire array of points is evaluated, and output is automatically put in QUIET mode.  If switch 78 is turned on before the process starts, the array will consist of 118x118 data points instead of 59x59.  (The finer resolution will only be visible in the COLOR display, however, so it makes no sense otherwise.)  It will also take four times longer to run.

 

After the array of points is evaluated and the plot is displayed, you can change the display orientation and height by entering the PA3 command with "AGAIN" in word 2.  In this case, do not use the same MACro, since only a single line of input is required.  (You cannot change the default oblique-perspective or color view into a contour plot, and vice-versa, however.)  So, if your plot was shown with the surface drawing and you want to show it again with the COLOR option, you could simply enter

 

 PA3 AGAIN COLOR

 

after the first display appears.  You can also do the reverse.

 

In the event that the loop terminates abnormally because of an unforeseen error condition, the system may no longer respond to user input.  This is because it is still trying to run PA3 instead of the command interpreter.  To recover from this situation, just enter PA3 QUIT.

 

The optional [NO]SMOOTH directive controls whether the oblique-perspective view is smoothed or not.  If your data show discrete points on a continuous surface, then smoothing should be turned on (it is the default).  If your data contain discrete values which are better shown as a step function, then smoothing should be turned off.

 

To illustrate this feature, we enter the following MACro:

 

PA3 LOOP 3.5 L

RZ1 1 0

RZ2 -.05 .05

XLAB "HBAR 1 TO 0"

YLAB "BTH -.05 TO .05"

ZLAB "1/RMS"

LOOP

CHG

BTH Z2

END

RMS P Z1 200

Z4 = FILE 1

= 1/Z4

Z3 = FILE 1

PA3

 

In this sequence, we vary the BTH parameter from -.05 to .05, thereby changing the focus position slightly, using variable Z2.  We also change the field point with variable Z1, which shows up in the HBAR position in the RMS command.

 

To evaluate the effect of these changes, we then execute the RMS command (in the MACro), which puts the RMS spot size into file location 1.  Now things get a little tricky.  A good image has a low RMS, but on the plot that would appear as a hole instead of a hill, and would not be visible due to the hidden-line removal algorithm.  So we instead put the value into Z4, find the inverse of that quantity with the implied calculation, and then put the results of that calculation into Z3 for the plot.
