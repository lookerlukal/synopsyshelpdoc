3.0 LENS DATA INPUT

 

The specifications for a lens are initially entered into SYNOPSYS by means of a data file of the structure described below.  Lens units are in inches unless switch 24 is ON (making the default units mm) – or unless units other than inches are specifically declared (see 3.2).  Most lens file data may optionally be entered by interactive input to the dialogs SYS and SPS, although it is faster to enter them as described below.

 

 

3.1 RLE Data:  System Input to RLE

 

RLE

ID [sample lens ident.]

[ID1 ' text ']

[ID2 ' text ']

[ID3 ' text ']

[ID4 ' text ']

[ID5 ' text ']

 

[ LOG lognumber ]

[ ILINE ]

END

 

RLE

Read LEns.

 

ID

Enter up to 33 characters of lens identification.  The log number of the most recent run in which the lens was changed is also output whenever the lens ID is printed or plotted.  ID1 to ID5 give up to 5 additional lines of lens identification, which may contain up to 76 characters, which are listed in the RLE and LEO files.  These lines will appear in the LE editor, where they may be changed, and also in a special dialog that you can reach via the Worksheet or with the command ID.  The Al program can also change or display the current ID, with input of the form ID = 'example lens file', or ID?

 

You can also annotate lens files as you save them, and the annotation will be displayed when the files are opened again.  See Section 3.7.2.

 

WAVL

Use this command to input three or five wavelengths, in uM; the first three NB entries are for long, middle, and short wavelengths. The CDF option automatically enters 0.6563, 0.5876, and 0.4861 um; these are also the default wavelengths.  Entering GHI specifies 0.4358, 0.4047, and 0.365 um, in the ultraviolet.

 

With this option, the second wavelength is always the primary color, and you must enter either three or five numbers. Weights are automatically set to unity for all wavelengths, but may be changed with WT1 input (or afterwards with MMC or SYS).

 

If the arguments do not specify three or five wavelengths, the program will interpret the line as WA1 input.

 

WA1, WA2

These options permit entry of any number of wavelengths from one to 10.  Weights are set to unity, and may be changed with the WT1 and WT2 input.  The color order is not set with this input.

 

At least one wavelength must be entered if the WA1 format is used.  To enter seven wavelengths, for example, one could use

 

WA1 .8 .75 .7 .65 .6

WA2 .55 .5

 

WT1, WT2

Use these options to enter the spectral weights of each of the entered wavelengths.   Any number except zero may be assigned to any wavelength.  To suppress any given color, use a very small weight, but not zero.  Thus, to evaluate the above data where you plan to filter out the light at 0.65 uM, you would enter

 

WT1 1 1 1 .0001 1

WT2 1 1

 

CORDER

This input determines the color order: the first entry is the primary color, which is used for paraxial raytracing and any paraxial solves that may be in effect.  It also is the color at which the chief-ray point is found for image analysis.  The second and third are the long and short colors, which give the wavelengths at which third-order chromatic aberration is calculated.  Multicolor image analysis is performed on all wavelengths, in this order.  If only the three required arguments are input, the program will fill in the remaining colors automatically.  The color order is automatically set to (2 1 3) with the WAVL option, but is not set by the WA1 and WA2 options, for which a separate CORDER input is required.

 

LOG

Whenever you enter or change a lens, it is assigned a lens number that is the same as the current LOG number.  This helps you keep track of which run created which lens, and the number is printed with the SPEC and other listings.  When you save a lens and then get it back another time, the lens is assigned a new log number, not necessarily the one that was in effect when it was first generated.

 

If you then wish to document the lens, you probably want the original log number to appear on the listings, not the current one.  If so, just put this line in a CHG file.  The lens number will be set to the entered value.

 

ILINE

This is a system specification intended for lenses that work in the ultraviolet region of the spectrum.  Modern microlithography lenses require finer resolution than can be obtained with light in the visible spectrum and instead work at shorter wavelengths near the iLine (0.395 uM).  But the default glass model utilized by GLM does not accurately describe real glasses in that region.

 

Activating this option puts into effect an alternate set of interpolation coefficients for GLM elements that is a closer match to the behavior of real glasses in the UV.

 

This option only works if three or more wavelengths are defined, and none specify a value greater than 0.45 uM.

 

If this option is selected, some of the glass-map analysis features are disabled, since they need the index at the C line (0.6563 um), which is outside the range of the new coefficients.

 

To remove this declaration, use NILINE in a CHG file.

 

WPOWER

Input power to a microlithography lens.

 

DCYCLE

Duty cycle for a microlithography lens.

 

 

Whenever new wavelength data are entered, the program COLOR (see 5.6) is automatically called following the RLE or CHG input to calculate indices of refraction for all media for which glass model, glass table, or glass coefficients (GLM, GTB, GLASS, GDF) have been specified.  If any material has a fixed index, it is an error to input more wavelengths than indices, and care must be exercised to ensure that if wavelengths are changed the indices are also changed to correspond.
