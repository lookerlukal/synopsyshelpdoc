10.3.5.2 Correction Input                                                        Next Previous TOC

 

In some cases you will want an aberration to be corrected exactly to a particular value.  The PSD damped least squares optimization algorithm employed in SYNOPSYS lends itself to implementing exact aberration correction through the method of Lagrange multipliers.

 

The specification is given by:

 

C TAR WT

 

where the WT entry is ignored since this option in effect gives the aberration infinite weight.

 

During optimization, SYNOPSYS will attempt to correct the aberration to the exact value of TAR.  However, you are cautioned that this will not occur unless the aberration (as well as any others controlled in this way) can actually be corrected exactly to the target value.  It is risky to try to control more than a single aberration with Lagrange multipliers because they often turn out to be incompatible: if the goals cannot be met exactly, the procedure will not converge.  This is distinct from normal optimization, which is designed to balance mutually incompatible goals.  We have seen cases where two correction requests resulted in a singular matrix, and if this occurs, use normal minimization with a high weight instead of correction.

 

If the correction does not converge, it is sometimes helpful to use both a correction and a minimization on the same aberration.  If you give a high weight to the minimization, program will then usually keep the lens in a region in which exact correction is possible – with no penalty to the merit function since the error becomes zero.  For example, suppose we want to keep the object-to-image distance exactly 100:

 

AANT

…

M 100 10 A TH0

A TOTL

A BACK

C 0 1 A ABR –1

…

 

The correction here has specified a target of zero for the previous aberration (which targetted the total distance to a value of 100).

 

We have observed that even though this kind of control yields an exact linear solution, in fact the lens is constantly changing and the solution so found will probably not be perfect at the next iteration.  So we use this feature with caution.
