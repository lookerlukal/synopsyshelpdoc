12.5.2 Including other quantities in MC                                        Next Previous TOC

 

It is sometimes useful for MC to evaluate the statistics of properties of the lens or the image that are not considered by BTOL.  Normally, MC considers only the same quality descriptors that were requested in the BTOL run that produced the budget that is being simulated.  But nearly any other property can be included in the MC evaluation, and the statistics of this property can then be printed and plotted by MC along with its other parameters.

 

To include another property in the MC analysis, the desired quantity must first be assigned to one of the Zn parameters, within the MC file MCFILE.MAC, and then included with the input

 

 MC IZn “label”

 

where IZn is the Zn parameter to be included, and label is an identification string of up to 15 characters that identifies this parameter, enclosed within quote marks.

 

For example, suppose you want to monitor the exact location of the image centroid in three colors at full field during the MC evaluation.  The file MCFILE.MAC might look as follows:

…

END

…

END

(adjustment parameters included by the PREPARE MC request)

RMS 1 1 300

Z2 = FILE 5

= Z1 + Z2

MC IZ3 “Centroid 1”

Pay attention to the use of the Zn parameters.  Here we use Z1 and Z2 for preliminary calculations, and put the desired result into Z3, Z4, and Z5.  MC will read these values when the MACro is finished, and they must not be used for anything else in the meantime or the values will be overridden.

 

And note the final "MC" command.  If you forget to include that, and your analysis is in quiet mode (which makes sense), the system will wait for the missing command and will seem to hang.
