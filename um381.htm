3.8.1 Relative, Remote                                                Previous Next TOC

 

Relative and remote tilts and decenters of optical elements are entered with parameters specifying a direction and a magnitude, and the number of surfaces involved.  With these commands a single surface or a group of surfaces may be moved as a unit, and the program will automatically compute the reverse motions required to place the surfaces following the group at their normal position relative to the elements before the group.

 

A tilt may be specified about a point not at the vertex of the surface with the remote options RAD, RBT, and RGT.  This feature calculates both the decenter and tilt of the group and does not accept separate DECENTER data.

 

Decenters:

 

SN DECENT XDC YDC ZDC NSURF

 

If NSURF, the number of surfaces involved, is not specified, or is input as 1, the asymmetry applies to that surface only.  If the number exceeds the remaining number of surfaces in the lens, the reverse motions are not applied, resulting in a permanent optical axis tilt or decenter.

 

Relative tilts may also be edited in the SPS dialog.  The input commands, which are included in the RLE or CHG file, are as follows:

 

Tilts:

 

(Alpha Tilt)

(Beta Tilt)

(Gamma Tilt)

 

(Remote Alpha Tilt)

(Remote Beta Tilt)

(Remote Gamma Tilt)

 

 

A special set can be applied to mirrors with the input

 

SN { MAT / MBT ) ANGLE.

 

See below for details.

 

The directions involved and the sign conventions are shown below.  The magnitude of alpha and beta tilts should not exceed 90 degrees, and gamma tilts should be less than 180 degrees.  The location of the tilt axis, if not at the vertex of the surface, may be given by the AXIS input if it is on the Z-axis, or by the remote tilt options RAT, RBT, and RGT, where the coordinates of the rotation point are explicitly entered with the arguments XR YR and ZR.  Note that the latter option is converted by the program into a conventional decenter plus a centered tilt, and no other decenter should be entered when using this option.  The equivalent decenters are printed with the ASY output.

 

Do not use the tilt angle of an RAT … RGT tilt as a design variable.  Since the program does not retain the remote coordinates after it has converted to a centered tilt, a different tilt angle will not correspond to the resulting decenter.  You may, however, vary the tilt angle when the tilt axis has been specified by the AXIS input, since that dimension is honored continuously.

 

Keep in mind that the paraxial programs ignore tilts and decenters.  If you position surfaces with this options, the focal length, f/number, etc. may have little meaning.  See GLOBAL.

 

 

You can vary the tilt angles with the PANT input

 

VY SN AT NSURF

VY SN BT NSURF

VY SN GT NSURF

 

and you can target any of these quantities in the AANT file with input such as

 

M 5.0 1 A TILT 4  ,

 

which specifies a target value of 5 degrees to the tilt angle on surface 4.

 

The input

 

SNA PAS SNB NSURF

 

picks up the tilts and decenters from surface SNB (with a sign change if SNB is negative) and applies them to a group of NSURF surfaces beginning with surface SNA.  Note that if surface SNB uses a non-zero AXIS location, the sign of AXIS follows that for tilts.  Surface SNB must not be GLOBAL or LOCAL.

 

The input SN RELATIVE will change a GLOBAL or LOCAL surface to a RELATIVE surface, if possible.

 

 

Several rules must be observed when using relative tilts and decenters:

 

1.

Only one tilt direction may be specified per surface (i.e., do not do both an AT and BT on the same surface).  Decenters may be in all three directions at once.  Coincident dummy surfaces may be used to provide compound tilts, or you can use the GROUP option, which allows tilts in three axes at the same surface.

 

2.

A surface or group tilt or decenter will automatically produce a compensating reverse decenter or tilt on the surface following the group.  This surface is called the “undo” surface, and is identified in printed listings of the asymmetries in the lens (see 4.6).  There is no restriction regarding additional tilts and decenters on this surface, and any number of prior surfaces may utilize the same undo surface.  See Section 2.4.1.1 for more information.

 

3.

To apply both a tilt and a decenter to the same surface, enter the decenter first.  The number of surfaces in the group must agree in both entries.

 

4.

The location of the tilt axis is negative if it is to the left of the surface (i.e., in the -Z direction).

 

5.

If a non-zero AXIS is entered, a resulting decenter will be computed and added to whatever decenter was input previously.  If a tilt entry with a non-zero axis is entered to change a previous tilt which also had a non-zero axis, the previous decenter will be dropped before the new one is calculated.

 

6.

A relative tilt may be placed on the object surface.  In this case the data are assigned to surface number 0, the axis location is always assumed to be zero, and the group size entry is unused; the tilt affects the object surface only.  You cannot pick up the object tilt with PAS or make anything coincident with it using COINCIDENT.  There is no GLOBAL or LOCAL option for a tilted object.  The object tilt may be varied in the PANT file and appear as a target in the AANT file, both using surface number 0.

 

The tilt option is not available for OBB, INFINITE, OBG, or OBD objects.  (It only works on finite object specifications.)

 

The image may be tilted using any of the tilt options.

 

7.

Tilt and decenter data must be entered on separate lines.  Tilts and decenters may be removed from a surface (and the associated UNDOsurface, if any) with the command

 

SN NAS                (No ASymmetry).

 

To remove all tilts and decenters in the entire lens, enter

 

NAS

 

in a CHG file with no surface number.

 

Deformations (see 3.3.2.2.2) are not removed by these instructions.  The input

 

SYMM

 

removes all tilts, decenters, and deformations from the entire lens.  No surface number is used on this command, which is entered as part of a CHG file. Note that the paraxial raytrace, and therefore pickup and solve computation, ignores all asymmetries.

 

 

Tilting Mirrors

 

When mirrors are tilted, one usually wants the axis following the mirror to get a second tilt equal to that of the mirror.  This is done in one of two ways:

 

1.  A dummy surface is placed at the same location, on the axis, following the reflector surface, and is assigned the same tilt angle as the mirror.  One usually employs the PAS option, described above, which makes that surface pick up the angle of the mirror.  The WorkSheet button Fold here can automatically insert both surfaces to set up this geometry.  If the mirror is to be inserted at the location of an existing dummy surface, the WorkSheet can optionally use that surface as the mirror and then add just one more dummy for the axis tilt.

 

2.  If you do not want an extra dummy surface located at the mirror, the format SN { MAT / MBT } ANGLE will assign the requested tilt angle to the mirror and then implement a negative undo on the next surface -- which has the effect of tilting that surface by an equal angle, putting it on the tilted optical axis. The WorkSheet can automatically insert a fold mirror with this geometry too.  The surface following the mirror is then available for any other tilt or decenter.  Note that this method does not allow an additional decenter on the mirror, since the paraxial raytrace does not have valid separations in this case.  If you need to decenter the mirror before it is tilted, use the two-surface method, above.

 

To change a mirror tilt to an ordinary relative tilt, first change the following surface (with the negative undo) to GLOBAL or LOCAL.  This will change the status of both that surface and the surface with the mirror tilt, leaving the locations and angles unchanged.
