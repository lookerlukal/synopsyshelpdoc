3.9.3 Custom Prism Definition                                Previous Next TOC

 

While the prism library provides many of the most often-used prism types, you may have a need for a prism with unusual geometry not found there.  To enter such a prism in the RLE file, use the following format:

 

SN PRISM CUSTOM SIZE

 

NAME 'prism name'

FACES

NFACES

 

X Y Z ALPHA BETA GAMMA NPTS FFLAG [ REFL / AIR ]        ! face 1

XAP YAP

XAP YAP

...

 

X Y Z ALPHA BETA GAMMA NPTS FFLAG [ REFL / AIR ]        ! face 2

XAP YAP

XAP YAP

...

...

 

LINES

NLINES

 

LFLAG X1L Y1L Z1Y X2L Y2L Z2L

...

 

END

 

In this input, declare the name you want to give to your prism in quote marks, and give the number of faces in NFACES.  Then, for each face, you must enter the (X,Y,Z) coordinates and tilt angle in ALPHA, BETA, or GAMMA, of that face in the coordinate system of the previous face. The entry NPTS gives the number of vertices that will define the shape if the aperture on that face, which is usually four but can be more.  The face flag FFLAG is currently unused, but will declare a roof surface in a future update.  The last entry on this line declares the index following that face as a glass type (default), AIR, or makes the face a reflector.

 

The face data must be followed by NPTS lines giving the vertex locations of a UAP 4 aperture that will be assigned to that face.

 

When all faces have been so declared, you can define up to 11 extra lines that will be used by the solid drawing programs to draw the extra surfaces that make up the parts of the prism not encountered by the light.  The NLINES parameter specifies the number of lines you wish to define, and this must be followed by NLINES lines giving the start and end points of each line, in the coordinate system of the first face.  The flag entry LFLAG controls how the line is interpreted: if the flag is 1, the line is a new line; if it is 0, the line defines a plane with the previous line and the drawing programs will draw it in front of anything behind it.

 

The input file must end with END, which ends the prism definition and is separate from the END that ends the RLE file.

 

The only tricky part of this is figuring out where each face is in the coordinates of the previous one; the program puts each face higher than the first in LOCAL coordinates.  One way to do that is first to set up a dummy system where the faces are all in GLOBAL coordinates, which may be easier to calculate, and when the locations, angles, and UAP apertures are all satisfactory, to type ASY LOCAL in the Command Window.  That will list the local positions for you.

We recommend you enter the prism in normalized coordinates, where the face width is exactly one lens unit, and declare the actual size with the SIZE input.  Then, if you later want use a similar prism on another project, you can just copy the input lines and adjust the size as necessary without recalculating any of the other data.  And if the prism is used in a space following an odd number of reflections, enter the same data as before; the program will reverse those coordinates as necessary automatically.
