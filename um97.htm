9.7 Extended Image Analysis (EIA)                                Previous TOC

 

Two features of SYNOPSYS can analyze the image from an extended source object, instead of just a single point.  The Image Tools Menu (MIT) can apply a given image to a wide selection of target types, including the Air Force resolution target and even your own photograph.  It makes the assumption that the image patch to be analyzed is isoplanatic, however, which means that the aberrations are assumed to be everywhere the same over that patch.

 

But this is sometimes not the case.  The Field Blur Menu (MFB) will show the image of an extended object with geometric aberrations accounted for at each point in the field, but that feature does not have the resolution to examine fine structure and produces no numeric analysis.  The feature discussed in this section is the Extended Image Analysis.

 

The EIA feature is intended to tell you what fraction of the rays traced from a grid of object points filling a desired object patch eventually gets to the final surface.  Only geometric raytracing is involved, and only the number of rays is considered, not apodization or absorption; also, it only applies to FOCAL lenses.  A unique set of rays is traced from each object point.  If the goal is to see how much of the energy gets to a detector, located at the last surface, you must put a hard aperture on that surface, since that is what will cause rays to be deleted if they do not fall inside that aperture.  All hard apertures are considered, at all surfaces, including the EFILE edges, if any.

 

Input is:

 

EIA { ICOL / P / M } NRYS

IOBJ { RECT GBAR HBAR / ROUND HBAR }

CENTER XOS YOS

GRID GRIDNUMBER

GO

 

Here, the color selection { ICOL / P / M } has its usual meaning: ICOL is an explicit color number, P is the primary color, and M requests a multicolor analysis.

 

The object patch dimensions are given by the IOBJ input, where you specify either a rectangular or circular patch, with dimensions given in normalized object space coordinates.  The dimensions for a rectangular patch give the total extent of the patch in X and Y, while for the round patch the HBAR entry is the radius of the circle.  There must be a skew field defined in the lens, or EIA will abort.

 

The center of the patch is located at the position (XOS,YOS), in object coordinates.  The grid number specifies how many object samples to take within the patch; thus, a grid of 3 would trace 3x3 or 9 points if the patch is rectangular, and fewer if circular.  For good accuracy you should experiment with the grid number and find the best tradeoff between speed and accuracy.

 

Here is a sample of the output from EIA:

 

--- EIA M 900

--- CENTER 0 0

--- IOBJ ROUND 1

--- GRID 11

--- GO

NUMBER OF RAYS ATTEMPTED       83517

NUMBER OF RAYS THROUGH         23587

GEOMETRIC EFFICIENCY     0.2824

SYNOPSYS AI>

 

This analysis says that only 28% of the rays reached the detector in this case.
