17.3 SYNOPSYS Features Utilizing Polarization                   Next Previous TOC

 

Once the system has been modeled with appropriate coatings and polarizers, and the object polarization has been entered, some of the features of SYNOPSYS can evaluate their effects.  Among these are the image evaluation commands

 

PTRACE, PRAY

DMTF, DMODEL, PSPRD

GMTF, GMODEL, TFG, KNIFE

TN, PSURF

 

The diffraction-based options perform polarization raytracing for each ray in the pupil, and then do a vector diffraction calculation to get the point-spread function.  This takes from three to six times longer than the usual (POLARIZATION OFF) analysis.  The geometrical options do polarization raytracing to get the intensity of each ray in the spot diagram, but do not consider the vector directions.  In either case, if the object is unpolarized (which is one of the polarization modes), the image will be computed twice, and the two added incoherently.  If the model options are used, the programs FOR and DETECTOR (see section 9.1) will evaluate the image as affected by polarization, since the models contain that information.

 

If the above options are selected, the program will indicate the energy that arrives at the image.  A sample summary reads:

 

Rayset   Thru lens   Energy/Ray        Energy Thru   *Polarization

   177     157      0.100000E+0  0.887006E+00    0.798800E+00

 

Here, 200 rays were requested, but 177 was the largest vignetted square array that would fit within the pupil.  Of these, only 157 reached the image, because of vignetting.  There was no apodization, however, so the energy per ray was still unity on that account.  Since some rays were vignetted, only .887 of the original energy got through, neglecting polarization.  When coatings are included, only .799 of the original energy got through.  Note that those IMAGE features that do not consider polarization, such as SPT, will not calculate the last item.

The Al program can evaluate the transmission of a ray, in response to a question like

 

Find the trans for yen = .5 at full field.

 

(See chapter 15 for a discussion of the grammatical possibilities of Al.)  A summary of commands that evaluate lens throughput in various ways is found in the dialog MTR; some of these consider polarization and absorption, while others do not, as labeled on that dialog.

 

Note that none of the features that take polarization into account also consider absorption within the lens elements, except for TN, which finds the net T/number with all effects accounted for, and ILLUM.  Absorption can be calculated separately with GTRANS and RTRANS if necessary, and factored in later.

 

 

An individual polarization raytrace may be requested with the command

 

PRAY { ICOL / P } HBAR XEN YEN { SURF / ESURF / E / 0 } GBAR [ ISTOP ]

 

This command gives a table (with SURF) of the two E-vectors as they are resolved at each surface.  The amplitude of the two may be followed through the system in this way.  (Note that the intensity of the ray is given by the sum of the squares of the amplitudes of these two vectors, times the apodization, if any.)  The ESURF option also gives a table, with ellipticity at each surface printed instead of the phase.  The E option also gives the ellipticity, but only at the last surface.

 

The optional ISTOP gives the surface at which the raytrace stops.  Normally one traces through the entire lens, and this input is not needed.  But if you want to control the individual S and P polarization components following a particular surface within the lens, you can give that surface number here and then monitor or target the output data found in the AI buffer after the PRAY command is finished.  The PRAY command can also be found in the dialog MTR.

 

To visualize the polarization state of a given ray as it propagates through a system, use the command

 

       PSURF { ICOL / P } HBAR XEN YEN GBAR

 

A sample plot is shown below.  In this system, linear polarization is turned into circular at surface 4, which is a waveplate with retardance of 1.25 waves.  Other surfaces are uncoated, so the intensity is reduced as the light goes through.  Note that the program shows a projection of the polarization state on the local X-Y plane.  It does not show the Z-component, if any.  If the ray arrives at a surface at a steep angle, this causes the X-Y projection to appear foreshortened.
