3.1.1.1 Noncircular Pupil                                        Previous Next TOC

 

Unless otherwise specified, SYNOPSYS assumes that the X and Y dimensions of the entrance pupil are equal: the fractional ray coordinates XEN and YEN refer to points on a circle (which may, however be inclined -- see 2.6.3).  If a nonzero value is entered for the parameter XMP1 with the object specification, a noncircular pupil may be defined.  In this case XMP1 need not equal YMP1, and all of the ray-based analysis features of SYNOPSYS will take the entered value of XMP1 as the "full aperture" dimension in the X direction.  If the ratio of X to Y in the pupil is far from unity, however, some caution is needed in using some of the diffraction-based image analysis programs: the image sample area is based on the Y-dimension F/number, and may be inappropriate in the X-direction.  A user-entered value of XMP1 does not take effect until the pupil has been declared either elliptical (EPUPIL) or rectangular (RPUPIL).

 

The wide-angle pupil options WAP 2 and 3 (see 2.6.3) adjust the values of YMP1 and XMP1 at each field angle, giving in most cases a noncircular entering beam.  The VFIELD option gives yet another way to define a pupil that is intentionally noncircular.

 

The OBG object can specify a noncircular gaussian beam directly.

 

To change a noncircular pupil created with an entered XMP1 to a circular pupil (causing XMP1 to be ignored), use the following sequence:

 

CHG

CPUPIL

END

 

All of this is explained in the Pupil Wizard dialog.
