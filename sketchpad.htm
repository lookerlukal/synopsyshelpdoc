SketchPAD

 

The SketchPAD is the most interactive feature of SYNOPSYS.  Here you can see the lens along with a display of image quality in several different formats.  This display can be automatically updated during the course of an optimization run, allowing you to monitor progress and halt the run by clicking if it becomes apparent that changes are in order.  See Optimization.  A related feature, the WorkSheet, permits changing the lens by entering new data into a dialog box or by moving sliders in an analog fashion with immediate graphical feedback.  It also provides tools to insert, split, and remove surfaces, elements, prisms, and fold mirrors. Additional features are the Vendor Catalog dialog and the Edge-definition Wizard.

 

The SketchPAD display is divided into an upper and a lower section.  Any of seven display types can be shown in either half, although some features require a drawing in the top part.  Once the display format has been requested, SYNOPSYS remembers this format and automatically displays it whenever the SketchPAD feature is activated.  If the lens is later changed, or if another lens is called from the library, the SketchPAD display will automatically update with the new lens the next time the screen is redrawn, or when you click the button .  You can define up to five separate display formats, and switch between them with a click on the buttons .

 

It is also possible to link the PAD display to a command string, which will be executed whenever the display is updated, letting you view any of the graphics output formats in real time as your lens is edited.  The command string must be no more than 89 characters long and contained in a single line.  (To run a multiline command, compress all the data onto a single line, with a slash (/) between the separate lines, as in FCO 100 / MTF P 1 0 0 0 P.)  It is wise to avoid commands that execute slowly, since the WorkSheet sliders will then exhibit an unwelcome lag in response.  This feature is activated by enabling the checkbox at the bottom of the PAD TOP or PAD BOTTOM dialog and entering the desired command string in the edit box there.  For more information, click here.

 

The SketchPAD may be activated by clicking on or with the SYNOPSYS command PAD.

 

You can move and resize the PAD display, and when you next open it, the position and size are remembered.  In the event that you want to revert to the default location and size, just close it and then open it with the command

 

PAD ZERO

 

or close it with the "X" button, shown below.

 

 

SketchPAD Raytracing Options

 

The PAD display can show a variety of ray patterns, selected with options on the PAD Top or PAD Bottom dialog.  These include the default rayset, a single ray, or a custom rayset.  The default rayset shows three rays each from the on-axis field point and full field.  If switch 10 is turned on, the lower field point will also be shown.  The single-ray option opens a small dialog where you can select the relative field point and relative aperture point, both in the meridional plane.  When the dialog is closed, the selected ray is shown.  The custom rayset shows a desired number of rays in a TFAN at the requested relative field point, which can be a skew point.

 

A special feature will show either the upper or lower rim ray at full field.  It is useful when you want to identify which ray is encountering feathered edges.  Just press and hold down the F7 key to see the lower rim ray, and the F8 key to see the upper rim ray.  When you release the key, the normal drawing is restored.  If the lens has a skew field defined, the object point taken for this ray is in the meridional plane at a distance from the axis equal to the diagonal distance to the corner of the field.

 

If you hold down the <SHIFT> key while you press the F7 or F8 keys, the rays will be traced from the lower field point instead of the upper; and if you hold down the <CTRL> key, the field is taken in the meridional plane, ignoring the skew field, if any.  This can be useful when the field is rectangular but you only want to see rays at the top and bottom of the field, not the corners.

 

Note that, depending on the sign of the object height and the pupil definition in effect, the "upper" rim ray may in fact be at the bottom of the pupil.  This feature makes it easy to see which ray is where.

You can also pan the display by clicking and dragging with the right mouse button, regardless of which tool is selected, and you can instantly magnify or minify at any point by clicking with the right button while holding down the Shift key or both the Ctrl and Shift keys.  You can also zoom in and out with the mouse wheel.

 

All of the PAD display formats have an optional scale factor, and if you leave it out the program will adjust the scale automatically.  But sometimes a fixed scale is called for, such as when you want to see the effects of a change and want the same scale each time.  You can of course click the PAD Top or Bottom buttons and enter the scale yourself—but there is a shortcut too: click in the PAD window, and then hold down the <Ctrl> key while you click the PAD Top or Bottom button.  The current display format stays the same, but the displayed scale becomes fixed at its current value.

 

The default printing option, which you get by clicking the print button or typing <Ctrl>P, sends a vector print to the printer, which has smooth lines but cannot show the fill colors, which require a raster capability that printer drivers do not have.  Another option, activated by <Ctrl>+<Shift>+ print button, encodes the display as a bitmap and sends that to the printer instead.  Then the picture shows all the fill colors, but lines are drawn as individual dots, which are not as smooth, since the screen resolution is lower than the printer resolution.  The quality can be maximized by enlarging the PAD display as much as possible before running this option.

 

Many of the features of the SketchPad and its family of tools require that the top display show a lens drawing (D, P, or PP).  PAD will show circled surface numbers on the first side of real lenses (those with an index change) if switch 52 is OFF, and will number all surfaces if switch 38 is ON as well.

 

As you move the mouse about on the PAD display, you will see the values of the (X,Y) position of the cursor displayed on a separate small dialog.  The values reported are calculated according to the scale factor of the drawing in the top display and the current lens units, whenever that drawing is of the form D or PP.  You can reset the (0,0) point by a <ctrl><right-click> at the desired location, as you would in a Graphics Window.  If the top drawing is not a profile drawing (D) or perspective view (PP), the scale factor will be either a default value or that resulting from the most recent valid adjustment.  This feature is intended to help you judge the size and locations of lens elements, as is the scale-bar option.

 

The “D” form of PAD drawing has an option that draws the lens elements filled with color.  You can turn this feature off (and get only a line drawing) by turning on switch 46 or by clicking the button , should you want to.  This kind of fill only works on the monitor, and not on any printers we have tested.  The D drawing is not recommended for lenses that have a combination of RAO and EFILE edges defined.  To show these complicated edge shapes, the PP (perspective view) should be used instead.

 

SketchPAD Operations – Command mode input:

 

PAD

activates the SketchPAD display.  The prompt changes to PAD>.

D

prompts for attributes of a Y-Z-profile drawing

PP

prompts for attributes of a perspective drawing

P

prompts for attributes of a paraxial drawing

F

prompts for attributes of a transverse rayfan display

FP

same as F, but sagittal fan is over the full pupil

F3

same as F, but over three fields

FP3

same as FP and F3 together

O

prompts for attributes of an OPD rayfan display

OP

same as O, but sagittal fan is over the full pupil

O3

same as O, but over three fields

OP3

same as OP and O3 together

S

prompts for attributes of a spot diagram display

A

prompts for attributes of a display of astigmatic field curves

X

erases the requested portion; used to show only one display instead of two

U

updates the display

ZOOM

zooms a ZFILE zoom lens to the zoom position in the 2nd word

END

ends the PAD interactive sequence (but does not close the display)

RESTORE

Adjusts the position and size of the PAD display so that it appears on the screen.  Some activities, especially resizing the SYNOPSYS frame, can result in the PAD window going out of the frame entirely.  This command will restore its position so you can see it.

 

The default display of rayfans and OPD fans shows 10 rays per sfan.  For higher resolution, turn on switch 69; then the fans show 20 rays.

 

The size and location of the PAD display are sticky: if you close and then reopen PAD it should come back just where you left it – but this may not be exact for some monitor settings and resolutions.  If the position does not come back exactly where you want, go to Options | Set Device Offset, Margins, Font, and enter a correction value in the X and Y boxes in the PAD section.  The units are about 0.01 inches.  You may enter either positive (right and down) or negative numbers.
