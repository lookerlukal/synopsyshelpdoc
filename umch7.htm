Chapter 7

 

7.0 LENS ANALYSIS ROUTINES – PLOTTED

 

Several kinds of graphical lens analysis routines are available in SYNOPSYS. These include image assessment routines, pictorial descriptions of the optics, and surface mapping.  See also section 5.1.1 regarding deformed surface plots.

 

 

7.1 Lens drawing (DWG)                                Next TOC

 

The entry

 

[ USE BTOL ]

DWG [ TSCF [ JSSS JSPS [ HBAR NB NB NB ]  [ OFFSET XOS YOS ] ]

 

Will produce a Y-Z profile of the optics, with the path of specified meridional rays shown.  Apertures appearing on the drawing will be those entered as part of the lens input, (see 3.3.1.1) if any, otherwise those computed with CAP; DWG will also honor edges defined with the EFILE, if switch 39 is ON (see 7.8).  For more control over the viewing angle or ray selection, the programs PER and RPER are also available.  The DWG feature may be run interactively from the MPL dialog.  Optional input is:

 

[ USE BTOL ]

This optional input will open the data file created by BTOL via the directive PREPARE ELD, which contains the details of the tolerance budget.  It is intended for creating an assembly drawing with tilt, decenter, and airspace tolerances.  See Section 7.13 for examples.

 

TSCF

Times SCale Factor.  The available space in which the lens is drawn is 9.5 in. long by 6.6 in. wide.  The default scale factor, produced by a zero or blank entry, gives an 8 in. long drawing.  If a negative TSCF is entered, no rays will be drawn on the picture.

 

JSSS JSPS

First and last surfaces on drawing.  If surface numbers are zero or omitted, all surfaces will be drawn unless BACK exceeds TOTL, in which case the image surface is omitted.  If switch 63 is on, the image will be drawn regardless of the size of BACK.  If JSSS is zero but JSPS is not, the object will be drawn as well, unless it is over 10**6 units away.

 

HBAR NB …

specifies up to three object points from which rays will be traced and included on the drawing.  The upper and lower rim rays, and the chief ray (if unvignetted) are drawn, all in the primary color.  If the HBAR entrie(s) are omitted, rays will be drawn from a single object at HBAR = 1.0, except with OBG systems, for which the field point is HBAR = 0.

 

OFFSET …

This input causes the lens drawing to be positioned on the page at a location offset from the default position.  Here, XOS shifts the picture to the right, and YOS moves it up.  This can be useful with folded systems that extend off the page.  You may enter either or both of the HBAR and OFFSET data at the same time.

 

 

Since the Y-Z profile of each surface is drawn in the plane of the paper, asymmetries resulting from beta or gamma tilts will not be shown.  To obtain a true perspective drawing which shows all tilts and decenters, the programs PER and RPER are recommended.  Also, lenses that have a combination of RAO and EFILE edges, which can produce a rather complicated geometry, may not be shown correctly by DWG.  The perspective programs are recommended in such cases.

 

Default edges are drawn on all lenses connecting the clear apertures (see section 6.2.1), except for the following cases: (1) those separated by AIR, (2) the surface immediately following a reflection if the medium is not AIR, and (3) surfaces having a tilt or decenter with respect to the other side of the lens.  The edges are drawn at angles of 0,45, or 90 degrees with respect to the local axis.  If switch 20 is turned on, DWG will omit dummy surfaces (no index difference) from the drawing unless they happen to be the first or last surface shown or have a user-entered aperture assigned.  For more explicit control of lens edges, see the Edge Wizard.
