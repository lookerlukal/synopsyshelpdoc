8.4 Footprint Options                                         Next Previous TOC

 

 

The ray pattern may be examined at locations other than the final image with the input shown below.  The dialog MFP provides an interactive way to request footprint plots.  In Command Mode or in a MACro the format is

 

 

The program traces a beam of approximately NRYS rays from the object at HBAR GBAR and plots its intersection with surface ISN at a scale of TSCF where 0.5 is half scale, etc.  The center of the plot corresponds to ( XOS YOS ) on the surface unless CR is entered, in which case it is the chief ray position for that field angle.

 

On the plot, the positive Y axis is oriented up, and the positive X axis is toward the right.

 

If NRYS is input as 1, a single ray is traced with fractional entrance pupil coordinates XEN YEN.  For NRYS greater than 1, the ray grid that is traced is defined by the pupil option in effect.  The default option causes the program to trace a grid of rays filling a circular pupil.  This will be a square grid if switch 90 is off, and a hexapolar array if that switch is on. You may request alternate patterns by the optional PUP input; PUP 2 specifies different spacing in the X and Y directions, usually 1 by N for a slit pupil.  In this case the NRYS entry on the TRACE command is overridden (but NRYS must not equal 1).

 

The first option, PUP 1, creates a square grid of rays and sets the grid number so as to approximate the total number of rays requested in the TRACE data.  If switch 90 is ON, the program traces a hexapolar grid instead of a square grid.

 

The first two options utilize the pupil diameter input as YMP1 in the RLE file, or adjusted by any WAP or FILLSTOP options in effect.  For studying the effects of physical apertures it is sometimes necessary to specify an oversize value for YMP1, to ensure that the stops are what limit the beam, and not the input pupil diameter.

 

The third option, PUP 3, allows you to determine the shape of the boundary of the vignetted pupil as seen from the entered object point.  The program traces a pencil of NRYS rim rays, with a starting pupil radius of YMP1, and moves each ray toward or away from the chief ray until it just clears all of the apertures or feathered edges.  The final ray pattern is then plotted.  When using PUP 3, at least one clear aperture must be specified or the pupil diameter would become infinite, and the lens must be in VIG mode.  The chief ray must not be vignetted, or PUP 3 will abort.  The entry NRYS (on the TRACE command) is overridden, but must not equal one.

 

PUP 4 is similar to PUP 3 except that a search is performed only if the starting ray is vignetted (i.e. the beam will not grow larger than YMP1, but can become smaller if vignetting is encountered). PUP 5 is the same as PUP 3, except that the center ray is taken at the ICR point.  This option is useful for systems where the default center ray will not trace.  The PUP specification applies to the TRACE footprint output, and may be used to specify a pattern of rays to the drawing programs PER, RPER, and SOLID as well.

 

The LINE option for PUP 3, 4, and 5 will cause each ray intercept point on the surface to be connected with a straight line to the previous point.  This is a good way to show the portion of a surface that is utilized by the beam from a given object point.

 

The footprints from additional object points may be plotted on the same drawing by merely entering additional TRACE data.  In this way the total footprint required to cover a given field of view, or the aperture required for a scanning system, may be determined.  A GET command may precede the TRACE command, to retrieve a lens with a rotated prism, for example, in the latter case.  It is sometimes useful to request a different color for each of several field points (or requested wavelengths) with the optional [ RED/BLUE/GREEN…] input.  If any of these are entered, the program will ignore any conflicting COLOR directive that was entered previously.

 

If switch 16 is turned on before the TRACE input, a list will be produced of all rays that are deleted because of ray failures and vignetting.  Switch 21 will cause such rays to be included on the plot as a small numeral, giving the surface number where they stopped.  This is a very handy way to find out where your beam is being vignetted.

 

 

Transmission calculations

 

A TRACE command may be used without a preceding PLOT request, to provide just a transmission analysis of the system, and no plot. TRACE will consider the effects of vignetting and apodization, while PTRACE will also consider the efficiency of coatings, if polarization mode is on (see Chapter 17).  Neither of these will consider absorption within the glass, for which the GTRANS, RTRANS, ILLUM and TN commands are used instead. FTRANS will estimate the Fresnel losses that occur at a cemented interface.  The AI program can also evaluate the transmission along a ray, with all effects accounted for.  The dialog MTR shows the eight commands that evaluate the transmission of your lens, and indicates which properties are taken into account, making it simple to select the most appropriate one.

 

The entry SGTH can be used in the PANT file to control the total glass thickness and thereby control absorption losses.

 

With TRACE and PTRACE, a listing is made of the number of rays traced, the number that passed through to the image, the fractional transmission of the rays that got through, the transmission relative to the incident ray set, and the reduction due to reflection losses (with PTRACE).  The raytrace statistics, and transmission, are put into the Al buffer (see 15.1.2.3).

 

If switch 27 is ON (default), the spots are plotted as a single dot.  If it is OFF, the dots are drawn with symbols denoting the ray color.  These symbols are the same as those listed on the SPT graphics, and it is simple to make a sample of the latter if you need to see the symbol key.  The size of the symbols can be set with the command

 

 SSSIZE NB

 

before the plot is created.
