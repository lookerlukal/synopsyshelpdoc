7.11 Graphical System Summary (GSS)        Next Previous TOC

 

This manual has described many different ways to analyze your system, some based on geometric optics and others on diffraction theory.  With those features you generally get one kind of analysis on a plotted page.

 

Another command will create a drawing with a collection of several kinds of graphs, including the diffraction point-spread function, spot diagram, pupil wavefront map, MTF, field curves, distortion curves, and lens drawing – all on the same page.  This command is intended primarily to help you document your final design by displaying lots of data all at once.  It can be accessed with the dialog MGS.  Because each of the graphical elements is quite small, you should generate the full-size version of the analysis if you wish to make measurements of the data.  Input is:

 

[ FCO CUTOFF ]

[ PSVISUAL MAGN GAIN ]

[ FRINGES [ GRID ] ]

 

GSS [ [ ICOL / P / M ] NRYS [ Q ] ]

[ZERNIKE]

[ HBAR HBAR  [ HBAR  [ HBAR ] ] ]

[ GBAR GBAR ]

[ DWG TSCF JSSS JSPS ]

[ PER EL AZ TSCF JSSS JSPS [ V ] ]

[ PSCF PSCF  [ R / L / 0 [ C ] ] ]

[ SSCF SSCF ]

[ PUSCF PUSCF [ WX WY ] ]

[ FSCF FSCF ]

[ DSCF DSCF ]

END

 

All input except the GSS and END lines is optional.

 

FCO specifies the cutoff frequency of the MTF, which is calculated by the routines described in Section 8.6.3.

 

PSVISUAL displays the diffraction point-spread function with its visual appearance rather than by oblique projection.  The data are described in Section 8.6.2.

 

FRINGES shows the pupil map as fringes instead of in oblique perspective.  GSS does not honor the CONTOUR directive, and will turn the latter into FRINGES if it is requested.

 

The analysis may be requested in a single color (ICOL), in the primary color (P), or in all colors at once (M).  The default is M. NRYS gives the number of rays to be analyzed.  This number is fit to the (usually circular) aperture, so the exact number may vary, and the number in each color of the spot diagram is reduced according to the spectral weight in that color.  The default is 600 rays.  The optional Q runs in quiet mode, where output to the Command Window is suppressed.

 

Field points to be analyzed are given with the HBAR input, where you may request up to three fractional field points.

 

If the lens has a nonzero skew field, you may specify the fractional field point in the X-direction with the GBAR input.

 

The optional ZERNIKE directive causes the program to use the Zernike wavefront expansion for the MTF portion of the analysis.  This option is intended for cases when the cutoff frequency FCO is too low for the normal MTF analysis.  (You will get an error message if that is the case, raising the FCO to the limit, and advising you to use the Zernike option.)

 

The default drawing is the lens profile (DWG), which will be drawn at the scale given by TSCF and will include surfaces JSSS through JSPS if those data are entered.  In the event that the requested scale would cause the drawing to exceed its somewhat limited area of the picture, or if no scale is entered, the program will calculate a scale that is within this area.  If no surfaces are entered, the program will draw the entire lens.  Since this is a Y-Z profile drawing, it will not display tilts or decenters that are out of the plane of the paper, and caution is recommended if that is the case.  The program will fit the drawing horizontally into its assigned area, but it there are asymmetries that go above or below this area, you are advised to use the PER option instead.

 

If you enter the PER data, the drawing will be a perspective view instead of the Y-Z profile of DWG.  You may select the view angles with EL (the elevation) and AZ (the azimuth).  The drawing will be at a scale of TSCF if entered, or a default scale if not, and will span surfaces JSSS through JSPS if those data are entered.  Note that this feature takes only a single line of perspective data, unlike the PER command itself; the rays to be traced are always the chief ray and the upper and lower rim rays at whatever HBARs have been requested.  The optional V in word 7 will cause the input light to come down from the top, rather than from the left.  The perspective feature draws rims around the edges of the elements; if both view angles are zero and you don’t want the extra rim lines on the drawing, turn on switch 18 before running GSS.

 

PSCF gives the height, in inches, of the peak of the point-spread function.  This value should be no greater than 1.0, since the area for each diagram is small.  If you enter no data for PSCF, the program will adjust the height of each pattern to illustrate relative intensities.  The pictures may be rotated left (L) or right (R), and may be displayed with colors that map the local color of the image at each point (C).  If the optional PSVISUAL has been entered, you should enter a height unless the image is very well corrected at all field points.  (The program normalizes the heights before it converts to the visual appearance, and the intensities are then normalized and the spot will be easily seen – while a poor image with no height given may well be too dim to see clearly.)

 

SSCF gives a reference dimension, in lens units, for the spot diagrams.  If the lens is AFOCAL, this dimension is in radians.  The program draws, for reference, the size of the Airy diffraction disk, calculated for the major color and the actual F/number in X and Y at the first requested image point.  This may be defeated by turning on switch 80.  Switch 85 affects the colors displayed. See Section 8.5.1.1 for a discussion of this special color option.

 

PUSCF give a reference dimension, in waves, for the PUPIL plots.  If FRINGES has been requested, you may also enter a wedge in X and Y, in fringes, with WX and WY.

 

FSCF gives a reference dimension, in lens units, for the field curvature plot.  If the lens is AFOCAL, this dimension is in diopters.

 

DSCF gives a reference dimension for the distortion plot.  This is a fraction, relative to the GIHT.

 

Since many of the items displayed by GSS involve diffraction effects, this feature should only be used with systems whose image is corrected well enough to make this analysis appropriate.  Here is an example of the GSS output, where the input was

 

GET 4

GSS

PSCF 0 L

END

 

 

Here we entered the number 0 for the PSPRD height, so the program would scale the diagrams according to intensity, and requested a left orientation on the drawings.

 

The next example shows the visual appearance of the diffraction patterns for an eyepiece, with the pupil displayed as fringes with a 6-wave wedge in X added for clarity.  Here is the input:

 

GET 9

SSS .002                ; small spots requested, because the pictures are small

PSVIS 3 2

FRINGES

GSS M 600 Q                ; quiet mode requested

DWG 0 2 18

PUSCF .1 6                ; fringes for the pupil, requested above; the scale is ignored

PSCF 1 L                ; and visual display for the image, normalized intensity

END
