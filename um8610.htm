8.6.10 Through-focus PSPRD (TFP)                        Next PreviousTOC

 

A composite picture of the diffraction point-spread for up to 5 focus positions at up to 3 fields of view may be obtained with the input shown below.  It may also be obtained interactively from the dialog MTP.

 

[ PSVISUAL MAG GAIN ]

 

 

TFPSPRD NRYS [ HEIGHT  [ R / L ] ]

 

 

[ DF { 1 / 3 / 5 } DELTAF ]

(defocus data)

 

[ HBAR HBAR1 HBAR2 HBAR3 ]

(fractional field height)

 

[ GBAR GBAR ]

(fractional X object)

 

[ ICOL { ICOL / M / P } [ ICOL … ]

(up to 5 color numbers)

 

PLOT

 

 

where NRYS is the number of rays to be traced in the pupil, and HEIGHT is the height of the highest point in the central slice through the image.  The program will generate a WPSPRD for each HBAR that is entered (default: 0, .75, 1.0) in each color ICOL (default: the primary color only), and plot them on a single picture, oriented as seen from the right or left if the optional R or L is entered.  “M” requests a multicolor plot in all defined colors, and “P” is just the primary color.  If the number of defocus positions entered on the DF line (default: 1) is 3 or 5, the program will also calculate the image as it would appear at distances of plus and minus DELTAF from the image surface.  These images are then plotted alongside the first.  The procedure is repeated a second time if five defocus positions have been requested.

 

If just the word TFP is typed, the program will prompt for the other data items.  If HEIGHT is omitted or zero, the program will calculate a scale factor such that a perfect image has a height of about one inch on the plot, and poorer images show a reduced height that indicates the reduced peak intensity at that image.

 

If the lens is AFOCAL or ACCOMODATED, the program will interpret the DELTAF entry as a defocus in diopters rather than lens units.

 

The individual pictures are rather small, since we have to fit up to fifteen of them on a single page, and the lines used to display the image are quite close together as a result.  We recommend that you use the smallest pen width for hardcopy plots, so the lines do not run together.  (See Options | Graphics | Set Pen Width, in the top menubar.)

 

If the TFP command follows a PSVISUAL command, the pictures will show the visual appearance of each image instead of an oblique-perspective drawing.  You may control the area displayed for each image in this case with the MAG argument, where a value of 2 would zoom into the image by 2X, for example, and by the GAIN, which amplifies the image, bringing out the fainter fringes while possibly saturating the peaks.

 

If you enter a value for HT other than zero on the TFPSPRD command, the program normalizes each image so that the highest value of any of the RGB color coordinates equals 255*GAIN.  In this case each image is normalized separately.  If you enter no value, or zero, for HT, then the program calculates the normalization required for the first image it creates and uses this scaling for all of the other images.  Since the first image is usually the in-focus, on-axis, (and best) image, this option will show the reduction in intensity that occurs at other field points and focus positions.  Often, however, some of the other images will be too low to see clearly, due to aberrations and defocus, and in this case you will need to enter a value for HT, such as 1.0, to see them.

 

Two examples of TFP output are shown below.  In the first, we did not specify an image height, and you can see that the defocused images are of lower peak intensity.  The second example is a PSVISUAL form of TFP, where we specified a MAG of 2 and a GAIN of 20 – but no height – to show the faint image at the upper right while saturating the better images.
