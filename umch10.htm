10.0 AUTOMATIC DESIGN PROGRAM

 

Automatic optimization is performed on SYNOPSYS using the PSD algorithm(1,2), which is superior to the familiar damped-least-squares (DLS) method.  This method has proved to be one of the most versatile and powerful methods for optimizing a nonlinear system ever discovered.

 

The program is versatile because it allows you to specify many kinds of quantities as "aberrations" to be controlled during the design of a lens, including construction parameters, paraxial quantities, object or pupil coordinates, ray intercepts, optical path differences, and even the diffraction MTF.  Using convenient mnemonic names, you can combine these quantities in arithmetic sums, differences, quotients, etc., specifying explicit targets or one-sided boundaries for the result, or give an equation describing the quantity you wish to control.  You can select from an automatically generated set of rays with flexible weightings, or construct your own merit function with rays you select yourself.

 

By assigning proper weights to each of the components of the merit function, it is possible to evaluate the tradeoff between image quality and mechanical requirements such as overall length or lens diameters.  By manipulating the weights, you can push the design toward any desired balance among residual defects.  Using such a tool it is possible to design an optimum system, and not merely an optimum image.

 

The program is powerful because the algorithm consistently converges faster than the DLS method or its other variants.  Automatic features control the damping, second derivative approximation, derivative increment, variable metric, and boundary conditions.  It is seldom necessary for you to input anything other than the starting lens, variable list, and merit function definition.

 

Many of the most useful features of the optimization program can be run from interactive dialogs that begin with MOM.  This feature provides a convenient way for a beginner to exercise many of the optimization possibilities, and by studying the commands and data that MOM produces, one can quickly learn the formats required to run with the faster command mode, which will be preferred by experienced users.

 

The input for an optimization run is organized into several sections, which must be entered in sequence.  The order is not important, but you must not break the sequence with a command not related to optimization.  If you do so, the program will reinitialize all of the optimization settings when you reenter with the other sections.

The only commands that may interrupt this sequence are the QUIET, LOUD, ACON, and ON and OFF commands.

 

Note that optimization data are not part of the lens definition itself, as is the case in some other software packages.  The input described above is usually entered in a MACro editor and saved with a desired file name.  It is very easy to create a number of such files, one optimizing with transverse aberrations, another with OPDs, and so on – and then to switch from one to another just by running that MACro.  We see no reason to associate one particular set of data with the lens file in a way that makes it clumsy to switch to another set.  But you can also associate any desired MACro with the final lens, for archival purposes, with the toolbar button , which appears on the main toolbar as well as the MACro editor toolbar.  Clicking on the former saves the lens with a name equal to the current LOG number and a file type of .RLE, while the latter saves the MACro with the same name and a type .MAC.

 

To illustrate this format, here is an example of a very simple optimization MACro:

 

PANT        ; DEFINE THE VARIABLE PARAMETERS

VLIST RAD 1 2 3 4 6

VY 1 TH 20 3

VY 2 TH

VY 3 TH 20 3

VY 5 TH

VLIST GLM 1 3

END

 

AANT        ; DEFINE THE ABERRATIONS TO BE CORRECTED

AEC                ; AUTOMATIC EDGE CONTROL

ACC                ; AUTOMATIC CENTER THICKNESS CONTROL

GSR .5 5 3 2 0

GNR .5 1 3 2 .5

GNR .5 1 3 2 .7

GNR .5 1 3 2 1.

GSR .5 5 2 1 0

GSR .5 5 2 3 0

GNR .5 1 2 1 1

GNR .5 1 2 3 1

END

 

SNAP                ; REQUEST SNAPSHOTS AS OPTIMIZATION RUNS

SYNO 25        ; REQUEST OPTIMIZATION FOR 25 PASSES

 

The data in the above lines are described in the later sections of this manual.
