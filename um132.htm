13.2 the MACro Editor                                                        Previous Next TOC

 

Much of your input to SYNOPSYS will consist of a set of commands that you will want to run more than once.  For this purpose it is most convenient to create a MACro.  You can open as many MACro editors as you want, each containing a different set of commands, and then run them at any time just by clicking on the RUN MACro button or the RUN Alternate MACro button .  You can assign names to any or all of them, and the contents will be saved on disk whenever you click on the SAVE MACro button – or when you run them.  A MACro may contain any number of lines of data.

 

When the editor opens a file, or when you make certain changes to the file in the editor, it scans the contents to identify lines that are commented out or are part of a SKIP section; those lines are then displayed in a different color.  This makes it easy to distinguish lines that will be run from those that will be skipped.  There is some overhead involved, however, and if your file is quite long, you may notice a delay after each line is typed.  If that is a problem, just turn off the automatic scanning by deselecting the button , which is selected by default.  You can also turn it off with switch 41.  (If you turn it off, you can still scan for comments; just <right-click> whenever you want to update the comment scan.)  Note that this feature is only intended to help you read your MACros; the command parser works whether the file is scanned or not.

 

The scan also tries to identify lines that start with a valid SYNOPSYS command, and those that are located are displayed in red type with a grey background.  This is a convenient way to distinguish lines that are commands from lines of data or AI sentences.  The process is not perfect, however, since it has to determine if a given input line is a command or is part of an input file -- in which case it is rendered as usual.  To do so, it looks for the end line of that file, usually END, PLOT, or GO. When that line is found, it can again identify commands that follow.  But if you structure an input line with a form such as  "chg/2 dec 31.75 0 0 1/end", for example, the "end" will not be found on a lower line.  So this kind of formatting is not recommended.

 

Lines in the MACro that begin with a comment character "!" or ";" will be skipped, as noted above.  If you want to comment out a group of lines, simply select those lines and then click the Comment button on the editor toolbar.  The selected lines will all get a "!" character in column one.  To uncomment a group of lines, select the group and click the Remove Comment button .  Any leading "!" or ";" characters will be deleted from those lines.  (For this to work, the selection should start at the beginning of a line.)

 

The command

 

 EE

 

will open the most recently used editor window, or a new one if there is none.

 

You can load a new MACro editor with a named MACro with the command

 

LM filename                (Load Macro …)

 

 

Where filename is the name of a MACro that was previously saved.  You can run a saved MACro (without loading it into an editor first) with the command

 

EM filename                (Execute Macro … )

 

and you can load a MACro into the computer’s memory with the command

 

LTM filename                (Load This Macro …)

 

This command does not load the file into an editor, and does not open the editor.  It is intended for when you wish to run the MACro at a later time with the GDS command or with a DO MACRO loop.  See Section 15.1.2.1 for a description of this powerful feature.

 

 

Once you have run a MACro, you can re-run it, even if the editor has been closed, with

 

GDS                (Get Data Set)

 

 

A MACro can call another MACro in a chain fashion, if you put the EM … command at the end of the file, specifying the new filename, and it can also run another MACro and then return to where it left off if the command is EAM … (Execute Alternate Macro).  To re-run the alternate MACro, the command is

 

GADS                (Get Alternate Data Set).

 

If you want to erase the memory copy of the last MACro (which is used by the GDS command), use the ZDS (Zero Data Set) command.

 

You can also use the Windows menus and toolbar features to do many of these same things: When a MACro window is active, the File menu gives options to open a saved MACro file (in the same window) or to save the current contents.  In addition, the MACro menu item contains similar tools, as do some of the top toolbar buttons

 

 

It is sometimes useful to prepare a MACro entirely with commands.  The sequence is

 

DATA

(any lines you want in your MACro)

…

EDS                (End Data Set)

 

The EDS command can also be put into a MACro created with an editor.  The result is that execution of the MACro is terminated when the EDS line is reached.  This is a convenient way to disable a MACro partway through, when you do not want the whole thing to execute.

 

A MACro prepared in this way exists only in memory, not in a MACro editor.  It can be executed with the command

 

GDS                (Get Data Set).

 

 

It is sometimes convenient to rerun the most recent commands that were typed into the Command Window.  The command

 

 LLB, ELB                (Load Line Buffer, Edit Line Buffer)

 

will load the last 20 commands into an editor.  You can also click on the menu MACros | Show the last 20 commands.

 

The command

 

 PMA filename        (Print MACro)

 

Will print the MACro on the Command Window, but does not execute it.

 

 

The command LMM will load the MACro editor with a copy of the commands that were most recently produced by a SYNOPSYS dialog.  Then you can change, run, or save the file without going to the dialog again.
