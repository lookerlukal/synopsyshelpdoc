3.1.7 Intensity Falloff Table (FTABLE, FCOEF)                        Previous Next TOC

 

Object type OBA permits input of a table of intensity as a function of angle with respect to the optical axis, which may be useful for modeling an LED with measured output properties.  One can then freely use the HBAR and GBAR parameters to specify a point in the field, as for an array of LEDs, as well as pupil coordinates XEN and YEN.

 

There are two forms:

 

1.  You can enter a table giving four angles and intensities.  For example, if an LED comes with measured data given by a table,

 

 

you could select four angles and enter the data as shown below:

 

CHG

FTABLE 4

20 .9551

40 .8298

60 .6495

80 .4495

END

 

The program will fit the entered data to a set of four coefficients that will be used thereafter to model that distribution as a transmission change with aperture.  The coefficients are saved with the lens, but not the table data.  Note that exactly four data points are required, and the number "4" in word 2 of the FTABLE line is a reminder of that fact.

 

2.  You can enter the four coefficients directly, as

 

CHG

  FCOEFF   0.9999343     -0.3744452      5.3659603E-02 -3.2903864E-03

END

 

This is also the format that appears in the saved lens file.
