5.38 Clocking element wedge errors                Previous Next

 

When lens elements are manufactured, all dimensions are in error by some amount, and to offset the effects of these errors, one often resorts to adjusting the lens in some way.  Perhaps the simplest adjustment is to orient the lenses in their cells so that the effect of the wedge error in one is compensated by the others.  One practice is to alternate the orientation of the elements, aligning the thick edge of one with the thin edge of the next, and so on. MC can evaluate the statistics of a batch of lenses that are aligned in this way, according to the state of switch 14.  A better idea is to clock the individual elements at angles that are computed to minimize the overall effect. MC can also evaluate the statistics of that process.  By “clock” we mean the orientation of the thickest edge with respect to the local Y-axis, which we call 12:00 or zero clock angle.

 

This procedure can be very effective.  The UCLOCK feature will calculate the best angles and provide a list and a plot of the results.  Input is as follows:

 

UCLOCK [ QUIET / LOUD / TEST [ MONO ] ]

[ WEIGHT WBORE WDISP WCOMA ]

SN WEDGE

SN WEDGE

…

GO

 

[ UCLOCK [ LIST / PLOT ] ]

 

The surface numbers SN in the above list must all be at the second side of each element to be considered.  The WEDGE value is the measured wedge angle, in arcminutes.  The program assigns the entered wedge to the element specified, and also assigns a gamma tilt to the first side of the element.  It then varies the gamma-rotation of each wedge in the list so that the accumulated effects of the wedge errors are minimized.

 

All of the surfaces to be assigned a tilt or rotation must be free of tilts or decenters when the command is issued.

 

By default, the defects to be minimized include the real-ray coordinates of the axis ray in X and Y at the image (to model boresight error), the separation between the rays in the long and short colors for the on-axis object (to model dispersion at the axis), and axial coma, which is approximated with coefficients number 9 and 14 in the COE expansion of the wavefront.

 

The relative weights of these items can be entered with the WEIGHT parameters, which have default values of 1.0.  If MONO is specified, or the lens has fewer than three wavelengths defined, the dispersion effect is ignored.  The optional QUIET input suppresses output from the optimization but displays the results.  The default is LOUD.  A listing and a plot are produced by default, and you can request another copy with the additional input shown above.

 

The optional TEST input adds the specified wedges to the elements, but then stops, letting you evaluate the effect of the wedges on image quality, with no clocking.

 

Cemented groups are treated somewhat differently: you can assign a wedge to the second side of each element as usual, but the program only calculates the clock angle of the group as a whole, not of the individual components.  For this reason, it is wise to enter the wedge with opposite signs on the components, which is common shop practice and which is recommended.  Then the program will clock the orientation of the group.  The result is a list of the optimum clock angles for each element, as well as a plot showing the orientations.

 

To illustrate this feature, let us start with the lens saved as X33.RLE.

 

 

First, let us examine the effect of wedges with the orientations implied by the input, without clocking.  Here, we assume that the elements have been manufactured and the wedge angles measured.  The angles are given in the data below.  Because surfaces 3, 11, and 14 are each on the second element of a cemented pair, we have specified a negative wedge angle, as pointed out above, for those surfaces.  The input is

 

FETCH X33

UCLOCK TEST

WEIGHT 1 1 1

2 4.2

3 -6

6 1.5

8 6.3

10 4.8

11 -2.1

13 6.6

14 -4.2

GO

 

 

Here, we assign the wedge of the first part of the first doublet, surfaces 1 and 2, to surface 2, the second side of that element.  The wedge on the second part goes on surface 3, which is the second side of the second element, and so on.  The wedges are now assigned (but not clocked, due to the TEST input), and we run the PSPRD command to see the effect.

 

 

Here we see a sizable amount of axial coma introduced by the wedge errors.  We wish to eliminate this error by clocking the elements.  To do so we rerun the UCLOCK command, replacing TEST with QUIET.  (The FETCH command is repeated, since UCLOCK requires untilted surfaces to start with.)

 

FETCH X33

UCLOCK QUIET

WEIGHT 1 1 1

2 4.2

3 -6

…

GO

The output confirms that the program did not clock the second of each cemented doublet, since there was already a tilt on the first side of the element.  This is the meaning of the DEFEATED comment.

 

The picture above shows a graphical display of the results:  Element 1 (and 2, since this is a cemented doublet) by default has its wedge oriented at zero degrees, that is, the thickest portion of the element is at the top, or 12:00 o’clock (the first element is always unclocked).  The third element (surfaces 5 and 6) has a wedge of 1.5 arcminutes, and the clocking gamma tilt of 137.92 degrees is assigned to surface 5, with a group size of 2.  (You can check the ASY listing if you are curious.)  Element four (7 and 8), with a wedge of 6.3 arcminutes, is clocked at –14.33 degrees, and so on.  These data will guide the assembly technician when the elements are put into the cell.

 

The listing shows an improvement in all of the measures that are controlled: the boresight error went from 0.2323 with no clocking to 0.0966 with the gamma tilts in effect.  Let’s see how the image has improved as a result of this clocking.  Here is another PSPRD of the clocked lens.  As hoped, the axial coma has been much reduced.

 

 

It goes without saying that you must be ready and able to run this program as each lens is assembled, in order to take advantage of improvements like these.

 

Finally, note that UCLOCK creates and runs a special MACro to reoptimize the system.  As a consequence, if you try commands like FINAL or SYNOPSYS, the program will not report on or run the last optimization that you ran yourself, which you might have expected, since the new optimization has overwritten those data.

 

You may run your UCLOCK command file via the MACro editor, but do not run it as the alternate MACro.  UCLOCK creates its own alternate MACro to do the optimization, so this must be vacant.

 

The optimization in UCLOCK works better if switch 67 is turned on and will turn it on temporarily if it was not on already.
