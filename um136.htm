13.6 Interactive Directory (DIR)                                        Previous Next TOC

 

The command

 

DIR { RLE / MAC / FIL / TPL / SSP / BTO / ZMX / SEQ / CFL }

 

will produce a listing of all of the files stored with the SAVE commands (RLE), the MACRO editor (MAC), thin-film files (FIL), online testplates, graphics files saved by a Graphics Window (SSP), or tolerance data produced by BTOL.  In the case of RLE files, the ID line of each file is also printed to help you identify the lens.  File types ZMX and SEQ contain lens information as produced by third-party programs Zemax and Code-V.

 

Useful displays of some of these file types are also given by the commands MWL (RLE files) and MWM (MACros).

 

Note that the program supports a specific directory structure, and if you depart from that structure some features may not work.  When the program is installed, it creates a directory C:\SYNOPSYS, which holds the program data files, and a sub-directory C:\SYNOPSYS\USER which contains some startup files read by the program to initialize switches and AI symbols.  You are encouraged to do all your design work in that directory -- or in a new one, which you can create with the CHD command, described below.  You can create as many of these directories as you require, typically one for each new project or customer.

 

The DIR command with no other argument will display the path of the current directory.

 

You can select a different directory by clicking on

 

File|Select Directory.

 

or with the command

 

 CHD [ “full pathname” ]. (Note the quote marks, which are required.)

 

example: CHD “C:\SYNOPSYS\MYPROJECT\”.

 

A CHD command with no argument (or File|Select Directory) will display the Select a new default directory dialog, where you may select an existing directory by clicking on the Browse button and selecting the directory you want.  Or you may type the full path of the desired new directory into the dialog yourself, for example, C:\SYNOPSYS\USER\.  (Note the final "\" in the name.)

 

In either case, the desired directory must exist before you can use this command.  The command form is convenient when you change directories frequently, since the command and pathname can be declared a symbol in the CUSTOM MACro, and then just typing this symbol sends you to the other directory.

 

Creating a new directory on your local PC

 

Another useful feature lets you create a new directory just by typing the desired name and clicking the top "Create" button.  The named directory must not already exist.  After it is created, the program will automatically copy three MACro files that are usually needed:  SYSTART.MAC, CUSTOM.MAC, and ICONS.MAC.  The originals of these files are taken from C:\SYNOPSYS\USER\.  This feature is available in the Select a new default directory dialog, described above.  The new directory will be created as a sub-directory of C:\SYNOPSYS.

 

If you change directories with the dialog, the program will automatically run the startup MACro SYSTART.MAC in the new directory, while the command form simply goes to that directory but does not automatically run anything.  Should you then wish to run the startup MACros in the new directory, use the command RESET. (See Section 2.9.1)

 

Creating a new directory on a remote network

 

The Select a new default directory dialog will also let you create a directory on a network drive, with the bottom "Create" button.  In this case, the new directory is named exactly as you state, and does not automatically become a sub directory.  The remote drive must be configured to allow full access to all users.  For example, to create a new directory on network drive Z, one could enter Z:\REMOTE_DIR in the dialog box and then click the Create button.  That directory is created on the network drive and then becomes your default whenever you start SYNOPSYS.

 

We have found that the program executes more slowly when using a network drive.  This is to be expected, since accessing your local hard drive is much faster than sending data through a network cable.

 

We note that, even though SYNOPSYS can access user's directory on a network drive, the program itself can only be installed on your local PC.
