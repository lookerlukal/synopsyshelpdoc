3.11 Lens Arrays                                        Previous TOC

 

All of the surface shapes (except prisms) that can be defined in SYNOPSYS can also be declared to be arrays of identical lenslets.  This kind of element is finding use in molded plastic sheets, for example.  When the surface has been given the shape you want, simply declare it to be an array with the input (in the RLE or CHG file)

 

SN ARRAY NXARRAY NYARRAY XSPACING YSPACING

 

To remove the array attribute on a surface, use

 

SN ARRAY OFF.

 

The NULL directive also removes any array designation in effect.

 

To create an array of identical lenslets in a 3-by-3 grid on surface 2, with the lenslets spaced a distance of 0.1 inches apart, for example, you would input

 

2 ARRAY 3 3 .1 .1

 

There are restrictions on the use of this feature.

 

1. A user-entered CAO or RAO aperture is required, to define the total size of the lens array.  This aperture may be decentered.  If it is not entered, the program will create an RAO aperture that will encompass the entire array.  Do not use any of the UAP aperture options on the array.  Note that this aperture applies to the entire array, not to an individual lenslet.

 

2. The grid number should be odd.  The central lenslet will be centered on the optical axis.

 

3. You cannot put any tilts, decenters, or local or global coordinates on an array.  The raytrace will automatically calculate a temporary decenter as required to put each ray at the correct location on the nearest lenslet, and this will conflict with all of the above options.  Use dummy surfaces before the array if you need to position it with these options.  The surface following the array may be assigned these attributes.

 

4. To show the array properly as an element, the other side should be assigned a CAO or RAO identical to that of the array.  If the other side also involves lenslets, then that side must also be declared an array: the designation applies to the individual surface, not to the element, and both sides must therefore be declared.

 

5. An array cannot be declared CAPTURE or LOOSE for nonsequential raytracing.  Use dummy surfaces before or after if you need to use them in this way.

 

6. Do not place a real stop anywhere following an array.  There is usually not a single unique chief ray in this case, and the pupil search will very likely not converge to what you want.

 

7. SOLID will show the shape of the lenslets best if you select the greyscale option and turn on switch 60.

 

8. All of the input parameters should be non-zero.  If you enter zero for any, the program will substitute reasonable defaults.

The ARRAY attribute is assigned to the HOE itself.  The program automatically assigns identical array attributes to the substrate of the HOE (surface 3 in this example), so that it will be rendered correctly by the drawing programs.  The RAO aperture on 3 (and 4) must be entered by the user.
