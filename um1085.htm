10.8.5 When "OPTIMUM" is not good enough                                Next Previous TOC

 

When a design has gone as far as it can the program will quit with a message stating that the last improvement was less than .01% of the merit function.  Then it wise to try to optimize for a few more iterations, because sometimes the lens will start to improve some more.  If that doesn’t work, you may try one of the following:

 

1.

KICK the lens.  This is a feature that can jump out of a local minimum by altering the design parameters by a small amount and then reoptimizing.  Even better, see item 2.

 

2.

Run the simulated annealing feature.  This will kick the lens repeatedly, sometimes finding a new solution region.  Really!  This feature sometimes works so well it’s scary.  We have learned to anneal the lens with every design job we do, and we encourage all users to do the same.

 

3

Run the BFO feature.  This is an easy way to see if there is a better solution that can be found by flipping the bending of a meniscus element.

 

4.

Run the STRAIN feature, to identify elements that are contributing so much power (and such large aberrations) that the other elements cannot compensate.  Click the above link for information on this useful feature.

 

5.

Eliminate something from the merit function -- especially mechanical constraints.

 

6.

Add some variables.

 

7.

See if one or two aberrations are much larger than the others. If so, these are penalizing the system heavily.  In this case it is instructive to run the system with much greater weight on the worst aberrations.  The resulting design will not be very good, but you will see which aberrations had to get worse to improve the others, and sometimes you can think of a different way to improve them.

 

8.

Sometimes, for reasons that remain obscure, it helps to optimize the lens with fewer variables for a few iterations.  If this yields a substantial improvement, you can then put the other variables back in.

 

9.

Explore the output from the DSEARCH or SPBUILD programs, if your job meets the restrictions of these powerful features.  You may discover a new starting point that results in a better lens.

 

10.

If your optimization run terminates on the first iteration with a message that the damping has reached its limit or no solution could be found, it means that the ordinary DLS method did not work.  Try turning on switch 82, which makes the program calculate first and second derivatives on that pass, and see if you get better results.  If not, try the simulated annealing feature, which will kick the lens out a local minimum and possibly find a better solution.

 

11.

Often, one gets to an optimum solution and then finds that a mechanical problem has occurred: lenses colliding or edges out of control that were not given targets in the merit function, or something of the kind.  So you add the required target and reoptimize – and the lens will not move at all.  It is easy to see why.  Imagine a flat plain with a low spot.  That’s your minimum.  Now raise and tilt this plain by a small angle.  That’s the effect on the merit function of adding a mechanical goal that is rather far off.  There is another minimum a great distance away, but you cannot see it from the bottom of the hole.  In other words, the local minimum is still a local minimum, even though the merit function became much larger.

 

This is where the simulated annealing is most powerful.  Nearly any random change to the lens kicks it out of the hole, and then it can head downhill toward the minimum you really want.  Get in the habit of running this feature after every optimization run, even if it seems to have converged.  You sometimes find a much better solution that way.

 

12

If the optimization fails on the first iteration, which can happen for obscure reasons, try turning on switch 86 and running the optimization MACro again.  That switch causes the program to automatically run the simulated annealing program, at a temperature of 100 and a cooling of 5, when the first pass fails, which may result in a satisfactory optimization.
