8.6.3.6 MTF over Field (MOF)                        Next Previous TOC

 

The convolution MTF may be plotted across the field with the input

 

 [F][Z]MOF { ICOL / M / P / S } 0 FREQ { GRID / 0 } GBAR [ Q ] [FREQ FREQ FREQ]

 

Here, MOF utilizes the exact-ray diffraction MTF calculation, and ZMOF interpolates over the pupil to approximate the OPD at each point, using a linear spline fit to the wavefront.  See 8.6.3.1 for an explanation of the difference.  Briefly, the interpolation method is required for very low spatial frequencies.

 

The FMOF and FZMOF options do a full trace from HBAR = -1 to +1, while the other options trace from 0 to +1.

 

This family of features allows a color number "S", which analyzes the MTF in each color separately.  This would be appropriate for cases where a digital display has phosphors of several colors, and you want to make a sharp image of the whole array -- but you can compensate for lateral chromatic aberration in software.  In this case, you do not have to correct lateral color especially well, which can make the design work better in other respects.  If this option is selected, the program will only honor the first of the FREQ requests.  The other color requests operate as usual: M analyzes all defined colors (referenced to the major-color chief-ray point), and individual colors analyze just that one.

 

FREQ is the first spatial frequency to be analyzed.

 

GRID is the grid number, from 20 to 80; this applies only to the interpolation options.

 

GBAR is the fractional skew field point.  A skew field must have been defined for this to mean anything.

 

Q requests quiet operation: the program will then not list all of the MTFs that are plotted.

 

FREQ … give up to three more spatial frequencies to be analyzed.

 

The program calculates the MTF for each of 21 field points, and plots the results, as shown in the example below.  The input for this example was

 

 MOF M 0 20 80 0 Q 40 60 80
