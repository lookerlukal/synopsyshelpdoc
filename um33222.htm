3.3.2.2.2 Power-series Aspherics                        Previous Next TOC

 

SYNOPSYS has many features to help with the design and analysis of aspheric surfaces.  The most basic option adds a power series to a base spherical radius plus optional conic constant.  That option is described in this section.

 

Others that may be useful include the following:

 

ASY                to print basic information about asymmetries in the lens, including aspheric shapes

ADEF                to analyze the shape of an aspheric surface

ADSTAT        to list statistics of the shape

SPROFILE         to show an aspheric shape with optional contours or fringes

GSAG                 to calculate the influence of a single aspheric term

AGT                to test whether adding aspheric terms improves the merit function

SFIT                calculates the aspheric terms required to match a set of measured sag values over a surface

FFA                analysis of free-form surface

SLOPE        analyze the slope of a surface at the current aperture

MAP                can map the aspheric shape of a surface in global coordinates via GSAG

 

CSAG                fits a set of measured sag values to a rotationally symmetric shape using USS type 21

AAA                automatic aspheric assignment; finds the best place in a lens to insert an asphere

ENM                to alter a surface according to a polynomial representing environment changes, such as temperature gradients, on a surface

USS                provides a menu of many unusual surface shapes        

RZS                recast a surface into Zernike format

RNS                recast a surface as a NURBS surface (USS 15).

RXN                recast a surface as an XNURB surface (USS 24).

RAS                recast a surface as a G-series aspheric

 

The basic aspheric shape is described below.

 

(These options go into an RLE or CHG file, and may be entered interactively with the dialog SPS.)

 

SYNOPSYS will accept aspheric polynomial terms of the following form, the data are input with the DCn or ATn format:

 

Z =

  G(1)R**2

+ G(2)Y

+ G(3)R**4

+ G(4)R**2Y

+ G(5)Y**2

 

+ G(6)R**6

+ G(7)R**4Y

+ G(8)R**2Y**2

+ G(9)Y**3

+ G(10)R**8

 

+ G(11)X

+ G(12)R**2X

+ G(13)R**4X

+ G(14)X**3

+ G(15)XY

 

+ G(16)R**10

+ G(17)

+ G(18)R**12

+ G(19)R**14

+ G(20)R**16

 

+ G(21)R**18

+ G(22)R**20

 

 

 

 

where R is the radial distance from the axis.

 

This polynomial permits expressing non-rotationally-symmetric surfaces as well as the more common aspherics using even powers of the radius.  The same input fomat is also used for unusual surface shapes (USS).  Data are input as follows:

 

SN DC1 G1 G3 G6 G10 G16

(even powers of radius)

SN DC2 G2 G4 G5 G7 G8 G9

(higher order terms)

SN DC3 G11 G12 G13 G14 G15

(terms without bilateral symmetry)

SN DC4 G18 G19 G20 G21 G22

(highest terms)

 

(To remove surface deformations, enter SN NDEF or SN NULL.)  Term G 17 is used only by the tolerancing program BTOL to model certain thickness tolerances.

 

The deformations so entered may be applied to flat, spherical, conics, or aspheric toroids, but not to biconics, torics, splines, biradial conics, or Zemikes.

 

Note: aspheric coefficients must be entered on a separate line from other input.  This means you cannot input, say DC1 data, on the same line as the RD or CC.

 

Thus, to add a G(3) term to surface 1 of a lens, one could enter

 

CHG

1 DC1 0 .002

END

 

Another form of input is permitted, which has the G terms in numeric order.

 

 SN AT1 G1 G2 G3 G4

 SN AT2 G5 G6 G7 G8

 SN AT3 G9 G10 G11 G12

 SN AT4 G13 G14 G15 G16 G17

 SN AT5 G18 G19 G20 G21 G22

 

 

If switch 68 is turned on, LEO, LE and SAVE will output the deformation terms in this format as well.

 

Some aspheric surfaces use the XDD and XLD mnemonics, as described in the appropriate section of the USS family of shapes.

 

Surfaces controlled by a curvature solve, such as UMC, are allowed to have aspheric coefficients as well as a conic constant, but they cannot be assigned any of the other shapes, such as toric, biconic, etc.

 

We note here that there are two algorithms in the program to find the intersection point of a ray with an aspheric surface.  The default method is very fast, using a coarse sample to find a crossing point and then a quadratic search to zero in.  This is recommended for the usual situation where the shape is determined largely by the radius of curvature and conic constant, and the aspheric terms depart from that shape by a relatively small amount.

 

But if the shape itself is largely determined by the aspheric coefficients, another method is sometimes called for.  This can happen with strongly aspheric cell-phone camera lenses -- especially if the shape flies back and forth, creating a situation where a given ray might have two or more intersections with the surface.  If this happens, the fast routine can sometimes zero in on an intersection you do not want.  In that case, you are advised to turn on switch 97, which activates a slower binary search algorithm that proceeds more cautiously and usually avoids the unwanted intersection.

 

Aspheric terms may be varied for optimization as explained in Section 10.2.

 

A useful feature to help you determine whether your lens will benefit from aspheric terms is the AGT command.

 

To illustrate the use of this shape, we load the MACro GNR.MAC.  This starts with a poorly corrected triplet and runs the optimization program.

 

! GNR.MAC

! THIS SHOWS A SIMPLE OPTIMIZATION RUN

 

GET 6        ; THE STARTING LENS, POORLY CORRECTED

 

PANT P        ; DEFINE THE VARIABLE PARAMETERS

VLIST RAD 1 2 3 4 6 

VY 1 TH 20 3

VY 2 TH

VY 3 TH 20 3

VY 5 TH

VLIST GLM 1 3

END

 

AANT P        ; DEFINE THE ABERRATIONS TO BE CORRECTED

AEC         ; AUTOMATIC EDGE CORRECTION

ACC        ; AUTOMATIC CENTER THICKNESS CONTROL

GNR .5 1 3 2 0

GNR .5 1 3 2 .5

GNR .5 1 3 2 .7

GNR .5 1 3 2 1.

GNR .5 1 2 1 0

GNR .5 1 2 3 0

GNR .5 1 2 1 1

GNR .5 1 2 3 1

END

 

SNAP         ; REQUEST SNAPSHOTS AS OPTIMIZATION RUNS

SYNO 25 ; REQUEST OPTIMIZATION FOR 25 PASSES

 

The lens that it produces has a merit function value of 0.01177.

Now we add a new variable to the PANT file and run the MACro once again:

 

PANT P        ; DEFINE THE VARIABLE PARAMETERS

VLIST RAD 1 2 3 4 6 

VY 1 TH 20 3

VY 2 TH

VY 3 TH 20 3

VY 5 TH

VLIST GLM 1 3

VY 1 G 3

END

 

This will vary term G 3 on surface 1, the R**4 term, and since that surface is not yet declared an aspheric surface, it by default will use the polynomial given above.  After running this version, the lens is much better, with a merit function of 0.00466.

 

 

If the surface had already been defined as a different kind of aspheric, the G-variable would have applied to the formula appropriate for that shape, not the polynomial above.
