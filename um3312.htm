3.3.1.2 Other Special Surface Options                        Previous Next TOC

 

(These options go into an RLE or CHG file, and may be entered interactively with the dialog SPS.)

 

SN TAG

produces surface output on subsequent calls to RAY and PRT.

 

SN RVT

removes TAG.

 

SN NDEF

removes general aspheric deformations from this surface.

 

SN SPH

like NDEF except that it also removes conic constants, pickups and solves.

 

SN NAS

removes relative tilts and decenters from this surface.

 

SN NULL

removes all surface declarations; this must be entered before certain complex shapes can be changed to another type.

 

SN REFLECTOR

declares the surface a reflector

 

SN TRANSMIT

removes the reflection designation

 

SN INTERSECT { NEAR / FAR / AUTO }

 

selects the rule for determining which intersection of a ray with a conic surface will be preferred.  Only applies to spheres, conics, (which may have deformations), BIRADIALs, and ASTORICs. See 2.5.2.

 

SN SURFACE { NEAR / FAR }

 

 

controls which vertex of a surface will be drawn by the lens drawing programs, and also the intercept, if a surface is deformed and the desired intercept is past the hemisphere point.

 

SN ADD NB

 

adds NB to all following surface number references that give a surface number greater than or equal to SN.  This is different from the system option ADD NB, which changes all references, regardless of the given surface number.

 

SN DPROP [ D / G A ]  See Section 9.6.

 

 

SN SID 'identifier'

 

You may input up to 12 characters of identification for a surface.  This will be listed by most of the lens data output programs, and will show up on footprint and MAP plots as well.  To remove this item, enter only blank characters.

 

SN [ REAL / DUMMY ]

This parameter applies only to dummy surfaces (with no index change.)  Ordinarily, dummies are drawn differently from ordinary surfaces, especially by SOLID, which just shows a dotted line at the perimeter.  But if the surface in question is a real surface that happens to not be present in the current configuration (a flip mirror that is out of the way at the moment, for example), you can put it in position with no index change.  It will then have no effect on the raytrace, but if you declare it REAL, the drawing programs will show it in the same way as other real surfaces.  This will help to document the position and size of that mirror for the mechanical designers.

 

Use this option if you want a dummy surface to show up on STEP file drawings.  SOLID and other renderings show them by default and do not need this declaration.

 

The default setting for dummy surfaces is DUMMY

 

SN PEFILE { +/- } JSN

This input will pick up the EFILE edge definitions from the lens that starts with surface JSN.  See the above link for details.

 

SN DRAW, NODRAW, EDGE, NOEDGE, RAY, NORAY

control how surfaces, edges, and rays are shown on lens drawings. NODRAW deletes the surface from the drawing; NOEDGE deletes the extra lines that normally define the edge of the lens, and NORAY causes the portion of the ray path at the surface to be deleted, but not the surface itself. Default is DRAW, EDGE, and RAY.

 

SN HYPER

This entry is used only for hyperhemispherical surfaces assigned a hard clear aperture.

 

The program will sense if a spherical surface extends beyond the hemisphere point and will trace rays and draw the surface correctly, provided there is no hard aperture assigned.  If you assign a hard clear aperture, however, you must declare the surface HYPER if your intent is to place the hard aperture beyond the hemisphere point.

 

There are restrictions on this unusual surface type.  Only pure spherical curves are supported, with no aspheric coefficients, and a centered CAO is the only aperture that may be assigned.  Thus, inside apertures (CAI) and unusual apertures via UAP are not supported.  EFILE edges may not be assigned to a HYPER surface.  The program SOLID does not support this kind of surface, but RSOLID does.  A warning message is displayed if you attempt to show the system with SOLID.

 

The CAO assigned to this type is not examined when the program checks for vignetting.  Normally, if a ray intercept exceeds the CAO value, the ray is blocked -- but this is not the case with hyperhemispherical surfaces; so the test bypasses such surfaces.

 

The element drawing program ELD shows fewer reference lines and data boxes with this type.  You are advised to add whatever annotation and dimensions you need with the Annotation Editor.

 

To turn off the HYPER flag, enter

 

SN SPHERICAL.

 

See Section 3.12 for more information about this kind of surface.

 

In the event that the optimization program bends a surface so strongly that it becomes hyperhemispheric, and especially if that surface is to be aspheric, it is may be wise to add an operand that will prevent the curve from becoming so steep.  This can be done with the RSLOPE aberration, described in Section 10.3.1.2.

 

SN FLAG

 

SN FL1

SN FL2

...

SN FL9

Up to 9 surfaces in the lens may be assigned a FLAG attribute.  This is used to identify an important surface at which targets are to be given in the AANT file.  One can then specify that surface on the aberration input line with the word "FLAG" instead of a surface number.  For most lenses this is hardly necessary, since entering the number itself is just as good -- but if you want to run the AEI feature, which will insert elements here and there, the FLAG method is recommended since the flag is reassigned when elements are added in front of that surface.  Thus, the target always applies to the flagged surface.  For example, the input

 

M 0 1 A P YA 0 0 1 0 FLAG

 

will control the axial marginal ray at the flagged surface (which might be an internal reticle) and that ray will be corrected there even if elements are added in front of it by AEI.  It goes without saying that this form only works on those aberrations that can be assigned a surface target.

 

This flag also works in the PANT file, where one might specify, for example

 

VLIST RAD ALL EXCEPT FL4

 

This would then vary all radii except the one assigned that flag.

 

If more than a single surface is to be flagged, the mnemonics FL1 through FL9 may be assigned in a similar manner.

 

There are other mnemonics that refer to specific surfaces too.  These are described in Section 10.3.

 

 

SN STATIC / UNSTATIC

This designation may be assigned to individual surfaces that you want to not be changed in optimization.  You can still change the parameters manually, but the PANT file processor will ignore most variables requested for that surface.

 

Curvature variables are ignored if the surface is so declared, and TH variables as well, except for airspaces following a STATIC surface, which ignore this declaration.

 

This is intended mainly for running AEI, where you might want some elements to remain unchanged, while others can be.  In this case, it is inconvenient to call out each varying parameter in the PANT file, since the surface numbers will keep changing and the file will be incorrect very quickly.  But if you assign this property to the elements you want to remain static, then the PANT file can consist of VLIST ... ALL requests, and the static elements will remain unchanged no matter what the current surface number.

 

DSEARCH automatically assigns this property to the surfaces in the front end of a lens if USE CURRENT is requested without the "ALL" modifier.  It also assigns it to the surfaces of a COVER glass, if that is requested.

 

To remove this declaration, use the UNSTATIC version.

 

SN NOFFILL

This option turns off the PAD flood-fill feature for the lens at surface SN.  It is intended for cases where unusual edge geometry causes the outline to not be completely closed -- which makes the flood fill leak.  The program senses this situation and turns off the flood fill feature when it occurs.  But if a particular element is the offender, you can turn off the option for that lens.  To turn it back on, use SN FFILL.

 

SN SEDGES

This option is intended for rare cases when the program fails to connect surfaces with reasonable edges.  Normally, it draws one or more lines where the edge should be, according to a set of rules involving the geometry, aperture definitions, obscurations, EFILE data, and more.  But unexpected combinations do show up, and the edges may not look as you would like.  This option connects the curves on either side of the element with a simple straight line.

 

To remove this option and get the default edges, use SN DEDGES.
