10.2 Parameter Input (PANT)                                        Next Previous TOC

 

Only those design parameters specifically input as variables are available to the optimizing program.  Up to 400 variables may be specified in an input file of the form

 

       PANT [P]

 

 [ RDR FRACTION ]

 [ CBOUNDS ND1 VD1 ND2 VD1 ]

 [ FBOUNDS ND1 VD1 ND2 VD1 ]

 [ CLIMIT UPPER LOWER ]

 [ TLIMIT UPPER LOWER ]

 [ SLIMIT UPPER LOWER ]

 

 [ CUL CROWNLIMIT ]

 [ FUL FLINTLIMIT ]

 

 [ CLL CROWNLLIMIT ]

 [ FLL FLINTLLIMIT ]

 

 VY SN parameter [ UPPER LIMIT LOWER LIMIT [ INCREMENT ] ]

 VLIST parameter SN SN SN …

 VLIST Parameter SN THRU SN

 

 VLIST RAD ALL [ EXCEPT SN SN SN ...]

 VLIST CSUM ALL  [ EXCEPT SN SN SN ...]

 VLIST CDIFF ALL  [ EXCEPT SN SN SN ...]

 

 VLIST TH ALL

 VLIST TH ALL EXCEPT SN SN SN ...

 VLIST TH ALL OVER VALUE

 VLIST TH ALL OVER VALUE  EXCEPT SN SN SN ...

 

 VLIST TH ALL GLASS

 VLIST TH ALL GLASS  EXCEPT SN SN SN ...

 VLIST TH ALL GLASS  OVER VALUE

 VLIST TH ALL GLASS  OVER VALUE   EXCEPT SN SN SN ...

 

 VLIST TH ALL AIR

 VLIST TH ALL AIR  EXCEPT SN SN SN ...

 VLIST TH ALL AIR  OVER VALUE

 VLIST TH ALL AIR  OVER VALUE   EXCEPT SN SN SN ...

 

 VLIST GLM ALL [ EXCEPT SN SN SN ...]

 

 VLIST CC ALL [ EXCEPT SN SN SN ...]

 

 VLIST G ALL [ EXCEPT SN SN SN ...]

 

 VY SN NURBS

 

 VY SN XNURBS

 

 VY SN ZERNIKE [ SYMM / RSYMM / NLSYMM / RRSYMM ]

 

 VY SN DOE [ SHAPE ] [ UPPER LIMIT LOWER LIMIT INCREMENT ]

 

 VY SN DCA [ SYMM / RSYMM ]

 

 NDTEST

       

 END

 

Where parameter is a code word identifying the parameter, taken from the list below.  The upper and lower limits give the range through which the parameter is allowed to move, and the increment is used for calculating the derivative with the finite difference method.

 

The VLIST options utilizes default limits and increments for variables so entered.  There are several flavors of the VLIST.  The ALL format, which works with many variable types, automatically scans the lens and assigns variables to those surfaces that meet certain requirements.

 

For RAD variables, those are:

 

 1.  The surface is currently spherical or conic and not flat.

 2.  There are no pickups or solves assigned.

 3.  The object and image surfaces are not varied.

 

For TH variables, the requirements are:

 

 1.  There are no pickups or solves involved.

 2.  It is not part of a HOE, DOE, or GRATING

 3.  It is not inside a PRISM.

 

For GLM variables, the requirements are:

 

 1.  It does not pick up another index.

 2.  It is already defined a GLM material.

 

One can exclude surfaces from the ALL variables by declaring them following the EXCEPT mnemonic, which is in word 4 of the line.  Thickness variables generated by the ALL request can be restricted to air spaces, glass thicknesses, or only thicknesses currently over a stated value, by selecting the appropriate option.  Thus, if the lens has some airspaces at the default minimum value of 1 mm and you do not want those to be declared variables with the ALL format because they always want to get smaller but cannot do so, then you could declare

 

 VLIST TH ALL OVER 1.1.

 

This option examines the absolute value of the current thickness, and bypasses that variable if the value is less than the declared threshold.

 

The VLIST GLM ALL ... option will vary the glass model only on surfaces that already have a glass model assigned.  This is in contrast to the form VY SN GLM, which forces a glass model on the requested surface whenever possible, regardless of its current status.  If the surface does not already have an assigned model, the latter option will attempt to find model parameters that duplicate the index values currently in place.  This search is likely to succeed only if the current wavelengths are not very different from the CDF wavelengths.  If it fails, the variable will be deleted, and you are then advised to insert a reasonable glass model yourself before trying to use it as a variable.  A common use of this format is when you want to replace glass models with real glasses one at a time, reoptimizing each case before changing the next.  The VLIST ALL GLM format will then continue to vary those models remaining in the lens, but will leave any assigned glass-table glasses as they are.  So you do not have to remove each of the affected variables, as you would if you declared the GLM variable separately.

 

The VLIST CC ALL ... option will vary the conic constant on all surfaces that already have a conic constant.  These include the normal aspherics as well as Zernike surfaces, linear and cubic splines, and aspheric toroids.  It does not apply to any of the USS types, some of which have a conic term assigned to a G coefficient.  Those must be explicitly varied with the appropriate G term.

 

The VLIST G ALL ... option will vary all G-coefficients that are already nonzero on surfaces declared aspheric of the form DC1, etc.  It does not vary G terms on any other surface shape; to vary those you must input explicit variables on each term of each surface.  NURBS and ZERNIKIES, have other options as well.

 

The VLIST ... n THRU m form is similar to the VLIST ... ALL form, except it only examines and varies variables in the specified surface range.

 

The VLIST options only work with parameters described in the above paragraphs.

 

The form VY SN NURBS will vary all of the G-terms that apply to the Z-coordinates of the control points.  If the surface has been defined as symmetrical about the Y-Z plane (with the original input), only half of the terms will be varied; the other half will pick up those values.  The surface must already be defined a NURBS surface type USS 15.  The form XNURBS is similar, but applies to surface type USS 24 and varies the XLD variables.

 

The form VY SN ZERNIKE [ SYMM / RSYMM / NLSYMM / RRSYMM ] only works on surfaces already declared ZERNIKE.  It will vary the G-terms that apply to coefficients 1 through 37, unless the optional SYMM, RSYMM, RRSYMM, or NLSYMM is entered.  The former will only vary terms with bilateral symmetry, the second only those with rotational symmetry, the third is like the second except it does not vary the highest-order term G 36, and the fourth is similar to the second, except it does not vary term G 2, which is linear in Y and works much the same as an alpha-tilt variable (if one also varies the tilt, the effect is redundant, to first order).  To vary the center offset, use terms G 38 and 39; to vary the Y scale factor, use term G 51.  To vary the unit aperture, use term G 50.  If the ZVZ  option is in effect, none of these forms will vary term G 37, which is managed by that option.

 

The form VY SN DOE [ SHAPE ] only works on surfaces already declare DOEs by one of four forms:  True DOEs, USS 16, USS 20, and USS 25.  Types USS 16 and USS 20 contain a description of the base shape and the OPD function defining the kinoform zones.  For these two, you may enter the optional SHAPE qualifier, and the program will vary only those terms describing that shape.  If that qualifier is omitted, it will vary the terms describing the OPD function instead.  Type USS 25 is by definition always applied to a flat substrate, and the program ignores the qualifier and varies the terms describing the OPD function.  If the surface is a true DOE, the terms to be varied depend on which polynomial has been selected to describe the OPD function.  If it is a G-series polynomial, it varies terms G 16 through G 31.  If it is an S-type polynomial, the terms are 16 through 36.  If it is an XDD polynomial, the terms go from 16 through 48.  Zernike terms cannot at present be varied with this input format.  (You must vary explicit LHG and RHG terms for that shape, as explained in Section 3.3.2.8.)

 

The form VY SN DCA [ SYMM / RSYMM ] is intended for surfaces that are presently spherical, conics, or flat.  It applies the basic aspheric terms to that surface and varies some or all of them according to the directive in word 4. SYMM varies only those terms symmetric in X, ensuring bilateral symmetry. RSYMM only varies terms that are rotationally symmetric.

 

In the default mode of SYNOPSYS, the optional [ P ] on the PANT line has no effect.  This mode gives the minimum amount of printout during optimization, and automatically includes a listing of the input data for PANT and AANT (see 10.3).  If mode switch 29 is turned off (see 10.5), the program will examine the PANT command for the [ P ] and will echo the input if this is present.  If the P is not present it will print a more lengthy, but readable, record of all variables for the run.  In other words, if you want a very short listing, turn on switch 29.  For an input echo, turn off 29 and include the P, and for a longer summary leave the P off as well.

 

You may enter upper and lower limits on each variable, and if you do not, the program will substitute defaults.  These boundary conditions constrain the excursion of the variable.  If a boundary is violated at a particular iteration, the program scales the solution vector so that the variable moves exactly to that boundary, and then drops that variable.  If mode switch 1 is on, the variable remains at that value for the duration of the run.  If the switch is off, it will again be free to move on the next iteration.  Usually a variable will continue to violate a boundary, and it is somewhat more efficient to drop it permanently, by turning on this switch.  At the early stages of a design, however, when the design form is not yet determined, it may be wise to turn off this switch, since a boundary violation may then be only temporary.

 

The default boundaries for curvature and thickness variables may be set with the entries CLIMIT … and TLIMIT ….  Default limits are +/- 5.0 for curvatures (corresponding to a radius of 0.2), while for thicknesses and airspaces the limits are 1.0e5 and 1 mm.  The absolute value of thickness variables is compared with the limits to check for violations, as described above, so only positive limits need be entered.  Thus, if you are designing a system with very small elements, where the thickness may be less than a millimeter, you could enter

 

 TLIMIT 100 0.1

 

to set a smaller lower limit.  This limit applies to all VLIST TH variables, and all VY SN TH variables where explicit limits are not entered with the data.  The limits must be entered before the thickness variables are declared in order to affect those variables.

 

The limits applied to airspaces will be the same as the TLIMIT limits by default, but you can override those data with the SLIMIT input.  The data provided will then apply to airspaces that are variables.  For this reason the SLIMIT input, if entered, must follow TLIMIT.  The limits must be entered before the thickness variables are declared in order to affect those variables.

 

Another way of constraining a variable, which does not result in its being dropped from the variable list if a boundary is violated, is to target the value in the aberration array (AANT).  This type of control may give an exact target or a one-sided boundary (see 10.3.5).  This is the recommended method if you want a variable to move to a particular value and remain there, or if the variable must be constrained within a very narrow region:  if you try to constrain a variable with a narrow region using the parameter limits, it will likely be fixed at one boundary or the other on the first pass, even though the best solution may well be elsewhere.  For this reason it is better to give a wide boundary and control the value with the AANT file in such cases.

 

Proper choice of the increment size is important for success of any finite-difference program.  Too large an increment may produce error due to nonlinearities, and too small may produce inaccuracy due to problems with numerical precision.  Default values, described below, are usually appropriate, and these may be overridden when necessary.

 

The easiest way to put the best increments in effect is with mode switch 7 (which is a default).  This switch recalculates the increments at each pass, as explained in section 10.5.  Provided that the initial increments succeed at the first pass, the subsequent passes will use the new values.

 

Although the default derivative increment sizes are usually effective, it can happen that when a parameter is perturbed, some rays fail to trace due to TIR or MCS errors somewhere.  This occurs most often when the lenses are extremely small, and in that case one would like the default increments to be reduced by some factor.  This is the function of the optional RDR input, where FRACTION is the reduction factor desired.  Thus, to reduce the default increments to 1% of their normal values, one would enter

 

 RDR 0.01

 

in the PANT file.  This parameter affects only the starting increments, and if switch 7 is on, new values will be calculated at each iteration.

 

CBOUNDS and FBOUNDS are used to alter the default boundaries that apply to the dispersion part (VD) of glass variables of the form GLM, GBC, and GBF.  These data are most easily developed via the Glass Map.  Enter a pair of Nd and Vd values in each line, with the highest index values first, such as

 

CBOUNDS 1.88 40.43 1.49 78.53

FBOUNDS 1.92 22.16 1.50 62.67.

 

The boundaries so entered apply to all varying glass models except for elements designated PLASTIC.  In that case, narrow default boundaries will apply and the entered bounds are ignored.

 

If you enter non-default values for these boundaries, be sure to also supply a value for CUL, CLL, FUL, and FLL.  Otherwise the index may stray outside of the area you have selected, and results are unpredictable.

 

CUL and FUL give the upper limit for glass variables.  CUL is used for GLM and GBC variables, and FUL only for GBF variables.  The default limit for crown glass is 1.9, and for flints it is 1.92.  If you are designing an i-Line lens, for example, for which there are no suitable glasses with an index over 1.6, this is where you can restrict glass variables to that limit.  The input CLL and FLL similarly change the default lower limit on the index of crown and flint glasses. The default is 1.501, and you may increase that value, but do not decrease it since then the boundary enforcement calculations no longer work. Be aware that these settings remain in effect until you change them or restart the program.

 

The example above gives the program defaults; CBOUNDS defines the crown boundary, and FBOUNDS defines the flint.  See the above link for more information about the Glass Map and boundaries display.

Default values for the limits and increments are assumed unless specific values are input, except as noted below.  The data are defined as follows:

 

RD, RAD, CV

vary the radius of curvature of surface SN.  If the surface initially is a conic, this variable keeps the previous value of the conic constant.  If the surface was initially flat, the radius will be set to 10**8 units before optimization.

 

With any of these three, it is the curvature ( 1 / RD ) that is actually varied.  The limits on RD therefore actually apply to the curvature, not the radius itself.  Suppose you want the radius to be no smaller than 0.01 in absolute value.  You must then override the default limit, which is 0.1, with the input

 

VY SN RAD 0.01 –0.01.

 

Here it looks like the upper and lower limits are reversed – but they are not.  The upper limit on curvature is the reciprocal of 0.01, or 100; and the lower limit is similarly –100.  So the limits are correct when applied to the curvature, and have to be in this order to satisfy the syntax.

 

You can also specify a different default limit with the CLIMIT parameter, described above.  In that case you do not have to enter explicit limits on the variable declaration line.

 

The default increment of RD variables is 5.0E-5/CAO, which gives a slope change of about 2 arcsec.  If the surface does not have a CAO defined, the increment is 1.0E-5.

 

CSUM, CDIFF

The variables CSUM and CDIFF are an alternative to RAD variables.  If you vary the RAD on both sides of an element, you get two variables; if you vary CSUM, the program varies both radii at the same time, keeping the sum of the changes zero.  This is therefore a change in bending of the element.  If you vary CDIFF, the two radii vary in a way that changes the power but not the bending.  Mathematically, one would expect the result of varying the two radii to be the same as varying both CSUM and CDIFF -- but in practice many subtle processes affect the results, and due to the inherently chaotic nature of lens design, you usually end up with a very different lens.  It goes without saying that you must not vary the radii independently and also vary CSUM or CDIFF, which would probably result in a singular matrix.

 

This variable can only be assigned to the first side of an element, and most aspheric shapes are not allowed.

 

TH

varies a thickness or airspace.  The default limits allow the absolute value of the variable to grow to 10 000 units or shrink to 1 mm, keeping the same sign.  This rule gives a wide range to the variable, but the sign of the thickness is not allowed to change. If you want a thickness to be free to move to both positive and negative values, you must enter the limits yourself.  The default increment is 0.001.

 

You can also change the default limits with the parameter TLIMIT ….  The new defaults will then apply to all VLIST TH variables, and all VY SN TH variables where explicit limits are not entered.

 

If a starting thickness is exactly zero, the upper limit is applied to both the maximum value, and, with a sign change, to the lower limit.

 

See Section 10,3.8 for a discussion of how the monitors ACC and ACM can also be used to control thicknesses.

 

INDEX

varies the index at the primary wavelength.  The indices at other wavelengths remain unchanged.  This variable is useful primarily in monochromatic systems, where the dispersion of the material is not important.  The default limits on IND variables are (1.8, 1.46), and the default increment is 0.001.  For polychromatic systems, the glass variables in the next section, which vary all indices according to the glass model, are more appropriate.

 

VD, GLASS, GLM, GBC and GBF

 

all cause the refractive material of surface SN to be redefined as a model glass (see 3.3.4 and App. F).  The program attempts to find values for Nd and Vd so that the modeled values of the primary, long, and short indices are the same as the initial values.  (These variables all require at least three wavelengths.)  The Nd and Vd so found are then available as variables, depending on input.

 

Note that the glass model is appropriate only for materials resembling typical optical glasses, and only over the wavelength range of such materials.  Thus, you should not attempt to use it for infrared or UV work or with other materials.

 

You can see (and adjust) the glass variable boundaries via the Glass Table Display.

 

Glass variables entered with a VLIST are assigned default index limits (1.5 to 1.8, except for GBF variables, which can go up to 1.92).  If you want other limits, use the form VY … GLM, etc, and supply your limits there, or specify a different default with the CUL and FUL directives.

 

See the next sections for more information about glass variables.

 

VD

varies Vd only, keeping Nd unchanged.  The limits on Vd are a function of Nd, and correspond approximately to the crown and flint boundaries of the Schott glass map.  These boundaries can be viewed and adjusted in the Glass Table Display.

 

 

 

GLASS or GLM (or INDEX and VD in succession)

 

vary both Nd and Vd independently until Vd reaches one side of the glass map.  If this occurs, the two variables are merged and the glass moves in one dimension, up and down the appropriate boundary.  The message "GLASS PINNED, VAR. NO.___" is displayed when this occurs.

 

The glass model will not be employed, and the dispersion will not be accepted as a variable, if less than three wavelengths are defined or if the primary wavelength exceeds 2.0 μM.  If the initial glass is outside of the crown or flint boundary, it will be moved into it before optimization.  The message "GLASS  MODEL CHANGES INITIAL INDICES" will be printed when this occurs.

 

Entering variables INDEX and VD in succession is equivalent to entering GLM.

 

Since arbitrarily changing the initial glass properties will often lead to ray failures, one should ensure that the initial model is within or very near the boundaries before using this variable.

 

GBF

moves the initial glass over to the flint boundary, and subsequent optimization will move Nd and Vd together along that boundary.

 

GBC

moves the initial glass over to the crown boundary, and subsequent optimization will move Nd and Vd together along that boundary.

 

PGM

This variable turns on the plastic flag for the requested surface, and then varies the index input to the GLM glass model for that surface.  The flag causes the usual boundaries to be ignored and instead places the model on a line passing through the area on the glass map where plastics are to be found.  The index range is constrained to the region 1.488 to 1.663.

 

The line so modeled is similar to the GBF glassmap boundary set by the input FBOUNDS 1.92 12.76 1.50 80.27.  The program automatically uses the equivalent equation.

 

If the surface is already declared PLASTIC, then the GLM variable does the same thing as the PGM variable.

 

ASPH

causes the surface to be defined as a conic section of revolution (if it is currently spherical), and the semi-axes a and b to be made variables (see 3.3.2.2.1).  Default limits are plus or minus .025, and these apply to computed values of 1 / 2b and b / 2a**2.  (For a sphere, these correspond to radii between 0.025 and -0.025, going through infinity.)  The default increment is 1.0E-5.  This variable is equivalent to giving both the RAD and CC as variables.

 

The command AAA can automatically find the best location in a lens to introduce an aspheric surface.

 

AT, BT, and GT

vary either the alpha, beta, or gamma tilt on a surface.  If the tilt of a collection of surfaces is to be varied, the number of surfaces to tilt together is entered as NB. If a tilt is already present on that surface, NB must either be the same as the previously entered group size, or zero, which will retain the previous group size.  If no tilt is currently present, the surface is redefined as a tilted surface.  It must not currently be GROUP, GLOBAL or LOCAL.  Note that the rules for tilting surfaces listed in 3.8 apply, except that for tolerancing with the program TOL, a single surface may have an AT, BT, and GT simultaneously (since these are used one at a time).  If a non-zero value for AXIS (location of tilt axis) is desired, it must have been previously entered in an RLE or CHG file.  Default limits are plus or minus 90 degrees, and the default increment is 1 mr.  Limits may be input as angles in degrees, and the increment in radians.

 

XDC, YDC, and ZDC

 

vary the surface decenter.  Relative decenters may be specified in the same way as tilts, described above, with the number of surfaces to decenter together given by the NB parameter.  All three decenters may be varied for a single surface.  Default limits are plus or minus 100 lens units, and the default increment is .001.  This is not used for GLOBAL or LOCAL surfaces, which have other mnemonics (see below), and should not be used to vary the decenter of a surface entered with a nonzero AXIS value (see 3.8.1).  These variables can also be used for the decenter of GROUP surfaces, and in that case you do not need to enter the fourth word since the group size will have been declared already.

 

G NB

causes the surface SN to be described as a general aspheric (see 3.3.2.2.2), unless it has already been defined as TORIC, BICONIC, BIRADIAL, HOE, DOE, ASTORIC, ZERNIKE, USS, or GRATING, and the specified coefficients to be varied.  The following list gives the radially-symmetric coefficients of the general aspheric form.

 

G 1  multiplies ρ**2

G 3 multiplies ρ**4

G 6 multiplies ρ**6

G 10 multiplies ρ**8

G 16 multiplies ρ**10    

G 18 multiplies ρ**12

G 19 multiplies ρ**14

G 20 multiplies ρ**16

G 21 multiplies ρ**18

G 22 multiplies ρ**20

 

where ρ = ( X**2 + Y**2 )**1/2

 

Since some of these coefficients multiply the aperture to a very high power, the effect of the coefficients for an aperture that is less than unity is very different than for an aperture greater than unity.  For this reason, default increments are automatically computed based on the aperture size, unless explicitly entered on the input line.  Switch 7, which controls the increments during optimization, is recommended if the G-terms are to be varied.

 

The other G terms are non-rotationally symmetric.  Default limits for all those G-coefficients are + / - 1.0E10.

 

If the surface has previously been defined as TORIC, the variables RD and G 1 apply to the YZ radius and the quantity CX/2, where CX is the curvature in the X-Z plane.

 

If the surface has previously been defined as BICONIC, the variables G 1 and G 2 apply to the CCX and CCY conic constants.

 

If the surface has previously been defined as a HOE, the G variables apply as described in Section 3.3.2.4.

 

If the surface has previously been defined as a DOE, the G variables apply as described in Section 3.3.2.8.

 

If the surface has been defined as an unusual surface shape (USS), the G-terms apply in a manner that depends on the specific shape assigned.  See Section 3.3.2.7 for details.

 

 

 

If the surface is a BIRADIAL, variable G 1 applies to the quantity CX/2, where CX is the curvature in the X-Z plane.

 

If the surface is ASTORIC, the G-variables apply to powers of Y according to the table in Section 3.3.2.2.8 and G 17 to the quantity CX/2, where CX is the curvature in the X-Z plane.

 

If the surface is a ZERNIKE, the G coefficients 1 through 36 apply to the terms Z 1 through Z 36.  The term G 38 varies the X-offset of the unit circle, and G 39 varies the Y-offset.

 

 

TH0, YP0, YMP1, and YP1 vary the object parameters:

 

TH0

varies the object distance, as entered on the OBA line.  See the comments under YMP1, below.

 

YP0

varies the object height.  Note that a zero value of YP0 is never permitted, even as a starting point.  The derivative increment is initially equal to YP0/10,000, which may be a large number if the object is far away and YP0 itself is very large.  Use this form for object types OBA and OBB.

 

UP0

varies the object angle.  This is intended for OPD objects (wide-angle), where the YP0 variable has little meaning since it refers to a flat object surface and OPD creates a curved one.  It also works for OBC objects, where YP0 does not work.  The entry UB0 also works, to follow the mnemonic allowed for aberrations.

 

YMP1

varies the marginal ray height.  Varying YMP1 or TH0 is one way to change the object-space F/number.  (The other is to use the FILLSTOP option and vary the stop CAO.)

 

YP1

varies the chief-ray height on surface one (and also removes any APS specificafion in effect, using the current value of YP1 as a starting point).  This feature lets you find the chief-ray position that best meets your imaging needs without knowing in advance where the stop will be.  When the design converges, you then can examine the ray paths and select the best location for the stop.

 

The default limits for each of these variables is plus or minus 1.0E20, and the increment is 0.001.  For best results, infinite objects (OBB) should be redefined somewhat closer, with OBA or OBC, with TH0 = 1.0E4 or 1.0E5, prior to varying TH0 or YP0. Note that an entry for SN must appear on all parameter input including object parameters (for syntax error checking) although it is ignored in this case, as "VY 0 TH0".  Note also that if you vary the object distance you also change the object space F/number, and if this is not desired you must place a dummy surface number 1 at a convenient location and then vary 1 TH (or else use the system option OSNA (see Section 3.1.1.))

 

If the VFIELD option or a real pupil is in effect, you may not vary YP1, since the two options are incompatible.

 

The variable YP1 also works for systems with wide-angle object type OBD.  In those systems, the value gives the height of a real ray on surface 1 instead of a paraxial ray on the tangent plane, since the latter would go to infinity at 90 degrees.

 

ZTH0

This variable changes the object distance for a given zoom setting.  It is intended primarily for adjusting the object distance in MC analysis. To vary the object distance for a lens that is not a zoom lens, use the TH0 variable, described above.

 

Unlike the ZDATA variable (which is the only other variable that can be assigned to a particular zoom setting), this one can be applied to ZOOM 1 and higher.  (The ZDATA variable only applies to zoom 2 and higher, since in zoom 1 the offsets are always zero.)

 

This variable lets you focus a zoom lens by adjusting the object distance.  To focus individual zooms at the image, on the other hand, simply assign a YMT solve to the system; then all zooms will see the solve and will be refocused, and no variable is required.

 

The second word on the VY line in this case refers to the zoom position, not a surface number.  The VLIST format is not available for this variable.

 

PTH0

This parameter varies the value of TH0, or the object distance, with special application to zoom lenses.  When it is varied, the new value is automatically assigned to all defined zoom positions.  It is used to model a refocus at the object, where all zooms must work at the same distance.  (The variable ZTH0, above, is used when each zoom is allowed to adjust the object distance independently of the others.)

 

The surface number is not important, but a number must appear in word 2.  Thus you might enter “VY 0 PTH0”.

 

XMP1, XP1

are the same as YMP1 and YP1 but in the X-Z plane.

 

CC

designates the surface a conic section, and the value of ( 1 / 2b ) is varied keeping ( b / 2a**2 ) constant.  This is an alternate way of varying a conic section that will hold the axial radius of curvature constant while varying the conic constant.  (Note that varying RD on a conic will vary ( b / 2a**2 ) while keeping ( 1 / 2b ) constant.)  Default limits are plus and minus 20 for the variable (1 / 2b), and the increment is 0.0001.  (See App. C for a description of conic formulations.)  To vary both the radius and conic constant, use the simpler ASPH.

 

The command AAA can automatically find the best location in a lens to introduce an aspheric surface.

 

AP1 and AP2

vary the aberration coefficients of a HOE.  This feature may be used only if the system containing the HOE is ACON 1, and the HOE must already have had coefficients defined for the point P1 or P2 with the input A11 or B11, etc.  (See section 3.3.2.4). NB in each case is the G-term of the aberration polynomial to be varied.  Thus, to vary the fourth-order spherical aberration term of the beam from point P1 of the HOE on surface number 11, the input would be

 

          VY 11 AP1 3

 

The default increment is 1.0E-4, and the limits are plus and minus l.0E10.

 

ACCOM

varies the accommodation of the system (which must be AFOCAL).  The last surface will be placed at a distance of 1 / D from the previous surface, where D is the number of diopters of accommodation, and the image surface will be a spherical surface centered at the previous vertex.  See section 3.2 for a description of this feature.

 

XG through GG

 

vary the global X, Y, Z coordinates, and the global angles alpha, beta, and gamma.  If the surface is not presently GLOBAL, it will be made so at the current location. See section 3.8.2.

 

XL through GL

 

apply to LOCAL coordinates. See section 3.8.3 and the above.

XE through GE

 

vary the EXTERNAL position and angles.  EXTERNAL mode will be turned on if it is not already.

BTH

varies the offset from the paraxial focus, provided that the lens has a YMT solve defining the back focal distance.

 

ZDATA NZOOM

is used to move the zooming groups of ZFILE zoom lenses (see 10.7.1).  In this case, the entry SN refers to the group number, not surface number.  To vary the position of group 2 in zoom position 3, for example you would enter VY 2 ZDATA 3.

 

CAO

varies the hard clear aperture on the surface.  The aperture will become hard if the lens initially has program-defined apertures.

 

LHG, RHG

are used to vary the left-hand and right-hand half of a G variable.  This applies only to surfaces where two single-precision values have been encoded into the one double-precision address of G.  The only current case is that of a HOE with Zernike coefficients; these are loaded into the G-array as follows: Coefficients ZI to Z17 of point P1 are in the left half of G 16 through G 32, while coefficients Z18 to Z34 are in the right half.  Z35 and Z36 are in the left and right halves of term G33.

 

For point P2 the G address is increased by 18.  Thus, to vary term Z 19 of point P2 of the HOE on surface 3, the input would be "VY 3 RHG 35".

 

VZN

This variable changes the value of one of the AI numbers Zn.  It has no effect unless you are using the CLINK aberration form, where you input a command line or re-run the alternate MACro – and the data for that command or MACro contain one or more of the symbols Z1 through Z20 – or you are using a composite definition and want the Zn values to occur in an equation.  For example, to vary the value of Z6, one would put

 

VY 6 VZN

 

in the PANT file, and then use the characters "Z6" somewhere in the command or MACro, in the location where you want the numeric data to vary.

 

Since this variable takes effect only when the command line or alternate MACro is run, and not when Zn itself is altered, the CLINK aberration should be placed at the start of the AANT file, before any other aberration that may be altered by the change in Zn.

 

GOUT

This variable is used when you want to remove an element or change the index to that of AIR.  It declares the surface a fixed-index type, with all index values equal to the current major index on that element,  sets the upper limit on the index to the current CUL value, and the lower limit to 0.001.  These limits allow the value to drop to 1.0.  This is intended as a convenient way to see whether a lens element can be removed entirely.

 

(An even easier way is with the AED feature, described in Section 10.16.)

 

If one just removes an element, the lens will usually exhibit ray failures, and it is not always simple to proceed from there.  But if you vary the index with GOUT and target its value to 1.0 in the merit function, the lens will wind up with nearly zero power and you can then remove it without much effect.  And if the lens cannot be removed without a serious effect on the aberrations, the optimization program will not let the index drop all the way.  This is an indication that the element is serving a useful purpose and perhaps should be left in.

 

GPA, GPB, GPG

These variables change the group tilt angle.  They only work if a group tilt has already been assigned to the requested surface (unlike the relative tilt variables AT, BT, GT, which automatically put the tilt into effect if it is not there already).  All three directions are permitted on a given surface using the group tilt option.  The default limits are +/- 180 degrees, and the default increment is 1 mr.

 

CAX, CAY

These vary the decenter of a hard CAO in X or Y

 

 

The VLIST form of parameter input allows you to input a number of surfaces having a given type of variable, such as curvature or thickness.  When this form is used, default limits and increments will apply.  Any valid PAR that does not take an extra numeric argument may be specified, for example,

 

VLIST RAD 1 2 3 5

 

is acceptable, but not

 

VLIST AT NB 1 2 3...                (Invalid)

 

because the AT variable requires the extra argument, NB, specifying the number of surfaces to tilt as a group.  Any number of VY or VLIST entries may be input, up to a total of 400 variables.

 

By default, the program tests all the derivative increments before optimizing the lens and reduces any that cause ray failures, which is usually a good idea.  But if you have entered your own increments with the PANT data--and want those increments to remain unchanged--the directive NDTEST entered following the PANT file will defeat that step.
