5.10 Annotating output ( !, SKIP, MSG)        Next Previous TOC

 

Output may be annotated in three ways:

 

All entries on a line following a semicolon “;” or an exclamation point "!" are considered comments.  Such lines may occur nearly anywhere, including within data files (except when prompted for a filename, or glass type name).

 

A section of ordinary text may be included in the printed output if it is begun with the line

 

SKIP

 

and ended with the line

 

EOS.

 

If placed in a MACro, this will annotate the output.  No other action is taken on the contents of the SKIP file.  You may also enter input of the form

 

MSG any desired message using valid characters

 

at any command-level position (i.e., not within a data file) to cause the message to appear in the output listing.

 

(Previous versions also honored the format TEXT ... END to denote text lines.  That function still works, but since there are already three other ways to identify lines that are bypassed by the command interpreter, it seems redundant to provide yet a fourth.  So this format is scheduled for deprecation at a future date).

 

You can easily comment out a selection of lines, or uncomment the section, in the editor, with the toolbar buttons The former adds the "!" character at the start of each line, while the latter deletes that character (or the ";" character).
