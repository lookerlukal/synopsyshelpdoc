10.15 Automatic Aspheric Assignment (AAA)                Next Previous TOC

 

As a rule, one would like to avoid the use of aspheric surfaces when designing lenses, because they are much more difficult to fabricate and test than are spherical surfaces.  But for some purposes their use cannot be avoided – and they are in fact recommended for applications involving molded plastic elements.  One can sometimes eliminate up to three conventional elements by introducing a single aspheric surface, and a molded aspheric lens costs no more to make than a spherical one (although the mold is more expensive to make).

 

So one sometimes would like to know just where in the lens the introduction of an aspheric surface would do the most good.  The purpose of AAA is to find out.

 

Input is the same as for normal lens optimization, except the sequence requires the AAA command at the top:

 

AAA LLIB [ QUIET / 0 ] [ JSSS JSPS ]

 

[ EXCEPT sn sn sn ... ]

 

PANT

...

END

 

AANT

...

END

 

[ SNAP PASSES ]

[ DAMP DAMPING ]        ! allowed but ignored by AAA

SYNO PASSES

 

No other commands should interrupt this sequence.

 

The program will first execute a normal optimization run with the entered variables and merit function.  Then it will add the variable G 3 to each spherical surface in the range of JSSS to JSPS and reoptimize.  (Surfaces in an EXCEPT list are skipped.) When all surfaces have been tested, the program reports the surface number where the aspheric term reduced the merit function by the largest amount.  The best lens is loaded automatically into the PAD window, and a copy is saved in the lens library in location LLIB.  The optional QUIET in word 3 is recommended unless you really want to see all of the optimization output scroll by on the screen.

 

If the results indicate that the aspheric term G 3 significantly reduced the merit function, you should then consider adding variables for more of the available G-term aspheric terms, adding them a few at a time as long as things continue to improve.  If the radius of curvature is fairly short, you can also try varying the conic constant (with the CC variable).  Note that a CC variable has little meaning when assigned to a surface that is close to flat, and it should not be used in that case.

 

You can also ascertain exactly which aspheric shape to use at a given surface with AGT.

 

Dummy surfaces, those currently aspheric, and surfaces that pick up another curvature are not tested.

 

Running this feature therefore requires nothing more than adding the AAA command with its arguments to the optimization MACro.
