6.4 Ray Fans                                                Next Previous TOC

 

SYNOPSYS can trace ray fans and produce printed or plotted output, or both.  Plotted fans are discussed in section 7.3.

 

Individual fans of rays may be traced, and their transverse coordinates or OPD's at the final surface printed.  The input

 

{ SFAN / XFAN } NRYS { ICOL / P } HBAR GBAR

 

produces a sagittal fan of rays in color ICOL (or the primary color if “P” is entered) containing NRYS rays plus the principal ray.  (A sagittal fan consists of rays having X coordinates in the half pupil – see below.)  A negative entry for NRYS in an SFAN command will utilize rays in the left half of the pupil.  Entering XFAN instead of SFAN gives both halves. SFAN may be run interactively from the dialog MRR.

 

 

Output for each ray includes the fractional coordinate in the entrance pupil and final ray coordinates in the X and Y directions, measured from the chief ray point in the primary color. HBAR and GBAR are the same as for RAY input.

The input

 

{ TFAN / YFAN } NRYS { ICOL / P } HBAR GBAR

 

traces a tangential fan of rays.  These rays have Y coordinates in the entrance pupil as shown below.  Both TFAN and YFAN trace the entire pupil.

 

 

Output from TFAN includes the Y coordinate measured from the chief ray point in the primary color, and the X coordinate if the lens contains asymmetries (tilts or decenters).  Chief ray coordinates at the image are also displayed, for FOCAL TAP (see below) fans.  The output from TFAN is shown below. TFAN may be run interactively from the dialog MRR.

The command

 

FANS NRYS { ICOL / P } GBAR

 

traces T and S fans for four field points (HBAR =0, .5, .75, and 1.0) at the entered GBAR.  (Only the SFAN is traced on axis.)  You may also obtain plotted FANS output, as described in section 7.3. FANS may be run interactively from the dialog MRR.

 

If the entry

 

OPD

 

is input, all subsequent calls to FANS, TFAN, SFAN, XFAN, YFAN, and RAY will output optical path differences instead of transverse coordinates.  To restore output of transverse coordinates, use the command

 

TAP.

 

Transverse mode (TAP) is reset automatically following the commands GET and FETCH.

 

Rays that are vignetted because of edge feathering, clear aperture violations, or ray failure will be identified in the listing unless you have input the optional NOVIG in the RLE file, in which case only ray failures will be indicated.  The listing (but not the vignetting check) may be suppressed by placing switch 33 ON.
