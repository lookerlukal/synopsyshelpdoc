3.7.1 The Lens Library                                Previous Next TOC

 

The library can store up to ten lens descriptions.  This is the most practical place to store lenses under active development since some of the features of SYNOPSYS can read these data and their flexibility is thereby enhanced.  Normally it is up to you to store and get lenses, but a feature is available that will cause the program to reserve location number 10 for automatically storing the lens following an input or change sequence, and after optimization.  To use this feature you must place mode switch 42 on (enter ON 42; see 2.9).  The menu feature MLB can also be used to perform library functions.  In command mode the library commands are as follows:

 

PLB

lists the current contents of the library, along with the name of the current directory

 

STORE

stores the lens in the lowest empty location

 

STORE NB

stores the lens in location NB

 

GET NB

retrieves the lens from location NB

 

DLL NB

deletes the lens in location NB leaving that location empty.  To reset the merit function test enabled by switch 61, use DLL 10.

 

The lenses in the library are actually just lens files saved in the same format as any other RLE file, given the special names LENSLIB01.RLE up to LENSLIB10.RLE.  You may therefore access and display them with the MWL dialog, which is sometimes handy.

 

The command LDWG [ ALL / nb nb ... ] will prepare a drawing of all, or selected, lenses in the library.  See Section 7.10.
