5.56 Fitting aspheric terms to measured SAG data (CSAG)

                                                 Previous Next TOC

 

SYNOPSYS can fit a set of measured surface sag values to an aspheric surface.  This can be used to model a measured surface in order to analyze the performance of the as-build lens.  The shape is assumed rotationally symmetric, and the program assigns shape USS 21 to the surface.  (You can also model a non-symmetric shape, with SFIT.)

 

Input is:

 

CSAG sn

y sag

y sag

...

y sag

END

 

The lens must be current, and the surface to be modeled is given as sn.  The number of zones to be fit must be from 3 to 50.

 

The input includes a list of  apertures (y) and the measured sag at each point (sag).  The measurements must be normalized so the axis point has zero sag, but the entered input data start at the first non-zero Y coordinate.

 

When all the data are read in, the program submits an optimization run that targets the sag at each specified zone to the measured value, varying as many G-term variables as there are points to model, up to the maximum of 30 zones for this shape.

 

Example:

 

A vendor has measured the aspheric shape on surface 3 and provided a set of measurements.

 

The input will then be as follows:

 

CSAG 3

1 -.005145659

2 -.020512029 

3 -.045897901 

4 -.08100158

5 -.125465304 

6 -.178925032 

7 -.241055362 

8 -.311588336 

9 -.3902956

10 -.476908271 

11 -.570935297 

12 -.671380454 

13 -.776293867 

END

 

The program finds the following coefficients, and the merit function comes down to 1.616E-12.  This is a very good fit.

 

SYNOPSYS AI>ASY
