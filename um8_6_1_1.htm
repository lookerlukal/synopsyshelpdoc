8.6.1.1 Interferogram plot                                 Next Previous TOC

 

SYNOPSYS can plot the difference between two wavefronts in the same format as the PUPIL plot.  This requires first creating a reference wavefront with the command

 

IFR { ICOL / P } HBAR 0 SCF GBAR [ 0 0 AXIS ]

 

and then the actual wavefront, which may come from the same or a different lens, with the command

 

IFP { ICOL / P } HBAR 0 SCF GBAR [ WX WY  POST { X / Y / XF / YF } [ COMMON [ PHASE ] ] ]

 

The first wavefront is subtracted from the second, and the difference is plotted.  The program assumes that the two overlap perfectly – without shear or size difference.  These commands may be preceded by a CONTOUR or FRINGES request.  If the sequence involves any other command, such as GET or ACON, then both requests must be so preceded.  The optional WX and WY requests add a wedge to the IPF analysis in either X or Y.

 

This sequence should not be interrupted by any other event that involves OPD raytracing.  (That would alter some settings the program uses to keep track of the two wavefronts.)  Even activating PAD or a different ACON could possibly do that, if the display involves OPD fans.

The optional AXIS in word 9 of the IFR command makes the program ignore the reference chief ray data normally saved when OPDs are calculated.  The effect is to take the axis point as the reference instead of the actual image point.  This can be useful for analyzing the interference pattern between two channels of an interferometer, where the beams may have different image points or ray angles, and you don't want that difference to be subtracted out of the OPD patterns when they each find their own reference wavefront.

 

The optional POST ... in word 9 of the IFP command produces a trace through the center of the pattern, in either X or Y, or a Fourier transform of that trace, with XF or YF.

 

The optional COMMON in word 11 causes the IFP analysis to reuse the same reference sphere location and path length as the IFR analysis.  This is useful for analyzing interferometers where the two arms differ not only in wavefront shape but in phase as well.

 

The optional PHASE in word 12 adds the entered phase, in waves, to the wavefront calculated in the IFP stage.  This can be useful for modeling a Fourier-transform interferometer, where the absolute phase of the two arms changes and one wants to see how the signal is affected.

 

If the lens is in polarization mode, the program will trace the chief ray in each channel and compare the phases; then it adds the difference it found to the OPDs of the IFP case.  This is for cases where the phase is modified in one channel differently from the other, as happens when a half-wave plate is inserted in a beam with circular polarization.  This option assumes that the polarization state of the two channels is identical (linear, circular or whatever), and that all surfaces have been properly modeled with coatings.  If this is not the case, you are advised to turn off polarization mode.
