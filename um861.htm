8.6.1 Exit Pupil Wavefront Map (PUPIL)                Next Previous TOC

 

A plot of the OPD errors in the final wavefront may be obtained from the dialog MDI or using the command

 

[ CONTOUR [ SCF ] ]

[ FRINGES [ GRID ] ]

[ CLEVEL [ GRID ] ]

[ ZLEVEL [ GRID ] ]

PUPIL { ICOL / P } HBAR 0 SCF GBAR [ WX WY [ POST { X / Y / XF / YF }  [ EXPUPIL / MXSF [ RLIM ] [MULTI/ADD/FINAL/ ] ] ]

 

where the data are as follows:

 

ICOL

is the color number , or “P” for the primary color.

 

HBAR

is the fractional Y-object height

 

SCF

is the wavefront deformation that will be scaled to one inch on the plot

 

GBAR

is the fractional X-object height.

 

WX WY

give optional wedge components to be added to the pupil errors.  This is useful when preparing contour plots of the fringe pattem to be expected when testing a system, since interferometers usually are more readable with a wedge added to the wavefront. WX and WY are the wedge, in waves at the edge of the pupil, in X and Y.

 

POST

This entry in word 9 of the command will implement post processing of the fringe pattern.  It only works if FRINGES has been requested. X produces a trace through the pattern in the X-direction; Y does the same in Y; and XF and YF calculate the Fourier transform of the trace in X or Y.

 

In the Fourier analysis, the lowest frequency is a constant value, the next one corresponds to one cycle over the pattern, the third is two cycles, and so on.  The phase is given by the arctangent of the sine and cosine values.

 

The program also tests the transmission of each ray, if the lens is in polarization mode, and alters the intensity of the fringe pattern accordingly.

 

This feature only operates if the lens is in VIG mode.

 

EXPUPIL / MXSF

The entry EXPUPIL in word 11 causes the program to evaluate the fringe pattern at the exit pupil.  It will test the ray density at each point in the pupil and compare it to the density on the axis.  The intensity of the fringe pattern is adjusted if the two are different. This is intended for laser beam shapers, where rays are typically more concentrated at the edge of the pupil in order to compensate for the falloff of the Gaussian laser beam there.  If this feature is selected for an OBG system without beam shaping, the fringe pattern will show the Gaussian falloff in intensity.  If the beam shaping has been done correctly, the fringes will look more uniform.

 

The density test is performed on a flat surface, which is appropriate for a beam shaper, which is usually collimated.  But it is not recommended for a fast focal lens, where this mapping does not satisfy the sine condition and spurious falloff will be evident.

 

This feature only operates if the lens is in VIG mode, and only if FRINGES has been requested.

 

The MXSF option is similar, except that the position of each sample point is taken at the last surface in the lens rather than at the exit pupil.  This can be useful for a telecentric system, where the exit pupil is at infinity and the calculation becomes meaningless at that distance.  It is also called for in multi-beam designs where the several beams are decentered with respect to the optical axis and projecting the chief ray to that axis does not yield a meaningful pupil location.

 

Be aware that, if a beamshaper does not produce a smooth ray distribution at the output, the intensity pattern may see spurious peaks, especially if a caustic is present.  This looks like high intensity, and the program scales the entire pattern down to stay within the 256-level greyscale.  The pattern can then look entirely black, except for a white point here and there where the peaks occurred.  If this occurs, try the RLIM parameter, discussed below.

 

RLIM

This optional parameter is useful when the pupil remapping has nonlinear portions, or worse yet, a caustic.  That portion will give very high intensity, infinite right at the caustic.  This parameter is a limiting value against which all samples will be tested.  Any that exceed the entered value will be set to that value.  Then the well-behaved parts of the pupil should look more normal.  A value of from 2 to 10 often does the job.  Of course it's better to avoid caustics in the first place.

 

MULTI/ADD/FINAL

These entries in word 13 are used when you want to superimpose on a single plot the exit pupil (or MXSF pupil) from separate systems.  This is called for when you have a partitioned pupil, as from an array of coherent lasers, and want to see the ensemble showing all of them combined.  It only works in FRINGES mode, and all cases should otherwise have identical PUPIL parameters.

 

The first of the set should specify the MULTI parameter, subsequent cases should specify ADD, and the last one should specify FINAL.

 

 

The plot is made by drawing a series of horizontal lines spaced as though on an obliquely-seen aperture.  These are given a Y-offset corresponding to the value of the OPD error at each aperture point.  The pattern is shown in normalized entrance pupil coordinates; if XMP1 does not equal YMP1, the difference does not appear on this form of plot.  (However, the CONTOUR and FRINGES options do show the pupil shape; see below.)  Vignetting and obscurations are shown, but pupil distortion is not, since these are normalized coordinates.  To obtain a listing of all rays deleted from the pupil, turn on switch 16 prior to the PUPIL command.  To see the fringes at an intermediate surface, drawn in actual coordinates, the FIS feature is used instead.

The pupil map may also be plotted by a contour-plot program, instead of the normal oblique-perspective program.  To do so, enter

 

CONTOUR [ SCF ]

 

prior to the PUPIL command.

 

The program will generate a contour plot with an interval between contours of SCF.  Up to ten levels can be plotted, and if more than ten would be required to plot at the requested interval, a default interval will be substituted.  If the pupil is elliptical or rectangular, the shape will be shown on the plot.  An example of a CONTOUR plot is shown below.

 

 

The contours may be drawn as a line of level numbers instead of a line, by placing switch 26 ON.

 

 

To show the pupil map as a fringe pattern, the command

 

 FRINGES [ GRID ]

 

must be entered before the PUPIL command.  The pupil shape is shown on this form of plot.  This makes a picture as shown below.

 

 

The default grid index is 70x70, and you may enter any number for GRID from 50 to 500.  This form also shows the relative intensity of the fringe pattern as affected by a Gaussian object, pupil distortion,and  beam shaping (if RESAMPLE is selected), as well as polarization and coatings if the lens is in polarization mode.  To disable accounting for the influence of these effects, turn off VIG mode.  Then the plot will show only the OPDs and not the relative intensity of the pattern.  Post processing is available for the fringe pattern, as noted above.  Absorption losses within lenses is ignored by this feature.  If EXPUPIL is selected, the coordinates of the plot are taken at the exit pupil instead of the default entrance pupil.  The pupil shape may thereby be ascertained.

 

The PUPIL command can also show the shape of the wavefront with a color scale, where blue is the lowest and red is the highest point.  Enter the command CLEVEL before the PUPIL command.

 

 

If ZLEVEL is entered before the PUPIL command, the display will be annotated with the Zernike coefficients, as returned by the ZCOE option.

 

 

 

The PUPIL program can also give the wavefront shape resulting from a COE or ZCOE analysis. To do so, that request must immediately precede the PUPIL request.

 

Another form of the pupil plot shows the wavefront at several points in the field.  This is the FPU option, explained in Section 8.6.14.

 

Note that all of the graphics displayed by the forms of PUPIL (except the EXPUPIL and MXSF variety) refer to the entrance pupil.  The coordinates are normalized in the case of the oblique view, so the picture is always bounded by a circle.  The FRINGE option, on the other hand, shows the entrance pupil shape at the vertex plane of surface 1, and any change in that shape due to VFIELD data or the WAP options, shows up as well.  (An exception is made for object type OBD, since the paraxial dimension can become infinite as the angle approaches 90 degrees; the pupil with that object is shown in normalized coordinates.)  For example the triplet below

Here is an example of a multi-beam pupil calculated for an array of coherent lasers.  The several beams are called for in a CHG file, but could also be called by GET commands.

 

GET 3

CHG

ID TEST PUPIL MULTI

END

 

PP: 2

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP MULTI

 

CHG

1 DEC 0 -2 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 -1.5 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 -1 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 -.5 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 .5 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 1.0 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 1.5 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP ADD

 

CHG

1 DEC 0 2 0 100

END

FRING

PUP P 0 0 1 0 3 0 0 0 MXSF PP FINAL
