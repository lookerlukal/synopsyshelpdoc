13.3.1 SketchPAD graphics                                                        Previous Next TOC

 

There are seven kinds of display available in PAD, of which any two can be shown simultaneously, one in the top half of the PAD window and the other in the lower half.  Clicking on either of the buttons

brings up a dialog where you can select from

 

1.

A Y-Z profile drawing similar to DWG

2.

A perspective drawing similar to PER

3.

A paraxial lens drawing (useful when real rays will not trace)

4.

Transverse ray intercepts, similar to RPT

5.

OPD ray fans, similar to RPO

6.

Spot diagrams

7.

Astigmatic field curves

8.

A blank portion of the screen

9.

A single drawing, centered in the window.  This only applies to the profile and perspective drawing options.  The display form is given by that selected for the top display; the bottom display format is ignored if this option is chosen.  The default drawing scale is taken from the full window dimensions, rather than those of one-half of the window, so the display can be larger.

10.

PAD can also let you link a command line to the display; whenever the display is updated, the command is executed (and may create a picture in the current graphics window.  In this case, there are three displays that get updated together: two in PAD and one in the graphics window.

 

All of these display forms except numbers 9 and 10 can be initiated in command mode as well.  Type the command PAD, and then select from the commands

 

D

for the drawing

PP

for the PAD perspective drawing

P

for the paraxial drawing

F

for ray fans

FP

for ray fans over the full pupil (normally the SFAN covers half of the pupil)

FP3

for ray fans over the full pupil at three field points

O

for OPD fans

OP

for full-pupil OPD fans

S

for spot diagram

A

for astigmatism curves

X

to erase (cross out) a portion of the display

 

You will be prompted for additional data when you type any of the above commands.

 

In multiconfiguration work, it is sometimes convenient to assign a different PAD display configuration to different configurations.  To do so, be sure that switch 48 is OFF, and then define each of them the way you want when that configuration is active.  If this switch is ON, the PAD display is constrained to be the same for all of them.

 

The ray paths shown on the PAD perspective view follow the same rules as for the PER command.  (Switch 88 controls whether entering rays are shown.)

 

 

You can save the current PAD selections for all active configurations with the command

 

 SSU        (Save SetUp)

 

and get them back with

 

 RSU        (Restore SetUp)

 

These commands also save and restore the lens file that was assigned to each configuration.  This makes it simple to restore things the way they were before you exited from the program.  The lens files are named SSU1.DAT, SSU2.DAT, and so on up to SSU6.DAT, for each of the six configurations.  The RSU command automatically fetches all of these files and restores them to the configuration where they were before.
