2.7 OPD -- Numerical Considerations                                Previous Next TOC

 

Certain calculations performed by SYNOPSYS require determination of the optical path difference (OPD) between two rays, one of which is usually the chief ray.  The difference is usually desired to a precision of a small fraction of a wavelength of light, whereas the actual path lengths involved may be on the order of meters (millions of wavelengths).  Calculating this difference may in some cases result in loss of precision, particularly if the distances are unusually large.  An example might be a high F/number lens whose back focal distance is hundreds of meters.

 

Whenever possible, the AFOCAL option  should be used in these cases.  Since this is not always practical, however, a special OPD calculation routine is available which is largely immune to the numerical difficulties mentioned above.  This routine requires that the image surface and the surface immediately preceding it both be flat and free from tilts and decenters, and the back focus distance be over one meter in length.  When these two surfaces satisfy these requirements, the special OPD calculation is automatically invoked.

 

Numerical problems may also arise when a large number is specified for the object distance.  For infinite object distance, the object code OBB is recommended, with the angle UMPO = 0 (see 3.1.1).  This specification utilizes a special entering algorithm that does not require subtracting large numbers.  The program will treat the entering beam as a plane wave (for OPD calculations) if the value of TH0 exceeds 1.0E10.
