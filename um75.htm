7.5 Distortion Curve (DIS)                                Next Previous TOC

 

The command

 

 DIS SCF [ { P / 0 } [ { GBAR / 0 } { ICOL / P / M } {REAL / 0 } [ REFHBAR ] ] ]

 

 

will produce a table and a plot (if P is input) of the normalized distortion vs. fractional field of view for 25 points in the field.  For each point the real-ray intercepts in both X and Y are compared with the Gaussian image height for that field, and the differences are divided by the full-field GIHT.  The output gives two columns, with the transverse (Y) distortion along with the sagittal (X).  The values are thus fractions of the GIHT, in both directions.  If the optional word “GBAR” (not a number!) is entered in word four, the object points and calculation are taken in the X-Z plane.  A zero SCF gives default scaling.  This command can also be run from the MRR dialog.  A FOCAL lens with no distortion has an image height that is everywhere proportional to the object height.

 

A color number may be entered in word 5 (or “P” for the primary color or "M" for a multicolor analysis).  If the word “REAL” is entered in word 6, the distortion that is calculated will be the actual difference (not normalized) between the ray intercept and the paraxial value at that field point, in lens units if the lens is FOCAL, and in radians if not.

 

If you enter a nonzero reference field point in REFHBAR, the magnification is redefined so that the distortion is exactly zero at that point; the result then gives the distortion relative to that magnification.  This is useful, for example, if you want to calculate the departure of each field point from the ideal pixel location, where you try different values of REFHBAR to minimize the maximum departure.

 

In AFOCAL or ACCOMODATED mode the distortion is defined as above except that instead of the linear image height the program deals with the output angles.  Thus, if the lens has no distortion, one finds that the output angle is everywhere proportional to the input (object) angle. APERFECT mode, on the other hand shows the linear image height.  These issues influence the meaning of the distortion calculation, as explained in Section 7.5.1.

 

In lenses with object type OBD (for wide angles) the object coordinate given by HBAR always refers to a fractional angle, not linear height.  As a consequence, the distortion curve will appear perfect only if the image height is linearly proportional to the object angle.  This is the situation usually desired for so-called F-theta lenses.

 

Since the paraxial calculation of the ideal image point ignores tilts and decenters, the program DIS must be used with caution in systems with asymmetries.  In such cases, a value should be entered for REFHBAR, causing the paraxial results to be ignored.  The command CDIST will calculate the coefficients of a power-series polynomial that models the distortion over the field, and does not assume symmetry.
