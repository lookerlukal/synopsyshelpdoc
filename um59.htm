5.9 Thermal Soak Program                        Next Previous TOC

 

The RLE lens file permits optional input of the form TEMPERATURE TEMP, which causes the indices of refraction to be adjusted from the catalog values in accordance with thermal-index data contained within the catalog or entered explicitly by the user.  That feature therefore models the index of the lenses at the entered temperature – but does not affect any other lens data, such as radii, thicknesses, and airspaces.  This is where you give the temperature at which the lens will be manufactured if it is other than the default 20 degrees C.  The lens dimensions you enter will apply to the lens at that temperature.

 

If you need to calculate how the dimensions and performance of a lens will change as the temperature changes, and include the effect of mechanical expansion in the calculation, another feature, THERM, is available, described in this section.  There is also a very simple way to athermalize some systems using the glass map display.  Click that link to read about it.

 

The general case, where the lens experiences thermal fluxes and temperature gradients, is beyond the scope of SYNOPSYS.  That kind of analysis requires taking into account the conductivity and emissivity of the lenses and the cell.  A simple form of that analysis can be performed by ENM, which accepts polynomial coefficients describing surface and index departures from nominal, as you might find if lenses are hotter on one side than on the other.  See that link for details.

 

The simpler case, where the lens acquires a new, uniform, temperature can be examined with THERM.  The program can either change the current lens in accordance with the coefficients and the new temperature, or it can create shadowed versions of the current lens, in different configurations, that reflect the changes due to thermal effects – and which then automatically follow the current lens whenever that lens is changed.

 

This feature will only work properly if every refracting material in the lens has been assigned thermal-index coefficients.  All of the preferred glasses in the Schott and Ohara catalogs have them, and some in the U catalog as well.  See Section 3.3.4 for a discussion of the options.  There are several ways to verify that the materials are suitably defined.  One is to type PGA ALL and look for any for which the program says there are no coefficients.

 

Note also that lenses assigned a glass model (GLM) index have no thermal-index data, since that property varies widely between nearby glass types and is a function of wavelength as well -- and cannot reasonably be modeled.  So use the thermal program only when real glasses are present in the lens.

 

A simpler way is to run the command THERM TEST, which is a standalone command that does not require the rest of the normal THERM input file.  This will list all materials lacking the required coefficients.

 

Every glass in the glass tables that has assigned dndt data also has the CTE, or alpha, of that material, so you do not need to enter those data in the input file.  Lenses assigned an index pickup (PIN) will pick up the CTE from the pickup target as well.

 

To operate this program, each radius, thickness, or airspace is assigned an expansion coefficient, either automatically as mentioned above, or by the user.  Index coefficients are not entered with the THERM file, since the RLE file and glass tables already contain all the needed data.  Airspaces are automatically assigned the ALPHA value that was entered in the lens file, if any.  A new temperature is then input, and the program removes pickups and solves from the altered parameters and calculates the new dimensions.  Input data are of the form shown below.

Linear dimensions change with temperature according to the equation

 

Q' = Q (1 + α ΔT)

 

where Q is the parameter undergoing change, and ΔT is the temperature change.  The quantity α is the change per degree per lens unit.

When all the necessary coefficient values have been entered, they are assigned to individual surfaces by specifying RCHANGE or TCHANGE, and the CNB that applies.  On the following line(s), the surface numbers whose RD or TH have the specified α are listed. Relative surface decenters are recomputed with the coefficients assigned to DECENTER, but GLOBAL and LOCAL dimensions are not altered.

 

If the input file ends with the word TEST, the program prints a table giving the coefficients of each parameter to be altered, and specifies the type of dndt data that apply to each element.  In this case it does not make any changes to the lens.  This listing is more complete than the simpler THERM TEST version, which only lists problems.

 

The desired new temperature is entered with the TEMP line.  It must be in degrees Celsius.

 

The program automatically puts the system at the requested temperature and recalculates those mechanical dimensions that were assigned a coefficient.  Index values are recomputed at the new temperature according to the current dt/dt coefficients.  (Note that if your lens contains a material that is not in the libraries or does not have coefficients assigned – and many of the older glasses do not – you must enter the index coefficients yourself in the RLE file.)

 

Not all surface shapes can be altered by THERM at present.  If your lens contains one that cannot, you will see an error message.  Otherwise the radius is scaled up or down in accordance with the assigned coefficient, and aspheric coefficients are scaled by the power appropriate to each term.

 

Element thicknesses are altered according to the entered CNB, or according to the CTE in the glass table, if it contains coefficients for that glass type.  Airspace changes are more complicated.  If the lens file has been assigned an ALPHA value, the expansion coefficient of each airspace is assumed to take that value, unless a different value is entered for that surface in the THERM input.  If the final airspace in FOCAL systems has a thickness solve, it is not automatically assigned a coefficient, however, so that the solve will automatically be honored.  You must either remove the solve, or assign a coefficient yourself if the lens cannot be refocused with temperature.

 

The situation for ZFILE zoom lenses is more complicated, as explained in Section10.7.1.

 

The airspaces themselves are altered according to one of three rules, which depend on the user input TLEFT, TRIGHT, and TNULL.  These data describe how a change in a given thickness or airspace due to thermal expansion will affect other separations in the lens.  Briefly, if a surface J is assigned a TLEFT parameter of NB, then any change in that TH causes a change of the opposite sign in surface J - NB.  If the assignment is TRIGHT, then the affected surface will be J + NB.  Thus an increase in one place causes a decrease elsewhere.  There is a further complication, however: if the element to the left of an airspace is assigned a TRIGHT designation, then the program assumes that the lens mount design has a reference seat to the left of the element.  In this case, the reference airspace dimension that is to be altered is not just the TH assigned to the airspace; but extends all the way to that seat. (Here, left always means a lower-numbered surface, regardless of which way the light is actually going).  Similarly, if the element to the right of the airspace is assigned a TLEFT, then the lens seat is assumed to be on the right of that element and the reference dimension goes to that seat.  When these cases do not apply (i.e., TNULL), the program takes the reference dimension to be the actual TH altered by the surface sag values on the sides of the elements enclosing the airspace.  Thus, the dimension that is expanded by a thermal change extends across the spacer separating the elements at the clear aperture, not at the vertices, in that case.  This geometry is explained in the figure below.

 

 

In the above figure, the top lens mounting design requires the default spacing option, which is TNULL.  Here, if lens A expands, point a and everything to the right moves to the right.  So there is no compensation elsewhere, and each thickness and airspace expands independently of all of the others.  The airspace is altered by the amount that distance (ab) changes, since that is the presumed dimension of the spacer.  The inner diameter of this spacer is taken to be at the CAO of the element on each end, and the length is given by the nominal TH plus the sag distances at the two lenses.

 

The lower mounting scheme is different.  If lens C expands, point c stays where it was, and the airspace between the elements must decrease.  Similarly, if lens D expands, the airspace between them again must decrease, since it is mounted with a flange to its right and can only expand to the left.  Of course the airspace will expand as well, but the distance that is affected by this expansion is (cd), rather than the nominal airspace.  The change in this distance with temperature is then applied to the TH.  In this example, there are then three separate changes that are added together to find the effective airspace at the new temperature.

 

In all of these examples, we assume that a single shoulder on one side of the lens controls its position.  In practice there is usually a retaining ring of some kind on the other side, but this can easily overconstrain the lens.  If the lens mount expands at a higher rate than the element itself, then the element is underconstrained and will be loose in its cell.  If the temperature falls, the cell may be too small for the element, and this will cause stress in the lens, strain in the cell, and an index nonuniformity according to the stress-index coefficient.  SYNOPSYS does not consider any of these complications, and you are advised to design your retaining rings with enough flexure to avoid this situation.

 

There is one more complication.  As a spacer expands or contracts with temperature, the point at which it contacts the lens surface will change.  This too affects the resulting airspace, since the sag of the surface differs at the new contact point.  In addition, the sag is calculated at the new aperture for the new radius, which also has changed.  The thickness adjustments described above automatically take these effects into account as well, for surfaces that have been assigned a hard CAO.  That value specifies the aperture at the surface where the nominal spacer is in contact.

 

A rather complicated example is shown below.

 

 

Here, we want to athermalize a system by combining two materials with different expansion coefficients.  Let us assume that P is a reference point, and point Q expands away from P with a temperature increase according to the properties of aluminum.  From Q we want to go to the left with a plastic material, which has a higher expansion coefficient, and mount two lenses as shown.  Light comes in from the left.

 

Even this complicated system is not hard to set up, since the program does much of the work for you.  The procedure is to model all of the points where the expansion coefficient changes, and assign the CTE to each where needed.  Then couple each airspace where necessary to the location where an expansion leads to a corresponding contraction.

 

Thus, in the system we first put surfaces at P, Q, R, and S (which is a lens) in that order.  Assign the coefficient of aluminum to P, and that of polycarbonate to Q and to the spacer between the elements.  Assign TLEFT 1 to the surface at S, TLEFT 2 to the airspace between the elements, and TLEFT 3 to element F, first side, thickness.  Do not assign anything to thickness (RS).  In other words, we let the polycarbonate expand things from Q to R, and let the lens thicknesses and spacings control things from R to S.  This describes the physical model.

 

Now look at what happens when the temperature goes up:

 

1.  Q moves to the right

 

2.  R moves to the left relative to Q

 

3.  The thickness of E increases, and (RS) is reduced by the same amount (because of the TLEFT assignment) .  But here, (RS) is a negative number, so the decrease actually moves lens E away from R.  (Even though R is physically to the right of E, numerically it is to the left, and so the TLEFT applies here.)

 

4.  The airspace between E and F is altered by the amount that distance D changes.  This distance is greater than the actual airspace because the TLEFT on lens F tells the program that the mounting flange is on the right side of the element, at the CAO of the second side, and it is the total distance that must expand with temperature.  (RS) is given a compensating change by this expansion because of the TLEFT on the airspace.

 

5.  Element F expands, and (RS) is increased (made more negative) by the TLEFT on that element.

 

In this way, the two elements and the spacer have expanded to the left, while point Q went to the right.  By assigning the proper materials and distances in designs like this, one in principle can produce a lens whose focal point does not change with temperature.

 

Thermal Shadowing

 

 

The THERM input file is terminated in one of three ways: by the TEST directive, which lists a table of coefficients so you can check if everything is proper, but does not change the lens; by the new temperature directive TEMPERATURE, which changes the current lens; or by the input ATS, which activates thermal shadowing.  The latter feature requires that the current lens be in ACON 1, and it places the perturbed lens into the specified configuration(s) automatically.  This is a true shadow: if you change any of the parameters in ACON 1 that were assigned a thermal coefficient, the lenses in the other ACONs change as well in accordance with the thermal coefficients.  Thus you may run a multiconfiguration optimization to control thermal effects by correcting the image in all ACONs at the same time.  Shadowing is turned off by the command THERM OFF, or by any GET or FETCH command.  Bear in mind that the correlation between the nominal and shadowed lenses is defined with the original THERM file, and if you should later edit the nominal lens in a way that changes the surface numbers or materials, the shadowed lens will no longer make any sense.  When shadowing is turned on, the status pane shows the word “SHADOW” near the lower right of your monitor.

 

If you change any properties of the lens in ACON 1 that are not subject to a thermal coefficient, the alternate configurations will not shadow those data.  In that case, if you want those data to be transferred to the shadowed lens, you should run the THERM file again to reconstruct the shadowed version(s).  This applies, for example, when you change the object specifications or enter different wavelengths.

 

Any parameter that is perturbed by the THERM input will no longer be subject to a paraxial solve, if it had one before.  Solves are left alone on parameters that are not explicitly assigned a coefficient, however.  This means, for example, that if the nominal lens has a YMT solve on the image location, the shadowing (or altered) lens will still have a solve if that thickness it is not given a coefficient.  You would use this feature when one is permitted to refocus the lens as the temperature changes.  But if one cannot refocus, then you should assign a coefficient to the final airspace and let it vary with temperature as would any other thickness in the lens.

 

If a second case is to be analyzed, the THERM input file must be entered again, and for this purpose it is easiest to use a MACro.  If you want to monitor the system at up to six temperatures, you should use the ATS option, since that requires but a single setup for all of the ACONs.

 

Following the TEMPERATURE or ATS lines, the system will be recomputed and a paraxial raytrace executed, which will honor any pickups or solves not on affected surfaces.  A simple THERM input file is shown below.

Thermal shadowing works on ZFILE zoom lenses too -- but the shadowed copies are not zoom lenses anymore.  To see a shadowed lens in a given zoom, issue the ATS command when the lens in ACON 1 is in zoom 1.  Then you can change zooms in ACON 1 and the copies in the other configurations will be a shadowed version of that zoom setting.  Also, be aware that the shadowed versions will not have the same CAOs as the nominal lens unless those have been declared hard apertures.  Default apertures are calculated over all zooms, but this step cannot be done in a shadowed version since it is no longer a zoom lens, so the default apertures apply only to that zoom.  Likewise, any CSTOP or FILLSTOP  declaration will be deleted from a shadowed version, since it picks up the object parameters from ACON 1.  If your lens is modeled with GLM materials, be aware that those have no thermal data whatever, and a thermal analysis at that point has little meaning.

 

It is also possible to correct a zoom lens at more than one temperature.  To do so, follow these steps:

 

1.  Use the ATS format explained above to create a shadowed copy of the zoom lens in another configuration.  This can be done in zoom 1.

 

2.  In the AANT file, the aberrations normally start out in ACON 1 and ZOOM 1. You can specify ACON 2 as well, since at that point that ACON is a shadow of ACON 1 in ZOOM 1.  After those aberrations have been defined, you want to switch to the shadowed ZOOM 2 in ACON 2 in the AANT file.  But ACON 2 is not a zoom lens, so you cannot zoom it.  To synchronize both configurations, you have to put ACON 1 in ZOOM 2 first so ATS can modify it and update ACON 2.  Add to the AANT file the lines

 

ACON 1

ZOOM 2

M 0 1 A SYNC

 

The last line is a do-nothing aberration, the only effect of which is to actually switch to ACON 1 and put that lens in ZOOM 2.  Now ACON 2 is a shadowed copy of that setup.

 

3.  Now you can add aberrations for that configuration:

 

ACON 2

M TAR WT ....

 

Thermal shadowing cannot be used if ACON ... PICKUPS is in effect.

 

Let me finish this section with a warning:  When you are finished with the thermal analysis, be sure to turn off thermal shadowing with the command THERM OFF.
