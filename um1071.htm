10.7.1 ZFILE zoom lenses                                                Next Previous TOC

 

A lens can be defined a ZFILE zoom lens with the following input, which must be placed into the RLE or CHG file after all other lens data.

 

ZFILE NGROUPS        (number of zooming groups)

[ COBJ ]

[ CAM RANK RANK ]

[ CAM DAMP DAMP ]

[ CAM EXPONENT EXP ]

[ RSOLVES ]

[ ZCSTOP]

[ CUBIC ]

[ ZFOCUS TH0 JSN DT [ GIHT ] ]

FIRST LAST        [ PFOCUS / DFOCUS ]        (surface numbers enclosing first group)

FIRST LAST        [ PFOCUS / DFOCUS ]        (second group, etc.)

…

ZOOM 2                (defines zoom position 2.  The nominal lens is zoom 1.)

[ FNO FNUM ]

[ { OBA / OBB / OBC … / OBSAME } ]

 

ZDATA ZD1 ZD2 ZD3 … (gives shift along the axis of all the groups)

 

ZOOM 3                (defines zoom position 3, etc.)

…

END                        (this ends the RLE or CHG file)

 

In this input, NGROUPS is the number of moving elements or groups of elements.  It must be from 1 to 10.  The groups themselves are defined by additional data, one set for each group, giving the surface numbers that enclose the group, with the input FIRST and LAST.  The optional PFOCUS or DFOCUS may be used to designate a single group for adjustment by the CAM calculation to maintain paraxial focus.  See that link for details.

 

The CAM command can also set some of the above declarations.  See the above link for details.

 

Each zoom may be assigned its own object specifications, for cases where the object itself changes between zooms; or all zooms may be assigned the same object as zoom 1.  In that case, use the COBJ entry to signify a constant object.  If object data are not entered for any individual zoom -- and COBJ is not in effect -- that zoom will be assigned the same object as zoom 1.  Thus, each zoom can get a different object, which may or may not be the same as zoom 1.  COBJ acts like a pickup; if the object in zoom 1 is changed, all zooms will also change.

 

There is a practical advantage in designing the lens in such a way that the image geometry is constant rather than the object geometry.  Then you can use the CAM ZMAG magn feature to adjust the magnification if necessary, as well as the ZGROUP option in the AANT file, which applies the same aberration definitions to all (or a selected set of) zoom groups.

 

You may enter optional data RANK, DAMP, EXP, and CUBIC, which will affect how the CAM calculation executes a polynomial curve fit to the entered data.  See that link for more information.

 

The optional ZFOCUS entry will adjust the zoom lens to focus at a different object distance.  Enter the desired new value of TH0, the surface JSN whose airspace is to be altered to focus at that distance, and the desired axial shift of the first group, DT.  This option only makes sense if all zooms utilize the same object distance.  Thus, if group 1 consists of surfaces 1 through 4 and you want to focus the lens at 3000 mm by shifting that group by 20 mm, you would enter

 

 ZFOCUS 3000 4 20

 

ZFOCUS is also a command, and it can be included in an AANT file as well, for correcting aberrations at the altered object conjugate.

 

The program alters the object distance and the specified airspace and reduces the paraxial object height YPP0 by the same ratio as TH0, which keeps about the same object angle.  But this may not yield exactly the same GIHT as before, since the focal length of the lens also changes when the airspace is adjusted.  The optional GIHT entry causes the program to find the new GIHT and then adjust YPP0 by the ratio, resulting in a new GIHT that is very close to the entered value.  If this word is omitted, the program does not do the final adjustment.  That would be appropriate when roughing out a lens and the focal length and GIHT are initially not well controlled.

 

The optional RSOLVES input causes thickness solves to be active only in zoom 1.  This means that if zoom 1 has a YMT solve, for example, the last surface will be at the paraxial focus in that zoom -- but not necessarily in the others.  If this entry is not present, then thickness solves will be active in all zooms.  (Curvature solves are always active only in zoom 1.)

 

Many designers prefer to put the aperture stop at the last group, since then the CAO there will apply to all zooms and they all yield the same F/number.  But assigning the stop to a different group can sometimes produce a lens with attractive mechanical properties, smaller apertures, and the like.  But in this case the proper aperture on the (moving) stop will likely vary over the zoom range.  To enable this kind of design, we have implemented a special parameter, ZCSTOP, which goes in the ZFILE section of the RLE file.  (The lens must also have the CSTOP directive in effect for this to work.)  Then, to achieve a desired F/number, which can itself vary over the zoom, one adjusts the YMP1 parameter in the ZFILE object definition at each zoom so as to yield that F/number.  With the ZCSTOP directive in place, the program will then recalculate the stop CAO at each zoom, according to the paraxial beam diameter there, and if the lens is assigned WAP types 2 or 3, the beam of real rays will be adjusted so it just fills that stop aperture.  To remove the ZCSTOP directive, enter NZCSTOP in the ZFILE section of the RLE file.

 

When the groups have been defined, the zoom positions must be entered.  Up to a maximum of 20 positions may be described.  Each zoom setting requires zoom position data for each group and optionally an object definition for that zoom.  The object data are in the same format as an RLE object input line using the mnemonics OBA, OBB, or OBC.  If the object in all  zooms is identical to that in the first zoom or is controlled by FILLSTOP or FFIELD options, no object data need be entered for any zoom.  You may use the mnemonic OBSAME instead of an object specification, which also picks up the object data from zoom 1.  Those system options, and CSTOP, can in some cases maintain the desired object geometry, and they will be honored in each zoom position if they are in effect in ZOOM 1.  Note that the "OBSAME" option is not like a pickup; it gets values only when it is first entered, not continuously during optimization.

 

A restriction applies to object type OBC: if any zoom uses this type, all zooms must use it -- or the zoom slider will not work.  Object types OBA and OBB can be mixed.

 

You may also enter VFIELD data for each zoom.  Just enter the parameters with your input, following the object specification line.

 

The object line, if any, is followed by one or more lines beginning with the mnemonic ZDATA, defining the zoom shift in that zoom of each group according to the values of ZD1, ZD2, and so on.  Note that these data begin with zoom 2; the starting lens is always considered to be zoom 1.  The ZDATA do not give the new airspaces; they give the location of each group relative to its position in zoom 1.  Users of other programs should keep this in mind; those programs may define zoom positions by specifying the new airspaces, which is not the case with SYNOPSYS.

 

Paraxial curvature solves will apply only to zoom position 1, but thickness solves are normally active in all zooms.  Therefore, if the final airspace is assigned a YMT solve, be aware that that airspace will be different in each zoom position.  If you want it to be the same in all zooms, either use the RSOLVES option, or do not assign a YMT solve, but instead control the defocus in the AANT file for ZOOM 1.  Then all zooms will utilize the thickness that result. Never declare the image surface as a zoom group if it is also controlled by a YMT solve.  Then there would be two different rules applying to the back focus distance, and they would fight each other.  Use either a solve or a zoom setting, but not both.

 

We have had good results in cases where the number of zoom positions is between 5 and 10, but the most appropriate value depends on the situation.  It is better to correct at too many zooms than too few.

The input for this case might be as follows

 

RLE

…                        (lens description)

ZFILE 2

5 6

9 10

ZOOM 2

ZDATA .5 .5

ZOOM 3

ZDATA 1.0 1.0

ZOOM 4

ZDATA 1.5 1.5

ZOOM 5

ZDATA  2. 2.

END

 

To make sure that this lens is indeed optically compensated, one would have to include the difference between the group zooms in the AANT file, with a target of zero, as follows (see Section 10.3.3):

 

AANT

…

 

M 0 10 A ZDATA 1 2

; the zoom of group 1 at zoom 2

S ZDATA 2 2

; is equal to group 2 at zoom 2

M 0 10 A ZDATA 1 3

; do the same at zoom 3

S ZDATA 2 3

 

…

and so on

 

 

When the lens has been defined by the RLE file, it may be stored and retrieved with the STORE, GET, SAVE, and FETCH commands – and the zoom data remain with the lens.  To analyze the lens in a desired zoom position, enter the command (in command mode)

 

ZOOM NB

 

where NB is one of the previously defined zoom positions, or else click on the right toolbar:

 

(This toolbar is only visible when the current lens is a ZFILE zoom lens.)

 

When this command is read or the toolbar button is clicked, the program substitutes the object data for that zoom position, and moves the zooming elements the desired amount.  For example, the command "ZOOM 2" with the above lens would add 0.5 to the thickness (airspace) of surface 4 and subtract the same amount from thickness 6, and then repeat this for the second zooming element – thereby changing thicknesses 8 and 10.  To go back to the first position 1, enter "ZOOM 1".  The Al program will print the current zoom position with the question "ZOOM?"

 

It is therefore simple to go from one zoom to another, but there are several restrictions: the program will not permit you to make changes to a ZFILE zoom lens that is not in zoom position 1.  If you did, and then entered the ZOOM command to go to another zoom, the lens might not be correct since the zooms are all referred to zoom 1, not to the one you changed.  The one exception is in the WorkSheet, where you can alter the zoom data on a given group with a slider.  In this case, if the lens is not currently in zoom 1 and you click on or select a surface that is the start of a zoom group, the top slider (normally for CV) becomes redefined as a zoom slider, and the position of that group in the current zoom can then be altered with that slider.  (You can also change any of the ZFILE data from the WorkSheet by clicking on the button while in zoom 1.  Near the top of those data you will also find the CAM RANK, DAMP and EXPONENT values currently in effect, and which you can alter if you wish.)  If the lens is not in zoom 1 and you try to change anything else in the WorkSheet, the program will not let you.  Change to zoom 1 first, please.

 

To change the data in a ZFILE zoom lens, therefore, you must first go to zoom 1, make the change, and then zoom to the desired position.  Alternatively, you can remove the ZFILE characteristic with the command

 

ZFIX

 

when in command mode.  Once fixed in this way, the lens becomes an ordinary lens that may be changed in any way – but not zoomed.  Note that this input is a separate command, and not part of an RLE or CHG file.

 

When the command form of ZFOCUS TH0 JSN DT [ GIHT ] is entered, the program alters the object thickness and position of the focusing group, as described above.  Then you can analyze the image at the new conjugate.  To restore the nominal parameters, enter the command

 

ZFOCUS OFF.

 

Be aware that the ZFOCUS options will alter the specified TH, and if you use AEI to insert any elements before that TH, the thickness will then not apply where it did before, since now that location has a different surface number.  So, if you want to use AEI, be sure to give a surface range that does not include the front group..

 

The bottom button on the zoom toolbar opens a small dialog with a horizontal slider that becomes active when the PAD display is open.  Dragging the slider to the left or right will zoom the lens in small increments, interpolated between the zooms actually defined, to let you keep track of the image quality over the whole range.  This is a good way to check whether there will be image problems at zoom settings that were not controlled in the merit function.  This feature uses the same interpolation as the CAM routine, and you should read that section to be aware of the way the program fits a polynomial and the effect of the CAM RANK, DAMPING, and EXPONENT input, which apply to the zoom slider as well as to the CAM curve.

 

If a ZFILE zoom lens is stored or saved, the program puts it into zoom 1 first.  The program REVERSE will attempt to reverse the lens and the zoom adjustments, although some conditions are not treated, such as a zoomed image plane.  As for all reversals, you are advised to check the results, especially the object definitions afterwards.

 

In configuration 1, zoom 1, you can edit all parameters in the WorkSheet editor.  In other zooms, the sliders are disabled, as is the Update button.  In other words, if you want to change anything in the zoom lens, put it in zoom 1 first.  This restriction ensures that the other zoom positions, which get their nominal data from zoom 1, will be shifted properly.

 

If you are in a zoom other than 1 and click on the first surface of a zoom group, the top slider becomes active and controls the zoom position of that group in that zoom.

 

Special rules apply when a ZFILE zoom lens is subject to thermal shadowing.  The shadowed copy is put into an alternate configuration according to the ATS data, and the rules for which changes are allowed are then more restrictive than otherwise.

 

If you change to an alternate configuration that contains a thermal shadow of the lens in ACON 1, the zoom toolbar is then not available.  You must change zooms in configuration 1 to make the shadowed lens go to that zoom.  You should not attempt to edit other parameters of the shadowed lens either, since they will revert to the shadowed variables from ACON 1 whenever you activate that configuration.   In other words, use the shadowed version to check the layout and image quality – but don’t try to edit any parameters in that configuration independently.

 

See the next section regarding the use of ZFILE variables in optimization.

 

A summary of the ZFILE zoom data is listed by the command

 

ZOUT [ FULL ].

 

The summary includes all of the ZFILE data, including the object descriptions, if the optional FULL is included.  Otherwise, a brief summary is given as shown below.

 

ZFILE ZOOM LENS HAS      2 ZOOMING GROUPS,      5 ZOOMS

 

GROUP DEFINITIONS (FIRST,LAST):

( 5, 6) ( 9,10)

 

ZOOM SHIFTS BY ZOOM NUMBER:

GROUP:    1             2

----------------------------------

 1      0.000000      0.000000

 2      0.500000      0.500000

 3      1.000000      1.000000

 4      1.500000      1.500000

 5      2.000000      2.000000

----------------------------------

 

The program PAD will accept the command ZOOM NB at the PAD> prompt, thereby changing the zoom and updating the display (or you can click on the zoombar, shown above), and a composite drawing of all of the defined zoom positions may be obtained with the mnemonic ZDWG.

 

To alter the ZDATA in a ZFILE zoom lens, either use the LE (lens edit) feature (see 3.6.2), click on the WorkSheet toolbar button , or use the zoom data slider described above.  If you enter the ZFILE mnemonic in a CHG session, you must reenter the entire file of data, so it is easier to let LE and the WorkSheet prepare the current file for you, and then just edit this file with your new data.

 

One sometimes needs to analyze the performance of a zoom lens when the object distance is changed.  The simplest way is to use the ZFOCUS command, as described above, which changes the object distance and a designated airspace that is used for range focusing.

 

It is possible to analyze a ZFILE zoom lens affected by a change in temperature.  The THERM program can be used for the purpose, and the ATS (activate thermal shadowing) option is useful.  This will put a shadowed version of the lens in an alternate configuration, with dimensions altered by the temperature difference.  It will not be a zoom lens, however.  (The ZFILE data are not changed with temperature, so they would be inappropriate if used intact.)  But if you change the zoom setting of ACON 1, the new spacings will be transferred (and subject to thermal expansion) to the shadowed ACON, which will therefore model the lens, at the new zoom setting, at the new temperature.

 

The AI program can evaluate any property of interest, looping over the zoom positions.  It can also examine and change individual zoom positions -- with some restrictions.  The AI input

 

FIND ZDATA 1 2

 

will show the zoom position of group 1 in zoom 2.  You can also change things.  For example,

 

Z1 = ZDATA 2 3

= Z1 + 1

Z2 = FILE 1

ZDATA 2 3 = Z2

 

or

 

ZDATA 2 3 = -44.5

 

AI can also return the number of currently defined zooms, for example, FIND NZOOMS, or Z1 = NZOOMS.

 

AI does not support more complicated constructions, however, such as INCREASE ZDATA 2 3 BY 1.1 or ZDATA 2 3 = FILE 1.  The AI sentence can only contain numbers, the Zn parameters, and the ZFILE data.
