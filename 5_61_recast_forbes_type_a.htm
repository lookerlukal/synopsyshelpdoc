5.61_Recast_Forbes_Type_A

                                 Previous TOC

 

The command RFS will recast a surface shape into a Forbes type A surface.  The format is as follows:

 

RFS sn [ nterms / 0 r0 ]

 

This surface type can take up to 16 coefficients, and the number that will be used may be entered in nterms.  Entering a value of 0 causes all terms to be used.  The normalizing aperture may be entered in word four of the command.  If omitted, the aperture will be taken at a distance to the corner of a presumably rectangular aperture.  If the coefficients are to be fit over a circular area, you should enter the radius in word four.

 

The program calculates the sag of the original surface sn at an array of points over the surface.  Then it submits an optimization run that varies all the enabled coefficients and targets the surface sag at those locations.  When this completes, the surface has been redefined and has a shape one hopes will be very close to that of the previous.  Then one can optimize the new system, varying the G terms as usual.

 

To illustrate this feature, we start with an aspheric camera lens (example C44L3 from my book*) and cast surface 5 to a Forbes type A surface.

 

 

 

 

GET 3

RAS 5 0 0.8

 

The program fits the current aspheric shape and returns a set of coefficients.  In this case, the fit is essentially perfect.

Be aware that it is not possible to match many of the exotic aspheric shapes with this axially symmetric type -- which is why those shapes are available in the first place.  If the attempt does not converge to a very small merit function, it is likely you cannot substitute that shape in this way.
