5.52 Oslo Input File Converter (OSCONVERT)                Next Previous TOC

 

The command

 

OSCONVERT

 

will open a dialog where you can select a disk file with the extension “.len”. This is the output file produced by the lens design program Oslo™. SYNOPSYS™ can open this file and in some cases convert the data to the SYNOPSYS RLE format. When it finishes, it opens the new file in a MACro editor where you can examine the results, make any changes necessary, and then run the file. The file to be opened must reside in the current working directory.

Disclaimer: Since Oslo™ is a third-party program over which we have no control and for which the data are not fully transparent, we make no guarantee that the results of the conversion to SYNOPSYS format will be complete or accurate.

At present this is a work in progress; it works with ordinary lenses, spherical and even-power-series surfaces, and mirrors.  More exotic geometry is not supported, but simple tilts and decenters are.  Oslo files do not distinguish the glass catalog names very well, and many glass names will be converted as pointing to the Schott catalog.  Be sure to examine the RLE file and make any necessary changes to the GTB entries.  Oslo input does not order the wavelengths from long to short, as does SYNOPSYS, so you might have to change the order that is returned by the converter.

For zoom lenses, you'll have to create the ZFILE data yourself.
