9.3 Image Dissection (FOR...)                Next Previous TOC

 

A frequent use of the image sampling program is to generate a knife edge trace, slit trace, etc., where the image is really a multicolor image at a single field point.  For these cases input is available which differs from the DETECTOR command described above.  This form will simplify data preparation and can also produce plotted results.

 

To use this feature, at least one image model must be created with either GMODEL or DMODEL, as described in the previous sections.  The dialog MMO can create the model in interactive mode.  This may be either in a single color, in several colors using the M option, or an image patch.  The model may then be analyzed by the program FOR, which can be run interactively from the dialog MID or in command mode with the input

 

 

This input is quite flexible; the idea is to say what kind of analysis you want in plain English, supplying any data that are not obvious.  For instance, suppose you want to superimpose the image model on a slit and vary its width to see what fraction of the energy gets through:

 

FOR SLIT

VARY X SIZE FROM 0 TO .0015 BY .0001

PLOT

 

To superimpose the image on a circular detector, which then moves in the Y direction keeping the same size, you could enter

 

FOR CIRCLE

RADIUS .0015

VARY Y POSITION ABOUT 0 IN 50 INCREMENTS OF .00025

 

Any meaningful combination of the above input parameters may be entered, and you should provide a complete description of the problem: for a moving detector you should supply the size, and for a detector whose size changes you should supply the position.  For a knife edge the size is not relevant, and only the position should be varied.  For DMODELs the smallest detector that can be sampled should have a radius (or half-width) not less than 1/10 of the radius of the Airy disk.  Requests for a smaller size will be skipped.

 

The extra words on the VARY command are optional.  They make the printout readable, but you could, for example, interchange

 

VARY Y POSITION FROM -.00125 TO .005 BY .0001

and

VARY Y POS FROM -.00125 .005 .0001

 

If the increment size is omitted from a FROM request, the program will generate 100 increments filling the requested interval.  No more than 100 increments should be requested in any case.  The requested number of increments is taken on both sides of center for ABOUT requests.

 

If a plot is desired, the last line in the FOR file should be a PLOT command.  A GO command will generate only a printed list of values.

 

The optional SCALE and ORIGIN input define the units per inch on the plot axes and the coordinates at the origin.  If these data are not entered, the program will substitute a default.  Be aware that if you superimpose curves with the CPLOT option, you may not get meaningful results unless you override the defaults with these parameters.  This is because the default scale depends on the image quality, which can vary with color and field point.

 

This feature can also be used in conjunction with the OV5 / IFOV input described in the preceding section.  To do so, the image model and the IFOV are defined as in that case, but instead of the DET input you may enter a FOR input file.  In this way you can get, for example, a plotted diffraction knife-edge trace where the image is really a small patch rather than a single point-spread function.

 

 

Examples of FOR Input

 

1. In this example we will get a plotted diffraction knife-edge trace at the .75 field point.

 

DMODEL M .75 300

FOR KNIFE

VARY Y POSITION ABOUT 0 IN 25 INCREMENTS OF .001

PLOT

 

Note that the (0,0) position, which is the center position of the knife-edge, is at the primary color chief-ray point.

 

2.  This example shows a simpler way to get the slit trace that was obtained with the DETECTOR sequence in section 9.2.

 

MODEL M 1 200 1 .012

FOR SLIT

SIZE .001 10

VARY X POSITION FROM -.005 TO .005

PLOT
