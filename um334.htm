3.3.4 Index Options                                        Previous Next TOC

 

(These options go into an RLE or CHG file, and may be entered interactively with the dialog SPS.)

 

Each surface including the object may be assigned either explicit indices or to one of several categories that permit the index to be calculated by the program.  The data that are entered on a particular surface refer to the index of the medium traversed by the light following refraction or reflection at that surface.  If explicit indices are entered, the number of indices must equal the number of wavelengths that have been defined.  Input formats are:

 

SN N1 NB                index at wavlength 1

 

SN N2 NB                index at wavelength 2

 

…

 

SN N10 NB                index at wavelength 10

 

 

SN INDEX NB

enters data for the primary color.  This is color 2 in systems where the wavelengths were specified with WAVL input, and the first number in the CORDER input otherwise.

 

SN GLM ND VD

 

 

SYNOPSYS uses a glass model formula to approximate the partial dispersions of normal glasses, as explained in App. F.  The quantities ND and VD are input to the formula, and indices of refraction are calculated for all defined wavelengths.  This calculation is repeated each time that new wavelength data are entered.  The resulting indices correspond to the requested ND, VD to within about .0001 over the CDF spectral region for most index values.

 

The model parameters may be varied in optimization to let you determine where on the glass map the best correction is to be found.  This feature does not locate special glasses whose partial dispersions differ from the average, however, since the model is based on the average glass properties.  Another feature, GSEARCH can try combinations of real glasses, where the individual glass properties are considered.

 

If you leave off the numeric data, and the surface already has a glass-table glass or fixed indices assigned, the program will attempt to fit the current glass parameters to a model, which will enable you to see the difference.  The GTB glass must have been processed first, so only use this option in a CHG file.

 

The glass model is appropriate only for systems working in the wavelength range of normal optical glasses, roughly 0.35 to 2.0 um.

 

If the system operates only in the ultraviolet, from 0.35 to 0.45 uM, an alternate set of coefficients is more appropriate, and is more accurate but valid only over this region.  To activate this option, include the system parameter

 

ILINE

 

in the RLE or CHG file.  To remove this option, use NILINE instead.

 

 

 

SN GTB table / TYPE

 

or

 

SN GTB table  ‘TYPE’

 

This input retrieves data from the stored glass tables.  table may be any of entries { S / O / H / F / U / C / G / R / M / P / N} indicating Schoft, Ohara, Hoya, French Corning, Unusual, Custom, Chinese Guangming,  Russian LZOS,  SuMita, Private, or Nikon;  “Unusual” refers to a set of special materials including IR transmitting and crystals. The tables are listed in App. G, and may be displayed in PAD. Entry C refers to the custom table (see 5.20).  The TYPE is indicated on the following line, and must be spelled exactly as listed in the index.  It may also be given on the same line as the GTB entry,

 

The format is flexible, as shown in the following examples:

 

3 GTB S ‘N-BK7’

 

3 GTB S N-BK7

 

3 GTB S

N-BK7

 

3 GTB S

"N-BK7"

 

This format does not apply to birefringent materials (see 17.5).

 

SN GID / type

 

SN GID ‘type’

An optional word of up to 16 characters may be input for identification purposes only.  This will appear on SPEC listings to aid documentation of lens data.  The type may be entered on a separate line or else enclosed in quote marks.  If new index data are subsequently entered, the GID designation will automatically be replaced by the default GID of the new index.

 

SN GLASS / (6E13.7) Six index interpolation coefficients may be entered on a separate line.  The format of this line is very strict, and it is intended for data produced by a Fortran program with that format specification.  To enter the data yourself, use the GDF form, described below.  You may also use a separate command file, as shown in section 5.6.

 

 

SN GDF / Al A2 a3 / a4 a5 a6
       Same as GLASS except that the six coefficients are entered on two subsequent lines in free-form format.  These data refer to the glass coefficients published for years by the Schott Company, and not to the newer Sellmeier coefficients, which use a different formula.  To enter data for such glasses, it is best to use the GFIT form, described below, which calculates the coefficients in the desired format.

 

 

SN GFIT NWAV [ D ] / WAV IND / WAV IND...

With this input you may give the measured indices at from six to 20 wavelengths, and the program will calculate the six coefficients as though the data were entered with GLASS or GDF. NWAV is the number of wavelengths for which data are to be entered.  This command is followed by NWAV additional entries, each giving a wavelength and the corresponding index. If the measured data are imprecise it is wise to enter 20 values, because this will give a smoother interpolation curve than will six.

 

If switch 26 is turned ON, the program will also print a table giving the input data and the reconstructed index values given by the polynomial; this is useful to check the accuracy of the curve fit.  If the wavelength range is too wide for the material in question, the fit may not be sufficiently precise, and it may be necessary to break up the spectral range into smaller regions over which the fit will be better.

 

If the data underconstrain the polynomial, it may be necessary to add a damping factor, D.  Experiment with values of 1 or smaller until the fit is most accurate.

 

SN PIN NB        pickup index of surface NB.

 

SN N13 NB NB NB for indices less than 2.0, the “1.0” is omitted, and the three mantissae are entered.  You may enter up to five indices in the same way with N15, followed by five index values (without the mantissae).  You may also use the form N610 … for indices 6 through 10.

 

SN NIOP        removes any index pickup or index calculation (from a GTB, GLM, GLASS, or GDF request) fixing the current index values.

 

SN AIR        sets all indices of that surface to 1.0.  The default index is air for all surfaces.  If the system option VACUUM is in effect, the index of all airspaces will be 0.9997076 instead of 1.0.

 

SN VACUUM        sets the index of that surface to 0.9997076.

 

 

SN MELT Nd Vd        This input is valid only for elements with a glass-table index using the GTB format, and for lenses operating in the visual spectrum.  When a batch of glass is melted, the indices always differ somewhat from the catalog values.  Simply enter the measured values of Nd and Vd (after the GTB data are input) and the program will alter the catalog values accordingly.  To remove melt data, enter the GTB description once again.

 

SN MOLD RATE        This is valid only for those glasses designated as low Tg by the vendor.  Those types soften at a low temperature and are suitable for pressing into molds, which is a very inexpensive method of making aspheric glass elements once the (very expensive) molds are created.  The index of the glass then depends on the rate of cooling of the lens, which is usually done at a much faster rate than is the case with the "fine anneal" usually specified with cut lens blanks.  With this input you can specify the rate, which must be from 0.5 to 2.0 degrees/second.  The resultant index is somewhat lower than the catalog value for that glass, and the difference is implemented by the equivalent MELT parameters, which are then automatically applied to that element.  Not all vendors produce low Tg glasses, and not all provide the index change with cooling rate.  This input will not work on glasses lacking those data.  (At the current time, the Ohara catalog contains extensive mold data, while Schott only gives a single reduced index value, which cannot be resolved into different cooling rates.)

 

 

SN DNDT D1 D2 D3 [ W1 W2 W3 ]

 

When the RLE file is processed. the program loads the dn/dt data stored in the glass tables for all materials for which such data are present.  (Currently, this includes the preferred glasses in the S and O catalogs.  Most of the older glass types in those tables have no coefficients, and few in the other catalogs do.)  In the event that you need to evaluate thermal effects on a lens containing a material that is lacking these data, you can enter three dn/dt values with the DNDT input.  The default arguments are the relative dn/dt values listed at the wavelengths 1.06um, the e line (0.54607), and the g line (0.4358).  If those lines are not available or not appropriate for the wavelengths in the lens, you may select any three dn/dt values and follow those data with the corresponding wavelengths.  The program will interpolate between the entered values to find the dn/dt to apply to each wavelength in the lens file.  For example, if the lens contains the Ohara material PBL6Y on surface 9 and operates at the GHI wavelengths, the RLE input could be

 

9 DNDT 3.35E-6 3.8E-6 5.2E-6 0.47999 0.435835 0.365015 ; PBL6Y

 

(Since this glass is one of many for which dn/dt data are contained in the glass table, you would not need to enter the above data in this case.)  If you do not have three values, which is the case for many IR materials, you may also enter just a single dn/dt value, and no wavelengths.  That value will then be applied at all wavelengths.

 

The DNDT input may also be input for materials that do not use the GTB glass-table lookup, such as the fixed-index option N13, etc.

 

Before you change the temperature of the lens, it is wise to check whether all of the materials have been assigned the required coefficients.  The easiest way to check is to run the command THERM TEST, which will list all non-air materials lacking dn/dt data.  You can also list the attributes of each glass in the lens with PGA ALL, and the program will identify any that do not have thermal coefficients.  This check should be done before the temperature is altered and before running THERM.

 

To remove the DNDT coefficients, enter the line with no data, such as

 

9 DNDT

 

SN CTE ALPHA

 

You can also enter a thermal expansion coefficient for materials that do not come with that datum.

 

SN PLASTICThis entry identifies the glass model material as a kind of plastic instead of glass.  The only effect is that in optimization the boundaries of the glass map are restricted to the area where plastic materials are to be found, ignoring any CBOUNDS and FBOUNDS data that may be in effect.  Then, when you substitute a real plastic for the model, either manually or with ARGLASS, the match will work better since the lens was already close.  To remove this declaration, use SN NPLASTIC.

 

If the object is not in air, indices may be entered using surface number 0.  Any of the index options listed above may then be used, with the exception of GRIN, BIREF, and PIN.
