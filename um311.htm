One of eleven object formats must be selected for input.  The contents of each is indicated below, where the data items are identified in the figures.  Note that the definition of positive object height depends upon which object mode is selected.

 

 

A spherically-curved object may be specified in an RLE or CHG file by entering surface number 0 with RD or CV input.  The object is automatically assumed to be curved if selection OBD is used.  In that case the entered angle(s) may exceed 90 degrees.  See below.

 

The formats are as follows:

 

OBA TH0 YP0 YMP1 [ YP1 XP0 XP1 XMP1 ]

[ GAUSSIAN RBS ]                                        (See Section 3.1.4)

[  FTABLE ... ]                                                (See Section 3.1.7)

 

OBF TH0 YP0 YMP1 [ YP1 XP0 XP1 XMP1 ]

OBB UMP0 UPP0 YMP1 [ YP1 UXP0 XP1 XMP1 ]

OBC TH0 YANGLE YMP1 [ YP1 XANGLE XP1 XMP1 ]

OBD TH0 UPP0 YMP1 [ YP1 UXP0 XP1 XMP1 ]                (see below)

 

 

OBG WAIST [ RBS [ WAISTx ] [ RBSx ] [ M2 ] ] ]        (see section 3.1.2)

 

 

OBL TH0 YP0 ANGLE [ XP0 ]                        (see Section 3.1.3)

 

 

OBW SEMIAP MULTIPLIER                        (see Section 3.1.5)

 

OBI TH0 YP0 ANGLE XS YS NX NY                (see Section 3.1.6)

 

The above object and pupil descriptions require only a single line of input.  The object and pupil can also be specified using one OBJECT line, plus one REFERENCE line chosen from the list below.

 

OBJECT FINITE TH0 YP0 [ XP0 ]

OBJECT INFINITE UPP0 [ VPP0 ]

 

REFERENCE HEIGHT YMP1 [ YP1 XMP1 XP1 ]

REFERENCE ANGLE UMP0 [ YP1 VMP0 XP1 ]

 

TH0

object distance from surface 1, positive if to the left.

 

YP0

object height, in lens units.  Must be non-zero.

 

YMP1

axial marginal ray height on surface 1 vertex plane.

 

YP1

principal ray height on surface 1 vertex plane.

 

XP0

object height along X-axis, in lens units. Zero is permitted for XP0, but not for YP0.

 

XP1

principal ray height on surface 1 X-axis, from the object at XP0 or UXP0

 

XMP1

X-dimension of axial marginal ray (see below)

 

UMP0

paraxial marginal ray angle in degrees.  Used chiefly for infinite conjugate, for which UMP0 = 0.

 

UPP0

field angle in degrees of object on Y-axis, measured at the vertex of surface 1.  The value must be non-zero.

 

UXP0

paraxial chief ray angle in degrees for object on X-axis, measured at the vertex of surface 1

 

YANGLE XANGLE

These quantities are used only with the OBC object format.  They specify the object height in terms of an angle, measured from the vertex of surface 1, instead of a linear height, which is the case with format OBA.  OBC differs from the OBD format (which also specifies an object angle) because the object is plane in the case of OBC, whereas it is curved with OBD (and can extend past 90 degrees) – and object points are specified as a fractional height (as with OBA) instead of a fractional angle (as with OBD).

 

ANGLE

specifies the semi-angle of light from a Lambertian object

 

Note that if a stop surface has been defined, with APS SN (see section 3.2), and SN is not equal to 1, a pupil solve will be performed to recompute YP1 and XP1.  Otherwise, the input values of YP1 and XP1 will apply.  If "APS 1" is physically entered, the current values of YP1 and XP1 are set to zero and the stop is then actually on surface number 1.  Unless the pupil has been declared elliptical with EPUPIL or rectangular with RPUPIL, any user-entered data for XMP1 are ignored, and the pupil remains circular (the default is CPUPIL).

 

If the FILLSTOP option is on, the input value of YMP1 will be replaced with a value calculated so that the paraxial marginal ray will intercept the stop surface at the clear aperture (which must be input by the user).

 

If the FFIELD option is on, the program will adjust the value of YP0 so that the absolute value of the Gaussian image height is equal to the clear aperture on the image surface (which must be input by the user).  This option does not work with selection OBD, but you can solve for the corresponding object point with OCALC.

 

If the WAP 2 option is on (see 2.6.3), the values of YMP1 and XMPI are adjusted at each field angle so as to fill the clear aperture of the stop surface.

 

The WAP 3 option adjusts the values of YMP1 and XMPI to approximate the pupil formed by all of the user-entered apertures in the system. See 2.6.3.

 

The object coordinates can be varied in optimization as follows:

 

The object distance can be varied and targeted with the mnemonic TH0, as "VY 0 TH0".

 

The object height can be varied and targeted with YPP0 or YP0 if the object is type OBA or OBB.  For object types OBC and OBD, the mnemonic is "UPP0", "UP0", or "UB0".

 

Yet another way to control the value of YMP1 is with the input

 

OSNA NA

 

This input must follow any other object descriptor.  The program calculates the value of YMP1 (and XMP1 if the pupil is circular) so that the entering marginal ray angle satisfies the condition n*sin(UMP0) = NA, where n is the object-space index at the primary color, keeping that entering beam NA even if you later vary the value of TH0.  Most of the other options would keep YMP1 constant as TH0 varied, with the result that the object-space NA would constantly change.  OSNA is incompatible with the other options that recalculate YMP1 by various means, and may not be requested for a ZFILE zoom lens, for which the (possibly) changing value of YMP1 would not be seen by any zooms except number 1.  You may turn off this option with the input

 

    OSNA OFF.

 

The object beam can be rendered telecentric in two ways:  One is to assign the stop to surface 1 and then specify a value of YP1 equal to the object height.  The other is to include the line

 

    TCO

 

in the RLE file.  That declares the object to be telecentric, puts the stop on 1, turns off any WAP designations, and copies the object height to YP1 (and XP1 if there is a skew field).  If this is turned on, you cannot alter those parameters independently.  To remove this declaration, use

 

  TCO OFF.

 

The utility OCALC (see 5.17) can adjust the values of YP0 and XP0 so that the corner object is imaged at a desired location on the last surface.

 

Light rays are specified by their fractional entrance pupil and field coordinates, and the entrance pupil is normally assumed circular (see the following section).  The shape of the exit pupil will depend on the amount of vignetting that has occurred, in those programs that consider vignetting. In all of the object selections except OBD, the object is assumed to be flat unless a shape is explicitly given to surface 0.  All of the features of SYNOPSYS that refer to a relative field in this case specify the fractional linear field point with arguments HBAR and GBAR, and never specify the angle or fraction thereof directly.

 

For systems with a total field of view that exceeds 180 degrees, the OBD form is required.  In this case the object is automatically assigned a radius of curvature equal to the object distance TH0, and the input angles UPP0 and UXP0 may exceed 90 degrees. For this selection only, the arguments HBAR and GBAR give the fractional angle, and not the fractional linear object height, as in the other cases.  For an object at infinity, you may enter a large value for TH0, up to 1.0E21.  This option of course can also be used when the object field is less than 180 degrees, and is recommended when you want to specify object points as fractions of an angle, not fractions of a height.  If the object distance exceeds 10.E9, the program will treat the entering beam as a plane wave when calculating OPDs.  Since the object is specified as an angle rather than as a linear height, this form is useful for so-called F-theta lenses.  The distortion calculation automatically plots image height as a function of object angle.

 

If the lens with object OBD is to have a skew field defined (for use with MAP or GDIS, for example) the location of the object point that corresponds to a given (GBAR, HBAR) position is hard to visualize, since then we are dealing with spherical coordinates.  The best way to understand the geometry is to imagine the object as flat, with axes X and Y measuring angles in the XZ and YZ planes.  Take the spherical object surface, and stretch it flat.  The full-field chief ray makes an angle A from the axis.  Now, suppose the field is axially symmetric, and you wish to trace a ray from a skew field position that comes in at the same angle to the axis.  In the case of the linear object formats such as OBA, that field point could be found at (0.707. 0.707), which is at the same distance from the axis as the point (0,1.0).  In the case of OBD, that ray is described exactly the same way (0.707, 0.707).  This convention makes it simple to find and trace the ray you want.

 

The WAP 1 option is allowed for lenses with OBD object and is useful for modeling a wide-angle eyepiece from the eye point when there is no real stop in the system.  In this case, surface 1 is declared the stop, and if it is not actually at the eye point, the latter can be implied by a nonzero YPP1, which can be varied in optimization (with VY 1 YP1).  If the lens is corrected for distortion, the beam diameter is constant over the field and WAP 1 is then appropriate.  If the system actually has a stop somewhere else, the entrance pupil can realistically be found only with the WAP 2 or 3 options.  If you want to model a skew field (where GBAR is not zero) when the stop is on surface 1, you must declare it a real stop (APS -1) in order to achieve the correct pupil rotation.

 

Object form OBD also does not permit a HOE or DOE on surface 1; it uses a special ray-intercept formula that works for all angles, but for those shapes the intercept is actually on surface 2.  So add a dummy surface 1 if you want to use this object with that surface on the first element.  A DOE modeled as USS type 16 is permissible, since the intercept is on that surface itself.

 

OBF is identical to OBA except that it is intended for systems where the object beam is very fast.  In this case it is not appropriate to base a ray position in the entrance pupil on a linear fraction of the pupil height, as all of the other options do.  Instead, this quantity is converted into a fractional sine of an angle, and the ray height then follows what one would expect if the object beam satisfies the Abbe sine condition.  The effect is to produce a uniform ray distribution in the output beam of the lens.  Without this feature, the latter would be weighted unevenly, with more rays at the periphery of the pupil than at the center.  Both real and paraxial rays are affected.

 

OBG models a gaussian input, such as a laser beam.  Here the input is often very narrow, diffraction has a significant effect on the direction of propagation, and the intensity of the beam is not uniform.  See section 3.1.2.

 

If you want the system to be telecentric in object space, do not give an APS entry, and enter a value of YP1 that is equal to YP0.
