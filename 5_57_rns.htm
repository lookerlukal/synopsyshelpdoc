5.57 Recast NURBS or XNURBS Surface (RNS, RXN)

                                                 Previous Next TOC

 

 

When designing a system with NURBS surfaces (type USS 15), one specifies the number of control points in X and Y and the size of the array so defined.  It can happen that, as a design progresses, one would like to model the surface with a different array size or a different number of control points.  That is the function of the Recast Nurbs Surface (RNS) command.  Format is as follows:

 

RNS SN [ NX XMIN XMAX NY YMIN YMAX [ SYMM ] ]

 

Data are the same as described in Section 3.3.2.7,  The optional SYMM in word 9 sets term G 10 equal to 1.0, which signifies that the surface is symmetric about the Y-Z plane. The product of NX and NY must not exceed 81.

 

If the optional data are not entered, the program will automatically create the NURBS with a 9 x 9 array extending in X and Y by the current CAO value of the surface.  This action is taken automatically when the program creates a step file (see Section 7.2) and a surface is aspheric and rotationally symmetric.  (Non-symmetric aspheres or surfaces with non circular apertures are rendered with panels.)

 

The program calculates the sag of the original surface SN at each of the newly-defined control points and then redefines the surface with the new array data.  Then it submits an optimization run that varies all the control point Z-coordinates and targets the surface sag at those locations.  When this completes, the surface has been redefined and has a shape very close to that of the previous.  Then one can optimize the new system, varying the new control points as usual.

 

This feature can also be used to change an existing aspheric surface of any shape into a NURBS surface.

 

It can happen that a surface is so steep that the sag calculation fails at the corners of the square grid of points in X and Y.  When this occurs, the program deletes that target from the merit function and prints an informative message.  All of the control points remain variables, and in that case the number of variables exceeds the number of targets -- a situation usually to be avoided since the results are underconstrained and often exhibit pronounced ringing.  To reduce this effect, the program increases the initial damping, which tends to minimize the variable changes.

 

 

A similar feature can recast an existing surface into an extended XNURBS surface of type USS 24.  The command is

 

RXN SN NX XMIN XMAX NY YMIN YMAX { SYMM / 0 } SLOT

 

The arguments are the same as for the RNS command, with the addition of the SLOT entry in word 10.  This must be a number from 1 to 5.  For this command, all the data are required, and there is no default as is the case with RNS.
