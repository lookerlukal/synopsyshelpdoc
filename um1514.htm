15.1.4 Calculating with Al                                                        Previous Next TOC

 

Two features are available for doing simple calculations with AI.  The COMPOSITE method is compact but imposes strict format rules, while the method using the Zn parameters is free form.

 

15.1.4.1 Composite aberrations in AI

 

AI can evaluate a quantity in the format of a composite aberration, described in section 10.3.10.  In this case the idea is not to correct the aberration, but simply to print its value.  The command for this feature is

 

COMPOSITE

 

followed by the composite definitions entered in the format of an aberration definition.  For example, the input

 

COMPOSITE

CD1 TH 2

CD2 RAD 3

= CD1/CD2

 

would calculate the ratio of thickness number 2 to radius number 3.  There are nine CD's (CD1 to CD9), and this feature requires adherence to the input formats of an AANT file: the CD’s must be defined in order, and you cannot define them with an AI sentence.  I.e., “CD1 2 TH” would not work because that is not a valid AANT format (although AI itself allows more flexible input, the command interpreter does not).  See section 10.3.10 for a discussion of the formats of a COMPOSITE aberration.  Although this feature is less forgiving than the other AI features, it is more compact for some purposes.  It also can access any parameter that can go into a merit function, including some that are not otherwise known to the AI program.

 

To evaluate the difference between two ray coordinates, you might enter

 

COMPOSITE

CD1 2 YA 0 0 1 0 4

CD2 2 YA 0 0 1 0 2

= CD2 - CD1

 

This kind of sequence may be put into a MACro, but not into a multiline symbol, since the “=” sign must be in the first column of the input line, as in the case of aberration definitions – and multiline symbols retain their column placement.

 

As in the case of composite aberrations, the equation can involve the functions ABS, the SQRT and circular functions SIN, ASIN, COS, ACOS, TAN, ATAN, and the exponential and logarithmic functions EXP, ALOG, and ALOG10.  Angular arguments in every case are in units of radians, as in Fortran.

 

The results of the equation evaluation are displayed on the screen and automatically placed into FILE location 1.

 

When a COMPOSITE command is entered, all previously defined CDn's are deleted.  (If defined within an AANT file, they retain their values unless explicitly redefined.  See 15.1.4.)

 

If your composite definition is complicated, it helps to be able to print the values of all of the CDn parameters afterwards for troubleshooting.  This is easily done with the AI sentence

 

 SHOW CD.

 

Note that this feature uses the aberration calculation program that is also used by the optimization feature.  For this reason, any previously defined AANT file data are discarded when the composite data are calculated, and if you wish to continue a previous optimization, you must re-enter both the PANT and AANT files.  See the next section for a different format that leaves your AANT file intact.

 

Also, note that you may freely use any of the Zn parameters in your equation.  Those are handy when you just want to insert a numeric parameter, since you can vary that value with the VY NB VZN variable declaration in the PANT file.  This feature lets you do all kinds of parametric studies, should you even need that kind of power.

 

Note that you can also evaluate an equation with simpler input, using the Zn parameters instead of the CDn.  This is explained in the next section.
