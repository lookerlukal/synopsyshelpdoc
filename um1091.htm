10.9.1 Summary of optimization                                                Next Previous TOC

 

The command SUMMARY, following an optimization, will give a table of the value of the merit function at each iteration and the elapsed time at each iteration, for use when you wish to develop the most efficient merit function.  In the event that the optimization terminated for a reason other than reaching the maximum pass number or an optimum merit function, this command also lists the reason.

 

In the event you want to closely monitor the rate of improvement, there is a plotted form:

 

 SUMMARY P [ TIME / PASSES ].

 

The default is PASSES, which plots the log of the merit function against pass number.  The TIME request scales the abscissa in units of elapsed time, but otherwise draws the same curve.  Thus it is not strictly accurate unless each pass takes the same length of time, which is not always true.  A sample plot is shown below.
