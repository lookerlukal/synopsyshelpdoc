5.17 Object point search (OCALC, FCALC)        Next Previous TOC

 

It is sometimes necessary to find the point in object space that will be imaged at a particular location at the focal plane.  To do so, enter the command

 

 

OCALC XOUT YOUT

or

FCALC XOUT YOUT

 

where XOUT and YOUT are the desired image coordinates (angles, if the lens is in AFOCAL mode).  With OCALC, the program will adjust the values of the object coordinates YP0 and XP0 so that the chief ray from the object point HBAR = 1 and GBAR = 1 will trace to the desired image location.  Note that this feature is based on a real raytrace, unlike the system option FFIELD that adjusts YP0 to give a paraxial GIHT equal to the CAO on the image surface. OCALC will not work if FFIELD is in effect.

 

If you use the FCALC option, the program leaves the previous object definition intact but displays the values of GBAR and HBAR that are imaged at the requested point.  You can then use these data to analyze the image at that point, for example:

 

FCALC .255 .9

Z1 = FILE 1

Z2 = FILE 2

SPT M Z2 200 0 Z1

 

Here we find the object point that images onto (0.255, .9) at the image.  The AI commands next pick up the GBAR and HBAR coordinates that resulted and put them into variables Z1 and Z2 (See Section 15.1.4.2 for a description of the use of the Zn parameters, and 15.1.2.3 for a description of the FILE buffer.)  The spot diagram will then analyze the image at that point.

 

OCALC and FCALC do not work on lenses with object description OBG.
