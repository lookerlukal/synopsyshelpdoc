12.1.2 BTOL Adjustments                                                Next Previous TOC

 

Adjustments are changes that are made to the lens while BTOL is running in order to compensate for the ill effects of parameter errors.  With judicious selection they can relax the tolerances by a large amount.  The most basic is a focus adjustment, and no tolerance analysis is complete without it.  Often a paraxial thickness solve takes care of this automatically.  It goes without saying that to be realistic, you must plan on actually making exactly those adjustments in the shop as the lenses are later manufactured.

 

The simplest form of adjustment can be made directly within BTOL, using the input formats described in this section.  These execute very quickly and do a good job of accounting for the effects of an airspace adjustment or the tilt of an element.  A more complicated form allows you to execute a full-blown SYNOPSYS optimization as each variable is perturbed in BTOL.  Here is it possible to perform a sophisticated fabrication adjustment, and the improvement in image quality will be reflected in looser tolerances.  This is the REOPT option, which is more complicated to run and many times slower than the first.  It is described in Section 12.1.2.1.  Yet a third feature will emulate a series of fab adjustments and give the statistics of the whole process.  This is the most powerful of all, and is described in Section 12.5.3.

 

To help you find which parameters would be most useful as adjustments, BTOL has a special mode that searches for all reasonable combinations of group shifts along the axis and decenters in Y, and prints a table showing the effectiveness of each.  This is the ATABLE feature, described in Section 12.1.2.2.

 

BTOL can adjust up to four quantities while preparing the tolerance budget.  As each variable is incremented, the adjustment is performed in accordance with the stated goals, with the result that the adjustments are built into the derivative increments and the budget preparation can ignore them after the derivatives are calculated.

 

There is no optimization involved in this step; the adjustment goals simply specify that the spots at one or more field points should vary as little as possible from the nominal design.  The image is therefore refocused at one or more points, but aberrations are not rebalanced.  Also, the adjustments only involve the spot size, not the boundary targets given to MAG, DIS, or BORE, if any.  The adjustment applied to a lens that is part of a multiconfiguration tolerance analysis considers only the image in that configuration.  This means that if you want an adjustment to control the image quality in several configurations at once, you must use the reoptimize option described above.

 

 

Although you may request up to four adjustments, it is strongly recommend that you refrain from using adjustments that are redundant.  Curvature and thickness adjustments are only used to compensate for axially-symmetric parameters, while the others are only used to compensate for non-symmetric parameters, such as tilt and wedge.  But when two or more adjustments apply to the same parameters, solving for the best combined adjustment will often produce a singular matrix; i.e., the two do exactly the same thing.  SYNOPSYS can damp the results, preventing singularity, with the optional DAMP term in the BTOL command – but this has the effect of using the adjustments more or less equally, which is not often the best solution.

 

In a typical case, where two thickness adjustments are requested, it often happens that one of them is very useful for refocusing, while the other changes the aberration balance too much to be useful for this purpose.  If the first were the only adjustment, the tolerances might be quite loose, because the adjustment works.  If the second were added in addition to the first, the tolerances could actually be tighter, even though BTOL now has an additional parameter.  Although this is counterintuitive, it results from the fact that the lens is only refocused, not reoptimized, and the ill effects of the second adjustment are contaminating the good effects of the first.

 

If you are sure that an additional adjustment is going to be useful, you should consider the REOPTIMIZE feature, described in the next section.  Otherwise you are advised to experiment with the damping factor and run BTOL several times to find if the tolerances can actually be loosened in this way.  In fact this sometimes happens (I have seen a factor of two improvement!) – but bear in mind that the shop then has a far more difficult assembly problem, since they will in general have no way of knowing how to adjust several parameters at the same time.

 

You may specify that a given adjustment only be applied to perturbations of surfaces that are within a specified surface range, using the APPLY input.  This may be useful, for example, when you want to construct a system in sections, adjust each section separately, and then assemble the entire system later.  If you choose this option, be aware that you will not be able to run MC, since that feature is not structured for this kind of adjustment procedure: it always applies whatever adjustments are available, to the entire system.  If you want to use MC in such cases, you are advised to use the SEGMENT option instead.  The APPLY input must immediately follow the ADJUST request to which it is to apply.  Adjustments with no APPLY data automatically apply to the entire lens.

 

If your lens is a ZFILE zoom lens, you can specify how the adjustment is to apply.  If you request ALL, then the adjustment is actually calculated only at zoom 1, and that adjustment is used when evaluating the image quality at every zoom.  If you specify EACH, then a new adjustment is calculated at each zoom position before the quality is calculated.  The latter makes sense only if you can perform such an adjustment whenever the zoom setting is changed.

 

Adjustments cannot be assigned to surface 0 (the object surface).

 

Input is

[ APPLY JSSS JSPS ]

 

[ FOCUS { REAL HBAR GBAR / ALL / EACH } ]

 

where name is one of

 

RD

TH NSURF

THI NSURF

RAD

XDC

TILT NSURF

YDC

ADJUST SN CV [ AJMAX ]

 

gives the surface number of a curvature that is available for adjusting the ill effects of other tolerances.  It is implied in this case that the other variables will be fabricated to the tolerances in the budget, and measured, and a new value for CV then computed with the optimization program.  The resulting value for this curvature will then be substituted for the original value when the final lens is made.  The purpose of this and the other adjustments, described below, is to loosen up the tolerances of the other elements by permitting a reoptimization of the lens after manufacture.  The maximum permitted adjustment for correcting a single parameter error may be entered as AJMAX.  The default allows unbounded change in curvature, and the units are curvature. ADJUST RD and RAD are equivalent to ADJUST CV.  The surface to be adjusted must be spherical.

 

The way in which the program BTOL utilizes this and the other adjustments is covered in the section on FOCUS, below.

 

ADJUST SN TH NSURF [ AJMAX ]

 

 

gives the surface number of a thickness to be used for tolerance adjustment. NSURF is the number of surfaces that are to move as a group.  If omitted, only the given thickness or airspace dimension is altered; if entered, the thickness change is compensated by an opposite change of the thickness following the group.  Default maximum is 0.05 in.

 

ADJUST SN TILT NSURF [ AJMAX ]

 

specifies that a surface or element tilt is to be used to compensate for the ill effects of other surface or element tilts and decenters. NSURF is the number of surfaces to be tilted as a unit; the default is one.  The focal plane may be included as a tilt adjustment.  If the surface SN has a tilt or decenter prior to entering the BTOL program, the entry NSURF is ignored and the same number will be in effect as was before.  Tilt adjustments utilize the relative tilt feature, and are subject to the restrictions of that feature regarding the availability of the UNDO surface following the group.

 

Note that a surface specified as a tilt adjustment will not be examined for surface wedge or element tilt or decenter tolerances.  If you want the wedge to be toleranced, and the surface has a tilt adjustment, you must enclose the element in dummy surfaces and adjust the tilt of the whole group.  The surfaces within the group will then be toleranced as usual.

 

Also note that a tilt adjustment is used only to control image quality degradation via the FOCUS command, not for boresight, magnification, or distortion correction.  This compensation is applied only to tolerances of non-rotationally symmetric errors; i.e., a tilt adjustment is not used to compensate for a radius error.  The default maximum is 1 degree; AJMAX should be input in degrees.

 

ADJUST SN DECENTER NSURF [ AJMAX ]

 

 

specifies a surface or element decenter adjustment in the Y-direction. The comments above regarding TILT apply also.  Default maximum is 0.05 in.

 

Note that for a centered non-tilted spherical surface, tilt and decenter are nearly equivalent, since decentering a curved surface changes the slope angle at the axis.  For this reason you should not adjust both a tilt and decenter of a surface, and if one is adjusted the other should not be toleranced.  If two or more adjustments have the same effect on the refocusing adjustment, they can produce a singular correction matrix, which is solved linearly, with optional damping.  The result can be unsatisfactory, and you are advised to eliminate one of the adjustments in such a case before proceeding.

 

ADJUST SN { XDC or YDC }

 

 

specifies the decentration direction.

 

TH0

It is possible to adjust the object distance to maintain focus in BTOL.  The surface number is then irrelevant, and you would use

 

ADJUST 0 TH0 …

 

Be warned, however, that when you vary the object distance in a lens with a fixed YMP1 (usually the case), the F/number in object space changes with TH0.  If this F/number is large, the effects are minor – but with a fast object, like a microscope objective traced from the short conjugate, this can be a major perturbation to the lens.  In this case, one should use the OSNA option, which adjusts the value of YMP1, keeping the numerical aperture constant.

 

ACCOM

This will adjust the afocal accommodation.  The lens must be in AFOCAL mode already, but it does not have to be accommodated.  Any surface number may be given.

 

Note that MC cannot adjust the accomodation, since it is a parameter, not a variable, for the optimization.  If you want to use MC to evaluate your tolerances afterwards, then instead of this adjustment, simply put the lens in ACCOMODATE mode and add a YMT solve on the next-to-last surface.  The final surface will retain the curvature set by the original ACCOM, but the distance to that surface will adjust itself to the paraxial focus.  Then do not declare the ACCOM as an adjustment.

 

ZDATA

This will adjust the position of a group in a ZFILE zoom lens.  In this case, SN input refers to the group number instead of a surface number.  The adjustment will automatically be applied to each zoom, since it is intended to model the common procedure where one adjusts one of the groups on a lens bench as the lens is assembled, and then cuts the cam curve for that group to the measured best position.  In this scenario it makes little sense to apply an adjustment of a given group in a given zoom to a different zoom, so word 5 of the input is ignored.

 

Since the ZDATA adjustment necessarily only applies to zooms 2 and higher, the program cannot use it to adjust zoom 1 -- which is usually desired as well.  So it assigns a Z-decenter to the zoom group, and varies the ZDC in zoom 1, while using the zoom position adjustment in the other zooms.  The result is a budget that reflects the above procedure and can be analyzed by MC.  In that program, the adjustment is divided into ZDC for zoom 1 and the ZDATA adjustment for higher zooms.

 

Note that, if the ZDATA adjustment is requested for a zoom lens, the program is restricted to systems with no more than 190 surfaces.

 

ZAW

gives the weighting of the various zoom positions during the adjustment calculation. FNUM weights the zooms by 1 / FNUM, and NB WT gives the weight at that zoom. UNIFORM weights all zoom positions equally.

 

FOCUS …

 

 

In order to utilize the adjustments given by the ADJUST input described above, there must be a specified adjustment goal.  Many lenses include paraxial pickups and solves as part of the normal lens description, and these are automatically honored by BTOL.  If a lens already has a paraxial solve on the final airspace, it is not necessary to specify an adjustment parameter or a goal, since the solve performs the function automatically.

 

Only a single FOCUS entry is permitted.

 

FOCUS REAL

 

 

gives a field angle at which a focus adjustment is to be performed.  The goal is to compensate for the defocus caused by each parameter as it is evaluated by BTOL, not to reoptimize the lens or rebalance aberrations.  The program traces four rays at the 0.1 zone in the primary color (and three more in the short color if this wavelength is defined) and changes the specified adjustment parameter so as to maintain the transverse intercepts of these rays as nearly constant as possible, measured from the chief ray intercept.  The effect is nearly the same as a paraxial focus goal for the on-axis image, but may be specified off axis if desired.  Any field point may be requested, and it need not be the same as any of the points at which the quality descriptors are defined.

 

Do not specify a real-ray focus goal on axis and a paraxial focus solve simultaneously.  These calculations control very nearly the same thing and satisfying them both can produce bizarre changes in the lens.

 

FOCUS ALL

 

specifies that the change in focus for all field points for which a quality descriptor has been specified (with the FOV input) will be controlled simultaneously by the adjusting variables.  The relative weighting of each field point is considered in finding the best compromise focus.

 

FOCUS EACH

 

 

specifies that each field point is to be adjusted independently with the allowed variables.  This situation would be appropriate, for example, for an eyepiece where the user could focus his eye at each point in the field as needed.

 

 

Be aware that there is a utility that can adjust the orientation of individual lens elements in their cell in order to minimize the effects of element wedge.  This is not part of BTOL or MC, but runs as a separate command.  Read about UCLOCK in Section 5.38. MC can also evaluate the statistics of a batch of lenses that will be clocked at assembly.
