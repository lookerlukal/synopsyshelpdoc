9.1.2 Image model coordinate systems                        Next Previous TOC

 

Except for the case described below, the origin of coordinates of single-color models is the principal ray point in the requested color.  For images created with the M option it is the chief-ray point in the primary color.  A detector at coordinates (0,0) will be centered at this point, and images in other colors will be decentered by an amount equal to the lateral chromatic aberration of that field point.  If the image is defined by an image patch, the (0,0) position is the center of the patch.

 

If the optional AXIS entry appears in word 8 of the command (which is valid only for the GMODEL), the center of the image model coordinate system is on the axis of the image surface instead of at the principal ray.  Another option, also available only with GMODEL, lets you combine several geometric images at different field points and or colors.  Subsequent analysis will then examine the ensemble of all of the images so created, and this makes sense only if all of the models use the same coordinate system.

 

To use this option, each GMODEL must specify only a single color (not the multicolor M option), the first GMODEL command should include 0 AXIS in words 7 and 8, and all but the first must contain the entries ADD AXIS in words 7 and 8 of the command.  The relative weighting of each image model is entered with the WT input, which is only used when combining images and can be zero otherwise.  Each model will be added to the composite image structure, referenced to the optical axis.  You may request as many image models as you wish to combine in this manner.  (This option requires command-mode input, and is not found in the image model dialog MMO.)
