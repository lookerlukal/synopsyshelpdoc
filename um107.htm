10.7 ZOOM Lenses, Multiconfigurations                                Next Previous TOC

 

Two families of features are available for optimizing zoom lenses.  The first applies only to true zoom lenses, where only the object parameters and surface spacings are allowed to zoom.  A lens using this feature is called a ZFILE zoom lens.  The feature is relatively simple to use, requires only a single configuration for optimization (see below), and has the advantage that the zoom information is part of the lens description itself, and follows the lens when it is stored or saved.  Up to ten zoom positions may be defined in this way.  This type of zoom lens is recommended unless you absolutely need other parameters to change besides the element spacings.  In that case, the second family, the multiconfiguration feature, is required.

 

The multiconfiguration feature lets you specify up to six completely independent lenses in the form of alternate configurations, or ACONs, all of which may be optimized at the same time.  Input formats are provided to tie the parameters of one configuration to those of another, so for example all of the radii and thicknesses could be identical in the several configurations and would remain so during optimization.  Although the ZFILE form would generally be preferred when designing a true zoom lens, if there are any parameters that differ besides airspaces, such as when lens elements are flipped into or out of a system to change magnification, the ZFILE form is not suitable, and the alternate configuration, or ACON form, must be used.  It is also required for more complex tasks, such as modeling the HOE construction optics at the same time the HOE system is optimized. ACON zoom lenses are in effect separate lenses, and there is no information in one that ties it to any of the others, except during optimization.  The ZFILE zoom option is described first.
