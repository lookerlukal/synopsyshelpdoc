Version 16.010

 

The update to version 16.010 incorporates the following changes:

 

Features:

 

1.  A new options is available in ZSEARCH.  The entry PRISM PTH will add a block of glass at the end of the zoom lens, assigned a glass model equal to type N-BK7 with thickness PTH.  This is intended for when the lens will be followed by some kind of prism whose thickness will contribute aberrations that should be accounted for during the search.  When the search is complete, you can then redefine that block of glass as whatever prism you want.  (Note that a similar capability is also available in DSEARCH, where it is called COVER.)

Version 16.009

 

The update to version 16.009 incorporates the following changes:

 

Features:

 

1.  A new operand is available.  IPVAR will target the wavefront variance, taking the image centroid as the OPD reference point.  It thus differs from GNV, which takes the chief-ray point as the reference.  It also differs in format:  This form requires the M tar wt ... format as do ray aberrations.

 

2.  The CFIX command has been enhanced.  CFIX {C/E/R} will redefine all current hard apertures, making them either circular, elliptical, or rectangular.  It is intended for when FFBUILD requires noncircular apertures via the XCAP directive, and you want to change the definitions afterward.

 

3.  A new option is available for ZFILE zoom lenses.  If ZCSTOP is included in the ZFILE definitions, and CSTOP has been declared in the lens file, then the hard CAO on the stop will be recalculated at every zoom position according to the ray path in that zoom.  (Without this declaraion, a CSTOP aperture is calculated only at zoom 1 and applies to all zooms.)

 

Bugs Fixed:

 

1.  The MSS dialog and the User's Manual had different listings for some switches.

Version 16.005

 

The update to version 16.005 incorporates the following changes:

 

Features:

 

1.  A new monitor is available.  ALA will control the CAO of all lenses, ignoring mirrors.  (The AAC monitor applies to the CAO of both lenses and mirrors.)

Version 15.97

 

The update to version 15.97 incorporates the following changes:

 

Features:

 

1.  SSU and RSU will now accept a number and only operate on that configuration it is entered.

 

2.  A new tolerance desensitization operand is available.  SDU will evaluate the sum of the squares of the change in paraxial angle UA at each surface.  If this is large, then even if it is compensated elsewhere, the surfaces will have tight tolerances.

 

3.  Operands have also been defined for RUD (the difference between the incidence angle of the full-field upper and lower rim rays), which is squared and may help reduce contributions to off-axis aberrations, and DAA, the difference in SA3 contributions at adjacent surfaces.  If this is large, then tolerances may also be tight on those surfaces,

 

4.  A new command, RUDIFF, calculates the difference in angle between the upper and lower rim rays, compared to the surface normal, in radians, and prints the sum-squared value.

 

5.  The optimization selection QPLUS now adds an additional ray at XC 1 1 0.  This will avoid failures when the meridional rays will trace but a sagittal ray will not.

 

Bugs Fixed:

 

1.  The command LLB, which loads the last 20 commands into an editor, did not work properly.

 

2.  The ZSPACE option in ZSEARCH did not work as it should.  It now works better after applying a much higher damping.

 

3.  If you changed ACONs and varied the GLM in the new one, the program tested for a glass model in ACON 1.

 

Bugs to be Fixed:

 

1.  The PAD scan feature does not work with a zoom lens that has been refocused via the ZFOCUS command.

Version 15.89

The update to version 15.44 incorporates the following changes:

Version 15.30

 

The update to version 15.30 incorporates the following changes:

 

Features:

 

1.  SYNOPSYS can now model certain environmental changes to a lens system, including surface distortions and index changes within a lens.  This is an improvement over the TOSP evaluation, which models those changes on a differential basis but is only honored by a few of the image analysis routines. ENM, as the new feature is called, implements the changes with Zernike polynomials on the surface and a special GRIN designation for the index, so the changes are honored everywhere.  This is intended for the case where a lens is hotter on one side than the other, for example.

 

Input for this feature is a set of coefficients for a power-series expansion of the parameters affected.  A future update will accept an array of data points from a third-party program such as ASAP or NASTRAN, but currently we do not have an example of data from those programs.  If any users can provide those examples, we will add that capability too.

 

2.  The precision of the RZS calculation has been enhanced.  Now the result should be a more exact fit to the nominal surface.

 

3.  The display of numeric data on graphical output has been changed.  Now it will display small numbers with E-format instead of F.  This means you will see more decimals of precision, which seems like a good idea.

 

4.  A new feature is available in BTOL.  To enhance finding tolerances for aspheric surfaces, we have implemented a new declaration, PFTEST, which is appropriate when a surface will be tested with a profilometer rather than a null test.  In the latter case, one wants the tolerance of the fringes one sees in an interferometer, and those are given by the irregularity and rolled-edge tolerances.  But profilometers do not give a set of fringes, instead producing a table of sag values over the surface.  With those data, one can deduce the standard deviation of the surface compared to the ideal shape, and that is the number that can now be toleranced.  This declaration can be given for aspherics defined with either a power-series expansion or Zernike polynomials.  If it is not declared, the surface will be toleranced as before, which is appropriate when one can examine fringes.

 

5.  You can now assign EFILE edges to Zernike surfaces.  This only makes sense if the surface is rotationally symmetric, but the previous release would not let you do it even then.  Now you can.

 

6. ADEF has been improved.  Before, it found the sphere that best fit an asphere by examining the sag only along the Y-Z meridian.  Now it evaluates a 21-21 grid of points over the entire surface.  ADEF sn PLOT now displays data from the bottom to the top of the aperture, instead of only from the center to the top as before.  Also, the input has been simplified; there is no provision for specifying the range examined anymore, since it is always the entire surface now.  (If you have any MACros with ADEF commands, you will have to revise them according to the new format.)

Version 15.27

 

The update to version 15.27 incorporates the following changes:

 

Features:

5.  ZSEARCH now accepts a ZFOCUS declaration with the input goals, allowing you to find a zoom lens that works at two different object distances.

 

6.  The maximum number of aberrations allowed in the AANT file is increased to 15000.

Version 15.6

 

The update to version 15.6 incorporates the following changes:

 

Features:

 

1.  A new feature in the free-form surface analysis routine (FFA) now shows the fringes in double pass relative to a given reference sphere.  This should greatly help in testing these unusual shapes.

 

2.  The Foucault test routine MFK now has a Ronchi option that shows the pattern when a ronchi grating is present instead of the usual knife edge.

 

3.  A new AANT file entry GDR will target the locations of a grid of rays at the image.  This is intended to control distortion in systems with a skew field, where image points must be controlled in both X and Y.

 

4.  A new search routine is available. AGT will test selected aspheric terms on a given surface and retain only those that make a desired improvement in the merit function.  If you want to know whether adding general aspheric terms is a good idea, this will tell you.

 

5.  The default CAO routine has been enhanced.  Now you can enter the system declarations XCAP and YCAP.  The former makes default CAOs become RAOs (rectangular apertures), with sizes found by tracing rays at the top, bottom and sides of the field of view.  It is intended for strip-field or free-form systems, where the portion of the lenses that is actually used can be strongly non-circular.  YCAP only considers the meridional field and returns circular apertures.  (The default setting returns circular CAOs based on the diagonal extent of a rectangular field.)

 

Bugs Fixed:

 

1.  The FFA analysis did not work if the system was in OPD mode.

2.  Some problems emerged related to multithread operation, including ray failures for some aspherics.

3.  A surface assigned a mirror alpha tilt (MAT) plus a Y-decenter did not come back correctly after a SAVE.  This geometry is more easily set up as a simple change of TH, and the combination is no longer allowed.
