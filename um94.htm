9.4 Diffraction Program (DIFF)                        Next Previous TOC

 

The program DIFF is similar to the PSPRD program available in IMAGE except that it is not limited to the 5.74-Airy-disk sample region of the latter, and it produces a diffraction knife-edge trace or a table of local energy density at many sample points in the image, as well. PSPRD calculates the diffraction intensity at sample intervals of one-half the radius of the Airy disk (or one-fourth, depending on the setting of switch 55) and employs an interpolation algorithm to plot the complete image, while DIFF calculates every point in the image exactly.  For this reason, DIFF takes longer to run than PSPRD.

 

A table of energy densities may be requested in two ways: if switch 36 is turned on before running DIFF, the program will print a table giving energy densities in a central slice through the image.  Normally, you will specify that the image is to be calculated (and plotted) at many points in a two-dimensional array of samples.  But in the event that only a single column is requested (SLICES = 1), the program will assume that the image has circular symmetry for normalizing the energy calculation.  This allows you to obtain the energy at far off-axis points without examining a greater area of the image.

 

If switch 51 is on, the program will print a table giving the image intensity at each sample point on the plot.  If these data are captured into a disk file (see Section 13.10), they may be read by a user-written program for additional image assessment.

 

DIFF has two modes of operation: in one, a polynomial is calculated (with COE) to describe the wavefront OPDs, and the diffraction integral is evaluated by interpolation over the pupil.  In the other mode, a collection of OPDs is calculated exactly with DPF.  The COE option is faster.  Both options operate on only a single wavelength at a time.

The COE feature has been used to analyze an image that had a large amount of tangential astigmatism but was otherwise diffraction limited, and was designed to fall on a narrow slit. Ordinarily it is impossible to analyze such an image with the diffraction programs, so a special feature was installed: by placing switch 19 on prior to the DIFF input the term G(5) is set to zero.

 

The input required to create the coefficients and to run DIFF in COE mode is as follows:

 

COE { ICOL / P } HBAR 0 0 GBAR

DIFF REGION SLICES SAMPLES HT NPUP [ LOG ]

[ KEEP / COMBINE ]

END

 

where

 

REGION

is the region about the chief ray intercept to be included in the output, measured in Airy disk diameters. (For REGION >20, running time can become long.)

 

SLICES

is the number of horizontal lines to be used in drawing the point-spread function.

 

SAMPLES

is the number of discrete data points per line, giving the horizontal resolution.  Note: maximum SLICES * SAMPLES = 5000, maximum SAMPLES = 2000.

 

HT

is the height, in inches, of the peak value of the calculated point spread function in the central two lines.

 

NPUP

is the array size into which the pupil is to be partitioned to generate the point spread function.  For a 10x10 partition, for example, NPUP = 10.  This input applies only when using the COE … option; the ray number is explicitly entered with the DPF input, if that option is selected, and NPUP must then be -1.

 

For best accuracy, you want the total number of rays traced to be close to, but not more than, 20,000.  If the system is obscured, you may have to experiment with NPUP to find the value that works best.  If the aperture is not obscured, a value of 31 is appropriate.

 

LOG

This input calculates the logarithm to the base 10 of the individual intensities.  When plotted in this way, you can see faint structure that otherwise would not be visible.  An example is given in the next section.

 

To use the vignetted exit pupil created with DPF (with no interpolation), the input is:

 

DPF { ICOL / P } HBAR NRYS 0 GBAR

DIFF REGION SLICES SAMPLES HT –1 [LOG]

 

Both the DPF and COE methods will honor any OBS input.  DPF, because it traces every ray exactly, will also honor the CAl data.

 

An additional feature of DIFF will calculate the relative efficiency if the diffraction pattern were used in a heterodyne detection device such as a laser radar.  After the image is obtained, DIFF prints two numbers identified as "SUM I, SUM I * * 2" which give, for all of the sample points over the image, the sum of all of the individual intensities and the sum of the squares of these values.  Experts in this field can utilize this information to estimate the relative efficiency of the device compared to a perfect system.  The total energy is normalized to equal the total transmission of the optics, with vignetting and apodization (but not absorption) considered.  If the user enters the command

 

KEEP

 

following the DIFF input, the current image is stored.  If a second image is later computed, you may enter the command

 

COMBINE

 

following the second DIFF command, and the current image will be combined (mathematically, not graphically) with the stored one.  The values of I**2 then refer to the products of the two local intensities while the sum of the I's gives the total intensities of the two, divided by two.

 

Note that this calculation is meaningful only if the sample points in each image are identical, and for this reason an alternate form of input for REGION is permitted: if a negative number is entered for this quantity, it refers to the actual lens units of the sample area, measured from the chief-ray point in the requested color, rather than the sample size in Airy disk units.

 

DIFF may also be accessed via the MDF dialog.

 

If you need a multicolor slice through a diffraction image, use the feature MIT, which can create a variety of images of a variety of targets and plot the central slice in X and Y.
