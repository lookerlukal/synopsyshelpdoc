5.47 Automatic Real Glass Insertion (ARGLASS)         Previous Next TOC

 

Several routines in SYNOPSYS™ can change glass types or insert real glasses into a lens. IRG can assign a desired glass to a specified surface or automatically pick the closest one from a given glass table, and GSEARCH can try all permutations of a list of candidate glasses to find the best combination.

 

The feature described in this section will automatically replace any or all of the current glass model glasses with real glasses, subject to certain preferences.  It can only be run after a normal optimization run, since it will repeat that optimization with the same merit function and the same variables – except for any GLM, GBC, or GBF variable on the surface under investigation, which will be deleted first.  It can also be run from the dialog MRG.  If the optimization job was run in multiconfiguration mode, the main configuration must be in ACON 1, since ARG will restore that configuration before optimizing the candidate glasses in that case.

 

Input is:

 

ARGLASS LIB [ QUIET ]

[ SEQUENTIAL / SORT / REVERSE ]

CAT { S / O / H / F / G / R / M / U / N }

[ NRM ]

INCLUDE JSSS [ TO ] JSPS

EXCLUDE SN SN SN …

[ PRICE PTIMES ]

[ BUBBLE BLIMIT ]

[ STAIN SLIMIT ]

[ ACID ALIMIT ]

[ ALKALI ALLIMIT ]

[ HUMIDITY HLIMIT ]

[ PREFERRED ]

[ SAFE ]

[ ILINE ]

[ ANNEAL TEMP DELTA ]

GO

 

LIB

This gives a library location where the program will save the starting lens, and then  the final lens, with real glasses inserted, when the process is finished.

 

SEQUENTIAL / SORT / REVERSE

This option sets the order in which glass models are replaced.

 

SEQUENTIAL matches them starting from the first element and ending at the last. REVERSE does the same, in the opposite order.

 

SORT examines the current model glasses and adjusts the order so that the model whose closest qualifying real glass is furthest from the model will be matched first.  This will allow other glass variables to compensate for the disruption when that model is replaced by a glass that is not very close. When the other models are later matched in turn, presumably there will be less need for more variables since they are closer to a real glass and the lens is therefore changed by a smaller amount.

 

We have found that sorting the glass types sometimes, but not always, yields better results.  You are encouraged to try all three options and select whichever works best.

 

The default is SORT.

 

CAT

Specifies the desired glass catalog, which must be one of S for Schott, O for Ohara, H for Hoya, F for Corning France, G for Guangming, R for Russian LZOS,  M for Sumita, U for the unusual catalog, or N for the Nicon catalog.  It does not work with the C (custom)  catalog.

 

If the U catalog is requested, the program will treat the PREFERRED designation as a request to match only plastic materials, and only surfaces marked PLASTIC in the RLE file will be matched.  If any other catalog is requested, surfaces so marked will automatically be excluded.

 

NRM

This optional entry (No Range Message) will disable display of the error message when the useful range of a material does not encompass the wavelength range of the lens.  If you are matching plastic elements to the U catalog, for example, and the ranges are slightly different, it is no matter and can be ignored.  When optimizing with switch 99 on, every instance of SYNOPSYS can show the message, which is then a nuisance.  This option will turn it off.

 

INCLUDE

Specifies the range of surfaces to be treated; the default is the entire lens.  Only one INCLUDE directive may be entered.

 

EXCLUDE

Gives a list of surfaces you do not want matched to real glasses.  More than one EXCLUDE directive is allowed

 

PRICE

Here you give an upper limit to the price of the real glass, in multiples of the price of glass type N-BK7.

 

BUBBLE

This is the maximum bubble class you will accept

 

STAIN

This gives the maximum stain class you will accept.

 

ACID

 

The maximum acid class.

ALKALI

The maximum alkali class.

 

HUMIDITY

The maximum humidity sensitivity class.

 

PREF

Enter this option if you want to match only preferred glass types.

 

SAFE

Use this directive to match only environmentally safe glasses (without lead or arsenic).

 

ILINE

This filter will only match glass types identified by the manufacturer as suitable for use in the ultraviolet, where many other glass types darken with prolonged exposure.  At present, only the Ohara catalog has a sufficient quantity of such types to be useful.  Most of the other catalogs have few or none.

 

ANNEAL

Here you can request that each case should be annealed at the specified temperature and cooling rate.

 

GO

Begins the matching process.

 

The program starts at the beginning of the range JSSS to JSPS, identifies the next surface presently assigned a glass model (GLM), finds the closest glass in the specified catalog that meets the selection criteria, and reoptimizes the lens after adding targets to the merit function that move the GLM parameters close to the selected real glass.  That step should reduce the likelihood of ray failures when the glass is inserted in place of the model.  Then it replaces the glass model with that glass type, removes the glass model variable on that surface, and runs the optimization program again.  Then it loops on the next candidate surface, continuing until all requested surfaces have been matched.

 

It is an error to run ARGLASS without previously running an optimization run, and it is only intended for systems operating in the visible, where the glass model is valid.

 

If the selection criteria are too restrictive, the program might fail to find any good solutions.  Many glasses are not assigned all of the named attributes by the vendor, and in those cases the catalog has a value of zero for the missing data.  Such glasses will therefore pass the tests of ARGLASS, even though the glass itself would perhaps not qualify if its true attributes were known.  It is therefore recommended that you verify with the vendor the values of the attribute(s) that are important to your project.

 

The U catalog (unusual materials) is treated somewhat differently from the others.  Those materials do not have data such as price, acid sensitivity, etc., but since that catalog contains all of the plastic materials, which are useful for molded optics, that one is supported as well.  In this case the Preferred designation is automatically selected and singles out plastics instead of the usual availability level.  All GLM materials will be matched to plastics in this mode, and if you want specific elements to not be matched, if they are to be made of optical glass, for example, you have to list the surface numbers of the model in the EXCLUDE section.

 

To illustrate the use of this feature, we will start from nothing and design a seven-element lens with DSEARCH.  Then we use ARGLASS to substitute real glasses for the models returned by DSEARCH.

 

First, create a DSEARCH MACro to find an attractive lens form.

 

ON 998

CORE 13        ! use multicores if you have them

QUIET

 

DSEARCH 5  QUIET

SYSTEM

ID DSEARCH SAMPLE

OBB 0 20 12.7

WAVL 0.6563 0.5876 0.4861

 

UNITS MM

END

GOALS

ELEMENTS 7

FNUM 3.57

BACK 50 0

TOTL 0 0

STOP MIDDLE

STOP FREE

RT 0.5

FOV 0.0 0.5 .7 1

FWT 10 5.5 5.5 3

NPASS 40

ANNEAL 0 0

SNAPSHOT 10

QUICK 20 40

END

SPECIAL PANT

 CUL 1.9

 FUL 1.9

END

GO

LOUD

 

When this finishes, we edit the optimization file that DSEARCH has prepared, adding the ARGLASS input at the bottom, increasing the number of passes requested, and then requesting an MTF analysis over the field:

 

PANT

 CUL 1.9                                                                                

 FUL 1.9                                                                                

VY 0 YP1

VLIST RD ALL

VLIST TH ALL

VY   1 GLM

VY   3 GLM

VY   5 GLM

VY   7 GLM

VY   9 GLM

VY  11 GLM

VY  13 GLM

END

AANT P

AEC

ACC

GSR     0.500000     5.000000      4  2    0.000000

GNR     0.500000     3.000000      4  2    0.750000

GNR     0.500000     1.000000      4  2    1.000000

GSR     0.500000     5.000000      4  1    0.000000

GNR     0.500000     3.000000      4  1    0.750000

GNR     0.500000     1.000000      4  1    1.000000

GSR     0.500000     5.000000      4  3    0.000000

GNR     0.500000     3.000000      4  3    0.750000

GNR     0.500000     1.000000      4  3    1.000000

M   0.500000E+02  0.100000E+01 A BACK

END

SNAP/DAMP 1

SYNOPSYS 100

STORE 3

 

ARGLASS 3 QUIET                ! Start of ARGLASS input.

CAT S                                ! Specify the Schott glass catalog.

INCLUDE 1 TO 13              ! Do all surfaces.

PREF                               ! Only use preferred glass types

SAFE                                ! and environmentally save glasses.

GO    

 

MOF M 0 40 80 0 Q 30 20 10      ! Calculate the MTF over field.

 

We run this file on the best of the DSEARCH results that came back, and end up with this design, complete with real glasses.  The entire process required only about a minute of computer time.

 

 

 

These excellent results came about partly because we ran the job several times, adjusting some of the input parameters to find the best result.  Powerful as the program is, there is still room in this world for human judgment.  That’s your job as a lens designer; the software lets you make tradeoffs, but it does not do your job for you.  It will make it much easier, faster, and fun, however.
