3.3.2.7 Unusual Surface Shapes (USS)                Previous Next TOC

 

(These options go into an RLE or CHG file, and may be entered interactively with the dialog SPS or the WorkSheet.)

 

SYNOPSYS currently has several types of unusual surface shapes.  The description is always entered with a set of coefficients in the following formats:

 

 SN USS TYPE

 SN DC1 …

or        SN AT1 …

or        SN XDD …

 

The TYPE must equal one of the numbers in the list below – and determines which of the shapes the other terms refer to.  The input formats are defined in Section 3.3.2.2.2.

 

Most of these forms utilize the 17 G-terms, which can be entered in the same manner as for normal power-series aspherics with either the DCn form or ATn input forms.   Types 8, 12, 14, 15, 16, 20 and 21 require additional data, and in this case the XDD format is required, described below for that type.  The XDD form may also be used for any of the other USS types.

 

All of the USS shapes contain a full description of the shape; the Z terms apply to an otherwise flat substrate.  In some cases those terms also specify a curvature plus a conic constant which apply in addition to the various power-series coefficients. Note that, if you use any of the USS shapes, you do not specify a RAD or CV as usual. The shape is described entirely by the G terms.

Type 1; conic plus power series:

 

This is a common power-series shape with terms that can go up to the 28th power of the aperture.

 

G1 is the axial curvature

G2 is the conic constant

G3 is the 2nd power aspheric term

G4 is the 4th power term, and so on up to G16, which is the 28th power term.

 

 

Type 2; General power series with cross terms:

 

This is a simple form of a general power-series with cross terms, up to the fourth power of aperture.

 

Z = G(1) + G(2)*X + G(3)*Y  + G(4)*X**2 + G(5)*XY + G(6)*Y**2 + G(7)*X**3 + G(8)*X**2Y + G(9)*XY**2 + G(10)*Y**3 + G(11)*X**4 + G(12)*X**3Y + G(13)*X**2Y**2 + G(14)*XY**3 + G(15)*Y**4

 

 

Type 3; Sawtooth Fresnel, constant Y separation:

 

This is a sawtooth Fresnel surface where the zone width is entered in term G16, and the zigzag shape of the surface is explicitly modeled.  The other terms are the same as for type 1.  The substrate of this kind of Fresnel surface is flat.  See section 3.3.2.3 for another kind of Fresnel specification that may have an arbitrary substrate shape but does not model individual zones.  The zones of type 3 have flat slopes, modeling the polynomial at the breakpoints only.  To model a Fresnel surface whose base is not flat, use type USS 23.

 

 

Be aware that if your lens contains a Fresnel surface of types 3, 4, or 23, there will be zones where rays cannot get through.  In the illustration below, a ray hits the surface right at the edge between two facets.  The raytrace program cannot evaluate the path of such a ray, and it will fail with an MCS error.  If you see holes in your TFAN plots, this is the reason why.

 

 

One should also be aware that when Fresnel lenses are manufactured, the precision of molding is not adequate to maintain phase coherence of the light from adjacent zones.  The result is that over the aperture of the beam the phase is rather chaotic, and one cannot reliably compute the OPD of any given ray.  The program will generate a value if requested, but this is probably of little meaning in practical cases.

 

 

Type 4; Sawtooth Fresnel with constant sag:

 

This type is identical to type 3, except that the zones are given equal sag heights, not equal zone widths, and the zones have curved slopes, modeling the polynomial exactly.  Term G 16 in this case gives the zone height, not the width as in type USS 3.

 

 

The remarks given above for type 3 regarding ray failures and OPD evaluation apply to this type as well.

 

 

Type 5; Power series; symmetric in X:

 

This type uses a polynomial with higher powers, some odd powers of Y, that is symmetric in X.  The form is

 

Z = G(1) + G(2)*Y + G(3)*Y**2  + G(4)*Y**3 + G(5)*Y**4 + G(6)*X**2 + G(7)*X**2Y + G(8)*X**2Y**2 + G(9)*X**2Y**3 + G(10)*X**2Y**4+ G(11)*X**4 + G(12)*X**4*Y + G(13)*X**4*Y**2 + G(14)*X**4*Y**3 + G(15)*X**4Y**4

 

 

Type 6; Conic plus cosine ripple:

 

This type describes the surface with a curvature plus conic constant, and adds a deformation that generates cosine-wave shaped zones, equally spaced across the surface.  This may be used to model some common manufacturing errors.  The variables are:

 

G 1 axial curvature

G 2 conic constant

G 3 value A

G 4 value B

 

Here, the sag is given by

 

Z = (conic contribution) + A cos(Bρ)

 

where ρ**2 = X**2 + Y**2.

 

Thus, to model a wavy surface on the primary mirror of the system saved in file 4.RLE (a 16-inch relay telescope) one could enter

 

GET 4

CHG

1 NULL

1 USS 6

1 AT1 -1.011634E-2 0 5.0E-6 1.5708

END

 

PANT

VY 0 BTH

END

AANT

M 0 1 A P YA 0 0 1

END

SYNO 5

 

In this input, we want to model two cycles of a wave of amplitude 5.0e-6 inches from the center to the edge of the aperture.  The axial curvature is nominally 0.01011634 and the conic constant is zero.  To get two cycles at ρ = 8.0, value B must be PI/2, or 1.5708.  This form introduces a small curvature at the center of the surface, which then oscillates up and down out to the edge, and we have to adjust the focus slightly, with the PANT and AANT files.

 

Next we find the effective curvature of the mirror with

 

 1 CV ?

 

and the answer comes back –0.01012127, which is slightly shifted from the nominal, as expected.  Now we can see the effect on the wavefront with

 

FRINGES

PUPIL P 0 0 0 0 3.

 

 

 

 

Type 7; Conic plus odd and even powers:

 

This type is similar to type 1 with the important exception that it involves odd as well as even powers of the aperture.   R in this expansion is SQRT(X**2 + Y**2), and it is always positive.  The resulting shape is rotationally symmetric about the optical axis.

 

The focal length of this kind of surface is zero if term G3 is nonzero, since in that case the curvature is infinite at the axis.  Since it makes no sense to consider that term in the paraxial raytrace, it uses only the G1 and G4 terms to calculate the power of the surface.

G 2 conic constant

G 3 R

G 4 R**2

G 5 R**3

G 6 R**4

G 7 R**5

G 8 R**6

G 9 R**7

G 10 R**8

In this table, R = sqrt(X**2 + Y**2).  Since one ordinarily wants the two curves to meet each other at the zone radius, the program calculates a sag offset for the outer zone that is added to the intrinsic sag of that zone at the given radius so that this is the case.  The offset is placed, for reference, in data XD28.  You should not edit this value.  A normalizing radius may be entered in XD24.  If it is nonzero, the value of R is divided by this value before the aspheric terms are factored in (but it is ignored when calculating the curvature and conic sags).  The same normalizing radius applies to both inner and outer zones.

 

A fine point must be mentioned.  The diffraction image analysis programs must estimate the value of the Airy disk radius, since that determines the size of the area that will be sampled for the point-spread function.  This is done by tracing some rays at the 0.2 zone and extrapolating to the full aperture (to avoid problems in case the marginal rays will not trace) to get the local F/number.  But this type of USS will usually produce two different F/numbers, one from the inner zone and one from the outer.  If the zone boundary is beyond the 0.2 zone, then the value returned at the inner zone is used.  If it is within, then the opposite occurs.  In either case, the diffraction programs may sample an area different from what you would prefer.

 

And there is a second fine point:  Since the vertex position of the curve assigned to the outer zone is shifted automatically in order to make the two zones contiguous, the paraxial focal point of that portion will in general not coincide with the real one.  This can make what should be a good image look somewhat worse.  For that reason you are advised to adjust the focus position by varying the appropriate thickness while correcting one or more OPDs of real rays, rather than relying on the more common YMT solve to do the job.  Also, be aware that the reference ray used by the OPD calculation by default goes through the center of the entrance pupil and will intercept the USS surface in one zone or the other, but not both.  Thus you are advised to adjust the ICR parameter if you want that reference to refer to the OPDs of the zone that was not chosen.

 

All of these parameters can be varied in optimization, using the G-term format.  Thus, to vary term XD30, one would request

 

VY SN G 30

 

in the PANT file.

Type 9; Forbes polynomial type A:

 

This type expands the surface shape with a set of coefficients derived by Greg Forbes, which we call Forbes type A.  It is defined as the sum of an orthogonal set of polynomials plus a curvature and conic constant. All of the terms, and hence the surface, are rotationally symmetric.

 

Input, entered with DCn or ATn terms, is as follows:

G 2        conic constant

G 3        a0

G 4        a1

G 5        a2

G 6        a3

G 7        a4

G 8        a5

G 9        a6

G 10        a7

G 11        a8

G 12        a9

G 13        a10

G 14        a11

G 15        a12

G 16        a13

G 17        R0

 

For a point at XARG, YARG on a surface of normalizing radius R0, the first six polynomials are defined as

 

Q0        1.0

Q1         –(5 – 6x)

Q2        15 – 14x(3 - 2x)

Q3        –(35 – 12x(14 – x(21 – 10x)))

Q4        70 – 3x(168 – 5x(84 – 11x(8 – 3x)))

Q5        -(126 – x(1260 – 11x(420 – x(720 – 13x(45 – 14x)))))

 

where x = rho/R0**2 and rho = XARG**2 + YARG**2.  The surface sag is then given by

 

 Z(rho) = (CV+conic term) + x**2*SUM(an Qn(x))  .

 

Up to 14 polynomials may be incorporated in this shape.

 

This formulation yields benefits similar to the more common Zernike polynomial, and it produces a more accurate shape calculation than the usual power-series expansion, with fewer decimal digits and less loss of accuracy due to cancelling terms.  Here is an example of the shape resulting from the Q5 term on a surface:

Type 10; Diffuser:

 

This type is used to model a flat diffuser.  The parameter G1 in this case gives the semi-angle of the cone through which light will be scattered, and G2 is either 0 or 1; a value of 0 causes light to be scattered uniformly within the requested cone, and a value of 1 models a surface where light is preferentially scattered in the forward direction.  Thus:

 

 G1 = semi cone angle

 G2 = mode (0 or 1).

Type 11; Forbes polynomial type B:

 

This type is another formulation by Greg Forbes, called type B.  Both this and USS 9 model the surface with a series of orthogonal polynomials; type 9 adds polynomials that are orthogonal over the aperture, and you can control the RMS sag departure from the base sphere with the AANT mnemonic FRMS.  Type 11 instead models polynomials in a form that makes it easy to calculate the average slope difference instead of the sag difference, which is appropriate for making sure that the fringes shown when measuring that shape are widely enough separated to be discernible with a digital detector of finite resolution.  See the paper referenced under Type 9 for details.

Unlike type 9, with this shape the paraxial curvature is not necessarily equal to the first G-term; the polynomials themselves contribute optical power on axis.  The paraxial programs take this into account automatically.  The best-fitting sphere is given by the first G-term, however, and the ADEF analysis should produce a best fit that is quite close to that value, depending on the match between the actual clear aperture and the term G 17.

Type 13; Aspheric cylinder:

 

This surface is described by even powers of X and Y and is primarily intended for aspheric cylinders.  Input is

 

sn USS 13

sn XDD 1 G1(CV) G2(CC) G3 G4 G5

sn XDD 2 G6 G7 G8 G9 G10

sn XDD 3 G11 G12 G13 G14 G15

sn XDD 4 G16 G17 G18 G19 G20

sn XDD 5 G21 G22

 

Term G1 is the base curvature, and G2 is the conic constant.

 

The surface sag is then given by

 

Z = (conic contribution)

Type 14; Bezier spline:

 

This surface is generated by rotating a Bezier spline curve about the Z-axis.  It differs from the other spline shapes in several ways:  First, while the other spline surfaces always pass through all of the entered (Y,Z) data points, this is usually not the case with a Bezier curve.  Instead, you give the program a set of "control points" and it creates a curve according to its own logic.  The resulting curve has continuous derivatives -- which is not generally true for the other splines.  This form can be considered a special case of a NURBS surface.  That formulation allows a non-rotationally symmetric shape and allows the specification of weighting factors on each of the control points, whereas this USS shape implies weights of unity and is rotationally symmetric.

 

The N control points constitute a table of (YZ) coordinates, entered along with some defining parameters with the extended aspheric format as follows:

 

SN USS 14

SN XDD 1 CV CC RNORM N

 

SN XDD 3 PY1 PY2 PY3 PY4 PY5

SN XDD 4 PY6 PY7 PY8 PY9 PY10

SN XDD 5 ...

SN XDD 6 ...

 

SN XDD 7 PZ1 PZ2 PZ3 PZ4 PZ5

SN XDD 8 PZ6 PZ7 PZ8 PZ9 PZ10

SN XDD 9 ...

SN XDD 10 ...

 

The control points are defined as the pairs (PY1, PZ1), (PY2, PZ2), and so on.  Up to 20 points may be so defined, up to the number N.  The numeric values of the Y-coordinates can range up to the value of RNORM, which is a normalizing radius that should be equal to or greater than the CAO of the surface.  Another control point at (0,0) is automatically assigned by the program, and that point should not be entered with the data.  Thus, if you define N equal to five, the program actually evaluates a total of six control points when the added axis point is included.

 

The resulting curve always goes through the points (0,0) and (PYN,PZN), which are the first and last control points.  It usually does not pass through any of the others.  The Z-values of the control points are not usually assigned by the user, but calculated by the program during lens optimization.

 

It is common practice for this surface to be defined at equally spaced intervals in Y, and to assist the user in setting up the data, the program will automatically calculate the N points in Y if no data are entered for YN (or the value is zero).

Type 15; NURBS surface:

 

This surface is an example of the most general of the free-form shapes available in SYNOPSYS, and is a special case of the type known as a NURBS (Non Uniform Rational B Spline).  As with type 14, you give a set of control points and the program finds the surface shape according to the NURBS theory.  Unlike type 14, however, the control points are defined in both X and Y.  In this case, the points are uniformly spaced in an (X,Y) grid, and all are assumed to have equal weight.  Thus it is not a true NURBS because of those restrictions, but it is appropriate for use in optics, where the shapes tend to be well behaved with no areas that exhibit a strong local departure from the overall shape and would require nonuniform spacing or weights to model.  The degree of the NURBS is taken from the number of points in X and Y.  A set of three points models a quadratic fit; four points a cubic, and so on.

 

The control points are uniformly distributed and are not entered by the user.  Instead, you give the locations of the corners of the rectangular (or square) array with input, which is of the form

 

SN USS 15

SN XDD 1 CV CC NX XMIN XMAX

SN XDD 2 NY YMIN YMAX [ 0 1 ]

 

SN XDD 3 PZ1 PZ2 PZ3 PZ4 PZ5

SN XDD 4 PZ6 PZ7 PZ8 PZ9 PZ10

SN XDD 5 ...

SN XDD 6 ...

 

CV is the local curvature at the vertex.

CC is the conic constant, if any.

NX is the number of points in the X-dimension of the control grid, from 2 to a maximum of 20.

XMIN is the most negative of the X points.

XMAX is the most positive of the X points.

NY is the number of points in the Y-dimension of the control grid, from 2 to a maximum of 20.

YMIN is the most negative of the Y points.

YMAX is the most positive of the Y points.

 

If word 8 of the XDD 2 line is the number 1.0, the surface is declared symmetric about the Y-Z plane.  In this case, XMAX is set equal to -XMIN, and the control points on the right side of the array will pick up the Z-values of those on the left side.

 

Be aware that a NURBS surface does not actually go through any of the control points.  To see the actual surface that results, SPROFILE is recommended.  Think of the control points as pushing and pulling the surface, but not touching it.

 

The product NX x NY must not exceed 81.  Thus a grid of 9x9 is acceptable, as well as 8x10, etc., but 10x10 is not.

 

Most of these data can be varied in optimization.  All the Z-values can be varied with the PANT input VY SN NURBS.  You can also vary any of them independently with VY SN G NB, where NB starts at 11 for the lower left control point.  Other terms that can be varied are as follows:

 

G 1 varies the base CV

G 2 varies the conic constant CC

G 4 varies XMIN

G 5 varies XMAX

G 7 varies YMIN

G 8 varies YMAX

 

One sometimes wants to control the magnitude of the PZ terms, which can become large, even though the surface appears smooth and controlled.  This may be accomplished by including an entry for NURBS in the AANT file.

Type 16; Simple DOE:

 

This type defines a surface as a simple form of diffractive optical element (DOE), also known as a kinform surface.  (Another way to model DOEs is described in Section 3.3.2.8.  That form is powerful and general -- but it is somewhat complicated to set up, requiring two surfaces to hold all of the potential coefficients.)

 

USS type 16 is less general but easier to use.  The surface is completely described by coefficients entered with the XDD form of input and contains only even powers in the expansion polynomial.  The program simulates a normal DOE, taking its data from the entered terms.  With even powers it can only describe rotationally symmetric shapes -- but for centered optical systems those are exactly the shapes that are most useful.  A related type, USS 20, models a multilayer DOE.

 

The index on either side of the DOE is normally taken from the index of the materials to the left and right, one of which is usually AIR.  If the DOE is to be made by applying a thin film whose surface has the zones molded or etched in, the index and Abbe number of that material can be input with the HIN data.  Otherwise the HIN value should be that of the substrate itself, which can be entered either with GLM values (INDEX and VNUM) or indicated with PICKUP; in the latter case, the program transfers the index on the side that was not AIR to the DOE..  The diffraction efficiency of the DOE is taken from the blaze height, wavelength, the HIN value, the diffraction order, and the ray angles, according to scalar diffraction theory.  A good summary of this theory is published at http://apollooptical.com/content/docs/diffractive_formulas.pdf. If the substrate index is not the same as the HIN INDEX, the efficiency calculation for this form is less precise than that of the more complete DOE designation (although the error is small if the index difference is small), and that form should be used if the results are critical.  The raytracing calculation is correct in all cases.

 

If the DOE is immersed, which means neither index is AIR, you must declare HIN PICKUP.  The shape of the zones then depends on which side had the higher index.  It is an error to specify the same material on both sides.

Type 17; Biconic Birad:

 

This type is used for a biconic birad surface, which has a different axial radius in X and Y as well as different conic constants in the two directions.  (The biradial conic and biconic shapes have three independent parameters, while type USS 17 has four.)

 

Input is specified as follows:

 

SN USS 17

SN AT1 Rx Ry Cx Cy

 

In this input, Rx and Ry give the axial radius of curvature in X and Y, while Cx and Cy give the conic constants in X and Y.  To vary these parameters, use variables G1 for Rx, G 2 for Ry, G 3 for Cx and G 4 for Cy.

 

 

Type 18; User-described surface:

 

Type 18 is unique: it is not described anywhere in SYNOPSYS but can be supported by a user-written DLL.  A separate document is provided with programming instructions for those who need this level of flexibility.  Put simply, you write your code in C++, copy the DLL file into the SYNOPSYS directory, and your surface will then be honored by SYNOPSYS.  Your DLL must be able to calculate the surface sag as a function of (X,Y), and the axial curvature in X and Y.  It is a good idea to also provide code that can reverse and scale your custom shape.  SYNOPSYS does the rest.  See Section 5.54 for more information about this feature.

Type 24; extended XNURBS:

 

This surface is another of the free-form shapes available in SYNOPSYS, and is a special case of the type known as a NURBS (Non Uniform Rational B Spline).  It differs from type 15, which is also a NURBS, in several important ways.

 

1. You are allowed up to 400 control points (type 15 allows 81).

 

2.  This shape must be allotted to one of only five slots.  This is an important difference.  All of the other shapes can be assigned to any of 200 surfaces, and they can all be different.  But assigning 400 data points to that many surfaces would make the memory footprint of the program too large, so we provide only five sets of data, which can be assigned to as many surfaces as you wish.

 

3.  Data are input with a special mnenonic, "XLD", which permits up to 82 index values of five terms each.

 

4.  This type cannot be assigned to the object surface.

 

As with type 15, you give a set of control points, and the program finds the surface shape according to the NURBS theory.  The points are uniformly spaced in an (X,Y) grid, and all are assumed to have equal weight.  Thus it is not a true NURBS because of those restrictions, but it is appropriate for use in optics, where the shapes tend to be well behaved with no areas that exhibit a strong local departure from the overall shape and would require nonuniform spacing or weights to model.  The degree of the NURBS is taken from the number of points in X and Y.  A set of three points models a quadratic fit; four points a cubic, and so on.

 

The control points are uniformly distributed and are not entered by the user.  Instead, you give the locations of the corners of the rectangular (or square) array with input, which is of the form

 

SN USS 24 SLOT

SN XLD 1 CV CC NX XMIN XMAX

SN XLD 2 NY YMIN YMAX [ 0 1 ]

 

SN XLD 3 PZ1 PZ2 PZ3 PZ4 PZ5

SN XLD 4 PZ6 PZ7 PZ8 PZ9 PZ10

SN XLD 5 ...

...

SN XLD 82 ...

 

SLOT is a number from 1 to 5.

CV is the local curvature at the vertex.

CC is the conic constant, if any.

NX is the number of points in the X-dimension of the control grid, from 2 to a maximum of 20.

XMIN is the most negative of the X points.

XMAX is the most positive of the X points.

NY is the number of points in the Y-dimension of the control grid, from 2 to a maximum of 20.

YMIN is the most negative of the Y points.

YMAX is the most positive of the Y points.

 

If word 8 of the XLD 2 line is the number 1.0, the surface is declared symmetric about the Y-Z plane.  In this case, XMAX is set equal to -XMIN, and the control points on the right side of the array will pick up the Z-values of those on the left side.

 

It is possible for a surface to PCV the curvature of a USS 24 surface.  It gets the same data in the same slot.  But it is not possible to do a reverse PCV.

 

Be aware that an XNURBS surface does not actually go through any of the control points.  To see the actual surface that results, SPROFILE is recommended.  Think of the control points as pushing and pulling the surface, but not touching it.

 

Most of these data can be varied in optimization.  All the Z-values can be varied with the PANT input VY SN XNURBS.  You can also vary any of them independently with VY SN XLD NB, where NB starts at 11 for the lower left control point.  If the surface is declared symmetric, then the number of variables assigned by the XNURBS input is half of the total number of control points.  Thus, for a 20x20 XNURBS, with 400 control points, you would get 200 variables.

 

It is wise to start with a small number of control points and increase the rank gradually, as the need arises.  The RXN command can be used to change the array dimensions.  Keep in mind that the program only allows a total of 400 variables in the PANT file.  So only use the smallest rank that will do the job.

 

Other terms that can be varied are as follows:

 

XLD 1 varies the base CV

XLD 2 varies the conic constant CC

XLD 4 varies XMIN

XLD 5 varies XMAX

XLD 7 varies YMIN

XLD 8 varies YMAX

 

 

One sometimes wants to control the magnitude of the PZ terms, which can become large, even though the surface appears smooth and controlled.  This may be accomplished by including an entry for XNURBS in the AANT file.

 

The Z-coordinates of the control points are given in serial fashion with the remaining XLD input data, starting at the lower left of the grid.  The figure below shows the numbering scheme used for those data in the case of a 4x4 grid.

 

Point PZ1 is at the lower left corner, and so on, as numbered in the figure.

A surface can be recast to this form with the command RXN. The current definition and a list of all control points is produced by the command LNC.

 

This shape can accommodate up to 400 control points (compared with 81 for the simpler type USS 15), and it is not practical to edit that many individual terms with the Surface Shape Editor.  For that reason. that editor does not support this surface type.  A surface can, however,  be defined as type USS 24 with the RXN command, and the data can be edited by the WorkSheet or the LE lens editor, should you need to alter individual terms.

 

 

Type 25; Extended Simple DOE:

 

This type defines a surface as an extended form of diffractive optical element (DOE).  (Type USS 16 is similar, but only allows rotationally symmetric OPD terms, and can have a curved substrate.)  This form allows a variety of non-symmetric terms and is therefore more general.  This is yet another kind of kinform surface.

 

The surface is completely described by coefficients entered with the XDD form of input and contains even powers in both X and Y in the OPD expansion polynomial, as well as cross products.  The program simulates a normal DOE, taking its data from the entered terms.

 

The index on either side of the DOE is normally taken from the index of the materials to the left and right, one of which is usually AIR.  If the DOE is to be made by applying a thin film whose surface has the zones molded or etched in, the index of that material can be input with the HIN data.  Otherwise the HIN value should be that of the substrate itself, which can be entered either with GLM values (INDEX and VNUM) or indicated with PICKUP; in the latter case, the program transfers the index on the side that was not AIR to the DOE..  The diffraction efficiency of the DOE is taken from the blaze height, wavelength, the HIN value, the diffraction order, and the ray angles, according to scalar diffraction theory.  A good summary of this theory is published at http://apollooptical.com/content/docs/diffractive_formulas.pdf. If the substrate index is not the same as the HIN INDEX, the efficiency calculation for this form is less precise than that of the more complete DOE designation (although the error is small if the index difference is small), and that form should be used if the results are critical.  The raytracing calculation is correct in all cases.

 

If the DOE is immersed, which means neither index is AIR, you must declare HIN PICKUP.  The shape of the zones then depends on which side had the higher index.  It is an error to specify the same material on both sides.

The aspheric and OPD terms are stored as coefficients G16 through G34.  Thus, to vary the 6th power OPD term, one would vary term G 28 (15 + 13), as given in the table above.  The OPD terms are always in units of cyces of retardation; thus a value of 1 produces one wavelength of OPD change.

 

It is also possible to vary the construction wavelengths, which are in variables G 1 and G 13.  Then the DMASK plots tell you what blaze depths resulted.  With these variables, one can maximize the transmission for selected fields and colors.  Do not vary any other G terms, since those are part of the DOE model.

 

The quantity RNORM is a normalizing aperture:  When the OPD is computed at a given (X,Y) location, the value of Rho, the radial distance from the axis at that point, is divided by RNORM before the OPD is calculated.  (It does not apply to the terms of the base curve itself, only to the OPD definition.)

 

By default, the diffraction order is -1, but you can specify any order you wish with the ORDER parameter.

 

A coating may not be applied to the USS surface, and the materials on either side must be real glass or plastic, not a GLM model..

Type 27; Plane grating:

 

Type USS 27 models a plane diffraction grating.  You can also use the GRATING surface type, which is slightly more complicated, using the HOE model, which requires two surfaces.  The USS 27 form uses only a single surface.

 

The grating frequency is entered with the XDD form of input.  The program simulates a USS 16 DOE whose OPD function is linear in either X or Y.

 

The index on either side of the DOE is normally taken from the index of the materials to the left and right, one of which is usually AIR.  If the DOE is to be made by applying a thin film whose surface has the zones molded or etched in, the index of that material can be input with the HIN data.  Otherwise the HIN value should be that of the substrate itself, which can be entered either with GLM values (INDEX and VNUM) or indicated with PICKUP; in the latter case, the program transfers the index from the side that was not AIR to the DOE..  The diffraction efficiency of the DOE is taken from the blaze height, wavelength, the HIN value, the diffraction order, and the ray angles, according to scalar diffraction theory.  A good summary of this theory is published at http://apollooptical.com/content/docs/diffractive_formulas.pdf. If the substrate index is not the same as the HIN INDEX, the efficiency calculation for this form is less precise than that of the more complete DOE designation (although the error is small if the index difference is small), and that form should be used if the results are critical.  The raytracing calculation is correct in all cases.  The theory of the efficiency calculation is valid only for small diffraction angles and becomes less accurate if that angle exceeds about 8 degrees.  For gratings that diffract over a larger angle, the transmission and PRAY calculation are less accurate and should not be relied upon.

 

If the DOE is immersed, which means neither index is AIR, you must declare HIN PICKUP.  The shape of the zones then depends on which side had the higher index.  It is an error to specify the same material on both sides.

 

Input is of the form

 

SN USS 27

[CWAV CWAV

HIN [ INDEX VNUM / PICKUP ]

[ ORDER ORDER ]

SN XDD 1 0 0 CY/UNIT, {1 / 0 }

 

The base shape is always flat, and the OPD coefficient that defines the grating is taken from the XDD data, where the grating frequency is taken from word 6 of the input, and the value of word 7 specifies the diffraction direction: 0 for diffracting in the X-direction, and 1 for diffracting in Y.  If the lens is in units of MM, the grating is in lines/mm; if the lens is in inches, it is lines/inch.

 

Although you can run DMASK with this surface type, it is probably not a good idea, since the number of lines is likely to exceed the resolution of the analysis.

Type 28; Isolated bump or hollow:

 

This type applies to surfaces that are spherical or conic and have a polishing error that leaves an isolated high or low spot.  The parameters are as follows:

 

G 1 axial curvature

G 2 conic constant

G 3 HEIGHT

G 4 X

G 5 Y

G 6 R

 

The error is modeled as a cosine wave centered at point (X,Y) with height HEIGHT measured from the ideal surface over a radius of R.  With this model, one can ascertain the effect on the image quality of the specified error.

 

Here is an example.  We get out the relay telescope and apply the USS 28 shape to surface 6, the first corrector lens, specifying a hollow of 0.0001 inches depth, located at (0.2, 0.4), with a radius of 0.2 inches.  It is a hollow because this is the first side of the element and the thickness is positive there, as is the HEIGHT.

 

GET 4

CHG

6 NULL

6 USS 28

6 AT1 .08394022 0  .0001 .2

6 AT2 .4 .2

END

 

To see the effect on the wavefront, we enter

 

FRINGES                                                                                   

PUPIL P 1 0 5 0 0 0
