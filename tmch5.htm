Chapter 5

Finishing Up the Basics

 

In the last four chapters you have learned how to run many useful features of SYNOPSYS.  In the process I showed typical examples of each feature, but in order not to get bogged down in details I avoided going over all of the optional input and special cases that may actually be important in some circumstances.  I recommend that before you try any feature on your own you read the section in the User's Manual describing that feature.  There you will find a great deal more information than what is presented here, some of which may be just what you are looking for.

 

 

SYNOPSYS Mode Switches

 

You have seen some examples of the use of the mode-control switches in the earlier chapters.  Open up the MSS dialog and see for yourself what options are there.

 

Defaults:

 

There are currently 98 switches that are in use.  When you start SYNOPSYS the program turns on a default set:

 

ON 2 4 5 7 8 27 29 32 39 45 54 65 91 95 97.

 

If this is not what you want, you can put your favorite settings into a MACro, which you name CUSTOM.  This will be run automatically when you start the program.  If you later change any of these switch settings while you are running SYNOPSYS and want to restore the values you set at startup, type the command RESET.  This command will execute the startup sequence, which runs the MACro SYSTART and also calls the MACro CUSTOM, to enable any settings that you have placed there.  (SYSTART is replaced at each update, while CUSTOM is not, so your settings are more permanent if placed there.)

 

Here are some settings that you may find useful.  A listing of all of the switches and their effect is given in App. B.

 

ON

OFF

Effect

24

Default UNITS will be mm instead of inches

27

To get spots drawn as symbols instead of a single dot, turn this OFF

55

DMODEL, PSPRD use 2x finer sampling of image (but 4X slower)

 

You can print the current switch settings with the command SSW.

 

 

Startup MACro SYSTART

 

This is what you will find if you load the startup MACro into an editor.  (The command LM SYSTART will load it.)

 

! DEFAULT SYNOPSYS(tm) STARTUP MACRO FILE

!

! TO CUSTOMIZE YOUR INSTALLATION, PUT SETTINGS

! INTO A MACRO NAMED CUSTOM.MAC, WHICH WILL BE CALLED

! BY THIS MACRO WHENEVER SYNOPSYS IS STARTED.

!

LOG

ON 10 28 35 41 42 62 64

AI

FF: FINAL 5

TT: EM TEST

BELL OFF

EAM ICONS

EAM CUSTOM

PLB

ZDS

 

Here the LOG command increments the current LOG number (giving you an index number for your lenses automatically), and some switches are turned ON. AI mode is turned on, and some useful symbols are defined.  Then the toolbar symbols are defined by running the file ICONS.MAC, and any custom settings you put into the MACro CUSTOM.MAC are set by running that file.  Then the contents of the current lens library are listed on the screen.  All installations should have a default SYSTART.MAC, and you should not create or edit it yourself.  It will be overridden by each update, if it is in the default USER directory, and you are advised to put your custom settings into the CUSTOM MACro instead.

 

Now suppose you always want to turn on switches 24 and 55.  To do so, you would put these settings into CUSTOM.MAC.

 

ON 24 55

 

The command EAM, which you saw in the startup MACro, deserves comment.  This means “Execute Alternate MACro”, and provides a way for one MACro to call another, as you might call a subroutine.  This loads the second MACro into memory and runs it, and then returns to the calling MACro to continue where it left off.  You can also chain MACros by placing an EM filename at the end of one to call the other -- but that form will not return.

 

Incidentally, the LOG command can be entered like any other command any time you want.  It is sometimes useful during a long session to keep track of the many stages of your lens development, since the log number becomes a part of the lens ID and shows up on listings and plots.  We generally put the command at the start of our optimization MACros, so each resulting lens gets a unique number.

 

 

Keeping Track of your Lenses and MACros

 

After you have used SYNOPSYS for a while, you will probably have several dozen MACros and lens files saved on disk.  There are a number of commands and menus to help you organize and use these files.

 

Saving your Lens as an RLE File

 

Since the lens library can hold only ten lenses, you will frequently need to save your data in another form.  With the triplet still current, enter the command

 

SAVE TRIPLET

 

The lens will be converted by SYNOPSYS into an RLE file and saved as a file with the name "TRIPLET.RLE".  You can enter any name you want, as long as it starts with a letter, or else is made entirely of numbers, without punctuation or special symbols.  The current release allows up to 64 characters in a filename.  If you leave off the filename, SYNOPSYS will prompt you for it.  Once a lens has been saved in this way, SYNOPSYS keeps track of the name you used, and after that you can then just type SAVE to save it again with the same name.  You can also click on the toolbar button , which saves the lens with the most recent name – or calls up the File Save As dialog, if you have not yet entered one.  To save with a different name, you can also use the File|Save Lens As menu.  To get the lens back again, type

 

FETCH TRIPLET

 

You can save any number of lens files in this way. Now type the commands

 

DIR RLE

DIR MAC

 

The first DIRectory command will list the ID's of all of the lenses you have SAVEd (a total of one so far, in addition to those the installation has saved for you), and the second will give the names of all of your MACros.  There are eight different kinds of file that the DIR command can list; type DIR and look at the TrayPrompt to see the list.

 

There are also dialogs that tell you about your files; if you type the command MLB you will see the PLB display in a dialog window; you click on a location in the library, select either the GET or the STORE option, and click OK.  The menu MWL gives a scrollable window with the ID's of all your SAVEd lenses along with a preview drawing pane; again, you select the line with the desired lens and click OK to FETCH the lens (or double-click the name).

 

MACros are manipulated in several ways. You have already saved a MACro from within the editor with the button.  You can get a dialog showing all of your saved MACros with the MWM command; from there you can either run or edit the file that you select.  When the EE editor is open, you can click on the toolbar button to open a window with a list of all your saved MACros.  When you select one from the list, a preview is shown in the editor, helping you identify the one you are after.

 

Other useful commands are

 

LM filename

to load the current editor with that file

EM filename

to immediately execute that MACro, without opening the editor

GDS

to rerun the last MACro that has been run

GADS

to rerun the last MACro that has been run with the Run Alternate MACro button.

 

SYNOPSYS keeps the lens file separate from the MACro you will use for optimization (described in Chapter 6) as well as the files that the tolerancing program can save.  This offers some advantages – but sometimes you will want to keep this information together.  There is an easy way.  Type SAVE LOG to save your lens with a filename that is just the current LOG number.  (You can also click the button  to do the same thing.)  Click the same button on the EE editor toolbar, and a copy of the current MACro is saved with the same name.  After you have run your tolerance analysis (described in Chapter 8), type BTOL SAVE and your tolerance budget is also saved with the same name.  You have just documented the current state of your project.

 

 

MACro Loops

 

One more word about MACros before we go on to some very friendly features found in the SketchPAD program.  Sometimes you will want a MACro to perform a given task a number of times, with one or more of the data entries varying each time.  There are two basic ways to accomplish this in SYNOPSYS; the first way is simply to put a GDS command at the end of the MACro.  When the program gets there it will loop back to the beginning.  What keeps it from looping forever?  You must arrange that when the program reaches a given line in the MACro it will stop and request operator input.  This is done with the command "NEW" at that place within the MACro.

 

MACro 1

 

CHG

NEW

END

 

VAR 2 1 300

GDS

 

Here is an example of a MACro that loops.  The program will wait for your input when it gets to the NEW line inside the CHG file.  You could then enter, for example, “2 TH 5.26”, and the CHG file would then continue and the image analysis would be performed.  Then it would loop back to the beginning and ask for another NEW line of input.  When you have run all of the cases you want, just type EDS and then END.  (The EDS ends the MACro execution, and END gets you out of the CHG file.)

 

 

AI MACro Loops

 

The second basic way to do loops of this kind is with an Al sentence, and it is much more powerful.  You have already seen some of the "Plot... for... = ... to ..." loops that Al can do.  It can also do a MACro within a loop:

 

Do macro for 2 th = 5.0 to 6.0

 

The MACro can contain anything except another “DO MACro" or an Al loop sentence, and when it is finished, AI increments the variable (2 TH in this example) and starts over. To use this feature, you first have to either run the MACro yourself at least once, which puts the contents into memory, or you can click the button

 

 

which does the same thing.  Then enter the “DO MACRO …” sentence.

 

It is also simple to ask Al to pick up a given quantity when it is in the loop and use it as data for a plot. For example:

 

DO MACRO AS THICKNESS 2 VARIES FROM 5 TO 6

 

MACro:

 

ORDINATE = SA3

ABSCISSA IS BACK FOCUS

 

Here, Al will run the MACro loop and pick up the third-order spherical aberration for the ordinate of the plot.  The abscissa will be the back focus, and the two will therefore be plotted as a function of thickness 2.  Of course, for this simple example it would not be necessary to use a MACro at all; Al could just as easily

 

PLOT SA3 VS BACK FOR 2 TH EQUAL 5 TO 6,

 

but there are times when you want to plot something that is not in the AI vocabulary, and then the MACro loop is needed.  To show how this works, first type a command like EDGE.  Then ask AI to show you the BUFFER:

 

BUFFER?

 

The current FILE BUFFER contains

    1        1.1451577        EDGE TH

    2        4.7136868        EDGE TH

    3        2.4921436        EDGE TH

  4 …

 

Many of the SYNOPSYS commands that produce a list or table of numbers put a copy of these data into a FILE BUFFER.  You can ask Al to show you the buffer, as above, or just a single entry:

 

WHAT IS FILE 1?

 

FILE NUMBER 1        EDGE TH.     1.1451577

 

With these data Al can plot nearly anything; here is how to plot the edge thickness between surfaces 2 and 3 as a function of thickness 2:

 

DO MACRO FOR 2 TH = 5 T0 6

 

MACro:

 

EDGE

ORDINATE IS FILE 2

 

Since file number 2 is the desired edge thickness, the ordinate of the plot contains the desired value, and the abscissa is by default the loop variable: 2 TH.

 

These examples of the Al MACro loop capability should give you an idea of the resources you have available.  I recommend you study Chapter 15 of the User's Manual, where many more possibilities are discussed.

 

 

The SketchPAD

 

Now we will explore some of the options available in the SketchPAD program.  This is a feature that shows you the lens and its image in real time, automatically updating as the lens is changed during optimization or in other ways.  Get out the triplet, and either type PAD or click on the button .  You will see the SketchPAD display, with a lens drawing on the top and a set of rayfans on the bottom.

 

 

There are seven kinds of display that you can ask for in PAD, and the display you see is the default.  You can easily select what you want.  Click on the button , and then click the option “RayFAN plots”, click on “Multicolor”, and then on OK.  This selection is just the default, so the display does not change.

 

The set of FANS shown on the screen is drawn to a default scale of .1 mm per unit height, determined by tracing a few rays (since you entered no scale factor yourself).  The three curves for colors 1,2, and 3 lie nearly on top of each other, and since this lens is well corrected for color; the numbers at the ends of the fans curves are similarly superimposed.  Note that rays vignetted by the clear apertures are indicated by separated dots, rather than by connected lines.  You can still use the command window while the PAD display is open, or click on any of the toolbar buttons.  Type the following lines

 

CHG

NOVIG

END

 

And then click the button

 

.

 

You will see the fans plots become continuous lines, since you just turned off the vignetting test, and rays that exceed the clear apertures now trace through the lens.  Any time the PAD window is redrawn, it updates the display with the current lens.  And if you close the display and later reopen it, the same choice of displays is shown again.

 

It is also possible to specify the kind of PAD display you want by typing into the Command Window.  Here you just type PAD, and then one of the following, and PAD will prompt you for the other parameters it needs.

 

PAD

activates the SketchPAD display.  The prompt changes to PAD>.

D

prompts for attributes of a Y-Z-profile drawing

PP

prompts for attributes of a perspective drawing

P

prompts for attributes of a paraxial drawing

F

prompts for attributes of a transverse rayfan display

FP

same as F, but sagittal fan is over the full pupil

O

prompts for attributes of an OPD rayfan display

OP

same as O, but sagittal fan is over the full pupil

S

prompts for attributes of a spot diagram display

A

prompts for attributes of a display of astigmatic field curves

X

erases the requested portion; used to show only one display instead of two

updates the display

ZOOM

zooms a ZFILE zoom lens to the zoom position in the 2nd word

ends the PAD interactive sequence (but does not close the display)

 

For the experienced user, it is possible to remember the sequence of prompted input you use for these features, and then enter that sequence into a MACro.  That way you can set up PAD the way you want it automatically, from the MACro.

 

If you want to save the display format you have defined so you can get it back later, type the commands

 

SSU (Save SetUp) or SPL (Save PAD Layout).  The former saves both lenses and the PAD layout; the latter just the latter.

 

Then, when you want to restore it or when you start SYNOPSYS again, just type RSU (Restore SetUp) and you will get back the lens and the display just as you defined it -- or RPL, which restores the PAD layout but does not get the lenses back.  You can define up to five different display formats for a given lens, using the buttons

and these also come back with RSU and RPL.

 

 

Changing the Lens in PAD

 

Pad also contains several features that let you change the lens and immediately see the effects on the image.  One useful tool is the button (which also appears in the SideBar); this opens the LE lens editor, where you can change anything in the lens, and when you exit from the editor (with the button ) the display is updated.  This option replaces the entire lens with what is in the editor when you run it, so be sure not to delete anything important.

 

 

WorkSheet Program

 

By far the most convenient way to edit your lens is with the Worksheet(tm) program.  First you must make a drawing on the top of the screen, if you have requested one of the other displays, since Worksheet will let you select surfaces to edit by just clicking on the lens drawing, and the drawing must be there.  When you have either a D, P, or PP display on top, click on the WorkSheet button .

 

 

The WorkSheet is highly interactive.  The edit box shows the data for surface 1 at the moment.  Click in the PAD display on a different surface (always click on the vertex, just past the surface you want).  Now you see the data for that surface.  Click the Checkpoint button so you can easily go back if you need to, Click in the edit box, and change a number, then click on the “Update” button.  The picture is updated, along with the image display, if any, which is on the lower part of PAD.  Now click the “Undo” button – and the lens is restored.  You can create a checkpoint any time by clicking the button, which then operates like a one-level undo.  (SYNOPSYS also has a 10-level undo feature, using Ctrl+Z (for text boxes) and Ctrl+U or F3 (for lens changes) – but these do not capture Worksheet changes.  See Chapter 13.)  WorkSheet only changes the data in the edit window, not the whole lens, although you can enter data for any surface in the edit window, in the format that goes into an RLE file; you are not restricted to editing only the surface that is currently selected.

 

With the original triplet on the display, click on surface 3, and then click on the thumb of the slider on the right that says “Slide element”.  Hold the mouse button down and slide this thumb to the left, slowly.  See the lens slide on the PAD drawing, and the ray paths and image change along with it.

 

To edit the object and system data, click one of the buttons to open the Object or Pupil Wizard, or to see the data in the WorkSheet editor.  Now you can edit those data.  You can also select a surface (or the system data) by entering the number (or 0) into the data box above the word “Curvature”, and then clicking “Update”.  Other WorkSheet toolbar buttons let you insert and remove surfaces and elements and perform other common tasks such as inserting fold mirrors or prisms..

 

 

Mapping Function Utility

 

To end this chapter I will describe briefly another of the IMAGE options.  This one lets you map your choice of one of 20 different quantities as a function of pupil position or field position.  With the original triplet the current lens, enter the following input:

 

MAP PUPIL OVER FOV

FGRID RECT 5 1

RGRID CIRCLE 20

PLOT

 

The results are shown below.

 

 

In this example you specified a field grid (FGRID) of 5 by 1 points and a ray grid (RGRID) consisting of a circle of 20 rays.  Look at the general input format for MAP to see all of the possibilities, and then type MMA to see the dialog, which puts most of these options on the screen.

 

There are more features in MAP than I can explain in a Tutorial Manual, but here are the basics:  Some of the functions can be specified at a given surface, others must be, and still others cannot be.  The grids that apply to the points in the field and points in the pupil are defined as follows:

 

 

The ray grid (RGRID) is identical to the field grid FGRID except that there are two kinds of circles: CIRCLE and MCIRCLE.  CIRCLE is like the PUP 4 option in TRACE, described in Chapter 4.  This circle will contract to pass clear apertures and feathered edges, but cannot expand.  MCIRCLE is like PUP 3: it can both expand and contract.

 

In the MMA dialog you will see the options ACTUAL and EXPLODED.  These determine whether the plot shows each pattern at the actual position where it would appear on the selected surface (or image plane), or is shown mapped into the FGRID or RGRID pattern.  For example, see the following two cases.

 

MAP UNI OVER PUPIL ON SURF 3

FGRID POINT 1 0

RGRID CREC 11 11

ACTUAL

SCALE 20

ANALOG

PLOT

CHG

CFOV

END

MAP DISTORTION OVER FOV

FGRID REC 9 9

EXPLODED

ANALOG 10

PLOT

 

 

 

In the first we mapped the incidence angle of a grid of rays from the full-field object point on surface 3.  The position of each fiducial is scaled to the actual ray position on the surface, at 20X magnification.  The angles are shown in analog fashion (as angles).  The pattern is noncircular due to vignetting.  In the second case we first redefined the field as circular with the CHG parameter "CFOV", since the lens originally had no skew field defined.  This makes XP0 equal to YP0, and the MAP sequence then shows the distortion of the lens mapped into object space, again in analog fashion at 1000X scale.  (In the first example, SCALE refers to the distribution of the fiducials; in the second, the ANALOG scale of 1000 refers to the length of the lines.)
