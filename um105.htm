10.5 Modes of Operation of SYNOPSYS                        Next Previous TOC

 

The design program can operate in any of several modes of operation, which are determined by the mode switches and set with ON and OFF commands (see 2.9). The switches that apply to optimization are

 

 1-9, 12, 29-32, 61, 62, 67, 82, 83, 99

 

These apply to SYNOPSYS operation in the following ways:

 

1.

causes the variables that hit a boundary during optimization to be dropped for the remainder of the run.  All variables have default boundaries, and the user may enter other values if required.  If a variable tries to violate a boundary at a given pass, it will usually continue to do so and is better dropped.  However, if a variable is dropped in the early passes when the variables are changing rapidly, it may later move away from that boundary if it is not dropped at that time.  If switch one is turned off, with “OFF 1” prior to the SYNO input, bounded variables will not be dropped.

 

2.

uses the PSD III optimization algorithm if switch 8 is also on (default).  This seems to be the most effective mode for optimization.  If off, the PSD II algorithm is used instead.

 

3.

causes subpasses to be performed to optimize a scaling factor that is applied in addition to the PSD damping, at each iteration.  This option gives the most improvement per iteration, but requires more time per iteration.  The default mode (2 ON) turns this off.  On average, this option does not give a net improvement in quality or time, but you can try it if you like.

 

4.

suppresses printing of the current aberration list at each pass and subpass. See switch 29, below.

 

5.

suppresses printing of the change list at each pass and subpass.  See switch 29, below.

 

6.

causes a listing of the aberration values during each evaluation, both for derivatives and for subpass solutions.  This switch is OFF by default, and is not recommended for most users since the output is lengthy.  See 29, below.

 

7.

causes new derivative increments to be calculated after each iteration.  The rule followed is to compute the largest aberration at each pass (A) and the largest change of any aberration (C) due to the previous increment DQ.  Then the new increment is given by

 

        DQ = DQ*0.1 A / C

 

This calculation is done for each variable.  The goal is to use an increment that will result in a largest aberration change equal to about 1/10 of the largest aberration.

 

This option usually generates very precise derivatives, and is on by default.  If your initial system is very poorly corrected, however, the derivatives may themselves become large and produce ray failures during optimization.  If you see a message to this effect, you may want to turn this switch off.

 

8.

causes the PSD algorithm to be employed.  Switch 2 controls which version is used.

 

9.

causes lengthy printout of the derivative array and matrix L to be listed each pass and subpass.  This is not a default mode and is not recommended for most users.  See 29, below.

 

29.

suppresses nearly all printout from the optimization.  If this mode is on (it is a default mode), switches 4, 5, 6, 9, and 31 are disabled and you will automatically receive only the following:

 

•

Input images for PANT, AANT, and ACON files.

 

•

A summary of alternate configuration locations.

 

•

A summary of the mode control switches, for reference.

 

•

A list of the initial aberration values.

 

•

Messages during the optimization regarding boundary violations if these cause variables to be dropped, and also GLASS variables that become pinned.

 

•

The merit function each pass.

 

•

The final merit function and aberration values.

 

•

A list of the variables that were not free at the end of the run.

 

30.

activates the variable-metric option.  This is an alternate optimization algorithm that you may wish to experiment with.  It is recommended that switches 2 and 8 be turned OFF if this switch is ON.

 

31.

prints a list (if 29 is OFF) of the metric parameters currently employed each pass.  This is not intended for the casual user.

 

32.

Causes the program to accept a solution that would normally be rejected on the first pass.  This is an effective way to overcome the limitations of the DLS method used for that pass.  Starting with the second pass the more powerful PSD method used.  This is a default mode.

 

61

saves the result of the optimization in library location 10 automatically.

 

62

compares variables with their boundaries and moves any that violate those boundaries into the valid region before optimization.

 

67

Requests double-sided derivatives.  Although slower than the default single-sided derivatives, this option can improve convergence as the design approaches optimum.

 

82

Calculate d2f/dX2 on first iteration.

 

83

Adjust automatic raygrid aberrations according to color weights in lens.

 

99

This switch makes the optimization program honor the CORE setting. All of the search program utilize multicore processing automatically, but the SYNOPSYS command, which runs a single optimization, normally executes on a single core.

If the process is lengthy, you might experience a speed improvement by authorizing as many cores as your PC has, and then turning this switch on. If the process runs in only a few seconds in a single core anyway, then this switch will likely make it run more slowly, rather than faster, owing to the overhead involved in starting and managing the several processes; so it should be used with discretion.

Switch 82 is not supported in this mode, and will be turned off if on.

 

Compare this list with the default settings in Section 2.9.  If you want different settings, you may enter them in the Command Window before running the optimization, or put them in the CUSTOM MACro so they will be set at program startup.

 

To illustrate the use of these mode switches, suppose that we desire to optimize a lens with narrow boundaries on some variables, and wish them not to be dropped if violations are attempted.  We would input:

 

OFF 1

SYNOPSYS...

 

This turns off switch 1, leaving all other switches unchanged.

 

A run on SYNOPSYS will terminate for any of the following reasons:

 

1.

Completion of the requested number of iterations.

 

2.

Improvement in the merit function less than .01% on any pass except the first.

 

3.

Reduction of the merit function below 1.0E-10.

 

4.

The damping became greater than DLIMIT.  This ordinarily does not happen, but if switch 8 is off (not default) the ordinary damped-least-squares algorithm is used, and in this case it might.

 

5.

A ray failed to get through the initial system.

 

6.

All variables became pinned to their boundaries.

 

7

The user clicked on the stop-sign toolbar button before the optimization was complete.

 

If mode control switch 29 is turned OFF prior to optimization, each iteration of SYNOPSYS will print out:

 

1.

pass number.

 

2.

present merit function.

 

3.

current values of increments DQ (if 7 is ON).

 

4.

current values of SECDR (see Section 10.1) if 8 on.

 

5.

with 3 ON (not default), successive trial DAMP's and resulting merit functions, followed by A, B, and C in the equation

 

        A X**2 + B x + C = Φ

 

where X = 1 / DAMP and Φ is the merit function.  This gives the best apparent damping factor for that iteration, DAMP = -2 A / B, also printed.

 

6.

predicted next merit function, based on a linear extrapolation from the present design.

 

7.

actual next merit function, using new solution vector.

 

 

A summary of the initial and final aberrations is provided at the start and end of the design run.
