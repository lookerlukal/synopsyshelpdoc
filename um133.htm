13.3 The SketchPAD                                                Previous Next TOC

 

The SketchPAD feature is a graphics window that can show either one or two displays simultaneously.  It is typically used to view the lens drawing and a display of image quality at the same time.  This is an interactive window that you can open, fill with your choice of display formats, and then update at any time with the button to see the current lens and its image immediately displayed in that format.  It is also updated whenever you GET or FETCH a lens, and during optimization if you have entered the SNAPSHOT command.  The display may be zoomed or panned, using the toolbar buttons at the top of the PAD window or with a shortcut, and the window can be minimized, resized and printed.  (Shortcut: pan by dragging with the right mouse button, magnify with a Shift-click of that button, and minify with a Ctrl+Shift click.)  You can drag-enclose a section of the display, and then click the button to view that section enlarged, and you can zoom in and out with the mouse wheel.  You can only have a single PAD window open at a time.  You can open a PAD window by clicking on the button or with the command PAD.

 

PAD lets you define up to five different display formats, which you can switch with the buttons .  Thus you could view a YZ profile along with rayfans using selection 1, a perspective drawing in two orientations with selection 2, and so on.  You can switch from one to another at any time, even during optimization.  The PAD layout for all selections and all ACONs may be saved with the command SSU or recalled with RSU.  There is also an option to link the PAD display to a command string, which will display any other kind of output that SYNOPSYS can create, in a separate graphics window, whenever the PAD display is updated.  That option is described here.

 

When the PAD window is open, you can click on the WorkSheet Lens Edit button , the Y-YBAR button, or the Glass Table button to open those features, or the Edge Wizard button to edit edge descriptions. You can also open the WorkSheet by typing WS, and the Edge Wizard by typing MEW.

 

A particularly useful feature of PAD is the SNAPSHOT feature, which shows the PAD display continuously as a lens is optimized.  This command is entered just before the SYNOPSYS command.

 

The LE Lens Edit button opens the LE lens editor, and after you have made your changes to the lens there and clicked on the RUN MACro button, the PAD display updates automatically.

 

When you close the PAD display or exit from SYNOPSYS, the program remembers the size and location of the window and attempts to restore it to that location the next time you open it.  If for any reason you want to restore the default location (such as after changing the monitor resolution) use the command DWL (default window location).  This also restores the default picture box location in SPS.  You can also enter the commands

 

PAD

RESTORE

 

which will move the current PAD display onto the monitor.

 

The PAD display can show a Y-Z profile drawing of the lens, with optional fill.  The fill makes the elements easier to see.

 

 

But sometimes roundoff error causes a single pixel at the end of a line to be missing, and then the fill color can bleed over the entire PAD screen.  The program will sense when this happens and turn off the flood fill for that display, testing again whenever PAD is redrawn.  You can disable the flood fill feature by turning on switch 46, or by clicking the button .

 

 

You can also try a feature that we have implemented for this purpose, which extends the lines that enclose the element drawing by a small amount.  The idea is to cover the gap, and this has proved useful with systems showing this behavior.  The command

 

FFF EXTEND

 

will extend the lines by the requested amount.  The default is 0.0, and you can try larger values if the flood fill does not show up, to see if they work better on your system.

The dispersion is taken from the model parameters for GLM glasses, but for real materials it is more complicated: the program finds the dispersion from the index values at the long, and short wavelengths as defined by the color order parameter CORDER, and If the wavelength range between these two is not the same as the difference from the C to the F lines, the V-number at the actual wavelengths will not be the same as the catalog values for the glass.  The program performs a linear adjustment in such cases, which usually reproduces the material colors fairly well.  The colors shown have little meaning in the case of IR materials.
