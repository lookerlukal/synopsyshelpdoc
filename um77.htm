7.7 Lens Element Drawing (ELD)                Next Previous TOC

 

A lens element drawing may be obtained with the input file

 

 [  USE BTOL ]

 ELD [ - ]jsss [ tscf ]

 

 [ MTH th ]

 [ DIA dia ]

 [ RAD rad ]

 [ BEVEL bc ]

 

 [ { 1/2 } { A a / SAG sag / BC bc / ABC ab bc / AAB a ab / ANG angle } ]

 

 [ ID up to 35 characters of title ]

 [ ID second line ]

 [ ID third line ]

 

 [ COAT "string" ]

 [ FINISH "string" ]

 [ DESIGNER " string" ]

 [ CHECKER " string" ]

 [ TWAV "string" ]

 

 PLOT

 

jsss is the lower of the two surface numbers defining the lens element to be drawn, and the scale factor tscf is optional.  The program will automatically scale the lens by powers of 2 or 1/2 until it fits the page if tscf is not entered.

 

ELD is also found in the dialog MPL.

 

The program draws a single lens element, honoring any edge description data that may be in effect.  This additional information can most easily be created with the Edge Wizard, but may also be created with an EFILE command or entered explicitly into the RLE input.  Data so defined become part of the lens description and are saved with the lens.  If no edge definition is in effect for the requested element, the program will substitute a default edge that is related to the CAO, as described below.

 

The font size of the three ID lines is adjusted according to the length of the line, with three sizes for lengths up to 20 characters, up to 27, and up to 35.

 

ELD creates a drawing with numerous boxes that can be filled in by the user, along with some that the program fills with data taken from the lens specifications.  Boxes for lens tolerances are filled in if you specify the USE BTOL option, described below.  Boxes for the coating, surface finish, designer name, checker name, and test wavelength are left empty unless you enter explicit character strings as listed above.  In that case, those strings are added as user annotation.  For example, to add the designer's name to the plot, you might enter DESIGNER "JOHN SMITH" with the ELD input.  Note that the quotation marks are required, to identify the input as a character string, not a command argument.  The above strings are limited to nine characters, except for the name or initials of the designer and checker, which will accept 12.

 

ELD may also be run from the dialog MPL.  This dialog does not permit entry of optional edge dimensions, however, so it is best to define the edges with the wizard (MEW) first.  (Actually, we always recommend using the wizard to define your edges.  That way you can see what you get, so you have immediate control.)

 

Some elements cannot be drawn in an appropriate manner by ELD, and in such cases you can elect to get an ELD drawing with most of the data boxes filled with the usual information but without the element itself.  To do so, simply give a negative surface number on the ELD command.  The program will then leave the scale factor and surface sag boxes empty and will not draw the element.  It will also not list aspheric terms, which would not include the non-symmetric terms, if any, and would be misleading.  Then you can add a PER or other drawing of the element and annotate it yourself; an example is given below showing how to do this.  If you want all of the data boxes empty, just turn on switch 34, which strips annotation from all graphics.

 

If a surface is not rotationally symmetric, the above paragraph applies.  The default ELD drawing will not show the asymmetry.

 

If the lens or mirror has been assigned a decentered CAO or RAO, ELD will draw the parent part, from which the actual piece will be cored out or sliced off.  Since one must usually make the parent first, this procedure gives you a drawing useful for that step.  You can then make a second drawing, adding a perspective view as required, annotated with the annotation editor, to inform the shop of the extra steps required.

 

The optional command USE BTOL, entered before the ELD command, will open the data file created by BTOL.  If that file contains tolerance values for the element to be drawn by ELD, those tolerances are added to the drawing automatically.  See Section 7.13 for examples.  (This option uses the data in the most recent file named BTOL_ELD.DAT.  If you do not run BTOL on the current lens, those data will apply to the last lens you toleranced, not the current one.  You probably don’t want those numbers, so be sure to run BTOL with the current system first, or else FETCH the budget if you prepared it at an earlier time.  See Section 12.1 for details. )

 

The tolerance shown for surface sag on the drawing is taken from the BTOL budget for the adjacent airspace, if any.  This sag error may or may not actually affect that airspace, however, depending on the mounting scheme you decide on later.  So be sure to examine the ELD drawing to verify that the sag tolerances are appropriate.  In the event that the adjacent airspace has no tolerance assigned, because it was either declared EXACT or will be used as an adjustment, the program assigns a default sag tolerance of 0.005 inches (0.127 mm) or 1/20 of the actual sag, whichever is smaller.

 

The options described below give a somewhat complicated way to define the edge geometry temporarily, within the ELD data file, and are probably not appropriate for most users, who will prefer the other features mentioned above, particularly the Edge Wizard (MEW).

 

To define the edge geometry temporarily within the ELD input file itself, data are entered defining the relevant quantities shown below, where three points of interest at each surface are labeled A, B, and C.  Data entered with the ELD file do not become part of the lens description.  They are used only for that drawing, and then discarded.  If no edge data are provided either with the lens file or in the ELD data, then default edges will be supplied by ELD, according to the rules below.

 

 

Point C is always at the same height for both surfaces, while A and B may be different. The default values for these quantities are as follows:

 

 

CONVEX

 

 

CONCAVE

C

larger CAO + .050 in.

 

C

larger CAO + .050 in.

B

C - .005 in. (BC = .005)

 

B

C -. 005 in. (BC = .005)

A

A = B

A

actual CAO + .005in.

 

If no data are entered to override the default lens edge definition, the program will add .050 in. to the larger clear aperture (as given on the CAP listing) to determine the lens diameter, and will give both sides a bevel of .005 in.  Convex surfaces will extend up to the start of the bevel, while concave surfaces will have a flat section from just beyond the CAO for that surface up to the bevel.

 

There are several ways to specify other values for the edge parameters.  In some cases specifying one will automatically redefine others as well.  In general, unless an option giving the quantity AB explicitly is entered (ABC or AAB), the values of A and B will always be the same for convex surfaces.  For concave surfaces, these quantities are generally different.

 

The simplest option is to specify the diameter of the element with DIA, or the radius with RAD (DIA = RAD *2).  The values of A and B will then follow the default rules: BC = .005 in., A = B for convex surfaces.

 

BEVEL BC gives a new bevel dimension to both sides of the lens.

 

More explicit control may be obtained by the optional surface input which begins with either 1 or 2; "1" referring to the leftmost surface on the drawing and "2" to the rightmost.  Normally (for positive thickness) the leftmost is also JSSS.

 

A

gives a new value to A for that surface.  Entered only for concave surfaces.

 

SAG

implies a new A for that surface: that aperture giving the desired sag.

 

BC

gives a new bevel dimension, thus redefining point B.  For convex surfaces, A is also redefined so that A = B.

 

ABC

gives explicit data for the distances AB and BC for that surface.  This input is valid for either convex or concave surfaces. A is recalculated, and is not necessarily equal to B in either case.

 

AAB

gives data for the value of A and the distance AB, thus implicitly redefining the bevel distance BC.  This may also apply to convex or concave surfaces.

 

ANG

gives the face angle for the section AB, which is normally at a 90 degree angle.

This is not to be confused with the bevel angle, which is always at 45 degrees.

Also included on the drawing are numerous boxes containing dimensions appearing on the drawing or in which you may enter tolerances, etc.  The input used to prepare this picture was

 

ELD 1 .2

MTH 3.0

BEVEL 0.1

2 ANG 30

2 ABC 3.5 .5

ID ELD EXAMPLE

ID BEVELED MIRROR

ID OSD, Inc

END

 

If the edge-definition data had been left out of this file, the mirror would have been drawn according to the current EFILE data, if any, instead of the entered dimensions.

 

Two lines of a title may be input with the first two ID lines.  These must begin in column 1 with the characters "ID", and the actual text characters will begin in column 4.  Any characters may appear in these lines except the (") character.  The program adjusts the text size to try to fit the entered characters in the space provided on the drawing, using three character heights: largest for up to 20 characters, smaller for up to 27, and smallest for 28 to 35 characters.  This may not be exact, since the proportional font uses more space for some characters than for others.

 

A single line giving your company name may be input with the third ID line.  Conic sections, torics, and power-series aspherics may be drawn, but not elements where one side is tilted or decentered with respect to the other – and only the rotationally-symmetric power-series terms will be annotated.  Also, Fresnel surfaces are not at present treated.  Only the Y-Z profile is shown, and for non-symmetric surfaces you will need to add another view by hand or by combining the ELD output with a PER drawing, possibly using a negative surface number to suppress the normal drawing as explained above.  Only the Y-dimension of RAO (rectangular aperture) data will be shown on the drawing.

 

Other annotation can be added to the drawing with the Annotation Editor.  One can define several lines of text that are used frequently and add them to the drawing as explained in that link.

 

Mirrors may be drawn as well as lenses.  The mirror thickness may be defined in the Edge Wizard or may be included in the ELD file.  If JSSS is a reflecting surface, you may follow the ELD file header command with the MTH input, which gives the desired thickness of the mirror.  This thickness is positive if the mirror is on the left, as shown on the above drawing.  If omitted, the mirror thickness will take a value equal to 1/10 of the clear aperture radius.  A flat dummy surface will be inserted for the back side, and all of the other edge parameters are available to specify bevels, face angles, etc.

 

If either side of the lens element has a CAl specification, the obscured portion will be shown as a dotted line on the ELD drawing.  If both sides have a CAl, or if the drawing is a mirror with a CAl, a perforation will be shown and annotated, as shown in the above example.

 

Glass types will be entered if the element has a glass-table glass or a PIN of a glass-table glass.

 

More complex edge shapes require a combination of ELD and PER or SOLID.  Here is an example of a lens element with a combination of EFILE edges plus an RAO aperture.  This element was edited with the Edge Wizard, which specified the values of points A, B, and C for both sides.  The RAO aperture could also have been added there.  For this example, we want the ELD drawing to show the element as well as two views created with SOLID.  To combine the plots we use the CPLOT command.  Here is the input MACro:

 

OFF 60                ; We want a stippled effect for shading

CPLOT                        ; This command says to combine plots

ELD 3                        ; Draw the element at surface 3

ID ELD EXAMPLE

ID COMPLEX APERTURE

ID OSD. INC.

END

 

SOL 0 90 1.5 3 4 -1.0 2        ; Now add a drawing showing the face

GREY                                ; Note the explicit center coordinates

PLOT

END

 

SOL 10 30 1.5 3 4 -1.5 -.5        ; Another view, in perspective

GREY

PLOT

 

QPLOT                        ; Finally, the plot is displayed

Prisms require a different drawing format.  In this case four views are shown, and dimensions are given where appropriate.  Angles and tolerances are listed in a table, along with surface shape tolerances, the latter as computed by BTOL and shown if the USE BTOL directive has been entered.  This drawing does not accept any of the optional data permitted for an ELD file except for three ID lines and the PLOT entry, which must all be entered, and in that order.
