Chapter 18

 

18.0 NONSEQUENTIAL RAYTRACING

 

All of the preceding chapters of this manual apply to systems in which light passes through the surfaces in numerical sequence (with the exception of some of the ghost image programs, which can reverse the light on a first-order basis).  Systems obeying this convention are called sequential systems.

 

It is also possible to define a system in such a way that the program will determine the sequence in which the surfaces are encountered.  Some surfaces may pass the light more than once, and others not at all, depending on the size, shape, and position of the surface and the nature of the light beam going though.  A system using this capability is called nonsequential, and the rules for raytracing and the interpretation of apertures and vignetting are different from the sequential case.

 

Because this feature involves a great deal more computation than sequential tracing, it should only be used when necessary.  It is possible to declare only certain surfaces nonsequential, thereby retaining the sequential rules (and raytracing speed) for the others.  In many cases, the sequentially oriented drawing programs DWG and PAD/D are not suitable for nonsequential systems; the programs PAD/PP. PER and RPER are recommended in these cases.

 

In all systems, light must pass from the object to surface number one.  From that point on, however, the rules may be different.  If a ray intercepts the final surface, it is always stopped.

 

STRANGE rays (see 2.5.1) are not treated at present, although they work in portions of a nonsequential system that are themselves purely sequential.

 

Since the nonsequential raytrace mode is so much slower than normal raytracing, it is not recommended for optimization, where the time required is longer than one perhaps would like.  For help in this situation, we have provided a mode switch that may be useful.  If switch 37 is turned on, then SYNOPSYS will perform sequential raytracing regardless of whether the lens system itself is nonsequential.  This is intended mainly for systems containing a roof prism – which must be nonsequential to display properly – but which give the same raytrace results, and image quality, regardless of which way it is traced.  If you turn on this switch, therefore, your optimization will run much faster and give the same results as with the slower raytrace.  But then you should turn off this switch when the design is finished and you want to prepare perspective drawings, which must show the correct nonsequential ray paths.  It goes without saying that if your system actually requires the nonsequential ray paths in order to determine the ray paths that you want to control, then you should not turn on this switch.
