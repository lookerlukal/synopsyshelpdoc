2.1 Input Commands                                                Previous Next TOC

 

When you first enter SYNOPSYS, you will be at COMMAND LEVEL.  This level accepts many commands, including some that will cause the program to jump to a higher level.  The prompt at command level is

 

SYNOPSYS>. or SYNOPSYS AI>   (if AI mode is turned on)

 

If you enter a command that goes to another level, the prompt will change, indicating new input requirements.  For example, if you enter an image analysis command, or just the command IMAGE, the program will accept another whole set of commands related to image quality assessment, and will prompt with

 

IMAGE>.

 

The valid input at this prompt is described in Chapter 8 of this manual.  Many of the dialogs contain data that go automatically to another level.  In these cases you do not have to be concerned about the command formats, since the dialogs handle that for you.

 

Much user input can be typed at the keyboard in real time, but if you want to type more than a few lines it is more convenient to use the MACro editor, which can prepare a group of lines and then run them all at once.  A few semantic terms should be defined before we get too far:

 

OPERATION COMMAND

(or just COMMAND)

a self-contained command.  Usually independent of the previous command, and not including AI sentences or interactive PAD commands.  See below.

 

INPUT FILE

a sequence of input commands used to exercise a given feature.  These require certain mnemonics in the first and last line.

 

FILE HEADER COMMAND

the first line of an input file.

 

The set of all input commands must end with the command

 

 EXIT,

 

which terminates program execution.  (You can also exit by clicking on the Close button at the upper right of the main window, or with File|Exit.)
