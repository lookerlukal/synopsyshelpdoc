5.58 Recast Zernike Surface (RZS)

                                 Previous Next TOC

 

The command RZS will recast most kinds of surface shapes into a ZERNIKE surface.  The format is as follows:

 

{ RZS / RZP / RZB } sn  [ SYMM  / RSYMM  [ CAO / 0 / rnorm [ RFIX ]]]

 

If the optional SYMM is included, only terms with bilateral symmetry will be used. RSYMM requests only terms with rotational symmetry. The optional rnorm is the aperture over which the polynomials are orthogonal; the default is one lens unit; if CAO is entered, the current clear aperture of the surface is used for rnorm.  The optional RFIX fixes the current RAD of the surface and only varies the Zernike terms.

 

If the command is RZS or RZB, the program calculates the sag of the original surface sn at an array of points over the surface.  Then it submits an optimization run that varies all the enabled coefficients plus the base curve (if not RFIX) and targets the surface sag at those locations.  When this completes, the surface has been redefined and has a shape very close to that of the previous.  Then one can optimize the new system, varying the G terms as usual.

 

The command RZP calculates the slope of the target surface at each array point and adds a target for the slopes in X and Y.  This may help avoid ray failures when the shape is changed.

 

The command RZB calculates both the sag and slope, and targets both.

 

To illustrate this feature, we start with the relay telescope and cast the primary mirror to a Zernike surface.

 

FET 4

RZS 1 SYMM

These aspheric terms have been added to a new spherical base curve that has a radius of -309.87495 inches.  The fit is essentially perfect, even though the original radius was -98.85 inches.  This happens because there is more than one combination that makes a good fit; indeed, Zernike terms added to a flat surface can also make a good fit.  So the program varies all the terms, plus the base radius, and optimizes all of them together.

 

If you want the base radius to remain the same, simple add the RFIX entry to the input.
