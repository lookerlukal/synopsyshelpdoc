10.21 Automatic test for G-variables (AGT)                                Previous TOC

 

The aspheric shapes in SYNOPSYS are expressed by a set of G-terms, the number depending on the shape involved.  Zernike surfaces, for example, allow 39 such variables.  When one suspects that adding aspheric terms to a given surface will improve the merit function (and it usually does), one must decide which terms to use; not all terms are equally useful.  Of course, one can simply vary all allowed terms for a given shape, but that is not always a good idea.  If some terms are actually doing little good, it is best not to vary them.  The raytrace and optimization usually run faster if fewer terms are involved.  The AAA feature can determine where in your lens an asphere will do the most good.  AGT is used to determine which G-terms are most effective on a given surface.

 

Input is as follows:

 

 AGT LLIB [ QUIET / 0 ] SN DELQ GN GN GN ...

 

 PANT

 ...

 END

 AANT

 ...

 END

 [ SNAP ]

 [ DAMP ]

 SYNOPSYS NPASS

 

In this input, LLIB is the library location that will receive the final lens. SN is the surface number in the current lens to test, DELQ is the minimum allowed fractional improvement to the merit function required before a given term is accepted, and the GN are the terms to be tested.  This line must be followed by the usual PANT and AANT files plus the optimization commands.  Up to 20 G-terms may be tested at a time.

 

The program will first optimize the current lens with the entered variables and merit function.  Then it applies each of the G-terms to surface SN in turn, optimizes the lens with this added variable, and if the fractional improvement in the merit function is greater than DELQ, that lens is retained and the next G-variable is investigated.  If the improvement is less than the threshold, that G-term is dropped and the next one in the list is tested.

 

When the process completes, the lens with the terms that were retained becomes current and is saved in LLIB.

 

Some cautions:  First, save this feature for when the lens is as good as it is likely to get otherwise.  It makes no sense to add aspherics to a lens that was still improving without them.  AGT will first optimize it with no added terms, but you should do that task yourself beforehand to be sure you are really at the optimum.  Second, when you see which terms were useful, you must add those terms yourself to the PANT file.  They remain in the lens, but they are not automatically added to the list of variables in the PANT file afterward.  Third, the terms you enter should make sense with the current surface shape.  If a surface has no aspheric terms, the program will interpret the G-terms as ordinary aspherics as described in Section 3.3.2.2.2.  If the surface has a different aspheric shape, that shape is retained and additional terms are applied as described above.  Lastly, we have found that this feature may not be appropriate for Zernike surfaces.  One can indeed often improve the image substantially by using that shape, but adding a single term by itself, as AGT does, can fail to see the improvement one gets by applying several terms at the same time.

 

Example:

 

! AGT.MAC

 

GET 6        ; THE STARTING LENS

 

AGT 5 0 6 .01 3 6 10        ; CHECK IF TERMS G 3, G 6, AND G 10 WILL BE USEFUL

 

PANT P        ; DEFINE THE VARIABLE PARAMETERS

VLIST RAD 1 2 3 4 6 

VY 1 TH 20 3

VY 2 TH

VY 3 TH 20 3

VY 5 TH

VLIST GLM 1 3

END

 

AANT P        ; DEFINE THE ABERRATIONS TO BE CORRECTED

AEC         ; AUTOMATIC EDGE CORRECTION

ACC        ; AUTOMATIC CENTER THICKNESS CONTROL

GNR .5 1 3 2 0

GNR .5 1 3 2 .5

GNR .5 1 3 2 .7

GNR .5 1 3 2 1.

GNR .5 1 2 1 0

GNR .5 1 2 3 0

GNR .5 1 2 1 1

GNR .5 1 2 3 1

END

 

SNAP         ; REQUEST SNAPSHOTS AS OPTIMIZATION RUNS

SYNO 44 ; REQUEST OPTIMIZATION FOR 44 PASSES

 

If the merit function improves by more than 0.01 of its previous value, the term in question will be retained.  In this example term G 10 was not retained because it did not improve the merit function by the required amount.
