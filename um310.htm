3.10 Edge descriptions                                        Previous Next TOC

 

The lens data described in this chapter are adequate for designing a lens and showing it with a variety of drawing features.  But the apertures that are shown will be just the clear apertures (CAO’s) that are in effect (either default or user-entered), with edges constructed by the drawing programs to make the elements look realistic.  For more precise control over the edge geometry, additional data are needed.  These data may be entered into the RLE file in the formats described below or created with a separate feature, the EFILE.  But the fastest and easiest way to define your edges is with the Edge Wizard, where you can create, delete, and edit your edge data while watching the results on the PAD display.  Some of the edge parameters are coupled to others, and the Wizard shows these connections onscreen, which greatly facilitates your understanding of the situation.

 

To enter EFILE data into the RLE file, you must select a mode from the following (see the link to the Edge Wizard for a description of how these options work):

 

SN EFILE OFF

No EFILE is in effect for this surface.

 

SN EFILE UP

Edge coordinates are calculated upwards, starting at the CAO

 

SN EFILE DOWN

Edges are calculated downwards, starting at the entered diameter

 

SN EFILE MIX

A combination of UP and DOWN

 

SN EFILE EX1 …

Edge data are entered explicitly; see below for data format

 

SN EFILE MIRROR …

For a mirror in air, this activates drawing the rear side as well

 

SN EFILE CMIRROR ...

Rear side of mirror copies curvature of front side.  Other EFILE dimensions do not apply to this type.

 

An edge description involves three coordinates at each surface, A, B, and C.  For a definition of these quantities, see the link above to the Edge Wizard.  The input formats for the RLE file are as follows:

 

SN EFILE DIA DIA

Specifies the value of (2*C)

SN EFILE ERAD C

Specifies the value of C

SN EFILE BEVEL BC

Specifies the bevel dimension BC

SN EFILE FACE AB

Specifies the face dimension AB

SN EFILE ANGLE ANG

Specifies the face angle ANG

 

The above entries apply to the modes UP, DOWN, and MIXED.  In some cases, entered data are overridden by other data, since the input possibilities exceed the number of degrees of freedom.  This is made clear in the Edge Wizard, where the unneeded options are grayed out as necessary.  After you have created your edge in the Wizard, these data are saved with the lens, and you can ask for a LEO listing to see which of the above options are actually required for your case.

 

Some options require additional data describing a MARGIN and a BEVEL, which are also explained in the Edge Wizard.  These data may be entered in an RLE file with

 

MARGIN MARGIN

 

and

 

BEVEL BEVEL

 

To specify the edge parameters explicitly, use the forms

 

SN EFILE EX1 A B C ANG1

SN EFILE EX2 E D ANG2                (with the same SN!)

 

And for a mirror, follow the above with the entry

 

SN EFILE MIRROR MTH, which gives the thickness to show up on the drawing.  (All three lines are required for this form.)  Here is the input one might apply to the file RTLOG141.RLE to add a beveled back to the large primary mirror:

 

1 EFILE EX1     8.011420     8.011420     8.061420     0.000000

1 EFILE EX2     3.000000     7.961420    25.000000

1 EFILE MIRROR    3.000000

 

The bevel now appears on the SOLID drawing:

 

 

The above example shows that with this type, the back of the mirror is axially symmetric.  That might not be appropriate if the mirror is strongly curved or decentered.  In such cases, the form

 

SN EFILE CMIRROR MTH

 

might be preferable.  The back surface is then a copy of the front surface, with the same borders, slope, and curvature.  It is also the only option for mirrors that honors a UAP aperture.

 

The explicit mode is the only mode that works properly for both elements in a cemented doublet.  The other modes transfer dimensions from the second element to the first, which is usually not what you want.

 

The explicit mode requires more than just the (ABC) coordinates of the other options, for the following reason: the RLE file is a surface-oriented input structure, while the complete edge description of an element requires the (ABC) coordinates of both sides of an element.  With the other options the program combines opposite faces to gather the full edge definition, creating points E and D from points A and B of the second side and modifying the data as required by the mode employed – while the EXPLICIT option lets you enter the entire definition yourself, for maximum control.  If you use this form, data for the second side are not strictly required, but should be entered anyway, with EX2, because the program requires that both sides of an element utilize the same EFILE mode and will give an error message if there is a mismatch.  If this sounds complicated, it is.  Use the Wizard.  You will find it at MPW.

 

Once entered, the EFILE data will appear on all of the lens drawing formats except PAD/P, provided switch 39 is ON.

 

See the link to EFILE, above, for more options including a listing of the current data.
