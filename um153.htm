15.3 Expert systems                                                Previous Next TOC

 

A second major part of the SYNOPSYS artificial intelligence capability is the expert systems program, XSYS.  This program can often find new lens configurations for you.  The DSEARCH program is also good at finding starting points for optimization.

 

Normally, in the field of AI, an 'expert systems" program is one that employs a tree-structured logic wherein decisions are derived from the responses of a number of experts in a particular field to a lengthy debriefing.  Remarkable performance has been achieved in some areas, rivaling that of a human expert.

 

The SYNOPSYS program takes a somewhat different approach: the program is presented with a number of finished lens designs, the products of expert designers, which represent the state of the art.  Using these lenses as models, the program analyzes the optical properties in great detail, determining for each lens the first and higher-order properties, and the aberrations that are present in the beam before and after each element.  In so doing, the program "learns" how a particular optical problem was solved.  The more examples it is given, the more it learns.

 

Then, when presented with a new problem or a lens that is not well corrected, it can determine if the current problem resembles one for which it knows a solution.

 

This feature is unique among all the programs in SYNOPSYS in that its performance is not built in.  When the program is first installed, and without an example file, XSYS is ignorant.  Most engineering programs give a certain output in response to a certain input, but XSYS has to be educated before it will work.  And if the example file contains only microscope objectives, for example, it will probably not be of use designing infrared telescopes: the examples should represent more or less the kind of systems for which the program will be used.

 

We should note here that XSYS is not the only feature of SYNOPSYS that can derive entirely new lens configurations.  See Section 10.14 for a discussion of the DSEARCH and SPBUILD programs, which utilize a procedure altogether different from that of XSYS.

 

When the program XSYS is entered, it prompts with the following menu:

 

Welcome to the SYNOPSYS expert systems program.

 

Select one of the following input commands:

 

 

 

M

(this menu again)

START

(to start a new lens from scratch)

ALTER

(find alternate starting point)

REENTER

to re-use the last match list)

LIST

(to list example lenses on file)

USE

(to use one of the example lenses)

INITIALIZE

(to generate a file of examples)

ADD

(to add a lens to the example file)

END

 

 

 

XSYS can be used for three broad purposes:

 

1.

Any of the example lenses may be retrieved intact for use as a starting point.

(Example: USE 4.)

 

2.

A general set of requirements may be input, and XSYS will find the lens (or portion of a lens) that most closely matches them (according to various weights input by the user) when the scale and aperture are adjusted as well as possible.  This lens can then be used as a starting point. (START)

 

3.

If you provide a starting lens, XSYS can analyze the beam everywhere in the lens and determine if a similar set of aberrations or first-order properties exists in any of the examples -- and if so, how the aberrations were corrected or how the first-order properties were achieved.  It will then suggest alternate starting points based on this information.  This feature is perhaps the most powerful ability of XSYS. (ALTER)
