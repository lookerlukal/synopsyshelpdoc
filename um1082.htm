10.8.2 Derivative calculation                                                Next Previous TOC

 

Any variable that yields the same merit function when incremented will be dropped for the remainder of the run.  This usually happens when a pickup or solve is accidentally left on that variable, but it can also occur when the increment is too small compared with the initial value of the variable.  For example, if you vary YP0 in a system with an infinite conjugate, the default increment (.001) would be inappropriate since the starting object height is on the order of 10**12 units.  In such cases, a finite object distance of about 10**4 and an increment of .01 should be input.  In general, the increment should be about 10**-5 times the variable.

 

If switch 7 is on, new variable increments will be calculated at each iteration except the first.

 

SYNOPSYS takes a single increment to find the derivative, unless switch 67 is on, which causes it to take steps on both sides of the current value for greater accuracy.
