MFB Field Blur Menu

 

SYNOPSYS has several features to evaluate the performance of a lens over a finite field.  The Image Tools Menu (MIT) can evaluate the effect of aberrations and diffraction on the image of a variety of extended targets.  This feature assumes that the extent of the target is sufficiently small that the image quality is unchanged over that target.  This condition is known as isoplanatism.  Another feature can plot the diffraction point-spread over the field.  This is MPF.  Another feature, MAP, provides a way to combine both goals: you can analyze the performance over an extended field, and get high-resolution geometric images at each point at the same time.

 

The Field Blur Menu (MFB) can generate a variety of targets, including grids of points and lines, and then evaluate what the image of this target would look like.  It can also evaluate the image of an extended source, such as a bitmap image of a photograph.  The calculation does not assume isoplanatism, and is valid over the entire field.

 

The MFB feature first samples the requested image with a resolution of 400 x 400 pixels, and then traces a pencil of rays from each of 400 x 400 points at the image plane, each starting from the object point that is conjugate to that image point.  The end points of each of these rays then accumulate in the bins at the image, and the total distribution of those rays is displayed as the processed image.

 

Because the resolution of the process (1/400 of the total object extent) is rather coarse by optical standards, it is unlikely that diffraction effects will ever turn out to be so large that they have a noticeable effect on the ray distribution.  For this reason, MFB is a purely geometric analysis.  It will show very nicely, however, the appearance of a target that is distorted by the lens, and this is its main application.

 

The program works in one of two modes:  If the lens exhibits axial symmetry, the aberrations of any point in the field are duplicated at the point opposite the axis, and the program only has to sample 1/8 of the total object extent.  If there are any tilts, decenters, non-symmetric aspheric terms, global coordinates, or decentered apertures in the system, however, it must evaluate the image quality at each object point independently, a process that takes about eight times longer.

 

Because the program must trace rays from skew field points for this feature, it will automatically alter the object description if necessary so that this becomes possible.  If the lens has no skew field at all, the program will reduce the value of YP0 by 0.707 and create a value of XP0 that is equal to this amount.  Thus, the lens then has a rectangular (square) field of view, the corners of which are at the same off-axis distance as the original meridional object, so the new square object is inscribed inside the a circle whose radius is the original YP0.

 

If the lens has a nonzero skew field originally, the program simply alters the value of XP0 so that it is equal to the original value of YP0, making the field square.

 

The program then evaluates the field points over the resulting values of YP0 and XP0.  If the lens has the CFOV attribute set, the program discards all field points that fall outside a circle inscribed in this square field of view.

 

There are numerous buttons in the MFB dialog:

 

SPOTS B/W

Creates a grid of small spots in black on a white background.

 

SPOTS W/B

A grid of white spots on a black background

 

GRID B/W

A grid of black lines on a white background.

 

GRID W/B

A grid of white lines on a black background.

 

Circles

Here you can specify the size and spacing of a grid of circles.  The arguments give the number of pixels of the radius and spacing, which must be from 1 to 400.  The screen is scaled so that the full-field point is at 200 pixels from the center.

 

Squares

Here you can specify the size and spacing of a grid of squares.  The arguments give the number of pixels of the width and spacing, which must be from 1 to 400.  The screen is scaled so that the full-field point is at 200 pixels from the center.

 

Open Bitmap

This button displays the File Open dialog, where you can select a file with the extension .bmp.  The program will discard any bits that exceed the 400 x 400 pixel size of the object array, so for best results you should use your graphics program’s resize feature to resample a larger image at this resolution, and then save the result.  A smaller file will only partially fill the object array.

 

Load from Clipboard

If a bitmap file has been placed on the Windows Clipboard, this button will load it into the object array, subject to the same size considerations noted above.

 

Show Raw

Displays the raw object array.

 

Process/

Show Processed

Combines the object bitmap with the spot diagram of the lens at each field point.  After processing is complete, this button changes to Show Processed, letting you flip from the raw to the processed image easily.

 

Print

This button prints an image of the entire dialog on the selected printer.

 

Help

Opens this section of the SYNOPSYS help file.

 

Close

Closes the dialog.

 

Image Tools

Opens the MIT dialog, which is suitable for high-resolution analysis of an isoplanatic image patch.

 

Clicking on this button puts a bitmap copy of the displayed image area on the Windows Clipboard, from which it can be imported into any graphics program.

 

Dither

If this box is checked, the program will interpolate the region near each object point, producing a display that is somewhat smoother than otherwise.  This effect is most pronounced when processing the grid options.

 

Be aware that if you click the Print button, the program creates a bitmap copy of the dialog on the Clipboard, and then prints it – and if you later want to load from the Clipboard, the new file will show up, which is most likely not what you had in mind.

MFB only works with object types OBA, OBB, OBC, OBF, FINITE, and INFINITE.

 

If your lens is monochromatic, you might expect that the displayed image will show up in that single color.  But that will happen only if you first change your photo to black & white. The reasons are complicated, related to the fact that we cannot display that single wavelength, but have to simulate it with the (RGB) color coordinates of that wavelength.  If the target is B&W, the problem goes away.
