3.1.4 Gaussian Input Profile                                        Previous Next TOC

 

It is sometimes convenient to model the input from a source with a Gaussian profile using object format OBA instead of OBG.  This applies, for example, to a laser diode, where the output is approximately Gaussian, but you wish to treat the beam with ordinary geometric optics rather than the special tools of OBG.

 

In these cases, one can simply follow the OBA descriptor with the additional line

 

 GAUSSIAN RBS

 

in the RLE file.  RBS is the relative beam size, specifying where in the Gaussian profile one finds the full-aperture points YMP1 and XMP1 (which can be different).  Thus, if the entered YMP1 dimension were to be exactly at the 1/e**2 point, then RBS would be 1.0.  This option differs from the OBG format in several ways:  You cannot ask for a BEAM analysis, and the first-order object properties are taken from the OBA description rather than recalculated to satisfy the Gaussian-beam requirements as described in Section 3.12.  Also, one can freely use the HBAR and GBAR parameters to specify a point in the field other than the axis – which is not appropriate for an OBG object.  This is how one might analyze a system with an array of laser diodes at the input.

 

This option affects raytracing in a special way.  If you ask for a geometric analysis that would ordinarily generate a set of rays spaced on a rectangular square grid, the program concentrates them toward the center so the ray density approximates the energy density of the Gaussian distribution.  For features involving diffraction analysis or the OPD of the rays, on the other hand, the ray distribution is regular but the intensity of each ray is altered to reflect the Gaussian profile.  This preserves OPD coefficients and the mapping required for convolution MTF analysis.  Thus, a footprint pattern, which is purely geometric, would look like this:
