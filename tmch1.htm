Getting Started with SYNOPSYS™

 

This Tutorial Manual is intended for beginning users of the SYNOPSYS optical design program.  Another volume, the User's Manual, (UM) contains a complete description of the features of the program and how they are used, but since that volume is over 500 pages in length, a new user might understandably feel overwhelmed at the thought of mastering so large a program.  This Tutorial Manual will help you to learn many of the most frequently used features of the program in only a few days.  All users are encouraged to work through this Tutorial from a printed copy. (Just type Ctrl+P when you link to each chapter.  That will print the whole chapter on your current printer.)  After you finish studying this Tutorial Manual, you are encouraged to examine the documents on our website for more detailed instruction.  Go to www.osdoptics.com  and click on Resources and then Knowledge Base.  You will find a list of over 50 lessons that cover many features of the program.

Preliminary Remarks

 

As you begin to use SYNOPSYS, there are of course many things to learn -- and some to unlearn.  If you have used another lens design program, you may have developed some habits related to the organization and features you are used to in that program.  Many of those are very different from the features in SYNOPSYS, however, so let us start off with a list of some of the major differences you can expect to encounter.  Understanding them now will avoid potential confusion later.

 

1.  A lens file is just a lens file.  The lens file in SYNOPSYS is identified by the file type "RLE", which means REad Lens.  That file does not contain any other information such as the definition of the merit function or tolerances.  Those are separate files in SYNOPSYS, and we find it more convenient to be able to edit them separately from the lens file itself.

 

2.  The spreadsheet is optional.  Other programs may show a spreadsheet all the time, but in SYNOPSYS, you have a choice of lens editors.  The spreadsheet (command SPS) shows some data for all surfaces, while a different editor, the Worksheet (WS) shows all the data for a selected surface.  They are both useful -- we use mainly the latter -- and we do not recommend keeping the spreadsheet open all the time.  Use it when you want to, and then close it.  You will have much more power at your fingertips.

 

3.  Lens optimization in SYNOPSYS is by far the fastest in the world.  So fast, that we rarely ask for more than 100 iterations, and usually less than 50.  Users of other programs are sometimes in the habit of asking for 500 iterations or more -- which makes sense with the slow and feeble optimization algorithm found in the program they were using before.  But now you are using SYNOPSYS, so take advantage of the superior power and speed.  You will get better results, much more quickly.

 

4.  SYNOPSYS has a command window (CW), and once you learn a few of the most common commands, you will probably prefer it to the much slower process of clicking on buttons, opening dialog, etc.  Other program make you navigate by clicking all over the place.  In SYNOPSYS, you can do it either way.  We use both approaches, depending on which is easier for a particular task.

 

5.  By all means become familiar with the many sliders that SYNOPSYS puts at your disposal.  There are several to be found in the WS, and the zoom slider is wonderful to behold (and found in no other program).  Zoom lens design was never so easy and so much fun.

 

6.  Read the sections of the UM about the global optimization features DSEARCH and ZSEARCH.  There is nothing like them in any other program in the world, and you should become fluent in their use as soon as you master the basic procedures of SYNOPSYS.

 

 

What You Need to Know Before You Use SYNOPSYS™:

 

To use SYNOPSYS properly some general knowledge is helpful. Look over the list below to see if you are familiar with these topics:

 

First-order optics (Gaussian image height, principal planes, Lagrange invariant, etc.)

 

Snell's law, Seidel aberrations

 

Elementary diffraction theory (Airy disk structure, interference, etc.)

 

Fourier theory (modulation transfer function, or MTF)

 

If any of these topics are new to you, I recommend that you review some of the basic optics textbooks listed at the end of this manual, since terms such as those in the list above are used throughout the SYNOPSYS manuals without any further explanation.

 

Ready to go?  The next few sections of this Tutorial will help you through some simple sessions with SYNOPSYS.  First you will get acquainted with the basic on-screen tools provided for communicating with the program and manipulating your lens.  Once you know how to get data into and out of the program, you will be able to try more sophisticated features with more complicated lenses.  Along the way I will introduce a modest amount of discipline with topics like coordinate systems and sign conventions.

 

 

The First Session

 

Okay, let's get started.  If you’ve reached this page you know how to get into the Help files.  (This Tutorial Manual is part of the extensive set of Help files available in SYNOPSYS.)  But you may not know all of the many ways, or even the easiest.

 

A quick approach for many users is just to hit the F1 key.  You can do this from anywhere, and if the help section that opens up is not what you want, just click on Contents or Index.  The whole Help file is then at your disposal.  Another way is to type HELP, which will go directly to the index.  And when you learn some of the SYNOPSYS commands, you can try HELP + that command mnemonic, to go quickly to that section.  For example, type HELP DWG for an explanation of the lens drawing feature DWG.  And once you are into the program and type a command, the format of that command is automatically displayed in the status bar at the bottom right. We call this the TrayPrompt.  Pressing the F2 key then goes to the help file section on that topic.

 

Once SYNOPSYS starts, you should see some lines of text displayed in the Command Window.  If your installation was not properly completed, or if you are in the wrong directory, you may get a message that says that some needed files are missing, such as the glass tables.  At installation, the program creates a directory called USER in the directory you selected for the installation (or the default if you did not select a different one.)  The parent directory to that one should contain many data files used by the program, such as the glass tables.  Review the installation instructions, and call OSD at 207/633-3711 or send an email to dilworth@osdoptics.com if you cannot resolve the problem.

 

The Command Window emulates a DOS window; this is where you will type commands and where you will see the result.  But it's better than a DOS window because it has scroll bars that let you back up to see what came before.  It will hold about 65000 characters of output, and there are two options that control what happens when it is full, selected with the menu item Options | Command Window.  If you select Prompt When Full, the program will stop when the buffer is full and you can then decide whether to erase the current window or open a new one.  The other option is Recycle Command Window, which will automatically erase the uppermost portion (which is out of sight) and keep writing at the bottom – so it will appear to scroll forever.  This is handier for most users, but if you have generated a large amount of data on the screen, you may find that information near the top of the buffer is no longer there.  In this case, you can use the Prompt option instead.

 

The lines displayed on the screen when you first open the program echo the startup MACro SYSTART.MAC, and the starting definitions of some AI symbols, from files that were installed along with the program.  We will cover these issues in a later chapter.  But first we want to do some simple operations that show you some of the features you have available.

 

When you run the program you usually have a choice of either typing commands directly into the Command Window or clicking on a dialog with the mouse and entering data in clearly labeled boxes.  Both forms of entry are useful, and individual users can choose which way they prefer.

 

We will start by using mouse clicks instead of commands, since this is probably easier for a beginner..  Click on the button in the SideBar that looks like . You will see a listing of the current contents of the lens library.  Click on the first one, which should say “ID MIT 1 TO 2 MICRON LENS” (if your library has not been changed since installation).  Then click OK.  Now click the button on the SideBar.  You’ll see a drawing of the lens on your monitor.  Now click on the button .  This takes you to the handy SPS spreadsheet lens data editor.  Here you can change anything you want, and then click on the “Close” button.  Be aware that SYNOPSYS does not expect the spreadsheet to be open all the time -- as it is with some other programs.  For editing a lens, the WorkSheet is much better, and we suggest you close the spreadsheet when you are through with it.

 

There are many ways to interact with SYNOPSYS, and the SPS feature you just tried is one that most people can use with little instruction.  Some of the others are actually faster and more powerful, however, and you will see examples of them in the following sections of this manual.  The most powerful of all is, of course, the command language.  As you learn to run the program, you will quickly learn which features are most easily run in which way.  This manual describes both methods.

 

Even if you never type a command yourself and do everything from the menus and toolbars, in the background nearly everything is really running from commands automatically generated by those features.  To use commands yourself, simply type them into the Command Window.

 

 

Command-Mode Input

 

A SYNOPSYS COMMAND is a string of characters that makes the program take a specific action immediately.  Type

 

INT

 

and press the <CR> (or "Enter") key.  (You must always press this key after typing a command string.)  You also could have typed the entire word "INTERACTIVE", but it is sufficient to enter only the first three characters.  (You will find later, using the AI (artificial intelligence) feature of SYNOPSYS, and using the glass tables, that for those features more than three characters are sometimes required.)  The INT command puts you in interactive mode, with the AI feature turned off (you will leave AI mode on all the time when you get more experience with the program).  You can also use the toolbar buttons and to turn the AI mode off and on.  SYNOPSYS commands work in either mode, but in AI mode you can do many useful tasks by just typing an English sentence, whether or not not there is a command for that task.  You will see examples of AI input later in this manual.

 

There is another variety of SYNOPSYS command that is not self-contained, but requires additional data on the next few lines. This is called a FILE HEADER COMMAND.  The program does not take any action when such a command is typed until the entire input file is read. But the program does indicate that it is in the middle of such a file by changing the prompt string. When you typed INT you probably saw the prompt

 

SYNOPSYS>

 

on the screen. This is the basic command-mode prompt.  In AI mode it changes to

 

SYNOPSYS AI>

 

When you are in the middle of an input file and see a different prompt, however, the program can at that point process only the remainder of that file, which usually ends with an END or PLOT entry.  So don’t try at that point to click on a menu item or toolbar icon.  You have to finish what you started first – or press the <esc> key if you want to terminate that file without finishing it.

 

The next thing you are going to do is enter the data for a singlet lens, in command mode.  You can easily enter this lens with the dialogs SYS and SPS (the boldface indicates that these are valid SYNOPSYS commands), which let you just fill in the blanks, but it’s really not hard to do with commands – and it’s a whole lot faster, once you know how.  As you learn to use SYNOPSYS you will find that this kind of input is usually typed into a MACro editor before it is sent to SYNOPSYS – but we will do it in real time at first, and use the editor later.  You are going to type the following lines of text into the Command Window.  If you make an error in a line, and find your error before you terminate the line with the <CR> key, you can use the backspace key to fix things up. Do not move the cursor with the arrow keys to try to change your line!  Remember, this is an emulation of a DOS window – which does not honor the arrow keys.)  If you have already pressed <CR> before you find the error, for this exercise you can probably just reenter the line correctly, since this is a lens input file where the order of the input lines is not particularly important. Type the following input lines:

 

 

SYNOPSYS input is almost entirely free form, which means you do not have to worry about which columns you are typing in – but there are some exceptions.  In the input file you just typed you were instructed to type the glass name "SK16" on a line by itself.  This is a case where the usual rules for input do not apply; since the names of most optical glasses require more than three characters, the program treats that line differently.  You can also enter a glass name with a line like

 

1 GTB S “SK16”

 

where the quote marks tell SYNOPSYS that this is a special word and it must consider all the characters.

 

Let me explain what has just happened. After you typed the RLE line, the prompt on the screen changed to

 

RLE>

 

indicating that you were in the middle of a lens file.  (Incidentally, the mnemonic derives from "Read LEns". SYNOPSYS commands are designed to be easy to remember.)  After that you entered a line of lens identification starting with the characters ID.  You can enter up to 39 characters on that line to identify your lens. This information will be printed on your lens listings and graphical plots, along with the current log number. Here you typed the word SINGLET.  This information is just for your convenience, to help you keep track of your lenses.

 

The next line gives a description of the object coordinates.  There are eleven kinds of object description that you can use, most described in Chapter 3,  and all listed in Section 3.1.1.  Probably the easiest way to enter those object data is with the Object Wizard (command MOW), which sorts things out for you.  For this exercise we show how to do it directly in the RLE file.  In this case we are using the OBB form of object, normally the most convenient when the object is at infinity.  The numbers on this line specify only three of the seven possible arguments valid on an OBB line: zero for the marginal ray angle at the first surface (because it comes from infinity); 1 for the semi-field angle in degrees, and 25.4 for the semi aperture of the axial beam on the first surface vertex plane.  There are actually not too many formats in SYNOPSYS that you should learn at this stage – and you can just use the SYS, MOW, or MPW dialogs to enter these data if you forget the sequence.  But you will save much time later on if you learn the first two object formats, OBA and OBB.  We put the lens in units of MM, and then defined three wavelengths with the code CDF.  This entry saves you the trouble of typing the actual wavelengths yourself, but there are other options as well.  The following lines are almost equivalent:

 

WAVL CDF

WAVL .6563 .5876 .4861

WA1 .6563 .5876 .4861

 

The WAVL and WA1 options accept from one to five wavelengths. After you have entered five wavelengths with WA1, you may enter up to an additional five with WA2 ….  This gives you up to ten wavelengths altogether.  Most of the image analysis features have an option to evaluate the image in either a single color or at all of the wavelengths currently defined, and this gives you a ten-color analysis if you have defined all ten wavelengths.  But there is a difference, related to the color order.  The WAVL input automatically sets the middle wavelength as the primary color, and the first and last as the long and short wavelengths for aberration analysis -- provided the number of colors is either 3 or 5.  You can read about this in Chapter 3.  The WA1 option usually requires an explicit color order definition, described in the same section, and this applies to WAVL as well if the number is not 3 or 5.

 

By default, the program weights all of the wavelengths equally, but you may enter your own weights with the options WT1 … and WT2….  The number of weights that are entered must equal the number of wavelengths that were entered.  These data may be entered interactively with the dialogs MMC or SYS, and can be generated automatically with the Spectrum Wizard MSW.

 

So far all this should be pretty easy.  The mnemonic input for surface data is mostly self-explanatory; RD means radius, TH means thickness, and GTB S stands for Glass TaBle, Schott – followed by the glass type on the next line.  The pattern used for most SYNOPSYS input is that used here: a mnemonic followed by one or more numeric entries.  The order of these mnemonic-argument pairs is in this case not important; you could have entered the TH before the RD, for example.  You will also notice that the data applied to a given surface usually start with the surface number, so in most cases it does not matter whether you enter these lines in any particular order either.

 

The mnemonic YMT deserves a little explanation.  This is an example of a thickness solve, where the actual value of the thickness is not entered by the user.  There are many kinds of solves available, described in Chapter 3, which can apply to either a radius of curvature or to a thickness, and which the program honors continuously.  The solve here controls the thickness of surface 2 (which is an airspace) by specifying that the marginal paraxial ray should have a zero height at the next surface, which is the image plane in this case.  Where did the zero come from?  Like most mnemonics, the YMT entry can accept an argument.  If we had entered, for example, YMT .5 on that line, the target value would have been 0.5 mm instead of zero. This is an application of a general rule: if a numeric argument is required after a mnemonic, but not entered, the program will take a value of zero for that quantity. There are also cases where some of the arguments following a mnemonic are optional, and then leaving them out results in a default value, not necessarily zero. To distinguish between these cases, the manual indicates optional data by enclosing the data description in square brackets [ ], while required data are in curly brackets { }.  So we have requested that the lens should always be in focus, paraxially.  After the whole RLE file has been read, the program will calculate the proper value for this airspace.  If we later change, say, the radius of a surface, this airspace will automatically change as a result.

 

The image plane is on surface 3, for which we entered no data at all except for the surface number.  Just entering the number declares the existence of the surface, and since no other data were entered, the program just assigns the default: a flat surface followed by AIR.  Many of the features of SYNOPSYS call for evaluation of the image quality, and this is almost always taken at the last surface in the system.  If, by mistake, surface 3 were left out, the program would then evaluate the image quality on the rear surface of the lens.  That's not a very good idea.

 

When the whole file has been read in, the program will respond with output from the paraxial-raytrace program, giving some first-order properties of the system.

Artificial Intelligence

 

Now let's have some fun. If you want to know, say, the F/number of the lens, you can always look at the SPEC listing -- but wouldn't it be nice to just ask the program for exactly what you want instead of picking through the output for it?  You can do that with SYNOPSYS, using the Al (artificial intelligence) feature.  If the command prompt does not read “SYNOPSYS AI>”, type “AI” to turn this feature on.  Then type the following on the keyboard:

 

FNUM?

 

The question FNUM? is valid input to the AI program.  It responds with

 

SYNOPSYS AI>FNUM?

The F/number or 1/Beam Diam. in afocal mode (FNUM) is        8.38043678

 

You can ask for, and change, a variety of quantifies using AI.  The formats are basically just plain English sentences, described in later sections of this manual.  For now, just for practice, type the questions in the sample session given below.  You type the BOLDFACE text and the program responds with the rest.

 

SYNOPSYS AI>PRINT BACK

The paraxial image distance (BACK) is        412.68628752

 

SYNOPSYS AI>WHAT IS THE SPHERICAL ABERRATION?

The third-order spherical aberration sum (SA3) is       -0.82349474

 

SYNOPSYS AI>SA3?

The third-order spherical aberration sum (SA3) is       -0.82349474

 

SYNOPSYS AI>WHERE IS THE STOP?

The aperture stop is on surface number 1

 

SYNOPSYS AI>PUT THE STOP ON 2

The aperture stop is on surface number 2

 

SYNOPSYS AI>2 TH?

The thickness or spacing of surface number 2 is     412.68628752

 

SYNOPSYS AI>2 TH = 412

The thickness or spacing of surface number 2 is      412.00000000

 

As you can see, the Al program is very flexible; many kinds of sentences will let you examine or change a quantity.  To see a vocabulary list, just ask Al to show you the valid subjects, verbs, and conditions:

 

SUBJECTS?

VERBS?

CONDITIONS?

 

Chapter 15 of the User's Manual describes the Al features at length.  For now, just notice that in the examples above we used the mnemonic SA3 as a synonym for third-order spherical aberration (many words have synonyms), and that when the explicit value of thickness 2 was entered, the program automatically deleted the YMT solve.  At the moment the lens has no focus solve, so let's put it back in. This is easily done with a CHG file, where you give the same kind of input that you could put in the RLE file.

 

 CHG

 2 YMT

 END

 

Even though the program is now in Al mode, it will accept all of the normal command-mode input as well.  For example, to get just the first-order part of a SPEC listing, you can type

 

SPEC SYS

 

which is another useful variation that you should try now.

If you prefer to use the dialog form of this command (so you don't have to type it yourself) just open the dialog MRR (Menu, Real Rays), and if you don't remember that, start with MMM (Main Menu Mode), select MLA Analyze the lens: Listings, plots, then select MRR.  Or you can start with the top toolbar menu Menus and follow the menu tree.  There are many ways to navigate around in SYNOPSYS.

 

The command you just entered is described in the User’s Manual in a format like this:

 

RAY { ICOL / P } HBAR XEN YEN [ SURF / 0 ] GBAR

 

where the underlined arguments are entries that must be replaced by numbers.  The mnemonics you see here are used throughout the manuals: ICOL is the color number, HBAR is the fractional field in the Y-direction, XEN and YEN are fractional pupil coordinates, and GBAR is the fractional field in the X-direction – which does not exist in this lens, since we defined only the field in the Y direction so far.  The notation  { ICOL / P } means that you must enter either a color number or the letter P.

One more word about AI before we finish this chapter.  In order to be user friendly, the AI program will tell you if you have typed a word not in its vocabulary, and let you just reenter that word, instead of retyping the whole sentence.  For example, if you were to ask

 

What is the rrd of surface 3?

 

the program would not recognize "rrd" and would ask you to reenter four characters starting at that word. If you then typed "rad " (four characters -- including the space --, as requested) the sentence would be correctly processed.  Since you are a beginner you will probably make input errors and see this message when you intended to enter a command that is not an AI sentence.  You can then just press the <esc> key to start the line over, or hit the Enter key without typing anything – or you can turn off AI mode if this turns out to be a problem.  (Type INT, or click on the button ).
