10.3.11 CLINK Command Link to Aberrations                                Next Previous TOC

 

Section 10.3.10 describes how to construct a composite aberration that is made of quantities that are normally available to the optimization program.  But sometimes you need to control the value of something that is not otherwise available as an aberration.  The CLINK operation can submit a SYNOPSYS command string during optimization, and then access the results via the AI output buffer.  (AI mode does not have to be turned on for this to work.)

 

The definition of this kind of aberration is very similar to that of the composite aberration.

 

M TAR WEIGHT CLINK

--- any single-line SYNOPSYS command ---

CD1 FILE nn

CD2 …

…

= equation

 

The word "CLINK" is entered instead of COMPOSITE, and the next line must contain a valid SYNOPSYS command.  Here there are restrictions.  You cannot use an AI sentence, and the command must be self-contained, without continuation characters such as "/".  You cannot use the EM or EAM command to execute a MACro, but you are allowed to use the GADS command to re-run any MACro that has been run with the EAM command or by clicking on the button before the optimization starts.  There cannot be any symbol substitution in the command other than the Zn parameters.  Only commands that place a copy of their output into the AI buffer will work, since that is where the data to be transferred to the CDn variables are found.  As with COMPOSITE aberrations, the CDn must be defined in numerical order, and the definition must end with an equation, starting with the "=" sign.

 

For example, suppose you want to control the phase difference between the S and P polarizations of the chief ray emerging from a lens.  If you run the PRAY command and then look at the buffer (with "BUFF?") you see the two phases in file locations 4 and 8.  So here is the aberration:

 

AANT

M 90 1 CLINK

PRAY P 1

CD1 FILE 4

CD2 FILE 8

= CD1 - CD2

END

 

This will target the phase difference to exactly 90 degrees.

 

Another example:  Suppose you want the weight of an element to be exactly 22 grams.  The WGT aberration is already available, but it does not honor EFILE edges, while the WGT command does.

 

M 22 1 CLINK

WGT 3

CD1 FILE 4

= CD1

 

This will control the desired weight.  Of course, EFILE edges must be assigned for this to be different from the WGT aberration, and switch 39 must be on for these edges to be honored.

 

If it is necessary to control additional quantities that have been placed in the AI output buffer, in subsequent aberrations, it is not necessary to re-run the command sequence that created that output.  Instead, use the COMPOSITE form on a subsequent aberration definition (this form does not use a command line in the syntax), and simply pick up the previous FILE variables with CD1 FILE nn, etc.  All of the entries in the AI buffer remain there until another command has changed them, and they may be accessed by as many COMPOSITE aberrations as you need, once the command has been executed.  But remember: each COMPOSITE or CLINK that has to redefine anything must start all over with CD1, CD2, in numerical order, which will overwrite the previous definitions.  Any CDs that were defined in a previous aberration and which are not overwritten may be reused, if the value is what you want, since they also stay defined until changed.

 

Note that if you want to execute a multi-line command with CLINK, you must put the sequence into a MACro before running the optimizing program, and run that MACro with the EAM command or with the button.  This executes the MACro and saves a copy of the command strings in the alternate MACro buffer.  Then run your optimization as the main MACro, and use the CLINK aberration with "GADS" in the command line.  Get Alternate Data Set  (GADS) simply re-runs whatever command strings are in that buffer.  (You cannot use the EM or EAM commands, however, in the CLINK aberration definition itself.)

 

This form of CLINK is more flexible than the single-line form, and the alternate MACro can contain many but not all of the usual SYNOPSYS commands and AI sentences.  For example, here is how you could minimize the weighted RMS spot size at three field points:  First make the alternate MACro, containing

 

! Calculate multi-color RMS at 3 fields:

 

rms m 0 300 0 0

z1 = file 1

Get the first RMS, on axis, and load the result into variable Z1

rms m .7 300 0 0

z2 = file 1

Do field point 0.7 …

rms m 1 300 0 0

z3 = file 1

and 1.0.

= (z1+z2+z3)/3

Calculate the average RMS for these field points.

 

Run this MACro with the button , and then make the optimization MACro as follows:

 

pant

(variables)

end

 

Define the variable parameters

aant

(other aberrations)

Start the merit function definition

m 0 1 clink

Create the CLINK aberration

gads

Execute the alternate MACro

cd1 file 1

 

The MACro left a copy of the calculation (z1+z2+z3)/3

In the first buffer location.  Here we assign that value to CD1.

= cd1

This completes the CLINK aberration, returning the value of CD1.

end

 

snap

syno 100

 

Completes the merit function definition

 

In this simple example you would get much the same result with GNR aberrations, provided the image centroid were close to the chief-ray point.  Even more the same would be the effect of the GNN aberration, which measures from the image centroid.

 

 

A final example of CLINK:  Here we want to minimize the departure of an asphere from the closest-fitting sphere.

 

M 0 10 CLINK

ADEF 1

CD1 FILE 6

= CD1

 

Here, the RMS departure has been loaded into FILE location 6 by ADEF, and we transfer that value to the variable CD1, which becomes the aberration to be minimized.

 

There are restrictions on what can go into the alternate MACro in a CLINK aberration.

 

The following are forbidden:

 

1.  Optimization files, PANT… and AANT….

2.  ACON pickup files

3.  BTOL files

4.  IRG, AIM, TPM, or any other command that runs its own optimization.  These will create files that destroy the optimization files containing the CLINK aberration.

5.  GET, FETCH, CHG, or other commands that change the lens.  You might get away with these if you get the lens back when the MACro ends.  The program has to see the effects of taking derivative increments, and these go away if you change the lens yourself.

 

See also Section 15.1.4.2, which describes how the Zn parameters may occur in the MACro to be run with CLINK.  Since the Zn can be optimization variables (with VZN), this lets you vary any numeric parameter that can be put into a MACro, while controlling any output that is placed into the AI output buffer.  The power and flexibility of this feature are quite wonderful.

 

The stop-sign button, normally used to terminate a lengthy operation that you do not wish to continue, is disabled while a CLINK aberration is being evaluated.
