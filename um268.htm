2.6.8 FNO option                                                                Previous Next TOC

 

Yet another way to adjust the paraxial value YMP1 is with the FNO solve.  The format is

 

 FNO value

 

where value is the desired F/number of the lens.  This parameter, which goes in the RLE or CHG file, should be used only when the lens is fairly stable, since the resulting aperture value might become chaotic otherwise.  It is not compatible with any UMC solves in the lens, since those also control the F/number.  The program adjusts YMP1 until the absolute value of FNUM equals the target value.  The target may be entered or modified with the SYS or MPW dialogs.

 

To remove the FNO solve, enter a value of 0.

 

Be advised that this method of controlling the F/number is very unstable.  There may not be a solution with just any set of curvatures, and it will almost certainly cause GSEARCH to hang up, since the lens is then by no means in a stable configuration.  Therefore, we recommend controlling the F/number with a UMC solve or with an FNUM target in the AANT file, instead with this parameter.
