10.4 The SYNOPSYS Command                                        Next Previous TOC

 

When a parameter file (PANT ... END) and an aberration file (AANT … END) have been entered, the automatic design program is initiated with the commands

 

[ DAMP DAMP DLIMIT ]

[ SNAPSHOT NPAS ]

[ DERIVATIVES ]

[ EVALUATE ]

[ SYNOPSYS NPASSES [ MULTICONFIGURATION / 0 ] [ FIX nb ]

 

 

DAMP DAMPING DLIMIT

The initial damping factor.  It is not usually necessary to change this for the PSD method. DLIMIT is the largest damping allowed.  The default limit is 1.0E10.

 

The initial default damping value is 1.0.  If switch 82 is turned on, – causing the program to calculate second derivatives on the first pass – a lower damping will usually work better, and the program uses a default value of 0.01 in that case.

 

The value you enter becomes the new initial value for the next optimization run.  The optimization program then adjusts the value as it runs.

 

You can ask for the current damping value with the AI sentence DAMPING? and set a different value with

DAMPING = number.

 

[ SNAPSHOT ]

requests real-time update of the PAD display.  See section 13.3.4

 

NPASSES

is the number of iterations to be made.  This must be a number greater than zero.

 

Users of older optics codes -- whose optimization is not as robust as that in SYNOPSYS -- may be in the habit of requesting a large number of passes, often 600 or more.  This is a mistake.  SYNOPSYS works so much faster than the other codes that a number from 20 to 50 is usually sufficient.  If one runs the simulated annealing program (which is almost always a good idea) the same logic applies.

 

It is often wise to ask for several short runs instead of a single long one.  Thus,

 

SYNOPSYS 30

SYNOPSYS 30

 

is often better than

 

SYNOPSYS 60.

 

This is particularly true if glass models are to be varied.  In those cases, the glasses can become pinned to the crown or flint boundary early on, but might want to go elsewhere later in the run.  A separate run starts with all variables free.

 

To illustrate the above advice, we ran a simple optimization, requesting 900 passes, and got a merit function of 0.01496 in 17.8 seconds.  Then we ran the same job, requesting 100 passes followed by annealing (ANNEAL 10 5 100) and got a merit function of 0.0133 in 10.8 seconds.  This is usually a better procedure than just asking for a large number of passes.

 

[ MULTI ]

requests a multiconfiguration optimization.  See section 10.7.

 

[ FIX nb ]

This option only affects cases where the initial system exhibits ray failures.  If that happens, it automatically executes the Fix Ray Failure feature, running it repeatedly until all of the rays trace, or up to the number of attempts given by nb. The result is the same as if you click the button one or more times.

 

If the initial lens is so badly constructed that the process does not converge in a reasonable time, you can also click the stopsign button to stop it.

 

The correction makes use of the ERROR ray parameter, as outlined at that link.

 

This option does not correct cases where the ray failure is really due to a real-pupil search failure.  That error can be caused by either a ray failure or the presence of a caustic in the beam at the stop.  Although such errors are also flagged as a ray failure error, they cannot be fixed in this way.  This is especially likely to happen if pupil option WAP 2 is in effect, since that option must iterate rays to the center and both top and bottom of the stop.
