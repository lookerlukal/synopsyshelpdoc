10.3.15 Automatic Zoom-lens airspace control (AZA)                        Next Previous TOC

 

When designing a ZFILE zoom lens, one naturally wants the moving elements to not collide with each other as a result of the ZDATA variables.  This can be done for an individual case by simply giving a target to any offending thickness in whatever zoom produces the collision.  A simpler way, which only works on a single configuration where the airspaces are supposed to be positive, is

 

 AZA [ TARGET WEIGHT [ WINDOW ] ].

 

The program will monitor the airspaces on both sides of every zoom group at all zooms, and compare the value to TARGET.  If the value is larger, there is no penalty.  The penalty goes as the square of the violation if the error is less than WINDOW, and linearly thereafter.

 

The program will also monitor the distance between the adjacent surfaces at the current CAO aperture and add a penalty if that is less than TARGET.  This monitor should therefore avoid collisions between moving elements and their neighbors.

 

This is one of several kinds of monitors that operate continuously during optimization.
