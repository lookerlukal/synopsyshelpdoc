15.2.2 Al Possibilities                                                        Previous Next TOC

 

The previous sections have explained the rules and concepts of AI in some detail.  But perhaps the best way to illustrate its potential is with a selection of representative examples of useful possibilities.

 

Plot back as wavelength varies from .3 to .8

 

(Note that BACK will not be affected unless a solve is in effect; and the indices should all permit recalculation.)  You also might plot one of the third-order aberrations as a function of wavelength, in this way.

 

Increase the field angle by .1

 

Normally, to change an object parameter, the entire OBA or OBB line must be retyped in a CHG file (or edited with the PAD features or by LE). Al can alter any of them individually, using the names given in section 3.1.1 (YMP1, UPP0, etc.).

 

What is ZA for YEN = 1 at full field on surface 3?

 

In order to evaluate a ray-based quantity, the RAY command may be entered in command mode, but the desired quantity must then be picked out from a listing. Al can find it for you.

 

@: Find YG for YEN = 1 on surface 6.

 

Here we have defined a symbol, "@".  If the aperture of surface 6 wants to exceed a critical space boundary, typing that symbol will give the current position of the troublesome ray intercept, in global coordinates.  This might be a simple way to keep track of the problem.

 

VV: VAR 2 0 200

 

Typing VV will go to the image program and evaluate the wavefront variance.

 

 FF: PANT / VY 6 TH / END / AANT / M 0 1 A DELF / END / SYNO 10

 

Here is a multiline sequence that will run a quick optimization to refocus the lens by varying thickness number 6.  All you need to type is FF to run this sequence.

 

Plot YC for YEN = .9 to 1

 

Suppose the lens has an annular aperture, and rays over most of the pupil will not trace.  This sentence gives a TFAN over only the last tenth of the pupil.  (For this example it might also be necessary to use the "ICR" option.)

 

Z1 = MXSF

= Z1 – 1

Z1 = FILE 1

steps = Z1

Do macro for AIP = 1 to 10

 

MACro: ord = CAO of surface AIP

 

This example shows the AIParameter used in a novel way:  Suppose the lens has 10 surfaces.  First, Z1 is set to 10 and then to 9 by the calculation.  Then STEPS is also set to the number 9.  The goal is for AIP to take on integer values corresponding to each surface number.  (One could of course just say “steps = 9” here, but this example is more general, and the above lines could be saved as a named MACro to be used with any lens, with any number of surfaces.)

 

When this MACro is run, symbol AIP will take on values 1, 2, etc., and these will be substituted for the symbol "AIP" in the MACro – which is the surface number of the CAO question in this case.  The plot will then show clear apertures as a function of surface number. (Note that 10 data points involve but 9 steps.)

 

Do MACro for 5 TH = 5 to 8

 

MACro: NARCISSUS

ORD = FILE 7

 

This will plot the narcissus on surface 7 versus thickness 5.

 

 

DO MACRO AS 9 TH VARIES FROM 0 TO 10

 

MACro:

GHOST R .00029

R .005 ALL

END

ORD FILE 1

 

This example takes a little explanation.  The GHOST program finds all of the ghost images, and then tabulates them in sequence, from the largest to smallest ghost.  It is the smallest ghosts that are usually a problem, since the energy is more concentrated than in larger ones.  For that reason, the GHOST program puts its output into the FILE buffer in reverse order: FILE 1 contains the smallest ghost radius.  (The ten smallest are stored.)  This sentence will plot the size of the smallest ghost (which may come from any surface) as a function of one of the lens thicknesses.

 

To see which FILE contains the quantity you want to plot, just run the feature (GHOST in this example), and then ask to see the buffer:

 

BUFFER?

 

Here is another example using the FILE buffer:

 

 

Do macro for AIP equal 0 to 1

 

MACro:

IMAGE

TRA 2 AIP 0 500

ORDINATE = FILE 4

 

If you do a TRACE, and then look at the buffer, you will see that the lens transmission is in file location 4.  This sequence plots the total lens transmission (which changes with vignetting) as a function of relative field, the parameter changing with AIP.  The example here, using the TRACE command, considers only apodization and vignetting.  To also consider reflection losses and coatings, change this to PTRACE.   Yet another variation uses the AI noun TRANSMISSION, which evaluates all effects, including absorption within the glass – but traces only a single ray.

 

If the system has an OBG object, you can thus see the gaussian envelope, which is modeled as an apodization of the input beam:

 

Plot transmission for yen = -1 to 1.

 

Here is an example of where a 3-character word would not work: the word "TRACE" is a cornmand-mode mnemonic, and "TRAN" is the shortest abbreviation of "transmission" that would not cause confusion.  "Find" and "FINAL" are another example of the same.

 

Plot the SAG of 3 for Y = .5 to 1

Plot the SAG of 3 for YEN = .5 to 1

 

These two examples differ slightly: the first specifies an absolute Y-coordinate, and the second a relative (to the clear aperture) coordinate.  If the CAO is 5, the second plots for Y = 2.5 to 5.

 

Plot YA on 16 at full field for wavl = .4 to .8

 

If surface 16 is the exit pupil, this sentence plots the chromatic aberration of the exit pupil.  Note that if a YPT solve is in effect, and "at YEN = 1" is added, the plot shows the spherochromatism of the pupil, since the defocus with wavelength is taken out.  If there is no YPT solve, and the loop parameter is the field point ("... for HBAR = 0 to 1 "), the plot shows the spherical aberration of the exit pupil.

 

 

Do MACro for AIP = 0 to 1

 

MACro:

increase 5 th by .125

ordinate = YA on surface 9 at YEN = 1

abs = thick 5

 

This example loops on the AlParameter, but does not use it in the MACro.  Thickness number 5 will increase in steps, and will be plotted against the ray coordinate, which is affected by the thickness change.

 

 

steps = 5

Do MACro for AIP = 22 to 22.5

 

MACro:

Change 15 th to AIP

EAM OPTIMIZE

FCO 300

DMTF 2 0 300

ORD = FlLE 10

ABSCISSA = THICKNESS NUMBER 15

 

Here we use AIP to change a thickness, reoptimize the lens (with the alternate MACro "OPTIMIZE.MAC" – the alternate comes back to the main MACro when finished), and ask for a DMTF.  The MTF values are automatically put into the BUFFER, and this case will plot the X-MTF at about 150 cycles/mm as a function of thickness 15.  In this example we ask for only 5 data points.  (We could also have said "... for 15 th = 22 to 22.5".)  The modulus and phase in both X and Y are placed into the buffer.

 

R1:        M 0 1 A 2 YA 0 0 1

R2:        M 0 5 A 2 YA 0 0 .6

R3:        M 0 1 A 2 YC 1 0 1

R4:        M 0 2 A 2 YC 1 0 .6

R5:        M 0 2 A 2 YC 1 0 -.6

R6:        M 0 1 A 2 YC 1 0 -1

R7:        M 0 1 A 2 XA 1 1

MF1: AANT/R1/R2/R3/R4/R5/R6/END

 

Here is an example of an entire merit function constructed with symbols.  You could define a large menu of rays in the startup MACro, and then simply refer to them by symbol when you make your merit function.  You can even make a menu of merit functions: MF1, MF2, etc. – each selecting the rays (or other aberrations) you want.  To use this merit function, simply enter

 

MF1

PANT

…

END

SYNO

 

In the event that the list of rays you want is longer than a single line (which is an AI restriction), you could break it up into sub-symbols:

 

P1:        R1/R2/R3/R4/R5/R6/R7/R8/R9/Rl0

P2:        R11/R12/R13/R14/R15/R16/R17/...

MF:        AANT/P1/P2/R56/R73/END

 

Aberrations can be made into symbols with symbolic weights:

 

w1: 1.0

w2: 0.5

w3: 5.0

r1: M 0 w1 A 2 YA 0 0 1

R2: M 0 W3 A 2 YC 1 0 -.6

 

and so on.  You could define a menu of merit functions, and just redefine the weights when you used them.

 

GG: GRAY 2 0 0 1 SURF 0

GG 6

GG 9

 

Here the symbol "GG" gives only part of a command.  The surface number is entered by hand each time the symbol is typed, giving the coordinate system for the GRAY raytrace.

 

**: RPT .001 10 2 1 3

 

Perhaps the most frequent use of symbols will be simply to streamline the input for often-used features.  The symbol “**" will generate a set of plotted ray fans.

 

The above examples should give you an idea of the extraordinary power of the Al feature, and also show why we find it difficult to describe in the manual.  If a program can only do a few things, the manual just lists them; but there is no end to the possibilities of AI, and it cannot be documented in the same way.  All we can do is encourage you to try it.  If you ask AI a reasonable question, and it does not yet know what it means, by all means send us a note; the feature is under continuous development, and you may have ideas that we haven't thought of yet.  The field of AI programming is virgin territory, and you can be a party to its growth.
