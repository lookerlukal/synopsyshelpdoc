3.1.3 Lambertian Object (OBL)                                Previous Next TOC

 

An important class of optical systems is designed to collect and distribute light in a desired manner, rather than form a sharp image.  This class often involves a light source such as a tungsten filament or an LED, both of which approximate a Lambertian emitter.

 

This kind of source can be modeled with the OBL object descriptor in the RLE file, described in this section.  For a single LED with no reflector around it, this format is appropriate.  To model an array of LEDs plus a reflector, the OBI format is used instead.

 

The format for the OBL object is

 

 OBL TH0 YP0 ANGLE [ XP0 ]

 

Here, the arguments TH0 and YP0 have their usual meaning, but there is no input of YMP1 as is usually the case.  Instead, you give the semi-ANGLE of the source, which is in degrees and must be greater than 0 and less than 90.  Since there is little light at extreme angles from a flat Lambertian source, it is usually best to give an angle of no more than 85 degrees (or less if the light source restricts one to a still-smaller value).

 

The program deals with this type of object in a unique manner: whereas the radiation falling on a distant plane would fall off as the cosine of the ray angle to the fourth power, we choose to model the ray distribution with this function and treat each ray with equal weight.  By so doing one can make a footprint of the ray pattern at a desired surface and get a good visual impression of the energy uniformity.  One can also use the GMODEL and image dissection routines to find the energy pattern explicitly.  If the rays all had different weights, (which would be the way to model a distribution with rays uniformly spaced) a visual impression would be harder to make.

 

As a result of this kind of model, it is not so simple to trace a given ray.  If you ask, say, for the ray at YEN = 1, you will indeed get the ray at the entered angle.  But other rays are remapped into the entrance pupil in a way that approximates the falloff as mentioned above, and as a result you will not usually get the ray that you expected.  If this seems complicated, remember that these systems are not intended for forming images, and one seldom has need to trace a particular ray.  Instead, one deals with overall ray distributions, as illustrated by the example in Section 5.37.

 

A good way to examine the illumination pattern at the final surface is with IPAT.

 

If no value is input for the skew field XP0, the OBL format automatically makes the object square; it extends in both HBAR and GBAR, so you can evaluate the flux from anywhere on a square LED.  The default clear apertures, which are calculated to pass the light at the corner field point, rather than from the object in the meridional plane at HBAR = 1 will therefore appear larger than you might otherwise expect.  Enter a small (but nonzero) value for XP0 if you want to model an object that has a negligible X-dimension.
