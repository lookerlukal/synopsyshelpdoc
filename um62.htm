6.2 Clear Aperture Programs (CAP, EDGE)        Next Previous TOC

 

 

6.2.1 CAP

 

When the input

 

 CAP [ JSSS JSPS ]        (Clear APertures)

 

       CAP [ SN ]

 

is read, the program determines the maximum clear aperture radius for each surface by tracing three rays, normally the full field upper and lower rim rays and the on-axis marginal ray, and prints these in a table.  If the lens has a skew field specified in the object description, the object point for CAP will be in the meridional plane at the same distance from the axis as the maximum skew object (where HBAR and GBAR both equal one).  If the lens has tilts or decenters, the calculation is repeated by tracing from the lower extreme object point.  The largest Y-coordinate required by any of these rays determines the aperture.  The calculation is not performed for any surface at which the user has entered a clear aperture explicitly (with CAO, RAO, or EAO).  If CFOV is in effect, the maximum object point takes HBAR equal to l and GBAR equal to 0. CAP may be run interactively from the dialog MLL.

 

Clear apertures may also be controlled or varied during optimization.

 

If surface numbers are entered, only that portion of the lens is listed.  An example of CAP output is shown below.

The apertures from CAP are utilized by DWG, PER, RPER, SOLID, TOL, and BTOL which calculate them automatically, as well as the SketchPAD feature.

 

The values printed by CAP are not necessarily appropriate for tilted or decentered surfaces for which the apertures are not round.  The footprint programs should be used to determine the aperture geometry in such cases. (See section 8.4 and the dialog MFP.)

 

Additional data may be entered to define the edges of lens elements for the drawing programs.  Those data comprise the edge-description file, or EFILE, and they will override the CAO data if they are present, when those programs are run.  If EFILE data are present, the CAP listing will show an asterisk at the affected surfaces.  To view the EFILE data, use the ELIST command.  See section 3.10.

 

For lenses which exhibit substantial vignetting at full field, either because of intentional stops or because of ray failures, an option is available to reduce the aperture that the program uses when finding the default CAOs.  This is the VFIELD, which may be included in the RLE or CHG file.  If the WAP or VFIELD option is in effect, the rays traced by CAP are based on the reduced pupil rather than the nominal pupil.

 

The CAO values calculated by the CAP program are not used for vignetting calculation for image analysis.  Apertures to be used for vignetting must be entered explicitly by the user, or fixed by the CFIX parameter.  The VFIELD parameters are honored by all analysis features that create a grid of rays.  (The nonsequential features in some cases will honor even default apertures, as discussed in Chapter 18.)
