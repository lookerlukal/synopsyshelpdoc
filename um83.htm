8.3 Influence of Apertures on Image Analysis       Next Previous TOC

 

All of the image analysis programs honor user-entered apertures, although the geometric analysis features, such as the spot diagram, cannot show apodization.

 

The number of rays that pass through the system must not exceed 20,000.  If many rays are vignetted, it is permissible to request more than this number when using the image analysis features, as long as the apertures eliminate the excess rays and the transmitted number is within the limit.  In the case where most of the pupil is vignetted, however, this procedure can require tracing many more rays than are actually needed.

 

 

For cases in which a large obstruction blocks most of the rays, for example, it is more efficient to eliminate the unused portion of the pupil before the rays are traced, rather than to trace a large number of rays, most of which are then discarded.  For this purpose the input

 

OBS [ P1 ] R1

 

should be used, where P1 is the fractional position in the pupil, measured along the Y axis, of the center of the obscuration, whose fractional radius is R1.  All rays within that portion of the pupil are skipped during the raytracing part of the image analysis.

 

The input OBS is distinct from the CAl specification in that the latter is entered into the original lens file (with RLE) and its effect is automatically taken into account by the image analysis and footprint programs (by tracing and discarding rays), while the OBS entry applies only to the group of image or footprint commands which immediately follows it.  Rays vignetted at the CAI surface are traced, while rays vignetted by OBS are not.

 

In cases involving a narrow beam passing through a much larger lens, the value of YMP1 should be about the same size as the beam.  You should avoid the practice of specifying a large value of YMP1 and a small CAO to restrict the beam size, since this is wasteful of computer resources; it makes no sense to trace 20,000 rays to generate a vignetted pupil containing 50, for example.  A second reason why the input beam should match the transmitted beam is that the diffraction-theoretic analysis options, described in section 8.6, first determine the F/number of the full aperture before generating the image model.  This reference is then used to determine the theoretical image dimensions and the sampling area.  If this does not match the actual image size – because a vignetted aperture produces a larger diffraction pattern – the sampling may be incomplete.

 

You are also cautioned that the final beam should have a large enough number of rays to properly describe the image or wavefront.  If you attempt to calculate a geometric MTF using an image of but 10 or 20 rays, for example, the coarse distribution will result in large unpredictable variations in the MTF at higher frequencies.  An image containing 400 or more rays is often adequate.  The upper limit is 20,000.

 

If the lens has OBG input (gaussian input beam), the IMAGE programs will automatically apodize the beam so as to take account of the gaussian intensity falloff with beam radius, and a message to this effect appears in the printout.  It is therefore not necessary for you to input any of the UAP options in order to model the falloff.
