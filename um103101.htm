10.3.10.1 Zn parameters in optimization                                        Next Previous TOC

 

Section 15.1.4.2 describes the Al capability to define a set of 20 parameters, called Z1 through Z20.  These are permanent parameters that can only be defined by Al or with the VZN variables, but once defined, they may be used in a composite aberration in exactly the same way as the CDn definltions.

 

For example, suppose you want the global Y-coordinate of surface 7 to equal the centroid position of a certain image.  SYNOPSYS can calculate the centroid of an image evaluated with the GNN option, but here is how to do it with the Zn parameters:

 

RMS 2 1 300

Z1 = FILE 5

Z2 = YA AT HBAR = 1

= Z1 + Z2

Z3 = FILE 1

 

PANT

VY 7 YG

END

 

AANT

M 0 1 COMPOSITE

CD1 YG 7

= CD1 - Z3

END

 

SYNOPSYS...

 

Here the value of Z1 is the Y-coordinate of the image centroid (from file buffer location 5, which is where the RMS command places it.  Run the RMS command and then ask AI “BUFFER?” to see what went where.), and Z2 is the chief-ray coordinate; their sum is the absolute centroid position, which is first calculated by the Al equation (which puts a copy of the answer into file buffer location 1), and then placed into Z3.  This is then subtracted from the global Y-coordinate of surface 7 in the AANT file.  The optimization run then varies the global Y-coordinate of surface 7 so that it becomes equal to the value of Z3.

 

(In a simple case like this, it would be better to let AI do the whole job; after the centroid position is in variable Z3, the AI sentence "Change YG of surface 7 to Z3" would suffice.)
