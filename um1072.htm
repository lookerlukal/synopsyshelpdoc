10.7.2 Alternate Configurations                                                Next Previous TOC

 

SYNOPSYS can maintain six completely independent lenses at the same time.  These are called alternate configurations, and it is simple to switch from one to another.  The command

 

 ACON NB

 

will change to configuration NB, which must be a number from 1 to 6.  You can also change configurations by clicking on one of the toolbar buttons .

 

When you have selected the configuration you want, you may perform any of the normal SYNOPSYS tasks, such as FETCHING a lens, etc.  If you later switch back to an earlier ACON, it will still be the same as it was before.  You can see which ACON is current by typing the command

 

 ACON

 

with no argument.  To delete a current ACON (causing it to be ignored by subsequent multiconfiguration optimization) use the command

 

 ACON NB NULL.

 

To immediately enforce any pickups that have been declared (see below), use the command

 

 ACON ENFORCE.

 

All features that can do multiconfiguration work will automatically enforce all pickups, but this command will let you do it yourself if you do not want to run any of those features.

 

The command

 

 ACON CLEAR

 

will delete any ACON pickup requests that may currently be in effect.

 

The command

 

 ACON BUMP

 

will copy the lens in the currently selected configuration and place it into the next-higher ACON.  Thus, if ACON 3 is currently active, this command will put a copy into ACON 4.  It will also transfer all of the PAD views to the new ACON and switch to that configuration.  It is intended for when you want to try something different on the current lens but do not want to change the lens and setup as they are at the moment.  The ACONS do not become coupled in any way by this command and may be treated a separate lenses.  This command will not work if the current ACON is number 6 of if the PAD display is not open. It can also be executed by clicking on the button .

 

The command ACON MBUMP works the same way, except it bumps to the left instead of to the right.  So if ACON 3 is active, this will put a copy into ACON 2.  This can be executed by clicking on the button .

 

The following sections explain how to use these configurations to design a complex lens system.  The most common situation is the zoom lens, where some of the airspaces vary to change the magnification.  If only the zooming airspaces are changing, then you do not have to use alternate configurations at all, since the ZFILE option described in the previous section handles everything for you in a single configuration.  The ACON feature is intended for more general cases, where some lens parameters differ between configurations while others must be the same.  This would be the case when you want the same objective lens to couple to several different relay sections, for instance.

 

SYNOPSYS can optimize all of the current ACONs simultaneously, but it will do so only it you specifically request it.  It does not automatically assume any relationship between the configurations, but you will often want them to correspond to each other in some way.  These relationships are defined in the ACON PICKUPS files, which must precede both AANT and PANT input files.  Data from any configuration may be picked up by any of the others.

 

Lenses to be assigned pickups must already be loaded into the intended ACON before pickups are assigned.  Just declaring the pickups does not initialize or load the lens.

 

When multiple configurations are to be designed, they must all be current before the optimization begins.  They all remain current after the optimization as well, when they may be listed, examined, or saved by any of the normal SYNOPSYS commands.

 

The relationship between each alternate configuration and the others is defined by one or more files of the format :

 

 

NB is the configuration to get the picked-up data (1 to 6), and ACONF is the configuration whose data are to be picked up (default is configuration 1).

 

Relationships are specified by the subsequent input data, where

 

SNA

is the surface number in configuration NB to receive the data.

 

PCV

indicates that the curvature, conic constant, and aspheric data are to be picked up.

 

PTH

indicates that the thickness is to be picked up.

 

PIN

indicates that an index is to be picked up. See below.

 

PCAO

picks up clear aperture data, including RAO, EAO, and UAP parameters.

 

PAS

indicates that the asymmetries (tilts and decenters) are to be picked up.  This will pick up all types of asymmetries: relative, local, and global.

 

PACCOM

picks up the afocal accommodation.

 

SNM

is the surface number in the source configuration from which the values are to be obtained.  A negative entry for SNM causes the sign of the picked-up quantity to be reversed.  If zero is entered for SNM, the pickup is removed.  See the comments regarding ZOOM, below.

 

PZDATA

Picks up the ZFILE zoom position data.  It does not pick up object data.

 

Up to four such lines may be specified for each surface in the alternate configuration.  If more than one alternate is desired, a second ACON ... END file may be entered, and so on.  Each file must end with END, and the ACON command must contain the word PICKUPS in word 3.

 

The input

 

ZOOM

 

in the ACON file causes all radii, thicknesses and airspaces, and indices to be picked up from ACON 1 by alternate configuration NB.  Tilts, decenters, and clear apertures are not picked up by this option.  This is useful when the lenses have the same parameters but have different tilts or decenters, as you might find in a scanning system.  The ZOOM entry may optionally specify a range of surfaces that are to pick up data from the source configuration beginning at a specified surface number.  For example, to cause surfaces 6 through 12 in configuration number 2 to be the same as the surfaces beginning with number 4 in configuration 1, the input would be

 

ACON 2 PICKUPS

ZOOM 6 12 4

END

 

In the event that you wish a certain parameter not to be picked up, but it is included in the range of a ZOOM command, you may follow the ZOOM command with a normal pickup request but specify 0 for SNM.  We repeat that this use of the ZOOM feature is separate from the ZFILE zoom, which is much easier to use and is recommended if the only thing that changes is the position of zooming lens groups.

 

The program tries to resolve inconsistencies between the parameters currently associated with a lens and the ACON commands.  For example, if a curvature pickup places a curved surface on a previously flat surface, the curvature flag on that surface is changed.  If the target of an ACON pickup is controlled by its own pickup or solve, that option will be deleted when the file is processed, since a surface cannot have two conflicting pickups in effect.

 

Index pickups are permitted for most of the possible index options.  Some combinations require altering the definitions, however, and the program will take corrective action as needed.  For example, if the source configuration has a glass model (or glass variable), the target will be assigned the same model parameters.  If the source uses the glass table or a GLASS or GDF option, the coefficients are transferred.  This permits the target to employ the same material as the source, but with different wavelengths.  If explicit indices are present in the source, the actual values will be picked up by the target.  In this case, since the indices cannot be recomputed, they are not meaningful unless the wavelengths are the same in both configurations, and an error message will be printed if this is not the case.  You cannot transfer BIREF or WPLATE index data with the ACON pickup file.  If the alternate configuration is to contain those materials, they should be inserted manually and no pickup included in the ACON file.

 

The request

 

 BZOOM …

 

in the ACON PICKUPS file is similar to the ZOOM feature described above except that it will also pick up asymmetries (tilts, decenters, local and global coordinates) as well as clear aperture and EFILE data.  This is intended primarily for multiconfiguration tolerancing with BTOL, where one must ensure that element tilts and wedges are picked up throughout the included surface range.

 

The option

 

 CZOOM …

 

is also similar to ZOOM, but picks up aperture and EFILE data but not tilts and decenters.  In both of these two options, the EFILE data that are transferred are converted to EXPLICIT form, so that a change in ray paths in the other ACON (which changes default CAOs) will not accidentally change the EFILE edges along with them.

 

Note that the ZOOM and BZOOM options pick up data only from ACON 1.  If you want to pick up from any other configuration, you must use the explicit forms (PCV, PTH, etc.).

 

Object data, including YMP1, YP1, TH0, YP0, XMP1, XP1 XP0 and the object radius of curvature may be picked up by the entry

 

1 POB 1 [ ACONF ]

 

in the ACON file.  This is required if any of these quantities are variables in the main configuration and you wish the alternates to use the same values.

 

An alternate configuration may be defined as the source of one or both of the beams used to create a holographic optical element (see Section 3.3.2.4).  If the mnemonic HP1, HP2, HZ1, or HZ2 is included in the ACON file, the aberration coefficient array will be computed for the alternate and transferred to surface number SNM (which must be a HOE) in the main configuration.  The position of the point P1 or P2 relative to the HOE will also be picked up from the known position of the image in the alternate configuration, measured from the next-to-last surface, which must correspond to the geometrical position of the HOE.  If SNM is negative, the signs of the coefficients are reversed.  Note that, in this instance, it is configuration NB that is the source of the data, while ACON 1 is the target.

 

In order for the program to transfer the aberrations to the HOE, the HOE must have been previously defined as an aberrated HOE, with the mnemonics A11 or B11, etc.  The aberration coefficients are then updated by the optimization program at each iteration.

 

The PAS option should only be used when the definition of AXIS is the same in both configurations, since the decenter resulting from the tilt will be recalculated as soon as the alternate undergoes a paraxial raytrace.  The PAS option will pick up all tilts or decenters in effect on the surface, including LOCAL and GLOBAL.

 

To enter variables and aberrations referring to the alternate configurations, simply enter the input

 

ACON JCONF

 

at any time in the PANT or AANT files.  All input within that file until the next ACON entry will refer to configuration JCONF, and so on.

 

Since the optimization program normally works on whatever ACON is current, and only that ACON, you must put the word MULTICONFIGURATION in word 3 of the SYNOPSYS command if you want to optimize more than one ACON at a time.  This causes the program to accept ACON entries are found in the AANT and PANT file, and to perform the pickups between the several configurations.  Thus to do 20 iterations of multiconfiguration optimization, you would use the command

 

 SYNOPSYS 20 MULTI

 

When the AANT and PANT files are processed, the program assigns the aberrations and variables to the current ACON until another ACON directive is encountered within that file.  Since one usually intends the first of these to apply to ACON 1, it is wise to either change to this ACON before entering these files, or to start the AANT and PANT files with an explicit ACON 1 directive.  Failure to do this can result in the data being applied to the wrong configuration, since the program has not yet seen the SYNOPSYS … MULTI command that is to follow, and it therefore defaults to the usual interpretation of the AANT and PANT data, which apply to the current configuration.

 

In other words, put an ACON 1 command at the start of your optimization MACro.

 

 

Once you have defined all of your configurations, you can easily save that setup with the command

 

 SSU [ nb ]        (Save SetUp)

 

After this is done, you do not have to set them up from scratch the next time you start SYNOPSYS.  You can of course save each of them to the library or a named RLE file before you exit from the program, but the above command will automatically save all current ACONs to a set of files, named SSU1.DAT, etc.  Then, when you want to restore that setup, just type the command

 

 RSU [ nb ]        (Restore SetUp)

 

These commands also save and restore the current settings of the PAD display.  (The commands SPL and RPL will save and recall the current PAD layout, but not the lenses in all the ACONs.)

 

If you specify a number in word 2 of the SSU or RSU commands, the program will save or restore only the lens in that ACON.  (The PAD display of all ACONs will still be saved and restored, however.)

 

The ACON ... PICKUPS feature is incompatible with thermal shadowing.
