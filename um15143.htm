15.1.4.3 Passing parameters with the Zn array                                Previous Next TOC

 

In addition to its use in a composite evaluation, the Zn array can also store and transfer numeric values.  Unlike the CDn composite aberration definitions, which are defined only within a COMPOSITE input file, and exist only during the optimization run or Al COMPOSITE sequence, the Zn parameters exist permanently and can be used like numbers in nearly all SYNOPSYS input.  For example,

 

Zl = 1

What is Z1?

Z2 = 2 th

4 th = Z2

increase th Z1 by 1.1

Z3 = FILE 4

 

Although the Zn parameters resemble in some ways the AIP parameter and other symbols, there is a formal difference: The Zn parameters are numbers, not symbols.  They can be set only in Al sentences (or in optimization), and are used like numbers – while symbols can be used for any input to SYNOPSYS.

 

Consider the following, for example:

 

Z1 = 0.4

a1: 0.4

change 2 th to Z1

change 2 th to a1

 

Here, both of these sentences would change thickness 2 to the value 0.4.  The symbol “a1" contains the string of three characters "0", ".", and "4", while Z1 has the value 0.4.

 

The Zn parameters should not be defined as symbols, since the numeric substitution takes precedence.
