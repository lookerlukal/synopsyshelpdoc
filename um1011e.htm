10.11 Naming aberrations: the AFILE, ALIST                Next Previous TOC

 

Section 10.3 showed how it is possible to assign a name to any of the permitted aberration definitions.  This section extends the concept further: a separate file is created that contains only aberration definitions and the names by which they are to be known; the AANT file can then refer to these aberrations by name, eliminating the need to retype the definitions at the time they are to be used.

 

The definitions are entered into the MACro editor exactly as they would be in an AANT file, except that the file begins with the mnemonic AFILE instead of AANT.  This MACro is never executed.  It is simply saved by the editor as a file with the special name "AFILE.MAC".  Each aberration, or group of aberrations, that you want to recall in a later AANT file should be identified with a unique name, of up to 8 characters, either all numbers or starting with a letter, and containing no imbedded spaces or punctuation.  Then the AANT file simply calls them by name.  For example, suppose the AFILE contains the entries

 

AFILE

NAME RAYSET1

M 0 1 A 2 YA 0 0 .6

M 0 1 A 2 YA 0 0 .8

M 0 1 A 2 YA 0 0 1.

NAME OFFAXIS

M 0 1 A 2 YC 1 0 1

M 0 1 A 2 YC 1 0 .8

M 0 2 A ABR -1

S ABR -2

M 0 1 A 2 YC 1 0 -1

MO 1 A 2 YC 1 0 -8

M 0 2 A ABR -1

S ABR -2

NAME AXCOLOR

M 0 1 A 1 YA 0 0.7

S 3 YA 0 0.7

NAME SPHER

M 0 1 A SA3

END

 

then a useful merit function could read simply

 

AANT P

AFILE SPHER

AFILE AXCOLOR 10

AFILE RAYSET1

NAME EDGEPOS

ECP .1 .01 2 4 5

END

 

where each named aberration is identified by the word "AFILE", followed by the name of that aberration, and optionally a weighting factor that multiplies the weight originally input to the AFILE.  For example, the first aberration in the above AANT file recalls the SA3 target, which originally was given a weight of 1.0.  The second, using the name AXCOLOR, gives a weight of 10, which multiplies the original weight of 1 on that aberration.  The third name used, RAYSET1, includes the three rays defined under that name.  The rayset OFFAXIS was not used by this AANT file, but an edge-control aberration was added, given the name EDGEPOS in the AANT file, at the time it was defined.

 

You may intermix AFILE references and locally defined aberrations at will, and optionally assign names to the latter.  When the optimization is complete, you may examine the resulting aberrations (and definitions) with the FINAL command or with the ALIST command, which gives an abbreviated listing that shows the name, raw value, and relative contribution of each aberration to the merit function.  The "raw value" of the aberration is the actual value of the quantity calculated, not the error measured from the target value.

 

For example, consider the following AANT file.

 

AANT

NAME RAY1

M .01 1 A 2 YA 0 0.7

NAME RAY2

M 0 1 A 2 YA 0 0 .9

NAME EDGEP

ECP 1 .1 3 6

END

 

After optimization, the FINAL list shows

 

SYNOPSYS AI>

SYNOPSYS AI>FINAL

ABERRATION LIST

       NAME            TARGET         WEIGHT            FINAL VALUE   R. EFFECT

 

  1 RAY1              0.0100000      1.0000000          0.140562E-02   0.005123

    A   2  YA    0.00000   0.00000   0.70000   0.00000

 

  2 RAY2              0.0000000      1.0000000         -0.111611E-02   0.003230

    A   2  YA    0.00000   0.00000   0.90000   0.00000

 

  3 EDGEP             1.0000000      0.1000000          0.195563E-01   0.991647

 UPPER R.R. EDGE CONTROL

    3

    6

SYNOPSYS AI>

 

and the ALIST gives

 

SYNOPSYS AI>

ALIST

 

ABERRATION SUMMARY LIST

 

NO. NAME       RAW VALUE  REL. EFFECT   NO. NAME       RAW VALUE  REL. EFFECT

______________________________________________________________________________

  1 RAY1          0.0114       0.0051     3 EDGEP         1.1956       0.9916

  2 RAY2         -0.0011       0.0032

SYNOPSYS AI>

 

The actual ray intercept of the first ray was 0.0114, which is 0.001405 more than the target value of 0.01.  The FINAL list gives the latter number (times the weighting, which is here 1.0), and the AFILE the former.  For aberration RAY2, the two are identical since the target was 0.  The ALIST command may be used even if no AFILE or local names are present; the names will be printed as blanks, with the aberrations listed in order.

 

In case it is not clear, you do not have to give names to your aberrations; this just to help you find them in what can be a rather long listing from FINAL or ALIST.
