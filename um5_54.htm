Creating and using a custom DLL file with SYNOPSYS™                        Next Previous TOC

Occasionally a user of SYNOPSYS has need for a feature that is not currently in the program.  When this happens, you are encouraged to contact us and request the new feature.  Be specific about what it will do: the input and output, and the applicable theory if it is complicated.  We usually respond quickly to such requests.

But not always.  If a new feature is judged to be of little interest to the general community, or if the theory is very complicated and there will be few eventual users, we might decline.  In such cases, you can write your own DLL and link it into SYNOPSYS.

Be advised that this step is not for the fainthearted.  You will have to know a great deal about programming in MFC with C++.  If you have not yet jumped into that whirlpool, you will find it cryptic, user-hostile, and frustrating.  Nonetheless, it can be done if you are an expert.  Here are the steps:

1.Ask for a copy of USERDLL.sln from OSD, LLC  This contains a rudimentary structure to which you can add your custom programs.
2.Get Microsoft Visual Studio 2010, which is the version that will link to SYNOPSYS.
3.The sample programs in USERDLL.sln contain several sections, most of which are involved with communicating between SYNOPSYS and the DLL.  Do not change any of those sections!  The section you can customize is called MyDLLFiles.cpp.
4.The sample code shows how to create a custom surface shape.  This shape will be defined entirely by a set of 43 aspheric coefficients, entered with the XDD input format in an RLE file.  Your code must be able to return the surface sag as a function of X and Y, the axial curvature in X and Y, and must be able to adjust the G-coefficients when the lens is scaled or reversed.  The example does these things for a shape with one coefficient, namely the curvature.
5.The sample also shows how to write code to analyze the image, either with transverse aberrations or OPDs.  It can calculate the RMS spot size and wavefront in a single color and field.  You can enlarge this capability as much as you wish.  (Of course, SYNOPSYS can already do this kind of image evaluation; the examples are given only to show you how to write this kind of program.)
6.The file fortranstructures.h lists the contents of most of the common blocks in SYNOPSYS, from which you can extract whatever lens data you wish.  It is not recommended that you change any of these data in your own routine, however.  Use the RLE and CHG formats instead.  Those have extensive error checking and initialize a host of arrays and constants needed by many features of the program, which your DLL file will not be able to do.  Be warned.
7.Your custom DLL will be able to write its results to the Command Window, using the sample code provided.
8.When your DLL is complete and compiles correctly, copy the file USERDLL.dll into the SYNOPSYS directory.  The program will look for it there.
9.When SYNOPSYS starts up, it  automatically loads your DLL if it is present.  To check, enter the command USER DLL, which will show a message telling you whether the load was successful or not.  Then, to use a custom shape, enter it as shape USS 18 and give the required coefficients as G-terms.  To use your custom image analysis, use the command USER DLL IMAGE NRYS ICOL HBAR GBAR.  (Or you can define your own data based on the input in the CARD() array.)  If that command is preceded by the OPD command, the OPD section of the analysis will be used.  Use the command USER DLL CUSTOM to run any other code you have put in the routine pfnDoUserFunc() in the USERDLL files.
10.If you succeed in getting your custom shape or image analysis working, by all means send us a copy of it.  We then might well incorporate it into the program itself.
11.We do not guarantee that your DLL will work or the results will be correct.  You do this at your own risk.
12.Best of luck to you.  We may or may not be able to help you diagnose bugs in your code.  As we said, this is not for the fainthearted.
