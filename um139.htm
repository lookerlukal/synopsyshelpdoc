13.9 Saving a plot as an HPGL, DXF, or Postscript file        Previous TOC

 

There are three ways to create a dxf file to be opened by a third-party program such as AutoCAD.  The first is described in this section; it only applies to 2D pictures.  The second uses the RPER wire-frame rendering, while the third is created with DSOLID.

 

After a plot is displayed on the screen, you can optionally save it in HPGL, DXF, or Postscript format.  These formats cannot be read by SYNOPSYS, but can be imported into some third-party applications that can read this kind of file.  Some of these applications can also translate the file into numerous other graphics formats.

 

The commands are

 

 HARDCOPY HPGL

 

 HARDCOPY DXF

and

 HARDCOPY POST.

 

You have some control over the name: the actual name is just the second argument of the command, of which the first three characters must be “HPG”, “DXF”, or “POS”.  So you could create a file named “HPGLENS1.PLT” with the command HARD HPGLENS1, or a file “POST55.PS” with the command HARD POST55.

 

If the application you use to read the HPGL file allows you to map the pen numbers to colors, to get the same colors as on the SYNOPSYS display, map them as follows.  (This applies only to HPGL and POST files; for DXF files with color, you must use the DXF button on the RPER dialog; see below.)

 

1

black

2

red

3

blue

4

magenta

5

cyan

6

green

7

black

 

 

You cannot save an RPER plot with the HARDCOPY command – but you can turn it into a DXF file by clicking on the button or an IGES file by clicking the button .  These files will contain all of the 3-D information, and the picture can be rotated by those commercial CAD programs with that capability.  The DXF format will show the picture at the current display angles, while the IGES format initially will show the lens at angles (0,0).  Also, the greyscale of the FRINGES or PSVISUAL and MIT plots cannot at present be converted with this command.  A dxf solid model may be prepared with the command DSOLID.
