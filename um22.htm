2.2 Notation Conventions                                                Previous Next TOC

 

Certain data entries described in this manual must be typed exactly as shown, while others represent choices of alphabetic characters or numeric input.  To prevent ambiguity, the following notation is used throughout this manual:

 

1. Capital letters represent identifier words whose first three characters must be typed exactly:

 

RLE

SPT

 

2. Underlined letters represent numeric data or words that must be supplied by the user

 

NB

SN SN SN

YMPI

 

3. Several items surrounded by curly brackets indicate a group, of which one must be selected and entered.

       

The format of the subsequent entries may in some cases depend upon which choice is made.

 

4. A group of entries surrounded by square brackets represents optional data.

 

DWG [ SCF ]

 

In such cases the program will assume default values for the absent parameter.

 

5. The letter O and the numeral 0 are printed as shown.

 

6. In the text, an asterisk (*) is used to denote multiplication, and (**) is used to denote exponentiation.

 

7. A slash (/) means that the input following is to be interpreted as if it were on the next line.  (Except lines beginning with the characters ID, PROJECT, or “=”.)
