FWAV

gives the (optional) wavelength at which the number of fringes to test plate and irregularity tolerances are given.  By default this is .6328 μM (HeNe).  This input also affects the RANGE and FIX tolerances that are specified in waves.

 

RANGE / FIX

These lines, which must precede all other BTOL commands except DEFAULT and FWAV, let you override the default range on a whole class of variables, or to input a fixed tolerance for that class.  Other variable attributes let you change them on individual surfaces, but these are global in scope, for when you want to change them all.  Any individual changes you later make will override the global setting for that surface.  The allowed parameters are:

 

RFR for radius error, in fringes.  The default range depends on the actual radius.

 

CONIC or CC for the conic constant.  The default range is 1.0.

 

TH thicknesses and airspaces.  The default range is 0.005 inches.  The range should be entered in current lens units.  In mm, the default is 0.127 mm.

 

INDEX The default range is 0.001.

 

VNO (or VNUM) The default range is 1.0.

 

IRREG for cylinder figure errors; the default range is 2.5 fringes of power.

 

REDGE for edge rolloff errors; the default range is 1.0 fringes at the edge of the aperture.  This is modeled by aspheric term G16, which multiplies the radius to the 10th power.

 

WEDGE specifies the wedge error on the second side of each element. The default range is 0.005 inches.  Input data give the angle in degrees.

 

WTR is similar to WEDGE, except the input gives the maximum TIR in lens units over an aperture of one inch.

 

DECENTER (or YDC) applied to the element as a whole.  The default range is 0.005 inches.

 

It is possible to find the assembly tolerance for a group of elements, in addition to those of a single element.  To do so, add dummy surfaces before and after the group and assign a relative tilt or decenter to the group (to define the group size).  The decenter or tilt angle can be zero initially.  Then supply your own RANGE for the decenter or tilt of the dummy at the start of the group.  This will force the program to add that parameter to the list of those to be toleranced.

 

ZYDC is valid only for ZFILE zoom lenses, and applies to the centration error of zoom groups.

 

TILT or TTR applied to the first side of all elements.  The default RANGE is 0.005 inches. This is always input as a TIR value.  See the above paragraph for comments about tilt tolerances for a group of elements.

 

DAMP

is an optional damping factor used when solving for the adjustments.  Since these may not be orthogonal, numerical difficulties may arise that can be offset by damping the solution.  If there is more than one adjustment, the default damping is 0.00001; otherwise it is zero.  See below for a discussion of the considerations that apply to this parameter.

 

FOV

gives the number of field points to be considered, where NFOV is a number from one to five.  Instead of specifying the individual points, as in TOL, you must select from the following list how many points you want.  For example, FOV 1 gives point 1; FOV 2 gives points 1 and 2, etc. The default is FOV 3, which gives the first three points, except for lenses with Gaussian object OBG, for which the default is a single field.

Note that the number selected affects the boundary conditions given tolerances by the MAG, DIST, and BORE options.  To tolerance BORE requires only the first field point. MAG requires the first two, and DIST requires the first three.  See section 12.1.3.5

 

MONO

Overrides the colors and weights declared in the lens file.  The BTOL analysis will be monochromatic at the entered color, or at the principal color if no color number is entered.

 

FWT

gives the relative weighting of each field point called for with the FOV input. The weights are input in the same order as the fields listed above.  These values are used in several ways, as discussed in the TOL commands section and the FOCUS section, below.

 

UCF

allows user-controlled field points.  Up to 5 points may be requested in NFOV and defined by subsequent HBAR GBAR entries.  If TOL BORE is also requested, the first of these points must be at (0,0); if TOL MAG is desired, the second is restricted to (1,0); and if TOL DIS is needed, the third must be at (0.5, 0).

 

ZOOM

specifies which zoom positions (in a ZFILE zoom lens) are to be considered by BTOL.  The default is all zoom positions.

 

IMAX

gives an optional limit to the maximum change in MTF or Strehl ratio that will be accepted when the derivatives are calculated.  To allow a 10% drop, for example, the input would be IMAX 0.1.  The default IMAX allows 50% drop.  Too large a drop gives less accurate derivatives, while too small may require a lengthy search with successively smaller increments.

 

GRID

specifies the grid number of the rayset for all image quality evaluations.  The default is 30, specifying a 30x30 rectangular grid.  User input must be between 10 and 80.

 

{ POWER / ZERNIKE }

applies to OPD-based image descriptors, to select the kind of interpolation used to analyze image quality. ZERNIKE, the default, fits the wavefront to a 36-term Zernike polynomial, while POWER uses a 17-term power-series polynomial, which is usually adequate and runs faster.  If the FANS curves are gentle and smooth, the POWER method should be adequate.  If there are strong flairs at the ends, then use the ZERNIKE method.

 

This parameter has no effect if the quality descriptor is MTF-based, since that always uses a linear spline interpolation method, or for spot-based descriptors, which involve exact raytracing.

 

GO                                                  

is the last input of the BTOL input file, and causes execution of the program.

EVALUATE

(instead of “GO") will compute the initial values of the quality descriptors and then terminate.

 

PREPARE MC  

 

This command, when placed before the GO input, causes BTOL to automatically prepare an adjustment file that can be used in a later MC run to check the statistics.  This file will contain PANT variables that duplicate the adjustment variables used in BTOL, as well as an AANT file that targets the same rays that the FOCUS command utilized.  See MC for a description of this powerful feature.

 

BTOL { SAVE / FETCH number/ LIST }

These commands are intended to help you document your lens and to save the tolerance budget produced by BTOL.  The command

 

      BTOL SAVE

 

will save a summary of the tolerances in a file whose name is just the current log number, and with the file type .BTO.  If you also save the lens with the command SAVE LOG, then you have a lens file and the tolerance file with the same name.

 

BTOL FETCH number will read the saved file in again.  This lets you use the features that add tolerance information to DWG and ELD drawings, even if you exited from SYNOPSYS after running BTOL.  Just be sure to FETCH the same lens that was used for the budget, since the saved budget applies only to that lens.  If you have not exited from the program since running BTOL, you do not have to save or fetch this file, since the budget is still current.

 

BTOL LIST will list the current tolerance budget.  This is done automatically when you run the program, but you can list it again if you want (to capture on the printer, for example).

 

MODIFY

This option is intended for the case when you already have run BTOL and have analyzed the statistics with MC.  In the event that the results are not as good as you had expected – because some image property not considered by BTOL has gotten out of hand, for example, or because the yield was smaller than predicted – you can modify that budget and analyze it again.

 

This input must be followed by one or more additional lines of input stating which tolerances are to be changed by how much.  The format of those lines is as follows:

 

{ list } MULTIPLIER

...

END

 

where list is one of the following,

 

ALL RD RAD RFR CV TH THI INDEX VNUM WEDGE IRREG YDC DECENTER TILT TTR XDC PEP REDGE RL1 RL2 PAX PAY CONIC ZYDC SSD 1 SSD 2

 

and MULTIPLIER is a value by which the given tolerance class is to be multiplied.

 

With this input, for example, if you want all wedge tolerances to be reduced to 75% of the value currently in the budget, you would enter

 

BTOL MODIFY

WEDGE 0.75

END

 

The selection ALL will multiply all tolerances in the budget by the requested value.

 

Once the budget has been altered by this option, you may save it, print it, or use it for MC analysis as you would any other budget.

 

CRITERION

If the entered number is less than 10, this is the confidence level of the budget, in sigma units: 1 for one sigma, 2 for two sigma, and so on.  If the number is greater than 50, it gives the percent confidence: BTOL 90 would specify a 90% confidence.  Note that there is no sigma value that corresponds to a yield of less than 50% – since in that case the program finds a budget where the mean value of the performance just equals the target, and half of the lenses would be better than that, if the budget calculation converges.

 

BTOL CRITERION MULTI

 

Use this form to calculate the tolerance budget for several configurations at the same time.  See Section 12.1.4.

 

BTOL EDIT number

This entry allows one to edit the budget that is returned by BTOL.  See Section 12.8 for details.

 

BTOL BUDGET { PTH ( AIR / GLASS } PCV }

This entry may be used for lenses where symmetry is an important part of the aberration correction.  In such cases you can assign pickups to the curvatures and thicknesses of elements in the second half and they will remain symmetrical with the first half.  BTOL will then honor those pickups but assign a tolerance to the resulting values.  (If you tolerance a lens with pickups without calling for this option, BTOL will assume that the picked-up values are exact and will not tolerance them.  This would be appropriate for a system where light encounters the same lens twice, as in a Mangin mirror, for example, where there can be only one set of tolerances for that element.)  This option also turns on the system option PCV CV for that lens, so the pickup does not transfer aspheric terms, which are used to model the irregularity and rolled edges, which must be done separately for both  elements.

 

You can specify whether BTOL assigns tolerances to picked-up airspaces, glass thicknesses, or curvatures, or any combination.

 

BTOL … RECALC

 

This option provides a way to quickly evaluate the budget consequences of several different quality descriptor values.  The input procedure must be followed closely for the results to have any meaning.

 

First, you execute a normal BTOL run without giving a RECALC request.  The program calculates all of the derivatives and produces the first budget.  Then you change the tolerance limit or degradation amount of the quality descriptor, add the RECALC request to the BTOL command, and run the entire BTOL input file again.  You must not change anything else, since the derivatives apply only to those variables and the previous kind of quality descriptor.  Thus, you could change DEGRADE SPOT 0.2 into DEGRADE SPOT 0.5, for example, but you could not change a spot descriptor into WAVE, STREHL or MTF.  Because the derivative calculation requires most of the calculation time, especially for REOPT adjustments, this reuse of the previous derivatives executes very quickly.

 

But there is a catch.  The program monitors the effect of each derivative increment, and will reduce the step size if the image degradation is excessive, with the goal of staying in a region of relative linearity.  So a different quality limit might require a change in increment size if it were run completely, while the previous derivatives of course were based on the size that was appropriate to the previous limit.  So after you have run the cases you want, and have selected the budget that is most suitable, it is wise to run that case again, without the RECALC option turned on, as a final check.

 

NODMESSAGE

As BTOL evaluates the effect on image quality of each variable, it will reduce the derivative increment if it finds excessive degradation; the purpose is to stay within the region of derivative accuracy.  It can be helpful to monitor the messages it prints when this happens, but if they are not useful, this option will suppress them.

 

SEGMENT

It is possible to estimate the total performance of a system when separate modules are evaluated separately.  This option causes the program to print some statistical data that is useful for that purpose.  See Section 12.3.1 for more information.

 

DEFAULTS

This entry causes the program to bypass the budget calculation completely, and treat the default tolerances as the budget.  This budget is output along with the performance statistics that would result if the lens were built to that budget.

 

The program will honor any RANGE, EXACT, or FIX input, and data so entered will override the default for this option.

 

SORT

Very complicated lenses, such as 30-element microlithography objectives that work in the UV, usually require extensive fabrication adjustment, where each element is made and measured in turn, and the entire system reoptimized for that exact lens before the next one is made.  The process works best if the elements with the tightest tolerances are made first.  This option will sort the previous BTOL budget, listing the radius and thickness tolerances, sorted from tightest to loosest.
