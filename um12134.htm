12.1.3.4 BTOL MTF quality descriptor                                        Next Previous TOC

 

You may select the convolution MTF with the following input:

 

TOL { MTF / YMTF / TMTF / SMTF / XMTF } FREQ MTF

TOL { MFF / YMFF / TMFF / SMFF / XMFF } FREQ MTF MTF …

TOL { MSF / YMSF / TMSF / SMSF / XMSF } MTF FREQ FREQ …

 

DEGRADE { MTF / YMTF / TMTF / SMTF / XMTF } FREQ FRACTION

DEGRADE { MFF / YMFF / TMFF / SMFF / XMFF } FREQ FRACTION …

DEGRADE { MSF / YMSF / TMSF / SMSF / XMSF } FRACTION FREQ FREQ …

 

TOL MTF

TOL MFF

TOL MSF

gives the minimum permitted MTF.  See section 8.6.3.1 regarding the assumptions inherent in this method.  The spatial frequency is given by the FREQ input, and the MTF is the average of the tangential and sagittal MTF.

 

The usual operation of BTOL is to insure that the quality descriptor does not exceed a given value; for MTFs (and the Strehl ratio) this needs to be modified since in this case the quality must not fall below a specified amount.  For this reason, BTOL actually tolerances the inverse of the MTF.  There are several subtle consequences of this action:

 

One is that if the quality were to be very bad after incrementing a variable, the descriptor could approach infinity and the derivative information would be meaningless.  In order to insure that the derivatives are suitable, the program will not allow the MTF to degrade to less than 0.5 times its initial value, or by the amount entered as IMAX.  If this degradation is exceeded, the program will iteratively reduce the derivative increment until it is not.  In rare cases this search may fail, and a warning message will be printed, indicating that that variable is too ill-behaved to tolerance with BTOL.  Note that this degradation is not the same as that given or allowed for the budget, and does not affect the range of the variable.  If IMAX is entered, it should be a positive number less than one.

 

The MTF option should only be requested if the lens has a nominal MTF greater than about 0.5 at the requested frequency, and less than about ten waves of aberration.  Since the MTF function will oscillate if the image is poor or if the modulus is low, it is not suitable for tolerancing in those cases.

 

Field weights may be entered for MTF tolerancing, if needed, with the FWT input.  The effect is to divide the (inverted) maximum descriptor by the weight.  If the minimum MTF were to be, say 0.4, the descriptor limit would be 1.0 / 0.4, or 2.5.  If a field weight of 0.5 were entered, this limit would become 5.0, corresponding to an MTF of 0.2.  Multicolor MTFs are calculated using the current colors and weights in the lens file.  Note that the field weights are defined in the order given in the UCF input, if any, and in the default order (0. 1.0, 0.5) otherwise.

 

TOL MFF specifies a different target for each field point, with the same special frequency for all fields.

 

The variants MSF, YMSF, etc. let you specify a different frequency at each field point, keeping the same target for each.

 

TOL { YMTF / TMTF / XMTF / SMTF /

YMFF /TMFF / XMFF / XMFF /

YMSF / TMSF / XMSF / SMSF}

 

specify that only the Y or X direction is toleranced.  T is the same as Y, S is the same as X.

 

DEGRADE MTF

gives the fractional reduction in the MTF.  For example, if the MTF is 0.8 and the fraction is 0.2, the MTF tolerance would be 0.64.

 

DEGRADE { YMTF / TMTF / XMTF / SMTF /  YMFF /TMFF / XMFF / XMFF /

YMSF / TMSF / XMSF / SMSF }

 

 

specify that the MTF is toleranced only in the specified orientation.

 

 

All of these descriptors involve interpolating over the exit pupil to find the OPD at many ray points.  The function used will be a linear spline fit to the wavefront.  See Section 8.6.3.1 for a discussion of the implications. The spline method is considerably slower but works well.  For additional accuracy, the interpolation grid is always set at 80x80 for the linear spline method.
