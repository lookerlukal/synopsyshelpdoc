8.5 Geometric Analysis                                 Next Previous TOC

 

In the geometric image analysis command formats described in this section, the desired number of rays is given in the input.  The number that is actually traced is determined by the geometry of a square grid superimposed on the entrance pupil, and is reduced by vignetting, ray failures, or edge feathering, unless you have input the optional NOVIG in the lens file.  In that case only ray failures will be considered.  Up to 20,000 rays may be in the final image.  To achieve adequate accuracy, the final image should contain at least 400.

 

The image is defined by the X, Y coordinates of the set of rays with the final surface, except for AFOCAL systems, for which the final ray angles are used instead (in radians).

 

Many kinds of geometric image analyses may be prepared interactively with the dialog MGI and subsequent links.

 

 

Many of the geometric programs (as well as the drawing programs PER and RPER) can draw individual ray positions as either a small dot or with a symbol denoting the color number.  You can control the size of the symbol with the command

 

 SSSIZE NB

 

where NB gives the size, in inches.  The default size is 0.020.  Note that switch 27 must be OFF to show these symbols; if it is ON, the rays are shown as a single dot.  This parameter may also be set via an edit box in the dialog MGI.

 

In addition to the features described in this section, a more general image analysis routine is available, which is described in section 9.1.
