3.7.2 SAVE, FETCH                                        Previous Next TOC

 

The lens library will store only ten lenses, but another storage option provides for an unlimited number.  You may save and fetch any number of lenses with the Windows menu items File|Save, File|Open, and the toolbar buttons normally used for file saving and opening.  The dialog MWL shows the IDs of all saved lenses along with a preview pane of the lens profile as you click on each one; you may select one with a click.  The commands for this purpose are as follows:

 

 SAVE filename and FETCH filename.

 

SAVE will write the lens file to disk storage in the format of an RLE file, and save it permanently as a file with the entered filename and a file type of ".RLE".  The filename may be typed either following the SAVE or FETCH command, or on the next line.  Any previous file with the same name and file type will be superseded (without a prompt).

 

File names in SYNOPSYS may be up to 90 characters in length, and may contain embedded spaces.  They must not use special characters such as “/”, “””, and “;”.  Some features, such as the MWL file-open menu, are limited to 50 characters, so you are advised not to exceed that limit.

 

For convenience, you may use the special name "LOG" or "log" instead of a filename with the SAVE command.  In this case, the program will replace those characters with a number equal to the current log number.  Thus you can easily document the design trail leading up to the current design: just be sure to put a LOG command in your optimization MACro (so that the number changes), and then whenever you get a design that you might want to go back to, just type SAVE LOG, and you have a unique file for that lens, with a name that is just the log number at that moment.  For convenience, we have provided a toolbar button to do the same thing.  Click on

 

 

and the command SAVE LOG will be executed, as described above.

 

A similar button appears on the MACro editor toolbar.  In this case, it does not save a lens file but rather saves the current contents of the editor as a file whose name is the current log number and whose file type is “MAC”.  If the editor contains the latest optimization MACro, then simply by clicking on this button on the editor toolbar and then clicking the same button on the main window toolbar you have documented this lens version and the MACro that optimized it.

 

If you have run BTOL, and want to save the tolerance budget as well, use the command BTOL SAVE, which will save the budget information as a file whose name is the current log number and the file type is “BTO”.

 

FETCH filename reads the file that was SAVEd.

 

A list of currently SAVEd files is produced by DIR RLE (see 13.6), and with the menu command MWL.

 

Note that operations requiring the lens library – such as MC – cannot access the files created with the SAVE command.  If a desired lens is not in the library it must be read with FETCH and placed into the library before such commands can be executed.

 

It is possible to save any number of lines of annotation with the lens file.  Turn on switch 58, and the SAVE command will prompt for text lines.  When you have entered all of the text you want, press the <esc> key.  These lines will appear on the screen when that lens file is read again by the FETCH command.
