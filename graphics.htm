Graphics

 

When a SYNOPSYS command or toolbar action has produced a picture, it is automatically shown in a graphics window.  This will be in the same window as the previous picture, or in a new window of its own, depending on the status of a menu option selection.  The graphics window may be resized, and a toolbar is provided for certain manipulations.

Bitmap options

 

The bitmap buttons , , and are used when you want to transfer data from a dialog or a display to annotation, or to convert a graphics window into a .bmp file which is then available offline.  For instance, suppose you want to include the Spectrum Wizard display in a report, to show the spectral sensitivity of your system along with a blackbody curve for the source.  This is easy to do:  When that display is open, click the button , and a bitmap copy of the display is put on the Windows clipboard.  At that point you can paste it into an open graphics window with the button, or you can open any third-party graphics editor and paste the picture in there.  (You can open a new, blank window with the command BLANK, if none is currently open.)

When the bitmap is shown in the graphics window, you can add annotation or print it as you would any other graphics display.  If you want to save the annotated bitmap, click the button and supply a filename when prompted.  This will save the current display as a .bmp file.

 

When you click the button , the program saves the picture as an SSP file, provided it does not contain a bitmap.  If it does, the program formats it as a JPG file, with all your annotation included, instead.  You therefore have a choice of saving as a bitmap or a jpeg; the latter makes a much smaller file, with minor loss of resolution.

 

Also, SYNOPSYS cannot open a .bmp file.  If you want to do so, first open the file in a graphics editor, and copy it to the clipboard.  Then paste it in as instructed above.

 

Some displays you might want to document via a bitmap include the onscreen GlassMap, the RPER display, and the SketchPAD display.  Since the latter two can be printed anyway, the main reason for loading the picture into a graphics window in those cases is so you can add your own text annotation, circles, arrows, and so on before printing.

 

Many of the dialogs that you might want to transfer to a Graphics Window are oriented in landscape mode (wider than high), while the Graphics Window is in portrait mode (the opposite).  If you want the picture to fill the entire window, it has to be rotated by 90 degrees first.  To do so, turn on switch 81 before clicking on the paste button.  If you want to add annotation, however, it is best not to turn on that switch, to ensure that the dialog text is lined up with the annotation.

 

For best results, we recommend that you maximize the display (where permitted) before creating the bitmap copy; this will give you the largest number of pixels and the best resolution.  Also, it is unwise to make a bitmap of an RPER display that has a black background.  The sizing algorithm used by Windows usually skip pixels here and there, and if those contained your data, you see breaks in the lines.  This seems not to happen with a white background.

 

When the annotated bitmap is the way you want it, you can transfer or save the composite to a new bitmap or jpeg image (enlarge the window first) if you wish.  This is an easy way to send a picture to someone via email.

 

Some users prefer to print the annotated graphics window to a PDF writer.  This works well and is probably easier than sending it to a bmp file as instructed above.  You can then attach the PDF file to an email.
